## 概念

本アプリは「作業」と「ナビゲーション」の二中核概念の上に組み上がる（§3）。

- **作業（Task）**: 特定の文脈下で、明示された開始条件で開始し、完了条件で終了する、状態を持った活動の最小単位（§3.1.1）。
- **ナビゲーション（Navigation）**: ステークホルダーが作業を実行・監督・改善するにあたり、現在地・目的地・経路・進捗・例外・再計算結果を、認知負荷を最小化して即時に把握できる状態を継続的に維持するための、情報提示と動的計算の総体（§3.6.1）。

地図アプリ・カーナビが「空間」を扱うのに対し、本アプリは「作業状態空間」を扱う。

## プロダクト構成

| コンポーネント | 技術 | 用途 |
| --- | --- | --- |
| 端末作業ナビアプリ | Tauri + React + SQLite（SQLCipher AES-256） | 製造現場の作業者が利用。Android／Windows |
| 設定 Web UI | React | 生産技術／システム部門がマスタ・フローを設定 |
| バックエンド | Rust（tokio）＋ PostgreSQL | REST API、基幹連携、端末同期 |
| アドオン SDK | Rust／AssemblyScript（WASM／Wasmtime） | 拡張機能（OPC UA／MQTT／通知 等） |

詳細は §7 §8。

## 競合上の戦略

- **圧倒する 7 軸（◎）**: 非エンジニア操作性／オフライン対応／対応端末／拡張性・アドオン／セットアップ容易性／ライセンス・TCO／規制適合（§4.4）。
- **同等以上を維持する 4 軸（○）**: ナビ粒度／ERP・MES 連携／メディア対応／国際化。
- **「劣る」（△・×）は許容しない**。検出時は §22 サイクルで是正（§4.4／§22.4）。

11 軸合計 33 点満点で **目標 ≥ 29 点**（§4.5）。
