# 生産技術向け 30 分ハンズオン（日本語）

> 対応 §: ロードマップ §25.1 §3.2.1.3 §10.2 §15
> 想定所要: 30 分

## 学習目標

1. 設定 Web UI（§7.2）でフローを最初から作れる
2. バージョニング・差分プレビュー・ロールバックを 3 ステップ以内で操作できる
3. 業界別テンプレ（automotive/pharma/food/electronics）を起点にカスタマイズできる
4. §10.2.3 ストレス予測スコアを試行版発行前に確認できる
5. パイロット → 本番昇格の経路を理解する

## ハンズオン手順

### 0. 準備（5 分）

```bash
# 設定 UI を起動
pnpm --filter @wna/config-ui dev
```

ブラウザで `http://localhost:1421` を開き、ログイン。

### 1. 業界テンプレからのインポート（5 分）

1. 「フロー」→「新規作成」→「業界テンプレから」
2. `examples/flow-templates/automotive/assembly-line.yaml` をインポート
3. ノード一覧／辺一覧が表示される

### 2. ノード追加と完了条件設定（10 分）

1. 「組立」ノードの後に「ねじ締めトルク確認」ノードを追加
2. 完了条件を `photo`（写真証跡）に設定
3. 標準時間を 30 秒に設定
4. ストレス予測スコア（5 軸）を確認

### 3. バージョニングと試行版発行（5 分）

1. 「保存」→ 自動で v2 にバンプ
2. v1 と v2 の差分プレビュー
3. 試行版発行 → 5 ステップ以内に発行完了

### 4. パイロット → 本番昇格（5 分）

1. パイロット端末グループで実行ログを確認
2. 不適合 0 件であれば「本番昇格」
3. 影響範囲（参照する作業件数）が事前に表示される

## §10.2.4 受入観点との対応

| 受入観点 | 本ハンズオンでの確認 |
| --- | --- |
| 5 分以内で最初のフロー発行 | 手順 1+2 の冒頭で達成 |
| ロールバックが 3 ステップ以内 | 手順 3 の差分プレビュー画面から実演 |
| 設定 UI と作業 UI が同等 UX | テンプレ操作で実感 |
| §10.2.2 14 観点 | UI 監査チェックリスト（[`../../../02_設計/設定UI監査.md`](../../../02_設計/設定UI監査.md)）を併読 |
