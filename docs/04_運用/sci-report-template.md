# SCI（Software Carbon Intensity）試算テンプレート

> 対応 §: ロードマップ §11.7.3 §11.7.4 §22.1
> 出典: Green Software Foundation SCI v1.0／ISO/IEC 21031:2024
> 対象読者: 導入企業のサステナビリティ担当、メンテナ
> 改訂サイクル: §22.1 半期サイクル

§11.7.3 の SCI 試算を導入企業ごとに実施するためのテンプレート。本書を `examples/sci-report-<企業名>-<年期>.md` 等にコピーして実値を埋める。

## 1. 計算式

```
SCI = ((E × I) + M) / R
```

| 変数 | 説明 | 単位 |
| --- | --- | --- |
| E | 電力消費量 | kWh |
| I | 電力の炭素強度（地域別） | gCO2eq / kWh |
| M | ハードウェア製造の embodied carbon（按分） | gCO2eq |
| R | 機能単位 | 完了作業 1,000 件 等 |

結果単位: `gCO2eq / 完了作業 1,000 件`。

### 2.2. 電力炭素強度 I（gCO2eq / kWh）

参考値（日本、2025 年実績ベースの一般値）:

- 全国平均: 約 430 gCO2eq/kWh
- 再生可能エネルギーミックス契約: 50〜200 gCO2eq/kWh

導入企業の電力契約に応じて差し替える。地域・契約種別を記入欄に明記。

## 5. 改善案

- 電力契約を再生可能エネルギーへ切り替え（I の削減）。
- アイドル時 CPU 使用率を下げる（§11.7.1 ≤ 1%）。
- 同期バッチ化（§11.7.1 省電力時間帯）。
- ログ・メトリクスのサンプリング率調整（既定 10%）。

## 6. 受入観点（§11.7.4）

- 本テンプレートが `examples/` または `docs/04_運用/` に同梱されていること。
- 導入企業が自社環境で実行可能であること。
- §16 観測ダッシュボードに「電力／推定 CO2eq」セクションが配置されていること（コード初期実装後）。
- §22.1 半期サイクルで SCI トレンドをレビューし、悪化時は §22.4 是正フローを起動すること。
