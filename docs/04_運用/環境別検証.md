# 環境別検証手順

> 対応 §: ロードマップ §9.5.5 §11.2.1 §6 §22.1
> 対象読者: メンテナ、半期実機検証担当
> 改訂サイクル: §22.1 半期サイクル

§9.5.5 受入観点「工場環境（80dB 騒音／低照度／手袋）を想定した検証手順が文書化されている」を満たすため、本書で環境別検証手順を版管理する。製造現場特有の制約下で UI／UX が破綻しないことを確認する。

## 1. 検証環境

| 環境 | パラメータ | 想定機種 |
| --- | --- | --- |
| 屋内（標準） | 環境音 50dB、照度 500lux、素手 | Zebra ET60 |
| 屋内（騒音下） | 環境音 80〜95dB（WHO 推奨上限）、照度 500lux | Panasonic TOUGHBOOK A3 |

## 2. 検証項目

### 2.1. 視覚

- [ ] 屋内輝度 500nits／屋外輝度 1,000nits でテキスト・アイコンが判別可能（§6.3）
- [ ] アンドンの 5 段階色（success／info／warning／alert／critical）が屋外でも区別可能（§9.3.4／design-tokens `color.andon`）
- [ ] 色覚多様性 P/D/T シミュレーションでコントラスト・識別性が成立（§11.2.2 axe-core）
- [ ] 形状・テキスト併用で色のみに依存しないこと（§11.2.2 WCAG 2.2 SC 1.4.1）

### 2.2. 聴覚

- [ ] 80dB 環境音下で警告音が知覚可能（design-tokens `sound.feedback.warning`、80dB @1m）
- [ ] 危険音（85dB @1m）が聴覚障害者にも視覚＋触覚で伝わる（§9.5.3 ／§11.2.3）
- [ ] BGM・ジングルが鳴らないこと（§9.2.4 ダーク・パターン排除）

### 2.3. 触覚（ハプティック）

- [ ] 厚手手袋でもタッチターゲット 11mm（推奨）で確実に押せる（§11.2.1）
- [ ] 警告ハプティック（強度・間隔）が手袋越しでも知覚可能（design-tokens `haptic.warning`）
- [ ] ハプティック非搭載端末でビジュアルフラッシュへフォールバック（§11.2.3）

### 2.4. 入力

- [ ] 水滴／粉塵付着下でもタッチが反応する（§6.3 IP65）
- [ ] 手袋モード（OS 設定）が利用可能であること
- [ ] 音声コマンド「開始／完了／中断／戻る／メモ／撮影」が 80dB 環境音下で 90% 以上認識（§10.4.3）

### 2.5. メディア

- [ ] 低照度（50lux）で自動露出補正が機能（§10.4.2）
- [ ] 騒音下でノイズキャンセリング音声が了解可能（§10.4.2）
- [ ] PTT（押している間のみ送話）が動作（§10.4.2）

## 3. 検証手順

### 3.1. 騒音環境

1. 騒音発生器（または現場相当の機械音）で 80〜95dB を作る。
2. 警告音／危険音を鳴らし、聞き取れるかを 3 名以上で確認する（OSS 公開後、コミュニティへ実機検証要請）。
3. 視覚等価物が同時表示されることを確認する。

### 3.2. 低照度環境

1. 照度計で 50lux 以下を確認する。
2. 端末画面の輝度自動調整がオフ／オンの両モードで識別性を確認する。
3. カメラの自動露出補正で被写体が判別可能なことを確認する。

### 3.3. 手袋環境

1. 薄手・厚手の作業手袋を着用する。
2. タッチターゲット 9mm／11mm の両モードで誤タップ率を計測する。
3. 大型タッチターゲット＋音声コマンド代替が機能することを確認する。

## 5. 受入観点（§9.5.5／§11.2.3）

- 工場環境（80dB 騒音／低照度／手袋）を想定した検証手順が本書で文書化されていること。
- §22.1 半期サイクルで実機検証記録が更新されていること。
- 失敗項目は §22.4 是正フローへ。
- §6.4 最小要件機種で全項目が緑になることが §22.1 半期で確認されていること。
- §9.5 デザイントークン（`color.json`／`sound.json`／`haptic.json`／`spacing.json`）と本書の検証項目が双方向参照されていること。
