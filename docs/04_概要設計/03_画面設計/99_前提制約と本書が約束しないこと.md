## 2. 本書が約束しないこと（対象外と判断する）

### D-UI-001: 個人別生産性ランキング画面

**対象外と判断する。**

個人別の作業速度・件数ランキングを表示する画面は、35 画面リスト（SCR-HA/MA/MC）に含まれない。BR-BUS-029（個人別成果指標の提示禁止）を画面設計レベルで技術的に担保する。監督者が利用可能な画面（SCR-MC-001 ダッシュボード）は SLI（サービスレベル指標）のみ表示し、個人識別情報を含まない。

### D-UI-002: 監視カメラ連動画面

**対象外と判断する。**

カメラ映像のリアルタイム表示・録画確認・人物識別を行う画面は、本システムのスコープ外とする（BR-BUS-028: 用途三限定）。カメラは写真撮影エビデンス（FR-EV-002）の取得のみに限定する。

### D-UI-003: 生体認証・顔認証画面

**対象外と判断する。**

顔認証・指紋認証・虹彩認証等の生体認証を使用した入退出管理・本人確認画面は本システムのスコープ外とする。認証は JWT + PIN（FR-AU-001）のみとする。

### D-UI-004: マルチテナント管理画面

**対象外と判断する。**

複数工場・組織を横断して管理するマルチテナント管理画面は本設計に含まれない。本システムは単一工場・単一テナントの構成を前提とする（要件定義フェーズで確定済み）。

### D-UI-005: 詳細設計レベルの UI 実装仕様

**対象外と判断する。**

本書はワイヤーフレームレベルの情報密度・配置・インタラクション方針を確定するにとどまる。React Native / React のコンポーネント実装詳細（Tailwind クラス名・アニメーションフレーム数・Jest テストケース等）は詳細設計フェーズで確定する。

### D-UI-006: 帳票レイアウトの詳細

**対象外と判断する。**

帳票（RP-001〜006）の紙面レイアウト・テーブル列定義・フォント指定の詳細は `06_帳票設計/` で確定する。本書の SCR-MC-009（帳票出力）は帳票出力フォームの UI のみを対象とする。

### D-UI-007: OS/ブラウザ別の実装差異

**対象外と判断する。**

Android / iOS / Windows の OS 別 UI 実装差異（React Native のプラットフォーム固有スタイル）の詳細は詳細設計フェーズで確定する。本書は論理的なワイヤーフレームを提供するにとどまり、特定 OS での描画結果を保証しない。

## 4. 図面一括検証手順（drawio-lint / svg-postcheck）

`03_画面設計` 配下の全 drawio/SVG ファイルに対して以下の検証を必ず実施する。

### 4-1. 単体検証（ファイル作成・更新のたびに必須）

```bash
# drawio 静的検査（ERROR ゼロが必須・WARN も撲滅努力）
drawio-lint --strict docs/04_概要設計/03_画面設計/img/<target>.drawio

# SVG エクスポート（drawio-lint 通過後）
# （drawio-authoring スキルの WSL エクスポート手順に従う）

# SVG 後検査
svg-postcheck --strict docs/04_概要設計/03_画面設計/img/<target>.svg
```

### 4-2. 一括検証（全ファイル完了後または CI 相当タイミング）

```bash
# 全 drawio ファイルを一括 lint
for f in docs/04_概要設計/03_画面設計/img/*.drawio; do
  drawio-lint --strict "$f" || echo "FAIL: $f"
done

# 全 SVG ファイルを一括 postcheck
for f in docs/04_概要設計/03_画面設計/img/*.svg; do
  svg-postcheck --strict "$f" || echo "FAIL: $f"
done
```

### 4-3. 目視確認（一括検証後）

1. GitHub ダークテーマで全 SVG ファイルを確認（ハイコントラスト文字が潰れていないか）
2. 代表 8 画面 SVG を 5×7 グリッドで並べて視覚的一貫性を確認
3. コミットは `.drawio` と `.svg` を**必ず同一コミット**で行う

### 4-4. ERROR 扱い一覧（drawio-authoring スキル規約準拠）

- 白背景矩形が存在しない → ERROR
- 矢印の交差・重なり → ERROR
- 1 ファイルに複数ページ（`<diagram>` 要素複数）→ ERROR
- ラベルのコントラスト比 7.5:1 未満（`#333333` 相当以下）→ ERROR
- ラスタ画像の埋め込み（`shape=image` 使用）→ ERROR

### 4-5. 許容 WARN — パレット逸脱の例外

`drawio-lint --strict` は `figure-layer-convention` の 4 パレット外の色を WARN に昇格させるが、以下のファイル群では**ブランドデザイントークン色の使用が仕様**のため、パレット WARN は受容する。ERROR（重大品質違反）がゼロであることが唯一の必須条件。

| 対象ファイル群 | WARN 原因 | 採用理由 |
|---|---|---|
| `fig_des_screen_*` (全 35 画面) | `#EBF3FE`, `#FFFBEB`, `#F8FAFC` 等 | 実装トークン色を忠実に反映したハイファイワイヤー |
| `fig_des_components_catalog_*` (5 枚) | brand.primary / brand.accent 各段階色 | コンポーネントの実際の彩色を図示するため |
| `fig_des_tokens_*` (6 枚) | カラートークン全段階 | デザイントークン表現そのものが目的 |
| `fig_des_brand_*` (3 枚) | ブランドカラー多用 | ブランドアイデンティティの視覚化が目的 |

**対処不要な WARN**（`--strict` でも放置可）:
- `fillColor=#EBF3FE / #FFFBEB / #F8FAFC / #DC2626` 等のセマンティックトークン色
- `opacity` 指定によるブランド色の半透明装飾（コーナーモチーフ等）

**対処必須 WARN**（アーキテクチャ図 / フロー図では撲滅する）:
- 矢印ラベルの bbox 干渉
- テキスト頂点同士の重なり
- ページ枠はみ出し

- **個人別ランキング画面（D-UI-001）・監視カメラ連動画面（D-UI-002）・生体認証画面（D-UI-003）を技術的・設計的スコープ外と宣言し、BR-BUS-028/029 の画面設計レベルでの担保を確定した。**
- **詳細設計フェーズへの責務移管事項（D-UI-005〜007）を明示し、概要設計の境界を確定した。**
