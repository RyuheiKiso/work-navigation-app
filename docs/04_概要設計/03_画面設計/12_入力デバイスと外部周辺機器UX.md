# 12 入力デバイスと外部周辺機器 UX

本章の責務は、ハンディ APP で使用するスキャナ・計測器・カメラの UX 方式（接続検知・フィードバック・エラーハンドリング）を確定することである。IF-005（スキャナ）・IF-006（計測器）・IF-007（カメラ）・FR-EV-002〜006 を本章で担保する。

---

## 1. QR / バーコードスキャナ UX（IF-005）

### 1-1. スキャンフロー

スキャンは SCR-HA-003（QR スキャン起動）で実施する。`CMP-HA-004 QrScannerView` が以下の UX を提供する。

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| スキャン待機 | カメラプレビューを全画面表示。「QR コードをカメラに向けてください」のオーバーレイテキスト表示 | FR-EV-004 |
| スキャン成功 | 緑枠を QR コード上に表示（0.5 秒）+ 「ピッ」音 + 触覚フィードバック（振動 100ms）| NFR-UX-004 |
| GS1 チェック成功 | 即座に TRN-004 発火し SCR-HA-004 へ遷移 | TRN-004 |
| GS1 チェック失敗 | 赤枠 + 「認識できないコードです（ERR-VAL-002）」スナックバー。3 秒後に自動クリア | ERR-VAL-002 |
| 手動入力モード | スキャン失敗 3 回後に「手動入力」ボタンを表示。テキストフィールドに GS1 AI 形式で入力 | NFR-UX-004 |

### 1-2. 接続デバイス優先順位

| 優先順位 | デバイス | 接続方式 |
|---|---|---|
| 1 | BLE HID スキャナ（ハンズフリー）| BLE |
| 2 | USB HID スキャナ | USB |
| 3 | 端末内蔵カメラ | OS カメラ API |

BLE スキャナ接続時は、カメラプレビューを閉じてスキャン待機モードに移行する。BLE 切断検知後 3 秒でカメラモードに自動フォールバックする。

## 2. 計測器 UX（IF-006）

### 2-1. 計測値入力フロー（SCR-HA-009）

`CMP-HA-002 NumericInputField` が以下の UX を提供する。

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| BLE 計測器接続待機 | 「計測器に接続中…」インジケーター表示 | FR-EV-006 |
| BLE 接続成功 | 「✓ {計測器名} 接続済み」バッジ表示 + グリーントースト | FR-EV-006 |
| 値受信 | 受信値を NumericInputField に自動入力。単位（UCUM）を隣接表示 | FR-EV-003 |
| 範囲チェック OK | 緑テキスト表示 + 「確定」ボタンをアクティブ化 | FR-EV-006 |
| 範囲チェック NG | 赤テキスト表示 + 「測定値が上限を超えています（ERR-BIZ-003）」バナー | ERR-BIZ-003 |
| BLE 未接続時 | 手動数値入力フォームを表示（テンキーのみ）| FR-EV-003 |

### 2-2. 単位表示方式

計測値の単位は UCUM コード（`m`, `mm`, `kg`, `N.m` 等）をそのまま表示する。計測器から受信した SI 基本単位を UCUM 変換テーブル（CFG 管理）で表示単位に変換する。

例: 計測器から `50000 g` 受信 → `50.000 kg` 表示

### 2-3. 計測器接続 UX の状態遷移

| 状態 | 表示 | 次のアクション |
|---|---|---|
| DISCONNECTED | グレーバッジ「計測器未接続」| 手動入力を促す |
| CONNECTING | スピナー「接続中…」| 待機 |
| CONNECTED | グリーンバッジ「{計測器名} 接続済み」| 値受信待ち |
| RECEIVING | 値が点滅表示 | 自動入力 |
| ERROR | レッドバッジ「接続エラー（ERR-EXT-001）」| 再接続ボタン表示 |

## 3. カメラ・写真撮影 UX（IF-007）

### 3-1. 写真撮影フロー（SCR-HA-008）

`CMP-HA-003 PhotoCapturePanel` が以下の UX を提供する。

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| 撮影待機 | カメラプレビュー全画面表示。Step のエビデンス要件（「ネジ頭を撮影してください」等）をオーバーレイテキスト表示 | FR-EV-002 |
| 撮影ガイド | 撮影対象に応じた枠線（CFG: 自由枠 / 矩形ガイド）をオーバーレイ表示 | FR-EV-002 |
| 撮影完了 | サムネイル + 「この写真でよいですか？」確認ダイアログ。「撮り直し」「OK」の 2 択 | FR-EV-002 |
| 解像度チェック | 端末側で即時チェック（1280×720 px 以上）。NG 時は自動再撮影を促す | NFR-UX-004 |
| アップロード中 | 進捗インジケーター表示。バックグラウンドで Exif 除去 + SHA-256 計算 + JPEG 圧縮実施 | FR-EV-002 |
| アップロード完了 | TRN-010 発火し SCR-HA-005 へ戻る | TRN-010 |
| オフライン時 | Outbox にエビデンスを蓄積し、同様の TRN-010 発火（同期は後続オンライン時）| P1（Offline-First）|

### 3-2. プライバシー設計

- Exif メタデータ（位置情報・端末情報）は端末側で撮影直後に除去する（IF-007 設計）
- 除去後の画像から SHA-256 を計算し、ハッシュ値をサーバーに先送りする
- 写真は作業エビデンスのみに使用し、個人監視目的への転用を API レベルで禁止する（BR-BUS-028）

## 4. 電子サインパッド UX（SCR-HA-010・CMP-HA-005）

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| サイン待機 | サインパッドキャンバス + 「署名してください」テキスト表示 | FR-AU-001 |
| PIN 入力 | テンキーで 4〜8 桁 PIN 入力（マスク表示）| FR-AU-001 |
| PIN 認証成功 | 「✓ 署名完了」グリーン表示 + TRN-013 発火 | TRN-013 |
| PIN 認証失敗 | 「PIN が一致しません（残り {N} 回）」赤テキスト表示 | ERR-AUTH-002 |
| 3 回失敗 | アカウントロック通知 + supervisor への通知（ERR-AUTH-003）| BR-BUS-035 |
| クリアボタン | サインキャンバスのみクリア（PIN は維持）| NFR-UX-004 |

## 5. デバイス接続状態の統合表示

デバイス接続状態は SCR-HA-015（設定画面）の「接続デバイス」セクションに一覧表示する。

| デバイス種別 | 表示情報 |
|---|---|
| BLE スキャナ | 機器名 + 接続状態 + バッテリー残量 |
| BLE 計測器 | 機器名 + 接続状態 + バッテリー残量 + 最終計測値 |
| ネットワーク | SSID + 電波強度 + Outbox 蓄積件数 |

---

**本節で確定した方針**
- **スキャナ（BLE HID 優先・カメラフォールバック）・計測器（BLE 自動入力・手動フォールバック）・カメラ（Exif 除去 + SHA-256 端末側計算）の 3 デバイスの UX フローを確定した。**
- **全デバイスで「接続成功・失敗・エラー」の状態フィードバックを視覚・触覚・音声の 3 要素で提供し、オフライン時の Outbox 蓄積フローを担保した。**
- **写真撮影時の Exif 除去と個人監視目的転用禁止を UX 設計レベルで明文化した。**
