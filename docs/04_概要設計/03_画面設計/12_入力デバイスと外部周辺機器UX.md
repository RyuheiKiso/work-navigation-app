# 12 入力デバイスと外部周辺機器 UX

本章の責務は、ハンディ APP で使用するスキャナ・計測器・カメラの UX 方式（接続検知・フィードバック・エラーハンドリング）を確定することである。IF-005（スキャナ）・IF-006（計測器）・IF-007（カメラ）・FR-EV-002〜006 を本章で担保する。

---

## 1. QR / バーコードスキャナ UX（IF-005）

### 1-1. スキャンフロー

スキャンは SCR-HA-003（QR スキャン起動）で実施する。`CMP-HA-004 QrScannerView` が以下の UX を提供する。

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| スキャン待機 | カメラプレビューを全画面表示。「QR コードをカメラに向けてください」のオーバーレイテキスト表示 | FR-EV-004 |
| スキャン成功 | 緑枠を QR コード上に表示（0.5 秒）+ 「ピッ」音 + 触覚フィードバック（振動 100ms）| NFR-UX-004 |
| GS1 チェック成功 | 即座に TRN-004 発火し SCR-HA-004 へ遷移 | TRN-004 |
| GS1 チェック失敗 | 赤枠 + 「認識できないコードです（ERR-VAL-002）」スナックバー。3 秒後に自動クリア | ERR-VAL-002 |
| 手動入力モード | スキャン失敗 3 回後に「手動入力」ボタンを表示。テキストフィールドに GS1 AI 形式で入力 | NFR-UX-004 |

### 1-2. 接続デバイス優先順位

| 優先順位 | デバイス | 接続方式 |
|---|---|---|
| 1 | BLE HID スキャナ（ハンズフリー）| BLE |
| 2 | USB HID スキャナ | USB |
| 3 | 端末内蔵カメラ | OS カメラ API |

BLE スキャナ接続時は、カメラプレビューを閉じてスキャン待機モードに移行する。BLE 切断検知後 3 秒でカメラモードに自動フォールバックする。

#### §1-3 ポカヨケ対象（工具・治具・計測器）スキャン UX

SOP の required_scans 配列が存在する Step では、SCR-HA-003 の上部に RequiredScansChipStack（CMP-HA-016）を表示し、照合が必要な全ターゲット（材料 / 工具 / 治具 / 計測器）をアイコン付きチップで一覧する。

**フェーズ別 UX:**

| フェーズ | 表示内容 | フィードバック |
|---|---|---|
| スキャン要求 | チップスタック全ターゲットを pending（灰色） | — |
| ターゲット完了 | 対象チップを `state.success.500` 塗りつぶし | snd-scan-ok + hap-medium |
| 誤工具検知（ERR-VAL-006）| 対象チップを `state.danger.500` シェイク + 赤バナー「指定された工具ではありません: 期待[XXX] / 読取[YYY]」 | snd-scan-ng + hap-error |
| 校正期限切れ計測器検知 | `state.danger.50` 全面オーバーレイ + ERR-BIZ-004 + 「保全担当を呼ぶ」CTA | snd-scan-ng + hap-error |
| 手動入力フォールバック（3 回失敗後） | tool_subtype ドロップダウン + scan_code 直入力フォームを表示 | — |

誤工具（スパナアイコン + 赤）と誤材料（部品アイコン + 赤）はアイコン・文言の両方で弁別する（色のみに依存しない）。
手動入力時は payload.scan_verifications[].is_manual_input = true を記録する。

## 2. 計測器 UX（IF-006）

### 2-1. 計測値入力フロー（SCR-HA-009）

`CMP-HA-002 NumericInputField` が以下の UX を提供する。

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| BLE 計測器接続待機 | 「計測器に接続中…」インジケーター表示 | FR-EV-006 |
| BLE 接続成功 | 「✓ {計測器名} 接続済み」バッジ表示 + グリーントースト | FR-EV-006 |
| 値受信 | 受信値を NumericInputField に自動入力。単位（UCUM）を隣接表示 | FR-EV-003 |
| 範囲チェック OK | 緑テキスト表示 + 「確定」ボタンをアクティブ化 | FR-EV-006 |
| 範囲チェック NG | 赤テキスト表示 + 「測定値が上限を超えています（ERR-BIZ-003）」バナー | ERR-BIZ-003 |
| BLE 未接続時 | 手動数値入力フォームを表示（テンキーのみ）| FR-EV-003 |

### 2-2. 単位表示方式

計測値の単位は UCUM コード（`m`, `mm`, `kg`, `N.m` 等）をそのまま表示する。計測器から受信した SI 基本単位を UCUM 変換テーブル（CFG 管理）で表示単位に変換する。

例: 計測器から `50000 g` 受信 → `50.000 kg` 表示

### 2-3. 計測器接続 UX の状態遷移

| 状態 | 表示 | 次のアクション |
|---|---|---|
| DISCONNECTED | グレーバッジ「計測器未接続」| 手動入力を促す |
| CONNECTING | スピナー「接続中…」| 待機 |
| CONNECTED | グリーンバッジ「{計測器名} 接続済み」| 値受信待ち |
| RECEIVING | 値が点滅表示 | 自動入力 |
| ERROR | レッドバッジ「接続エラー（ERR-EXT-001）」| 再接続ボタン表示 |

## 3. カメラ・写真撮影 UX（IF-007）

### 3-1. 写真撮影フロー（SCR-HA-008）

`CMP-HA-003 PhotoCapturePanel` が以下の UX を提供する。

| フェーズ | UX 方針 | 根拠 |
|---|---|---|
| 撮影待機 | カメラプレビュー全画面表示。Step のエビデンス要件（「ネジ頭を撮影してください」等）をオーバーレイテキスト表示 | FR-EV-002 |
| 撮影ガイド | 撮影対象に応じた枠線（CFG: 自由枠 / 矩形ガイド）をオーバーレイ表示 | FR-EV-002 |
| 撮影完了 | サムネイル + 「この写真でよいですか？」確認ダイアログ。「撮り直し」「OK」の 2 択 | FR-EV-002 |
| 解像度チェック | 端末側で即時チェック（1280×720 px 以上）。NG 時は自動再撮影を促す | NFR-UX-004 |
| アップロード中 | 進捗インジケーター表示。バックグラウンドで Exif 除去 + SHA-256 計算 + JPEG 圧縮実施 | FR-EV-002 |
| アップロード完了 | TRN-010 発火し SCR-HA-005 へ戻る | TRN-010 |
| オフライン時 | Outbox にエビデンスを蓄積し、同様の TRN-010 発火（同期は後続オンライン時）| P1（Offline-First）|

### 3-2. プライバシー設計

- Exif メタデータ（位置情報・端末情報）は端末側で撮影直後に除去する（IF-007 設計）
- 除去後の画像から SHA-256 を計算し、ハッシュ値をサーバーに先送りする
- 写真は作業エビデンスのみに使用し、個人監視目的への転用を API レベルで禁止する（BR-BUS-028）

| フェーズ | UX 方針 | 根拠 |
|---|---|---|

| クリアボタン | サインキャンバスのみクリア（PIN は維持）| NFR-UX-004 |

## 5. デバイス接続状態の統合表示

デバイス接続状態は SCR-HA-015（設定画面）の「接続デバイス」セクションに一覧表示する。

| デバイス種別 | 表示情報 |
|---|---|
| BLE スキャナ | 機器名 + 接続状態 + バッテリー残量 |
| BLE 計測器 | 機器名 + 接続状態 + バッテリー残量 + 最終計測値 |
| ネットワーク | SSID + 電波強度 + Outbox 蓄積件数 |

## 6. サウンド・ハプティック仕様

### 6-1. サウンドカタログ

WebAudio（Web）/ React Native Sound（HA）で実装。sine 波 + 減衰エンベロープによる合成音。

| サウンド ID | 周波数 / 波形 | 長さ | 用途 |
|---|---|---|---|
| `snd-scan-ok` | 880 Hz / sine 減衰 | 60ms | QR/バーコードスキャン成功 |
| `snd-scan-ng` | 220 Hz → 110 Hz グライド / sine | 200ms | スキャン失敗・認識不可 |
| `snd-shutter` | 1200 Hz click + 白色ノイズ burst | 80ms | 写真撮影 |
| `snd-success-soft` | 660 Hz → 880 Hz ツーステップ / sine | 160ms | Step 完了・一般成功 |
| `snd-error` | 440 Hz + 330 Hz 二音 | 80ms + 120ms | APIエラー・バリデーション失敗 |
| `snd-warning` | 660 Hz トリル 3 回 | 240ms | 警告・範囲外測定値 |
| `snd-andon-fire` | 1000 Hz / 1500 Hz スイープ × 3 | 600ms × 3 | アンドン発報（騒音 85dB 環境で識別可能）|
| `snd-notification` | 880 Hz → 1100 Hz 2 音 | 200ms | 新着通知（MC 向け）|

### 6-2. ハプティックカタログ（HA 専用）

| ハプティック ID | パターン（ms）| 用途 |
|---|---|---|
| `hap-light` | 30ms | 軽いタッチフィードバック |
| `hap-medium` | 60ms | ボタン press・スキャン成功 |
| `hap-strong` | 100ms | 確定・PIN 認証 |
| `hap-strong-3x` | `100-50-100-50-100 ms`（ON-OFF-ON-OFF-ON）| アンドン発報（注意喚起）|
| `hap-success` | `30-30ms` 二連 | Step 完了成功 |
| `hap-error` | 200ms 連続 | エラー・認証失敗 |

### 6-3. サウンド・ハプティックのミュートと例外

| 設定 | 対応 |
|---|---|
| `app_settings.sound_mute = true` | `snd-andon-fire` 以外の全サウンドを無音化 |
| `app_settings.haptic_mute = true` | `hap-strong-3x` 以外の全ハプティックを無効化 |
| アンドン（`snd-andon-fire` / `hap-strong-3x`）| ミュート設定に関わらず常に発音・振動（生命安全）|

## 7. スキャンフィードバック視覚仕様（トークン化）

### 7-1. QR スキャン成功フィードバック（CMP-HA-004）

| 要素 | 仕様 |
|---|---|
| 枠線 | `border.color.success`（`state.success.500`）/ 幅 4px / `radius.md` |
| 内側グロー | `box-shadow: inset 0 0 24px rgba(5,150,105,.25)` / 320ms フェードアウト |
| パーティクル | 小粒 4 個（`state.success.300` / 4×4dp）が 280ms 上方フェード |
| reduce-motion | 枠線点滅 1 回のみ（パーティクル・グロー無効化）|

### 7-2. QR スキャン失敗フィードバック（CMP-HA-004）

| 要素 | 仕様 |
|---|---|
| 枠線 | `border.color.error`（`state.danger.500`）/ 幅 4px |
| シェイク | `translateX(-6px → 6px → -4px → 4px → 0)` / 120ms |
| サウンド | `snd-scan-ng` |
| ハプティック | `hap-error` |
| reduce-motion | 枠線赤表示のみ（シェイク無効化）|

工具スキャン誤一致は `ERR-VAL-006` バナー（スパナアイコン付き）、材料スキャン誤一致は既存の `ERR-VAL-002/003` バナー（部品アイコン付き）で弁別する（FR-EV-013, FR-EV-004）。

### 7-3. BLE 接続状態ビジュアル（CMP-HA-002 + 設定画面）

| 状態 | バッジ色 | アイコン | アニメーション |
|---|---|---|---|
| DISCONNECTED | `neutral.300` / `neutral.500` テキスト | `BluetoothOff` | なし |
| CONNECTING | `brand.accent.50` / `brand.accent.600` テキスト | `Bluetooth` + スピナー | 回転 600ms linear |
| CONNECTED | `state.success.50` / `state.success.700` テキスト | `Bluetooth` + `Check` | チェックのポップイン 200ms |
| RECEIVING | `state.info.50` / `state.info.700` テキスト | `Activity` | 点滅 800ms |
| ERROR | `state.danger.50` / `state.danger.700` テキスト | `AlertCircle` | なし |

**本節で確定した方針**
- **スキャナ（BLE HID 優先・カメラフォールバック）・計測器（BLE 自動入力・手動フォールバック）・カメラ（Exif 除去 + SHA-256 端末側計算）の 3 デバイスの UX フローを確定した。**
- **全デバイスで「接続成功・失敗・エラー」の状態フィードバックを視覚・触覚・音声の 3 要素で提供し、オフライン時の Outbox 蓄積フローを担保した。**
- **サウンドカタログ 8 種・ハプティックカタログ 6 種・アンドン常時発動例外を確定した。**
- **スキャンフィードバックの視覚仕様をデザイントークン化し、実装での一貫性を保証した。**
- **写真撮影時の Exif 除去と個人監視目的転用禁止を UX 設計レベルで明文化した。**
- **工具・治具・計測器スキャンの UX を required_scans 配列駆動で構造化し、誤工具と誤材料をフィードバック上で明確に弁別することを確定する（FR-EV-013）。**
