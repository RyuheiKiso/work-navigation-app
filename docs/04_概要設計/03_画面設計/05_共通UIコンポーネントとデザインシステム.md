# 05 共通 UI コンポーネントとデザインシステム

本章の責務は、3 アプリで共有する UI コンポーネント（CMP-NNN）とデザイントークン（カラー・タイポグラフィ・スペーシング）を確定することである。NFR-UX 全 43 件の UI レベルでの担保を含む。

---

## 1. デザイントークン

### 1-1. カラーパレット

| トークン名 | ライトモード | ダークモード（夜勤）| 用途 |
|---|---|---|---|
| `color.primary` | `#0052CC` | `#4C9AFF` | 主要ボタン・リンク |
| `color.success` | `#006644` | `#57D9A3` | 完了・OK |
| `color.warning` | `#FF991F` | `#FFB154` | 注意・警告 |
| `color.danger` | `#BF2600` | `#FF7452` | エラー・緊急 |
| `color.background` | `#FFFFFF` | `#172B4D` | 画面背景 |
| `color.text.primary` | `#172B4D` | `#FFFFFF` | 本文 |
| `color.andon` | `#FF0000` | `#FF4444` | アンドン専用（緊急赤）|

全カラーはコントラスト比 4.5:1 以上（NFR-UX-007・WCAG 2.1 AA）を確保する。

### 1-2. タイポグラフィ

| トークン名 | サイズ | 用途 |
|---|---|---|
| `text.body` | 14pt | 通常本文 |
| `text.instruction` | 16pt | Step 指示文 |
| `text.heading` | 20pt | 見出し |
| `text.label` | 12pt | ラベル・補足 |

工場環境（振動・グローブ・強光）でも読みやすい Noto Sans JP（日本語）・Noto Sans（英語）を採用する。

### 1-3. スペーシング・タッチターゲット

- 基本グリッド: 8pt
- ボタン最小タッチターゲット: 44dp（NFR-UX-009 必須）・72dp（推奨・CFG-013）
- ボタン間マージン: 最低 8pt（指の太さ対応）

## 3. ダークモード自動切替（FR-UI-004）

```typescript
// 夜勤ダークモード（20:00-6:00）
function getThemeMode(hour: number): 'light' | 'dark' {
  // NFR-UX-005: 夜勤帯（20:00-翌6:00）はダークモード自動適用
  return (hour >= 20 || hour < 6) ? 'dark' : 'light';
}

// CFG で時間帯設定可能にする
const DARK_MODE_START = Number(getConfig('ui.dark_mode_start_hour', '20'));
const DARK_MODE_END   = Number(getConfig('ui.dark_mode_end_hour',   '6'));
```

---

**本節で確定した方針**
- **デザイントークン（カラー・タイポグラフィ・スペーシング）を確定し、全カラーで WCAG 2.1 AA（コントラスト比 4.5:1 以上）を保証する。**
- **CMP カタログ（HA-8件/MA-4件/MC-3件）を確定し、共通コンポーネントの責務・使用画面を明示した。**
- **ダークモード自動切替（20:00-6:00）を CFG で設定可能とし、夜勤帯の視認性を確保する設計を確定した。**
