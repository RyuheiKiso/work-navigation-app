# 11 多言語・国際化方式

本章の責務は、3 アプリ共通の多言語対応（i18n）・国際化（L10n）方式を確定することである。NFR-UX-013（日本語/英語対応）・FR-UI-001（言語切替）・FR-UI-003（やさしい日本語）を本章で担保する。

---

## 1. 対応言語と切替方式

### 1-1. 対応言語一覧

| 言語コード | 言語名 | 対象ユーザー | 提供 APP |
|---|---|---|---|
| `ja` | 日本語（標準）| 日本語母語話者 | HA / MA / MC |
| `ja-simple` | やさしい日本語 | 外国籍作業者・日本語学習中の方 | HA のみ |
| `en` | English | 英語母語話者 / 外国籍管理者 | HA / MA / MC |

`ja-simple` はハンディ APP の Step 指示文・ボタンラベル・エラーメッセージのみ対象とする。管理系画面（MA/MC）は `ja`/`en` のみ提供する（FR-UI-003 仕様）。

### 1-2. 言語切替方式

| アプリ | 切替方法 | 設定保存先 |
|---|---|---|
| ハンディ APP | SCR-HA-015（設定画面）にて選択 | SQLite `app_settings.locale` |
| マスタメンテ APP | ヘッダー右上の言語切替ドロップダウン | localStorage `locale` |
| 管理コンソール | ヘッダー右上の言語切替ドロップダウン | localStorage `locale` |

切替は即時反映とし、再起動・再ログインは不要とする（FR-UI-001）。

## 2. 翻訳鍵体系と管理方針

### 2-1. 翻訳ファイル構成

```
src/
└── i18n/
    ├── locales/
    │   ├── ja/
    │   │   ├── common.json       # 共通ラベル
    │   │   ├── handy.json        # ハンディ APP 専用
    │   │   ├── master.json       # マスタメンテ専用
    │   │   └── console.json      # 管理コンソール専用
    │   ├── ja-simple/
    │   │   └── handy.json        # やさしい日本語（HA 専用）
    │   └── en/
    │       ├── common.json
    │       ├── handy.json
    │       ├── master.json
    │       └── console.json
    └── index.ts                   # i18next 初期化
```

### 2-2. 翻訳鍵の命名規則

形式: `{scope}.{component}.{key}`

| scope | 対象 |
|---|---|
| `common` | 全アプリ共通（OK/キャンセル/エラー等）|
| `handy` | ハンディ APP 専用 |
| `master` | マスタメンテ専用 |
| `console` | 管理コンソール専用 |

例:
```json
{
  "handy.step.complete": "Step 完了",
  "handy.step.complete_simple": "おわりました",
  "handy.andon.trigger": "アンドン発報",
  "common.error.network": "ネットワークエラーが発生しました（ERR-SYS-001）"
}
```

### 2-3. ライブラリ

| 用途 | ライブラリ | バージョン方針 |
|---|---|---|
| React Native (HA) | `react-i18next` + `i18next` | 最新安定版（LSV）|
| React (MA/MC) | `react-i18next` + `i18next` | HA と同バージョン固定 |
| 翻訳型安全性 | `i18next-typescript-codegen` | 翻訳鍵の型生成 |

## 3. やさしい日本語（ja-simple）方針

### 3-1. 作成基準

「やさしい日本語」は以下の基準で作成する（FR-UI-003）:

| 基準 | 方針 |
|---|---|
| 文章長 | 1 文 40 文字以内 |
| 漢字 | 常用漢字のみ。それ以外はひらがなに変換 |
| 外来語 | 説明文を併記（例:「スキャン（QRを読む）」）|
| 敬語 | 「〜してください」「〜しました」の丁寧語に統一 |
| 主語 | 作業者を主語にする（「あなたは〜」ではなく「〜してください」）|

### 3-2. 対象範囲（HA のみ）

| 対象 | ja | ja-simple |
|---|---|---|
| Step 指示文（SOP コンテンツ）| 標準日本語 | やさしい日本語バージョンを別フィールドで保持 |
| ボタンラベル | 「次のステップへ」| 「つぎに すすむ」|
| エラーメッセージ | 「測定値が上限を超えています」| 「この数字が大きすぎます。もう一度はかってください」|
| OfflineBanner | 「オフライン：作業記録はローカルに保存されます」| 「インターネットにつながっていません。作業は続けられます」|

Step 指示文の「やさしい日本語」バージョンはマスタメンテ APP（SCR-MA-004）で master_admin が入力する。空の場合は通常の日本語にフォールバックする。

## 4. 数値・日時・単位の地域化

| 項目 | 方針 | 根拠 |
|---|---|---|
| 日時形式 | ISO 8601 で DB 保存、表示は `Intl.DateTimeFormat` で locale に従う | FR-AU-001 |
| 小数点記号 | locale に従う（ja: `1,234.5`、en: `1,234.5`）| NFR-UX-013 |
| 単位 | UCUM コードで DB 保存、表示はロケール変換なし（国際表記を維持）| FR-EV-006 |
| タイムゾーン | サーバー UTC 保存、表示は端末タイムゾーン自動適用 | NFR-ALC-007 |

## 5. SOP コンテンツの多言語対応

SOP の Step 指示文は多言語フィールドを持つ JSON 構造で保存する（TBL-008: steps テーブル `content` カラム）。

```json
{
  "ja": "ボルト M10 を 50 Nm で締め付けてください",
  "ja-simple": "ボルトを しめてください（50 Nmで）",
  "en": "Tighten bolt M10 to 50 Nm"
}
```

ハンディ APP は端末の言語設定（`app_settings.locale`）に対応するフィールドを取得する。対応フィールドが未設定の場合は `ja` にフォールバックする。

---

**本節で確定した方針**
- **日本語（標準）・やさしい日本語・英語の 3 言語を対応言語とし、i18next で実装する多言語方式を確定した。やさしい日本語はハンディ APP の Step 指示文・UI ラベルに限定する。**
- **SOP コンテンツは `{ja, ja-simple, en}` の多言語 JSON フィールドで保存し、端末言語設定に従いフォールバック付きで取得する方式を確定した。**
- **翻訳鍵は `{scope}.{component}.{key}` 形式で管理し、i18next-typescript-codegen で型安全性を保証する。**
