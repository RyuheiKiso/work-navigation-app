# 06 情報設計とナビゲーション原則

本章の責務は、3 アプリ共通の情報設計指針（1画面1タスク・現在地常時表示・誤操作防止）とナビゲーション設計原則を確定することである。NFR-UX 全 43 件のうち情報構造・ナビゲーションに係る要件（NFR-UX-001〜006/010〜015）を本章で担保する。

---

## 1. 情報設計の 5 原則

### 原則 1: 1画面1タスク（NFR-UX-001）

各画面は作業者が「今何をすべきか」を瞬時に判断できる単一タスクに絞る。複数タスクの混在は認知負荷を高め（NFR-UX-002）、誤操作リスクを増大させる。

| アプリ | 1画面1タスクの実装方針 |
|---|---|
| ハンディ APP | Step 1 件を 1 画面で表示（CMP-HA-001 StepCard）。前後 Step は非表示 |
| マスタメンテ APP | ツリー編集（SCR-MA-004）と差分表示（SCR-MA-010）は別画面とする |
| 管理コンソール | KPI ゲージとアラート詳細は別画面とする |

### 原則 2: 現在地の常時表示（NFR-UX-003）

作業者が「全体のどこにいるか」を常に把握できるよう、進捗情報を画面最上部に固定表示する。

- ハンディ APP: `CMP-HA-007 ProgressBar`（Step N / 全 M）を全 Step 画面の最上部に固定
- マスタメンテ APP: パンくずリスト（プロセス > オペレーション > SOP > Step）を最上部に固定
- 管理コンソール: サイドバーのアクティブメニューをハイライト表示

### 原則 3: 誤操作ゼロ設計（NFR-UX-004・BR-BUS-016）

工場環境（グローブ操作・振動）での誤操作を防止する設計を以下の 3 層で担保する。

| 層 | 方式 | 実装コンポーネント |
|---|---|---|
| 物理層 | 44dp 以上のタッチターゲット（NFR-UX-009）| CMP-HA-001〜008 全て 44dp 以上 |
| インタラクション層 | 破壊的操作に確認ダイアログ必須 | 中断・廃止・再投入の 3 操作 |
| データ層 | 完了済 Step の修正を ERR-BIZ-001 で拒否 | API-step-events-001 バリデーション |

### 原則 4: 緊急操作の最優先確保（NFR-UX-010）

アンドンボタン（CMP-HA-008）は Step 実行画面（SCR-HA-005/006/007）の右下に常設し、他要素で隠蔽しない。ボタンサイズは 72dp 以上（CFG-013）。

### 原則 5: オフライン状態の明示（NFR-UX-011）

`CMP-HA-006 OfflineBanner` は DISCONNECTED 状態以上で全画面最上部に表示。Emergency Mode（5 分超切断）時は赤帯に変化し、Outbox 蓄積件数を表示する。

## 2. アプリ別ナビゲーション設計

### 2-1. ハンディ APP: タスクフロー型ナビゲーション

線形ウィザード方式を採用する。作業者は次 Step への進行のみを行い、前 Step への任意の逆戻りは禁止（BR-BUS-011: Step 実行は順序厳守）。

```
ログイン → ホーム → [QR スキャン | 作業選択] → SOP 一覧 → Step 実行（1〜N）→ ホーム
                                                                ↕（アンドン・不適合・中断）
                                                              （副流画面）
```

図: fig_des_screen_ha_nav（img/ 配下）を参照。

**逆遷移の例外規則**:
- 中断（TRN-015〜016）: 理由選択後にホームへ戻る（前 Step には戻らない）
- アンドン（TRN-019〜020）: 発報完了後に元の Step 画面へ戻る（状態保持）
- 電子サイン（TRN-012〜013）: サイン完了後に同一 Step 画面へ戻る

### 2-2. マスタメンテ APP: ハブ＆スポーク型ナビゲーション

プロセス一覧（SCR-MA-001）を中心ハブとし、各機能へのスポーク遷移とする。一覧 → 編集 → 一覧の往復パターンが基本。

```
プロセス一覧（ハブ）
├── SOP 編集 → バージョン差分 → SOP 編集
│              └── プレビュー → SOP 編集
│              └── インポート → SOP 編集
├── レビュー依頼 → 承認サイン → 公開設定 → プロセス一覧
└── 廃止処理 → プロセス一覧
```

図: fig_des_screen_ma_nav（img/ 配下）を参照。

### 2-3. 管理コンソール: サイドバー型ナビゲーション

全機能を左サイドバーに列挙し、メインエリアでコンテンツを表示するダッシュボード型。ダッシュボード（SCR-MC-001）が常時起点となる。

サイドバー幅は固定 240px（最小 1024px 画面対応）。折りたたみ機能なし（管理操作の可視性を最優先）。

## 3. 情報階層と密度の設計基準

### 3-1. ハンディ APP の情報密度（NFR-UX-005）

| 情報要素 | 表示位置 | フォント | 根拠 |
|---|---|---|---|
| Step 指示文 | 中央大 | text.instruction（16pt）| 視認性最優先 |
| Step 番号/合計 | 最上部 | text.label（12pt）| 常時把握 |
| SOP 名 | 最上部（Step 番号の下）| text.label（12pt）| コンテキスト確認 |
| アンドンボタン | 右下固定 | —（ボタン）| NFR-UX-010 |
| OfflineBanner | 最上部固定 | text.label（12pt）| NFR-UX-011 |

1 画面の情報要素は 5 項目以内に制限する（認知負荷の制限: NFR-UX-002）。

### 3-2. 管理コンソールの情報密度（NFR-UX-006）

管理コンソールは PC ブラウザ専用。情報密度を高くして 1 画面で俯瞰的把握を可能にする。ただし SLI ゲージ（CMP-MC-001）は 3 指標以内に絞り、二次指標は折りたたみで表示する。

## 4. エラー・状態フィードバックの設計

### 4-1. フィードバック方式と視覚仕様

| 状態 | コンポーネント | 持続時間 | 位置 | サイズ | トークン |
|---|---|---|---|---|---|
| 成功（Step 完了）| `FRG-016 Toast success` | 2 秒 | 画面上部中央 / 12dp 下 | 幅: min(360dp, screen-32) / 高さ 52dp | bg: `state.success.50` / border-left 4px `state.success.500` / icon `CheckCircle2` |
| 警告（範囲外測定値）| `FRG-031 Banner warning` | 確認まで継続 | Header 直下（Sticky）| 高さ 48dp / 幅 100% | bg: `state.warning.50` / border-bottom 2px `state.warning.500` / icon `AlertTriangle` |
| エラー（API 失敗）| `FRG-017 Snackbar danger` | 5 秒 / 手動閉 | 画面下部中央 / 16dp 上 | 幅: min(360dp, screen-32) / 高さ 52dp | bg: `neutral.900` / text: white / icon `XCircle` |
| アンドン発報中 | AndonOverlay（専用）| 確認まで継続 | 全画面フルオーバーレイ | 100% × 100% | bg: `state.andon.bg` / text: `state.andon.text` / `shadow.popup` |
| Emergency Mode | `CMP-HA-006 OfflineBanner emergency` | 切断中継続 | 画面最上部 Sticky / SafeArea 下 | 高さ 32dp | bg: `state.andon.base` / text: white / icon `WifiOff` |

### 4-2. フィードバックのモーション仕様

| コンポーネント | 出現 | 消滅 | reduce-motion |
|---|---|---|---|
| Toast | 上から 12dp スライドイン 200ms `easing.decelerated` | 上へ 12dp スライドアウト 150ms `easing.accelerated` | フェードのみ |
| Snackbar | 下から 12dp スライドイン 200ms | 下へスライドアウト 150ms | フェードのみ |
| Banner | 上から展開 200ms（高さアニメーション）| 上へ折りたたみ 150ms | フェードのみ |
| AndonOverlay | `opacity(0→1)` 80ms | `opacity(1→0)` 200ms | 同左 |

## 5. ページ遷移・トランジション規定

### 5-1. アプリ別トランジション

| アプリ | 方向 | 遷移アニメーション | Duration |
|---|---|---|---|
| ハンディ APP（順進）| 前へ進む | 右→左スライド: `translateX(100%→0)` + 退場 `translateX(0→-30%)` 同時 | 200ms |
| ハンディ APP（逆進）| 前へ戻る | 左→右スライド: 逆方向 | 240ms |
| ハンディ APP（副流: アンドン / 不適合）| モーダル的 | 下から展開: `translateY(100%→0)` | 320ms |
| マスタメンテ APP | ハブ→スポーク | フェード + 8px Y シフト | 200ms |
| マスタメンテ APP | スポーク→ハブ | フェードのみ（戻り感を示す）| 160ms |
| 管理コンソール | タブ・メニュー切替 | フェードのみ（データ密度高・動きを最小化）| 160ms |

### 5-2. ナビゲーション部品のアニメーション

| 部品 | アニメーション |
|---|---|
| HA ProgressBar の進行 | 左→右 fill 200ms `easing.standard`（Step 完了時）|
| サイドバーアクティブ | 左ボーダー 3px が `brand.accent.500` で 120ms 色変化 |
| パンくずリスト更新 | フェード 100ms |
| Drawer 開閉 | 右から `translateX(100%→0)` 320ms `easing.decelerated` |

## 6. 操作安全性・三角配置禁止規定

### 6-1. 三角配置の禁止

主 CTA・破壊的操作ボタン・緊急操作（Andon）を画面上で三角形に配置することを禁止する。

**禁止パターン（旧設計の問題）**: 「中断ボタン左上」「Andon 右下」「Step 完了 中央下」の三角配置。グローブ着用・振動環境での誤タップリスクが極めて高い。

**許可パターン（新設計）**:
```
[ProgressBar + 中断]        ← 上部 Header（左端 secondary ボタン）
[StepCard コンテンツ]       ← flex メイン領域
[Step 完了 CTA]             ← FooterCTA（全幅・中央）
                      [Andon] ← FooterCTA 領域外・Floating（右下 shadow.2）
```

Step 完了 CTA と Andon Button の **最小物理距離: 24dp**（8pt × 3）。
中断ボタンと Step 完了 CTA の **最小物理距離: Header 全域分離**（同一 z-layer に置かない）。

### 6-2. 破壊的操作のキープアウト

| 操作の種類 | 最低物理距離 | 確認方式 |
|---|---|---|
| 廃止（CMP-MA 系）| 他 CTA から 48dp 以上 | 二段確認モーダル（`FRG-014 Modal destructive`）|
| 全件再投入（SCR-MC-007）| フッター CTA を専有（他操作との同居禁止）| 二段確認 + 入力チャレンジ |
| ログアウト（SCR-HA-015）| 設定リスト末尾・ 16dp 以上の Divider を挟む | 確認ダイアログ |
| アンドン（CMP-HA-008）| Step 完了 CTA から 24dp 以上 / Floating で z-layer 分離 | ホールド 300ms または誤防止カバー（実装注: 詳細設計で確定）|

---

**本節で確定した方針**
- **1画面1タスク・現在地常時表示・誤操作ゼロの 5 原則をアプリ別のナビゲーション方式に落とし込み、全 35 画面の情報設計基準を確定した。**
- **ハンディ APP はタスクフロー型（線形ウィザード）、マスタメンテ APP はハブ＆スポーク型、管理コンソールはサイドバー型の各ナビゲーション方式を確定した。**
- **エラー・状態フィードバックを 5 パターン（成功/警告/エラー/アンドン/Emergency）の視覚仕様（位置・サイズ・トークン・モーション）込みで確定した。**
- **アプリ別ページトランジション規定（HA スライド / MA フェード+Y / MC フェードのみ）を確定した。**
- **三角配置の禁止と破壊的操作のキープアウト距離（主 CTA〜Andon: 24dp 以上）を確定した。**
