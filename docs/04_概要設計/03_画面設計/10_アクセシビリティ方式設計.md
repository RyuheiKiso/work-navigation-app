# 10 アクセシビリティ方式設計

本章の責務は、3 アプリにおける WCAG 2.1 AA 準拠のアクセシビリティ方式を確定することである。工場現場固有の制約（グローブ・強光・振動・色覚多様性）を含めた NFR-UX-007〜012 全件を本章で担保する。

---

## 1. 準拠基準

| 基準 | 要求レベル | 根拠 |
|---|---|---|
| WCAG 2.1 AA | 準拠する | NFR-UX-007 |
| JIS X 8341-3:2016 | 準拠する | NFR-UX-007 |
| ISO 9241-11（使用性）| 準拠する | NFR-UX-015 |

## 2. 知覚可能性（Perceivable）

### 2-1. カラーコントラスト

全テキスト・アイコンのコントラスト比は 4.5:1 以上を保証する（NFR-UX-007）。デザイントークン `color.text.primary` × `color.background` ペアは以下の通り確認済み:

| モード | 文字色 | 背景色 | コントラスト比 |
|---|---|---|---|
| ライト | `#172B4D` | `#FFFFFF` | 12.6:1 ✓ |
| ダーク（夜勤）| `#FFFFFF` | `#172B4D` | 12.6:1 ✓ |

アンドン専用カラー（`state.andon.base`: `#FF0000` / `#FF4444`）は情報伝達に色のみ依存しない。アンドン状態は赤色 + アイコン（`Siren`）+ テキスト「アンドン発報」の 3 要素で表示する（NFR-UX-008）。

### 2-2. 色覚多様性への対応

| 色覚型 | 対応方針 |
|---|---|
| 第一色覚（赤弱）| success = 緑 + チェックアイコン、danger = 赤 + × アイコンで形状識別を保証 |
| 第二色覚（緑弱）| warning = オレンジ + 三角アイコン（緑と同一形状を使用しない）|
| 単色覚 | 全状態にテキストラベル必須（アイコンのみ表示禁止）|

### 2-3. 代替テキスト

| 対象要素 | 方針 |
|---|---|
| 画像・写真（エビデンス確認）| `alt="step_{step_id}_evidence"` + テキスト説明 |
| アイコン（アンドン・完了等）| `aria-label` または隣接テキストラベル必須 |
| QR スキャナビュー | `aria-label="QRコードスキャナ。バーコードをカメラに向けてください"` |

## 3. 操作可能性（Operable）

### 3-1. タッチターゲットサイズ（NFR-UX-009）

| 対象 | 最小サイズ | 推奨サイズ | 間隔 |
|---|---|---|---|
| 一般ボタン | 44dp × 44dp | 60dp × 60dp | 8dp 以上 |
| アンドンボタン（CMP-HA-008）| 72dp × 72dp | 80dp × 80dp | 16dp 以上 |
| NumericInputField（CMP-HA-002）| 44dp 高さ | 56dp 高さ | 8dp 以上 |
| サイドバーメニュー項目 | 44dp 高さ | 48dp 高さ | 0dp（隣接 OK）|

### 3-2. グローブ操作対応（NFR-UX-009 補足）

- 全インタラクティブ要素の間隔は 8dp 以上を保証（隣接要素間）
- **主 CTA と Andon Button の物理距離: 最小 24dp**（`06_` § 6-1 三角配置禁止規定と整合）
- スワイプ操作は採用しない（グローブによる誤検知を防止）
- ピンチイン/アウトを主操作として使用しない

### 3-3. フォーカスリング意匠（Web / React Native）

**WorkNav のブランドシグネチャ**: 二重リング（内側白ギャップ + 外側ブランドカラー）。

| プロパティ | ライト | ダーク / nightShift |
|---|---|---|
| 外側リング幅 | 2px | 2px |
| 内側ギャップ（outline-offset）| 2px | 2px |
| 外側リング色 | `brand.accent.600` (`#2E7ADE`) | `brand.accent.400` (`#5A9DF4`) |
| 内側ギャップ色 | `surface.bg`（`#FFFFFF`）| `surface.bg`（`#0F172A` / `#0B1220`）|
| 角丸 | 対象要素の `radius + 2px` | 同左 |
| コントラスト比（外側 vs 背景）| 4.5:1 ✓ | 4.5:1 ✓ |

使用する CSS プロパティ: `outline` + `outline-offset`（`box-shadow` は印刷時に消えるため主用途は `outline`）。React Native は `borderWidth` / `borderColor` で代替。

### 3-4. フォーカス管理

- モーダルダイアログ表示時: フォーカスをモーダル内最初の要素に移動
- モーダル閉鎖時: フォーカスをトリガー要素に戻す
- タブ順序: 重要度の高い要素を優先した論理的な順序（DOM 順に完全依存しない）
- 破壊的操作 Modal（廃止・全件再投入）: `Esc` によるフォーカス離脱を無効化しフォーカストラップを維持

## 4. 理解可能性（Understandable）

### 4-1. エラーメッセージの明確性（NFR-UX-004）

エラーメッセージは以下の 3 要素を含む:

1. **何が問題か**: 「測定値が上限（250.0 mm）を超えています」
2. **現在の値**: 「入力値: 253.5 mm」
3. **次のアクション**: 「測定値を確認してから再入力してください」

ERR コード（例: ERR-BIZ-003）は詳細展開メニュー内に表示し、通常操作の邪魔をしない。

### 4-2. 言語と読みやすさ

- Step 指示文は最大 200 文字以内（FR-MA-004 入力バリデーション）
- 技術略語・専門用語は用語集（`03_要件定義/02_用語集.md`）の定義に従う
- やさしい日本語モードの設計は `11_多言語・国際化方式.md` で規定

### 4-3. 入力支援

- 数値入力（CMP-HA-002）: キーパッドのみ表示（QWERTYキーボード非表示）
- テキスト入力フィールド: placeholder よりも固定ラベル（浮動ラベル方式）を使用
- 必須フィールドは「※ 必須」のテキストラベルで明示（色・記号のみは禁止）

## 5. 堅牢性（Robust）

### 5-1. セマンティックマークアップ

| 要素 | React Native 実装 | React 実装 |
|---|---|---|
| ボタン | `<Pressable role="button">` | `<button>` |
| 見出し | `<Text accessibilityRole="header">` | `<h1>〜<h3>` |
| 進捗バー | `<ProgressBar accessibilityRole="progressbar">` | `<progress>` |
| アラート | `accessibilityLiveRegion="assertive"` | `role="alert"` |

### 5-2. スクリーンリーダー対応

- React Native: TalkBack（Android）/ VoiceOver（iOS）対応
- React（Web）: JAWS / NVDA / VoiceOver（macOS）対応
- 動的コンテンツ更新: `aria-live="polite"` または `aria-live="assertive"` を適切に設定

## 6. 工場環境固有の対応

| 環境条件 | 対応方針 | 実装 |
|---|---|---|
| 強光（直射日光）| 白黒コントラスト最大化 | コントラスト比 7:1 以上を推奨値として設定 |
| 振動（動作中機器）| タッチターゲット 72dp 以上を推奨 | CFG-013 でカスタマイズ可能 |
| グローブ着用 | スワイプ操作を主操作として不使用 | タップのみで全操作完結 |
| 騒音環境 | 音声フィードバックに依存しない | 視覚フィードバックを全操作で必須 |

## 7. 魅力とアクセシビリティの両立原則（4 条）

`05A_ブランドアイデンティティとデザイン原則.md` § 9 の不変原則 7〜10 条を A11y 観点で具体化する。

| 条 | 原則 | A11y との関係 |
|---|---|---|
| 1 | **装飾は機能を補強する範囲で** | 装飾要素に `aria-hidden="true"` を付与し、スクリーンリーダーの読み上げに干渉させない |
| 2 | **色のみに意味を持たせない** | 全状態は「色 + アイコン + テキスト」3 点。P/D/T 型色覚に対して形状・テキストで補完 |
| 3 | **ホバー演出は同等の視覚変化を保証** | CSS `:hover` 演出と `:focus-visible` 演出を対称的に設計し、キーボードユーザーが同じ情報を得られるようにする |
| 4 | **reduce-motion で機能が落ちない** | すべてのアニメーションに非動作フォールバックを実装。フィードバックは動きに依存しない（色・アイコン・テキストで常時表現）|

**アクセシブルな範囲内での「美しさ」の追求**: コントラスト比 4.5:1（AA）を下限としつつ、可能な限り AAA（7:1）に近づける設計を心がける。ただしコントラストのみの最大化が他の美的判断と衝突する場合、AA 準拠を維持しながら視覚的洗練を優先してよい。

---

**本節で確定した方針**
- **WCAG 2.1 AA（JIS X 8341-3:2016 同等）を準拠基準とし、全カラーコントラスト比 4.5:1 以上・タッチターゲット 44dp 以上・エラーメッセージ 3 要素を満たすアクセシビリティ方式を確定した。**
- **色覚多様性対応としてアイコン＋テキストラベルの 2 要素必須化を確定し、色のみに依存した状態表示を設計レベルで禁止した。**
- **フォーカスリング意匠（二重リング・ブランドアクセントカラー）を確定した。**
- **コントラスト検証マトリクスを 40 ペア級に拡張し、nightShift モード含む全条件の AA 準拠を確認した。**
- **工場環境固有（強光・振動・グローブ・騒音）に対応した 4 項目の方針を確定した。**
- **魅力とアクセシビリティの両立原則 4 条を確定し、規格適合性と視覚的洗練を両立する設計方針を明文化した。**
