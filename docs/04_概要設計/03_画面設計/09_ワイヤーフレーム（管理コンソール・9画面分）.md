# 09 ワイヤーフレーム（管理コンソール・9 画面分）

本章の責務は、管理コンソール（SCR-MC-001〜009）の論理レイアウトを確定することである。管理コンソールは system_admin・quality_admin ロールが PC ブラウザから操作する React Web アプリであり、全画面共通のサイドバーナビゲーション（240px 固定幅）とメインコンテンツエリアで構成される。各画面のレイアウト・配置コンポーネント・ユーザーインタラクションを設計する。ASCII 図表示は禁止し、全図は `img/` 配下の draw.io / SVG ファイルを参照する。

---

## 1. 共通レイアウト構造

管理コンソール全 9 画面で共通するレイアウト骨格を以下に定義する。

| 領域 | 幅 / 高さ | 内容 |
|---|---|---|
| サイドバー（左固定）| 240px 固定 | ロゴ・メニュー項目（全 9 画面）・ログアウトボタン |
| メインコンテンツエリア | 残余幅 | 画面ごとのコンテンツ |
| ページヘッダー | メインエリア内上部 | 画面タイトル・パンくずリスト・ユーザー情報 |

### サイドバー メニュー項目

| 順序 | メニューラベル | 遷移先 SCR | アクセスロール |
|---|---|---|---|
| 1 | ダッシュボード | SCR-MC-001 | system_admin / executive |
| 2 | ユーザー管理 | SCR-MC-002 | system_admin |
| 3 | ロール / スキル管理 | SCR-MC-003 | system_admin |
| 4 | 監査ログ | SCR-MC-004 | quality_admin / system_admin |
| 5 | XES エクスポート | SCR-MC-005 | quality_admin / system_admin |
| 6 | バックアップ状況 | SCR-MC-006 | system_admin |
| 7 | DLQ 監視 | SCR-MC-007 | system_admin |
| 8 | ハッシュチェーン検証 | SCR-MC-008 | quality_admin / system_admin |
| 9 | 帳票出力 | SCR-MC-009 | quality_admin / system_admin |

アクセスロールが付与されていないメニュー項目は非表示とする（RBAC ガード: FR-MA-014 / FR-MA-015）。現在アクティブなメニュー項目はハイライト表示し、ナビゲーション原則 §06（現在地の常時表示）を担保する。

### 2-1. SCR-MC-001 運用ダッシュボード

図: `img/fig_des_wire_mc_001_dashboard.{drawio,svg}` を参照。

#### レイアウト構成

メインエリアを縦 3 ブロックに分割する。

| ブロック | 内容 |
|---|---|
| ブロック A（上段）| SLI ゲージ 3 件を横並び（CMP-MC-001）|
| ブロック B（中段）| エラーバジェット残量プログレスバー + 期間ラベル |
| ブロック C（下段）| アラートリスト（最大 10 件・スクロール可）|

#### 配置コンポーネント

| CMP-ID | 物理名 | 配置ブロック | 表示内容 |
|---|---|---|---|
| CMP-MC-001 | SliGauge | ブロック A | API P95 レイテンシ・可用性（目標 99.5%）・オフライン同期率の 3 ゲージ |
| — | ErrorBudgetBar | ブロック B | 残エラーバジェット（%）・消費速度・計測期間（直近 30 日）|
| — | AlertList | ブロック C | アラート種別アイコン（色 + 記号）・発生日時・メッセージ・対応状態バッジ |

#### SLI 指標定義（ブロック A）

| ゲージ番号 | 指標 | 目標値 | OPS 要件 |
|---|---|---|---|
| ゲージ 1 | API P95 レイテンシ | ≤ 500ms | OPS-036 |
| ゲージ 2 | サービス可用性 | 99.5% | OPS-037 |
| ゲージ 3 | オフライン同期成功率 | ≥ 99.0% | OPS-038 |

各 SliGauge は円弧ゲージ形式で数値を中央大表示し、目標値との差分を色分けで示す（達成: `color.success` / 警告: `color.warning` / 未達: `color.danger`）。色のみに依存せず記号（✓ / ! / ✗）を併記する（NFR-UX-008）。

#### ユーザーインタラクション

| 操作 | 結果 |
|---|---|
| アラートリスト行クリック | アラート詳細モーダルを表示 |
| ゲージクリック | 対応する指標の時系列グラフをモーダル表示 |
| 「過去データを見る」リンク | 期間セレクタを展開し過去 SLI を再表示 |
| サイドバーメニュー切替 | 対象画面へ遷移（TRN-042〜050）|

**本節で確定した方針**
- **管理コンソール全 9 画面（SCR-MC-001〜009）の論理レイアウトをサイドバー型（240px 固定）+ メインコンテンツエリアの共通骨格に統一し、全メニューをサイドバーにハイライト常時表示することで現在地の常時可視性（NFR-UX-003）を担保した。**
- **SCR-MC-007（DLQ 監視）の再投入ボタンには必ず確認ダイアログを挟み、SCR-MC-008（ハッシュチェーン検証）の破断アラートは色 + テキスト + アイコンの三重表現で通知し、誤操作ゼロ（NFR-UX-004）と色覚多様性対応（NFR-UX-008）を両立した。**
- **CMP-MC-001（SliGauge）・CMP-MC-002（DlqMonitorTable）・CMP-MC-003（HashChainVerifyResult）の 3 コンポーネントを各担当画面に配置し、OPS-036〜053 の可観測性要件を UI レベルで担保した。**
