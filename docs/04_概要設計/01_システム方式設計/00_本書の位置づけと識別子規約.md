# 00 本書の位置づけと識別子規約

本章の責務は、システム方式設計サブ（01_システム方式設計）が担当する IPA 共通フレーム 2013 の 2.3 タスク・カバーする要件 ID の範囲・本サブ固有の識別子規約を確定することである。本章に記述した責務境界・識別子割り当てを後続 4 章（§01〜§04）が遵守する。

## 3. 本サブで使用する識別子

### 3-1. SUB（論理サブシステム）

本サブが採番する SUB 識別子（付録/99 の SUB セクションで管理）:

```
SUB-001: フィールド層（Handy APP）
SUB-002: サーバー層（Backend + DB）
SUB-003: クライアント層（Web APP）
```

### 3-2. NODE（物理ノード）

本サブが採番する NODE 識別子（付録/99 の NODE セクションで管理）:

```
NODE-001: Active サーバー（Windows Server 2022）
NODE-002: Standby サーバー（Windows Server 2022）
NODE-003: NAS（ネットワークアタッチトストレージ）
NODE-004: Wi-Fi AP（工場 LAN 側）
NODE-005: L2/L3 スイッチ（管理 LAN 側）
```

### 3-3. ENV-ITEM（環境構成項目）

本サブが採番する ENV-ITEM 識別子（付録/99 の ENV-ITEM セクションで管理）:

```
ENV-ITEM-001: Windows Server 2022 LTSC（OS）
ENV-ITEM-002: WSL2（仮想化層）
ENV-ITEM-003: Docker Engine 26.x（コンテナランタイム）
ENV-ITEM-004: Docker Compose v2（オーケストレーション）
ENV-ITEM-005: IIS 10（Web サーバー・リバースプロキシ）
ENV-ITEM-006: PostgreSQL 16+（RDB）
ENV-ITEM-007: Rust Edition 2024（バックエンド言語）
ENV-ITEM-008: tokio（非同期ランタイム）
ENV-ITEM-009: axum（Web フレームワーク）
ENV-ITEM-010: React Native 0.74+（モバイルフレームワーク）
ENV-ITEM-011: React 18+ TypeScript strict（Web フレームワーク）
ENV-ITEM-012: SQLite + SQLCipher（ローカルストレージ）
```

### 3-4. ADR（設計判断記録）

本サブが発行する ADR 識別子（付録/05 に記録）:

```
ADR-SYS-001: Offline-First を基本設計方針として採用する
ADR-SYS-002: Active-Standby 手動切替を採用し自動フェイルオーバーを採用しない
ADR-SYS-003: WSL2 上の Docker を採用し別筐体の Linux VM を採用しない
ADR-SYS-004: OT ゾーンとのネットワーク接続を永続禁止とする（P4 原則）
ADR-SYS-005: IIS をリバースプロキシとして採用しコンテナ直接外部公開を禁止する
```

**本節で確定した方針**
- IPA 2.3 の全 11 タスクに対する本サブのカバー状態（完全カバー 6 タスク・委譲 5 タスク）を確定し、責務境界を明示する。
- NFR-AVL-001〜010・NFR-PRF-001〜010・NFR-OPS-036〜053 の計 34 件を本サブの設計根拠要件として確定し、新規要件 ID の採番を本サブ内で禁止する。
- SUB-001〜003・NODE-001〜005・ENV-ITEM-001〜012・ADR-SYS-001〜005 の識別子体系を確定し、付録/99 設計識別子採番台帳で一元管理する。
