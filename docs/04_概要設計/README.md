# 04_概要設計

IPA 共通フレーム 2013（SLCP-JCF2013）「2.3 システム方式設計プロセス」および「2.4 ソフトウェア方式設計プロセス」に準拠した概要設計成果物の格納場所。

上流の企画（`02_企画/`）と要件定義（`03_要件定義/`）の成果物を受けて「どう作るか」の骨格を確定する段階である。

---

## 位置づけ

| 層 | プロセス | 言語 | 問い | 文書 |
|---|---|---|---|---|
| 第一層 | 企画プロセス（構想） | べき論 | Why | `02_企画/システム化構想/` |
| 第二層 | 企画プロセス（計画） | する論（確定命題） | What/How-Frame | `02_企画/システム化計画/` |
| 第三層 | 要件定義プロセス | する論（検証可能命題） | What 詳細 | `03_要件定義/` |
| **第四層** | **システム方式設計 / ソフトウェア方式設計** | **する論（設計命題）** | **How 骨格** | **本書（04_概要設計/）** |
| 第五層 | ソフトウェア詳細設計 | する論（実装命題） | How 詳細 | `05_詳細設計/` |

## 閲覧順と依存関係

1. **まず本書を読む**: `README.md`（概要把握）
2. **全体規約を読む**: `00_本書の位置づけ` → `03_識別子体系` → `04_トレーサビリティ枠組み`

5. **UI・帳票を読む**: `03_画面設計` → `06_帳票設計`
6. **横断品質を読む**: `07_セキュリティ方式設計` → `01_システム方式設計`
7. **運用・移行・テストを読む**: `08_運用方式設計` → `09_移行方式設計` → `10_テスト方式設計`

## 共通執筆規約

本書全章（md ファイル）に適用される規約：

- 規制表記: **「準拠する」「対応する」「対象外と判断する」** の三段階のみ
- 禁止表現: 「考慮する」「検討する」「参考にする」「可能性がある」「場合がある」「なるべく準拠する」「準拠を目指す」「適宜対応する」
- 章末必須節: 「**本節で確定した方針**」（太字・3行以内）
- 章末必須節: 「## 参照業界分析」（必須/関連の区分）
- 図: アスキー表現禁止。drawio-authoring スキルで `.drawio` 作成 → `.svg` エクスポート。`img/` 配下に両方配置
- 図の命名: `fig_des_{area}_{topic}.{drawio,svg}`
- 図の参照: 「図: fig_des_xxx（img/ 配下）を参照」形式
