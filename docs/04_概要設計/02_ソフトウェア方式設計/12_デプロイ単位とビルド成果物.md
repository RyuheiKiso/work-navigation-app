# 12 デプロイ単位とビルド成果物

本章の責務は、Docker Compose による配置構成・IIS リバースプロキシ・各アプリのビルド成果物・バージョニング規約を確定することである。

---

## 1. Docker Compose サービス構成

```yaml
# docker-compose.yml（概念構成）
services:
  backend:
    build: ./crates
    image: wnav-backend:${VERSION}
    ports: ["127.0.0.1:3000:3000"]
    environment:
      DATABASE_URL: ${DATABASE_URL}
      JWT_SECRET_PATH: /run/secrets/jwt_rsa_private
    volumes:
      - wnav-files:/var/wnav/files  # NAS マウント（証拠ファイル）
    depends_on: [postgres]
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    volumes:
      - wnav-pg-data:/var/lib/postgresql/data
      - ./initdb:/docker-entrypoint-initdb.d
    restart: unless-stopped

  # Scheduler（cron ジョブ用別コンテナ）
  scheduler:
    image: wnav-backend:${VERSION}
    command: scheduler
    environment: (同 backend)
    depends_on: [postgres]
    restart: unless-stopped

volumes:
  wnav-pg-data:
  wnav-files:
```

---

## 2. IIS リバースプロキシ構成

```
クライアント（HTTPS）
  → IIS（443）
    ├── /api/*  → backend:3000（URL Rewrite）
    ├── /master/   → dist/master/index.html（静的ファイル配信）
    └── /console/  → dist/console/index.html（静的ファイル配信）
```

フロントエンド（FE-MA / FE-MC）は Vite で静的ビルドし、IIS の `wwwroot` に配置する。

---

## 3. ビルド成果物とバージョニング

| 成果物 | 形式 | バージョニング | 配置場所 |
|---|---|---|---|
| wnav-backend | Docker image | `v{SemVer}` タグ | `registry.wnav.local/backend:{tag}` |
| wnav-master-app | 静的 HTML/JS/CSS | `v{SemVer}` フォルダ | `C:\inetpub\wwwroot\master\` |
| wnav-console-app | 静的 HTML/JS/CSS | `v{SemVer}` フォルダ | `C:\inetpub\wwwroot\console\` |
| wnav-handy-app | Android APK / iOS IPA / Windows MSIX | `v{SemVer}` | MDM 配信または手動インストール |
| DB マイグレーション | SQL files（sqlx migrate）| migration タイムスタンプ | `src/backend/migrations/` |

---

## 4. SemVer バージョニング規約

`{メジャー}.{マイナー}.{パッチ}+{ビルドメタ}` 形式。

| バージョン上げ | 条件 |
|---|---|
| メジャー | API breaking change / DB スキーマ非互換変更 |
| マイナー | 新機能追加・後方互換 API 追加 |
| パッチ | バグ修正・パフォーマンス改善 |
| ビルドメタ | `+{YYYY.MM.DD.N}` でリリース日時を付加（参考情報）|

---

**本節で確定した方針**
- **Docker Compose で backend・postgres・scheduler の 3 サービスを構成し、IIS がリバースプロキシとして API と静的ファイルを振り分ける配置を確定した。**
- **ハンディ APP は MDM（モバイルデバイス管理）配信を推奨し、APK/IPA/MSIX の 3 プラットフォームビルドを成果物として確定した。**
- **SemVer でバージョン管理し、Breaking Change はメジャーバージョンアップと合わせて Git タグを付与する規約を確定した。**
