# 12 デプロイ単位とビルド成果物

本章の責務は、Docker Compose による配置構成・IIS リバースプロキシ・各アプリのビルド成果物・バージョニング規約を確定することである。

---

## 1. Docker Compose サービス構成

バックエンドは DB ロール物理分離・独立可用性・セキュリティ境界確保のために **2 サービスに分割**する。旧 `scheduler` サービスは廃止し、BAT（バッチ処理）は各 API バイナリの常駐 tokio task（Outbox Consumer 等）に内包する。

```yaml
# docker-compose.yml（概念構成）
services:
  terminal-api:
    build:
      context: ./crates
      target: wnav_terminal_api
    image: registry.wnav.local/terminal-api:${VERSION}
    ports: ["127.0.0.1:8080:8080"]
    environment:
      # app_event_insert + app_read のみ保有（app_write は設定しない）
      DB_EVENT_INSERT_URL: postgres://app_event_insert:${EVENT_INSERT_PASSWORD}@postgres:5432/wnav
      DB_READ_URL: postgres://app_read:${READ_PASSWORD}@postgres:5432/wnav
      JWT_PUBLIC_KEY_PATH: /run/secrets/jwt_rsa_public
    volumes:
      - wnav-files:/var/wnav/files  # NAS マウント（証拠ファイル）
    depends_on: [postgres]
    restart: unless-stopped

  master-api:
    build:
      context: ./crates
      target: wnav_master_api
    image: registry.wnav.local/master-api:${VERSION}
    ports: ["127.0.0.1:8081:8081"]
    environment:
      # app_write + app_read のみ保有（app_event_insert は設定しない）
      DB_WRITE_URL: postgres://app_write:${WRITE_PASSWORD}@postgres:5432/wnav
      DB_READ_URL: postgres://app_read:${READ_PASSWORD}@postgres:5432/wnav
      JWT_PUBLIC_KEY_PATH: /run/secrets/jwt_rsa_public
      JWT_SECRET_PATH: /run/secrets/jwt_rsa_private
    depends_on: [postgres]
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    volumes:
      - wnav-pg-data:/var/lib/postgresql/data
      - ./initdb:/docker-entrypoint-initdb.d
    restart: unless-stopped

volumes:
  wnav-pg-data:
  wnav-files:
```

## 2. IIS リバースプロキシ構成

端末（工場 LAN）からのリクエストは `terminal-api:8080` へ、管理 PC（管理 LAN）からのリクエストは `master-api:8081` へ振り分ける。`/api/v1/auth/*` は両バイナリに存在し、JWT の `aud` クレームで利用者種別（ハンディ端末 / 管理ユーザー）を判別する。

```
クライアント（HTTPS）
  → IIS（443）
    ├── /api/v1/events/*         → terminal-api:8080（URL Rewrite）
    ├── /api/v1/evidence/*       → terminal-api:8080
    ├── /api/v1/sync/*           → terminal-api:8080
    ├── /api/v1/iqc/inspect/*    → terminal-api:8080
    ├── /api/v1/rework/execute/* → terminal-api:8080
    ├── /api/v1/auth/*           → terminal-api:8080 または master-api:8081（aud クレームで判別）
    ├── /api/v1/sop/*            → master-api:8081
    ├── /api/v1/master/*         → master-api:8081
    ├── /api/v1/approval/*       → master-api:8081
    ├── /api/v1/users/*          → master-api:8081
    ├── /api/v1/audit/*          → master-api:8081
    ├── /api/v1/reports/*        → master-api:8081
    ├── /api/v1/ops/*            → master-api:8081
    ├── /api/v1/trace/*          → master-api:8081
    ├── /api/v1/alerts/*         → master-api:8081
    ├── /api/v1/capas/*          → master-api:8081
    ├── /api/v1/kaizen-proposals/* → master-api:8081
    ├── /api/v1/iqc/judge/*      → master-api:8081
    ├── /api/v1/iqc/concession/* → master-api:8081
    ├── /api/v1/rework/disposition/* → master-api:8081
    ├── /api/v1/scrap/*          → master-api:8081
    ├── /api/v1/return/*         → master-api:8081
    ├── /master/                 → dist/master/index.html（静的ファイル配信）
    └── /console/                → dist/console/index.html（静的ファイル配信）
```

フロントエンド（FE-MA / FE-MC）は Vite で静的ビルドし、IIS の `wwwroot` に配置する。

## 3. ビルド成果物とバージョニング

| 成果物 | 形式 | バージョニング | 配置場所 |
|---|---|---|---|
| wnav-terminal-api | Docker image | `v{SemVer}` タグ | `registry.wnav.local/terminal-api:{tag}` |
| wnav-master-api | Docker image | `v{SemVer}` タグ | `registry.wnav.local/master-api:{tag}` |
| wnav-master-app | 静的 HTML/JS/CSS | `v{SemVer}` フォルダ | `C:\inetpub\wwwroot\master\` |
| wnav-console-app | 静的 HTML/JS/CSS | `v{SemVer}` フォルダ | `C:\inetpub\wwwroot\console\` |
| wnav-handy-app | Android APK / iOS IPA / Windows MSIX | `v{SemVer}` | MDM 配信または手動インストール |
| DB マイグレーション | SQL files（sqlx migrate）| migration タイムスタンプ | `src/backend/migrations/` |

## 4. SemVer バージョニング規約

`{メジャー}.{マイナー}.{パッチ}+{ビルドメタ}` 形式。

| バージョン上げ | 条件 |
|---|---|
| メジャー | API breaking change / DB スキーマ非互換変更 |
| マイナー | 新機能追加・後方互換 API 追加 |
| パッチ | バグ修正・パフォーマンス改善 |
| ビルドメタ | `+{YYYY.MM.DD.N}` でリリース日時を付加（参考情報）|

---

**本節で確定した方針**
- **Docker Compose で terminal-api・master-api・postgres の 3 サービスを構成し、IIS がリバースプロキシとして各 API バイナリと静的ファイルを振り分ける配置を確定した。旧 scheduler サービスは廃止し、BAT 処理は各 API バイナリの常駐 tokio task に内包する。**
- **terminal-api（ポート 8080）は app_event_insert + app_read のみ、master-api（ポート 8081）は app_write + app_read のみの DB ロールを保有し、2 バイナリによる DB ロール物理分離を確定した。**
- **ハンディ APP は MDM（モバイルデバイス管理）配信を推奨し、APK/IPA/MSIX の 3 プラットフォームビルドを成果物として確定した。**
- **SemVer でバージョン管理し、Breaking Change はメジャーバージョンアップと合わせて Git タグを付与する規約を確定した。**
