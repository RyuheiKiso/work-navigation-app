## 3. 本サブで使用する識別子

### 3-1. MOD（モジュール）

| target | 説明 | 採番範囲 |
|---|---|---|
| `MOD-BE-NNN` | Backend Rust crate / module | BE-001〜 |
| `MOD-FE-HA-NNN` | Handy React Native feature/module | FE-HA-001〜 |
| `MOD-FE-MA-NNN` | Master React feature/module | FE-MA-001〜 |
| `MOD-FE-MC-NNN` | Console React feature/module | FE-MC-001〜 |
| `MOD-SH-NNN` | Shared 共通ライブラリ | SH-001〜 |
| `MOD-IN-NNN` | Infra（DB ロール・Docker・設定）| IN-001〜 |

### 3-2. BAT / MSG / ERR / CFG

本サブが一次採番する識別子:
- `BAT-NNN`: バッチ（§10）
- `MSG-NNN`: メッセージ/トピック（§05）
- `ERR-{layer}-NNN`: エラーコード（§07）
- `CFG-NNN`: 設定パラメータ（§06・§08・§09）

## 4. ALCOA+ × ソフトウェア設計 対応

| ALCOA+ 原則 | ソフトウェア設計での実現手段 |
|---|---|
| Attributable | 全書き込み処理で `worker_id`・`terminal_id` を必須引数として強制（アプリ層バリデーション）|
| Contemporaneous | `timestamp_client` はデバイス時計から取得・`timestamp_server` はサーバー受信時に付与（2 時計の分離）|
| Original | 証拠ファイルの SHA-256 は `MOD-BE`（API 受信時）で計算し、端末での事前計算と突合 |
| Accurate | JSON Logic 評価（§04）・数値バリデーション（§07 ERR-VAL）|
| Complete | ロックステップ進行（FR-NV-004）をドメインサービスで強制（§03）|
| Consistent | ハッシュチェーン更新を WorkEvent INSERT と同一トランザクションで実行（§08）|

---

**本節で確定した方針**
- **本サブは IPA 2.4 の全ソフトウェア方式設計タスクを担当し、86 FR の実装責任を MOD カタログ（§11）でモジュールに配分する。**
- **MOD 識別子 6 種（BE/FE-HA/FE-MA/FE-MC/SH/IN）を確定し、付録/99 採番台帳の MOD セクションで管理する。**
