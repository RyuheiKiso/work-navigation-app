# 00 本書の位置づけと識別子規約

本章の責務は、テスト方式設計サブ（10）全体の IPA 共通フレーム 2013 上の位置を確定し、カバーする要件範囲・他サブとの責任分界・TST-NNN（テストケース）および TC-NNN（テストケースバリアント）識別子の採番規約を権威的に定めることである。本章に記載した規約はテスト方式設計の全章に適用される権威規約であり、下位章での再解釈を禁止する。

---

## 4. DTM M5 / M6 との連携

### 4-1. DTM M5（NFR × 設計章）における本サブの位置

本サブは DTM M5 の検証列を担当する。各 NFR に対して、本サブのテスト方式設計がその非機能要件制約を「検証する章」として登録されることが M5 完全性条件（V-05）の充足に寄与する。

| DTM M5 での本サブの役割 | 内容 |
|---|---|
| NFR-QUA 群（品質保証） | 本サブ 01〜04 章が検証担当章として M5 に登録される |
| NFR-DQ 群（ALCOA+） | 本サブ 06 章が検証担当章として M5 に登録される |
| NFR-PRF 群（性能） | 本サブ 07 章が検証担当章として M5 に登録される |
| NFR-SEC 群（セキュリティ） | 本サブ 07 章が検証担当章として M5 に登録される（07_セキュリティ方式設計との共担） |
| NFR-AVL 群（可用性） | 本サブ 07 章が検証担当章として M5 に登録される |

### 4-2. DTM M6（BR-BUS × API/ERR）との連携

本サブは DTM M6 の各行に対して「統合テストでの BR-BUS 検証テストケース」を紐付ける。全 45 BR-BUS が少なくとも 1 件の統合テストでカバーされることを品質ゲート条件とする。

## 5. 識別子規約

### 5-1. TST-NNN 識別子（テストケース群）

TST-NNN は概要設計レベルのテストケース群識別子である。1 TST-NNN = テスト観点レベル（例: 「Append-only 制約テスト」「P95 ≤ 2000ms 性能テスト」）を表し、具体的な入力・期待値・手順は詳細設計フェーズで確定する（D-TST-001）。

**採番規約:**

| 規則 | 内容 |
|---|---|
| 形式 | `TST-NNN`（3 桁ゼロ埋め） |
| 開始番号 | TST-001 から採番 |
| 欠番禁止 | 採番した番号はスキップしない |
| 再使用禁止 | 廃止した TST-NNN の番号は再使用しない |
| 必須メタデータ | テストレベル（L1〜L4）・対象 FR-NNN/NFR-NNN/BR-BUS-NNN・検証観点・ツール |
| 採番台帳 | `04_概要設計/付録/99_設計識別子採番台帳.md` |

### 5-2. TC-NNN 識別子（テストケースバリアント）

TC-NNN は TST-NNN の下位に位置する個別テストケース（バリアント）識別子である。1 TST-NNN に対して複数の TC-NNN を持つ。TC-NNN の詳細設計は詳細設計フェーズで実施する（D-TST-001）。

**採番規約:**

| 規則 | 内容 |
|---|---|
| 形式 | `TC-NNN`（3 桁ゼロ埋め） |
| 開始番号 | TC-001 から採番 |
| 欠番禁止・再使用禁止 | TST-NNN と同じルール |
| 親識別子 | 各 TC-NNN は必ず 1 つの TST-NNN 親を持つ |
| 詳細設計フェーズで確定 | 詳細設計フェーズ完了まで TC-NNN は採番しない（D-TST-001） |

---

**本節で確定した方針**
- 本サブが担当する上流要件の範囲（NFR-QUA 10 件・NFR-DQ 9 件・FR-AU 6 件・FR 全 86 件・BR-BUS 45 件・TST 上流要件 3 件）を確定し、他サブとの設計責任分界を明示した。
- TST-NNN（テストケース群）および TC-NNN（テストケースバリアント）の採番規約を確定した。TC-NNN の詳細採番は詳細設計フェーズ委任とする。
- DTM M5（NFR × 設計章）および DTM M6（BR-BUS × API/ERR）に対する本サブの検証担当章としての位置を確定した。
