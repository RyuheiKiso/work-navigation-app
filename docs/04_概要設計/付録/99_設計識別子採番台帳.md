# 99 設計識別子採番台帳

概要設計フェーズで新設する全設計識別子の採番状況を一元管理する台帳。

各識別子の追加・廃止時にはこの台帳を更新する。各セクションで「次採番値」を管理することで欠番を防止する。

要件識別子（BR/FR/NFR/UC/EN/IF/RP/SCR 等）の採番台帳は `03_要件定義/付録/99_採番台帳.md` が権威台帳であり、本台帳は参照のみ。

---

## 凡例

| フィールド | 内容 |
|---|---|
| ID | 識別子（例: TBL-001） |
| 物理名 | テーブル名・モジュール名・メトリクス名等 |
| 状態 | Draft / In Review / Agreed / Frozen / [廃止: YYYY-MM-DD 廃止理由] |
| 次採番値 | 次回採番する番号。更新時に必ず書き換えること |
