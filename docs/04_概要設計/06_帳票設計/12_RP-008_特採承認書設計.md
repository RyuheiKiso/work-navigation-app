# 12 RP-008 特採承認書設計

本章は帳票 RP-008「特採承認書」のレイアウト・データフィールド・生成規則を確定する。RP-008 は IQC 不合格判定後の特採（CONCESSION）承認時に発行し、電子サイン・有効範囲・有効期限を法的根拠として記録する。

---

## 1. 帳票概要

| 項目 | 内容 |
|---|---|
| 帳票 ID | RP-008 |
| 帳票名 | 特採承認書 |
| 準拠規格 | ISO 9001:2015 §8.7（不適合なアウトプットの管理）/ IEC 62988 特採手続き |
| 主目的 | 品質担当による特採決裁の法的記録（電子サイン・有効期限・適用範囲） |
| 副目的 | 特採ロット使用時の警告バナー表示根拠（FR-IQ-010）|
| 出力形式 | PDF/A-3 |
| テンプレート ID | TPL-008 |
| 生成単位 | concession_approvals 1 レコードごとに 1 帳票 |
| 保存先 | NAS（`/reports/rp008/{year}/{month}/`） |
| 保存期間 | 7 年以上 |
| 担当要件 | FR-IQ-010, BR-BUS-037 |

---

## 2. レイアウト構造

### 2-1. Header

| フィールド名 | 表示ラベル |
|---|---|
| approval_id | 特採承認 ID |
| inspection_id | 入荷検査 ID（RP-007 リンク） |
| lot_id | 対象ロット ID |
| material_name | 材料名 |
| supplier_name | 仕入先名 |
| defect_summary | 不適合概要（自由記述） |

### 2-2. 特採条件領域

| フィールド名 | 表示ラベル |
|---|---|
| decision | 判定区分（CONCESSION 固定） |
| reason | 特採理由（TEXT NOT NULL） |
| validity_scope | 有効範囲（JSONB: 適用工程・製品・数量上限） |
| valid_until | 特採有効期限（DATE） |

### 2-3. 電子サイン領域

| フィールド名 | 表示ラベル |
|---|---|
| approver_id_hash | 承認者 ID（品質担当ロール）ハッシュ |
| electronic_sign_id | 電子サイン ID（ElectronicSign FK） |
| approved_at | 承認日時 |

---

## 3. 有効期限アラート機構

特採有効期限（`valid_until`）を超過した場合、スケジュールジョブ（BAT-009 拡張）が `lot_qc_states.qc_status` を CONDITIONAL_PASS → REJECTED に自動遷移させる（FR-IQ-010・§1-7 状態遷移参照）。
