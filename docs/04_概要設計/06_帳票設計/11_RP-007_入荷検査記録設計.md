# 11 RP-007 入荷検査記録設計

本章は帳票 RP-007「入荷検査記録（IQC）」のレイアウト・データフィールド・生成規則を確定する。RP-007 は JIS Z 9015-1（= ISO 2859-1）AQL 抜取検査の法的記録要件に基づき、入荷ロット単位で検査証跡を PDF/A-3 出力する。

---

## 1. 帳票概要

| 項目 | 内容 |
|---|---|
| 帳票 ID | RP-007 |
| 帳票名 | 入荷検査記録 |
| 準拠規格 | JIS Z 9015-1（ISO 2859-1）/ ISO 9001:2015 §8.4.3（外部提供者の管理） |
| 主目的 | 入荷ロットの AQL 抜取検査実施証拠（受入判定根拠・サンプル測定値・判定結果） |
| 副目的 | 仕入先品質管理・ロット追跡（後工程 QR スキャン照合の根拠） |
| 出力形式 | PDF/A-3 |
| テンプレート ID | TPL-007 |
| 生成単位 | incoming_inspections 1 レコード（入荷ロット 1 件）ごとに 1 帳票 |
| 保存先 | NAS（`/reports/rp007/{year}/{month}/`）+ PostgreSQL ハッシュ参照テーブル |
| 保存期間 | 7 年以上（NFR-DQ-010・ISO 9001:2015 §7.5.3 準拠） |
| 担当要件 | FR-IQ-001〜008, BR-BUS-032〜036 |

---

## 2. レイアウト構造

RP-007 は Header・Sampling Plan 情報・Measurement Body・Judgment Footer の 4 部構成とする。

### 2-1. Header（入荷ロット識別領域）

| フィールド名 | 表示ラベル | データ型 | ALCOA+ 原則 |
|---|---|---|---|
| inspection_id | 検査 ID | UUID | O（原本性） |
| lot_id | 入荷ロット ID | UUID | A（帰属可能性） |
| material_code | 材料コード | VARCHAR | A |
| material_name | 材料名 | VARCHAR | L（読取可能性） |
| supplier_code | 仕入先コード | VARCHAR | A |
| supplier_name | 仕入先名 | VARCHAR | L |
| received_at | 受入日時 | TIMESTAMPTZ | C（同時性） |
| inspector_id_hash | 検査員 ID（ハッシュ） | CHAR(64) | A・個人情報保護 |
| lot_quantity | 入荷数量 | INTEGER | AC（正確性） |

### 2-2. Sampling Plan 情報領域

| フィールド名 | 表示ラベル | データ型 |
|---|---|---|
| aql_value | AQL 値 | NUMERIC |
| inspection_level | 検査水準 | VARCHAR |
| sample_size_n | サンプルサイズ（n） | INTEGER |
| accept_number_ac | 合格判定個数（Ac） | INTEGER |
| reject_number_re | 不合格判定個数（Re） | INTEGER |
| severity_state | 検査の厳しさ | VARCHAR |
| sampling_plan_snapshot | AQL 表スナップショット（版番号） | VARCHAR |

### 2-3. Measurement Body（測定値明細）

サンプル 1 個ずつの測定値・合否フラグ・証拠ファイル参照を tabular 形式で出力。

| フィールド名 | 表示ラベル |
|---|---|
| sample_no | サンプル No. |
| measured_value | 測定値 |
| defect_flag | 不良フラグ（○/×） |
| evidence_file_hash | 証拠写真 SHA-256 |

### 2-4. Judgment Footer（判定領域）

| フィールド名 | 表示ラベル |
|---|---|
| defect_count | 不良数 |
| qc_status | 最終判定（PASSED / CONDITIONAL_PASS / SCREENING_REQUIRED / REJECTED） |
| judged_at | 判定日時 |
| quality_admin_sign_hash | 品質担当電子サイン（ハッシュ） |

---

## 3. 生成ロジック

AQL 自動判定結果（FR-IQ-007/008）確定時に BAT-007（document_hash_recorder）がトリガされ TPL-007 を適用して PDF/A-3 を出力する。`qc_status` が PASSED 以外の場合は「不合格ステータス」を赤ハイライトで強調表示する。

---

## 4. 改ざん防止

RP-003（不適合記録）と同一の帳票改ざん防止方式（`doc_hash` SHA-256 + BAT-007 記録）を適用する（`docs/04_概要設計/06_帳票設計/09_帳票改ざん防止技術設計.md` §2 準拠）。
