# 99 前提制約と本書が約束しないこと

本章の責務は、`06_帳票設計` サブの設計境界を明確にし、本書が担保しない事項を宣言することである。

---

## 1. 本書が担保すること

| 担保事項 | 根拠 |
|---|---|
| RP-001〜006 全 6 件の帳票要件を継承・設計した | 01_帳票一覧 |
| 帳票生成パイプライン（5 ステップ）とバッチジョブ（BAT-006〜010）の方式確定 | 02_帳票生成方式 |
| RP-001〜006 各帳票のレイアウト・データソース・生成トリガーの確定 | 03〜08 各章 |
| SHA-256 + PDF デジタル署名 + ハッシュチェーン連結による 3 層改ざん防止方式の確定 | 09_改ざん防止 |
| 7 年以上保管・セキュア廃棄・出力監査ログ・3 エクスポート手段の確定 | 10_保管・廃棄 |
| ALCOA+ 9 原則の帳票設計レベルでの実装担保 | 00_識別子規約・09・10 |

---

## 2. 本書が約束しないこと（対象外と判断する）

### D-RP-001: 個人別生産性帳票

**対象外と判断する。**

個人の作業件数・作業速度・Kaizen 件数等を個人識別情報と紐づけて出力する帳票は、RP-001〜006 のいずれにも含まれない。BR-BUS-029（個人別成果指標の提示禁止）を帳票設計レベルで実装的に担保する。集計は工程・オペレーション・SOP 単位にとどめる。

### D-RP-002: 監視カメラ映像の帳票化

**対象外と判断する。**

監視カメラ映像のキャプチャを帳票として出力する機能は本システムのスコープ外とする（BR-BUS-028: 用途三限定）。写真エビデンス（FR-EV-002）は作業者が主体的に撮影したエビデンス写真のみを対象とする。

### D-RP-003: 外部システム向けの帳票連携仕様

**対象外と判断する。**

ERP・eBR・MES 等の外部システムへの帳票自動連携（API 連携・ファイル転送）は本概要設計の対象外とする。eBR 代替自動昇格は設計フェーズで禁止済み（`01_システム方式設計/99` 参照）。外部連携が必要な場合は将来フェーズの要件として別途設計する。

### D-RP-004: リアルタイムダッシュボード

**対象外と判断する。**

本書の帳票はバッチ生成・手動要求生成の 2 方式に限定する。秒〜分単位のリアルタイム更新が必要な指標の表示は `08_運用方式設計/03_監視・メトリクス・アラート設計.md` の SLI 監視ダッシュボード（SCR-MC-001）が担当する。

### D-RP-005: 帳票レイアウトの実装詳細

**対象外と判断する。**

本書はレイアウト構成（Header/Body/Footer の情報要素・フィールド仕様）を確定するにとどまる。Jinja2 テンプレートの詳細コード・CSS スタイル定義・openpyxl セル操作実装は詳細設計フェーズで確定する。

### D-RP-006: 生産性以外の個人評価帳票

**対象外と判断する。**

スキル評価・人事評価・シフト管理等の個人を主体とした評価帳票は本システムのスコープ外とする。本システムが扱う個人情報は作業記録の帰属確認（ALCOA+ A 原則）のための最小限に限定する。

---

## 3. 前提制約の継承

| 制約 | 出典 |
|---|---|
| PDF/A-3（ISO 19005-3）準拠を全帳票で義務付け | NFR-ALC-008 |
| SHA-256 ハッシュと PDF デジタル署名（PKCS#7）を必須とする | NFR-SEC-007 |
| 個人識別情報はハッシュ変換後のみ帳票に出力する | BR-BUS-029 / NFR-SEC-009 |
| 全帳票を `hash_chain_blocks` テーブルに登録し週次検証対象とする | FR-AU-006 |
| テンプレートファイル変更は master_admin + quality_admin 二段階承認 | BR-BUS-015 |
| アスキー図の使用禁止、drawio + svg での図作成が必須 | `/docs/CLAUDE.md` |

---

**本節で確定した方針**
- **本書（06_帳票設計）が担保する 6 項目と対象外と判断する 6 項目（D-RP-001〜006）を明示し、詳細設計フェーズとの責務境界を確定した。**
- **個人別生産性帳票（D-RP-001）・監視カメラ帳票化（D-RP-002）を技術・設計レベルのスコープ外と宣言し、BR-BUS-028/029 の帳票設計レベルでの担保を確定した。**
- **外部システム連携（D-RP-003）とリアルタイムダッシュボード（D-RP-004）を明示的に対象外とし、将来フェーズへの責務移管を明示した。**
