# 03 RP-001 SOP実行記録設計

本章は帳票 RP-001「SOP 実行記録」のレイアウト・データフィールド・生成規則を確定する。RP-001 は品質監査のエビデンスとしての役割と、XES プロセスマイニングのベースデータとしての役割を同時に担うため、ALCOA+ 原則の全項目を帳票レベルで完全に充足しなければならない。

---

## 1. 帳票概要

RP-001 は SOP（標準作業手順書）の 1 回の実行を単位として生成される電子帳票であり、ISO 9001:2015 に基づく品質監査エビデンスおよびプロセスマイニング（XES 形式の前処理データソース）の双方を目的として設計する。

| 項目 | 内容 |
|---|---|
| 帳票 ID | RP-001 |
| 帳票名 | SOP 実行記録 |
| 主目的 | 品質監査エビデンス（ISO 9001:2015 要求事項 8.1/8.5.1 対応） |
| 副目的 | XES プロセスマイニングの集計ベースデータ（RP-002 へのデータソース） |
| 出力形式 | PDF/A-3（長期保存規格準拠）/ 内部データ参照用 JSON |
| テンプレート ID | TPL-001 |
| 生成単位 | work_executions 1 レコード（SOP 1 回実行）ごとに 1 帳票 |
| 保存先 | NAS（`/reports/rp001/{year}/{month}/`）+ PostgreSQL ハッシュ参照テーブル |
| 保存期間 | 7 年以上（NFR-OPS-035 準拠） |
| 担当要件 | FR-AU-004, NFR-ALC-001〜009, BR-BUS-015, BR-BUS-016 |

### 1-1. 主要な設計制約

| 制約 | 内容 |
|---|---|
| 電子サイン必須 | 全 Step 完了後に quality_admin ロールによる電子サイン（BR-BUS-015） |
| 完了 Step 修正禁止 | 生成後の帳票フィールドは不変。修正時は訂正帳票を別途発行（BR-BUS-016） |
| 作業者匿名化 | 作業者 ID は SHA-256 ハッシュ値のみ表示。氏名・平文 ID は帳票外 |
| XES 互換 | Body の各 Step 行は XES イベントに変換可能な属性構成を保持 |

## 2. レイアウト構造

RP-001 は Header・Body・Footer の 3 部構成とする。各領域の詳細は下図を参照。

> 図 2-1 RP-001 レイアウト構造図（fig_des_report_rp001（img/ 配下）を参照）

### 2-1. Header（帳票識別・対象物識別領域）

Header は帳票の上部固定領域に配置し、帳票の一意性と対象物のトレーサビリティを確保する情報を収録する。

| フィールド名 | 表示ラベル（日本語）| データ型 | 文字数上限 | ALCOA+ 該当原則 |
|---|---|---|---|---|
| company_name | 会社名 | VARCHAR | 100 | L（読み取り可能） |
| factory_name | 工場名 | VARCHAR | 100 | L（読み取り可能） |
| sop_name | SOP 名 | VARCHAR | 200 | A（帰属可能性） |
| sop_version | バージョン | VARCHAR | 20 | O（原本性） |
| product_code | 製品番号 | VARCHAR | 50 | A（帰属可能性） |
| lot_number | ロット番号 | VARCHAR | 50 | A（帰属可能性） |
| worker_id_hash | 作業者 ID（ハッシュ表示）| CHAR(64) | 64 | A（帰属可能性）/ 個人情報保護 |
| started_at | 開始時刻 | TIMESTAMPTZ | — | C（同時性） |
| completed_at | 終了時刻 | TIMESTAMPTZ | — | C（同時性） |
| terminal_id | 使用端末 ID | VARCHAR | 50 | A（帰属可能性） |
| work_execution_id | 実行 ID（帳票番号）| UUID | — | O（原本性）/ 一意識別 |

#### 2-1-1. worker_id_hash の算出方法

```
worker_id_hash = SHA-256(worker_id || salt_daily)
```

`salt_daily` は当日の日付文字列（`YYYY-MM-DD`）を使用する。同一日内で同一作業者は同一ハッシュとなるが、日付が変わると値が変化するため、長期的な名寄せを防止する。

### 2-2. Body（Step 明細領域）

Body は SOP を構成する各 Step の実行結果を 1 行 1 Step のテーブル形式で表示する。

| 列名 | 表示ラベル | データ型 | 文字数上限 | ALCOA+ 該当原則 |
|---|---|---|---|---|
| step_no | Step 番号 | INTEGER | — | L（読み取り可能） |
| instruction_text | 指示文 | VARCHAR | 100 | L（読み取り可能） |
| completed_at | 完了時刻 | TIMESTAMPTZ | — | C（同時性） |
| duration_sec | 所要時間（秒）| INTEGER | — | C（同時性） |
| evidence_type | エビデンス種別 | VARCHAR | 50 | O（原本性） |
| measurement_value | 測定値 | NUMERIC | — | A（正確性） |
| measurement_unit | 単位 | VARCHAR | 20 | A（正確性） |
| branch_result | 条件分岐結果 | VARCHAR | 20 | A（正確性） |
| e_sign_present | 電子サイン有無 | BOOLEAN | — | A（帰属可能性） |

#### 2-2-1. evidence_type の列挙値

| 値 | 意味 |
|---|---|
| `PHOTO` | 写真エビデンス |
| `MEASUREMENT` | 数値測定（センサー・手入力）|
| `SCAN` | バーコード / QR スキャン |
| `SIGNATURE` | 電子サイン |
| `NONE` | エビデンスなし（任意 Step）|

#### 2-2-2. branch_result の列挙値

| 値 | 意味 |
|---|---|
| `OK` | 条件分岐なし、または OK 経路 |
| `NG` | NG 経路（不適合フロー）|
| `SKIP` | 条件分岐によりスキップ |
| `N/A` | 分岐対象外 Step |

### 2-3. Footer（最終承認・改ざん防止領域）

Footer は帳票の下部固定領域に配置し、全 Step 完了署名・ハッシュ値・出力メタデータを収録する。

| フィールド名 | 表示ラベル | データ型 | ALCOA+ 該当原則 |
|---|---|---|---|
| final_sign_data | 全 Step 完了署名（quality_admin 電子サイン）| BYTEA | A（帰属可能性） |
| document_hash | SHA-256 ハッシュ値 | CHAR(64) | O（原本性）・C（一貫性） |
| exported_at | 出力日時 | TIMESTAMPTZ | C（同時性） |
| template_version | TPL-001 バージョン | VARCHAR | 20 | O（原本性） |

#### 2-3-1. document_hash の算出対象

```
document_hash = SHA-256(
    work_execution_id
    || sop_version_id
    || worker_id_hash
    || started_at (ISO 8601)
    || completed_at (ISO 8601)
    || CONCAT(step_no, completed_at, measurement_value, e_sign_present) for each Step
    || final_sign_data
)
```

## 3. データソースマッピング

RP-001 の各フィールドは以下のテーブルから取得する。

| RP-001 領域 | フィールド | 参照テーブル | 参照カラム | 結合条件 |
|---|---|---|---|---|
| Header | work_execution_id | work_executions | id | — |
| Header | sop_name | sop_versions | name | work_executions.sop_version_id = sop_versions.id |
| Header | sop_version | sop_versions | version_number | 同上 |
| Header | product_code | products | code | work_executions.product_id = products.id |
| Header | lot_number | lots | lot_number | work_executions.lot_id = lots.id |
| Header | worker_id_hash | workers | id (ハッシュ変換)| work_executions.worker_id = workers.id |
| Header | started_at | work_executions | started_at | — |
| Header | completed_at | work_executions | completed_at | — |
| Header | terminal_id | terminals | device_id | work_executions.terminal_id = terminals.id |
| Body | step_no | work_events | step_no | work_events.work_execution_id = work_executions.id |
| Body | instruction_text | sop_steps | instruction_text | work_events.sop_step_id = sop_steps.id |
| Body | completed_at | work_events | occurred_at | — |
| Body | duration_sec | work_events | duration_sec | — |
| Body | evidence_type | evidence_files | evidence_type | evidence_files.work_event_id = work_events.id |
| Body | measurement_value | measurements | value | measurements.work_event_id = work_events.id |
| Body | measurement_unit | measurements | unit | 同上 |
| Body | branch_result | work_events | branch_result | — |
| Body | e_sign_present | electronic_signs | id IS NOT NULL | electronic_signs.work_event_id = work_events.id |
| Footer | final_sign_data | electronic_signs | sign_data | electronic_signs.work_execution_id = work_executions.id AND role = 'quality_admin' |

### 3-1. 結合クエリ概略

```sql
-- RP-001 Header 取得
SELECT
    we.id                              AS work_execution_id,
    sv.name                            AS sop_name,
    sv.version_number                  AS sop_version,
    p.code                             AS product_code,
    l.lot_number,
    encode(digest(w.id::text || current_date::text, 'sha256'), 'hex')
                                       AS worker_id_hash,
    we.started_at,
    we.completed_at,
    t.device_id                        AS terminal_id
FROM work_executions we
JOIN sop_versions    sv  ON sv.id  = we.sop_version_id
JOIN products        p   ON p.id   = we.product_id
JOIN lots            l   ON l.id   = we.lot_id
JOIN workers         w   ON w.id   = we.worker_id
JOIN terminals       t   ON t.id   = we.terminal_id
WHERE we.id = $1;
```

## 4. ALCOA+ 準拠詳細

各フィールドの ALCOA+ 原則への適合状況を以下に示す。

| ALCOA+ 原則 | 原則の意味 | RP-001 での実装 | 担保するフィールド |
|---|---|---|---|
| A（Attributable）帰属可能性 | 誰が何をしたか記録 | worker_id_hash・terminal_id・e_sign_present | worker_id_hash, terminal_id, final_sign_data |
| L（Legible）読み取り可能 | 常に読み取り可能な形式 | PDF/A-3 出力・フォントサブセット埋め込み | 全フィールド |
| C（Contemporaneous）同時性 | 操作と記録が同時 | work_events.occurred_at をリアルタイム記録 | started_at, completed_at, 各 Step の completed_at |
| O（Original）原本性 | 最初の記録が保存 | document_hash で原本検証・NAS 書き込み後変更不可 | document_hash, template_version |
| A（Accurate）正確性 | 正確な値を記録 | 測定値はセンサー直接取得・手入力時は確認ダイアログ | measurement_value, measurement_unit |
| C（Complete）完全性 | 全 Step の記録が存在 | BAT-006 で全 Step 完了を確認後のみ生成 | 全 Body 行 |
| C（Consistent）一貫性 | 時系列の矛盾なし | started_at ≤ 各 Step completed_at ≤ completed_at を DB 制約で保証 | 時刻系フィールド |
| E（Enduring）永続性 | 保存期間中アクセス可能 | PDF/A-3 + 7 年保管（NFR-OPS-035） | 全フィールド |
| A（Available）入手可能性 | 要求時に即座に提供 | NAS + DB インデックス・SCR-MC-005 からのオンデマンド出力 | document_hash（参照キー） |

## 5. 生成トリガー

RP-001 の生成はバッチ処理 BAT-006 が担当する。

| 項目 | 内容 |
|---|---|
| バッチ ID | BAT-006 |
| 処理名 | SOP 実行記録 PDF 生成バッチ |
| トリガー条件 | `work_executions.status = 'COMPLETED'` への更新を PostgreSQL LISTEN/NOTIFY で検知 |
| 処理フロー | (1) work_execution_id を受取 → (2) 全 Step 完了チェック → (3) quality_admin 電子サイン確認 → (4) テンプレート TPL-001 へのデータバインド → (5) Rust PDF 生成エンジンで PDF/A-3 出力 → (6) document_hash 算出 → (7) NAS へ格納 → (8) document_hashes テーブルへ登録 |
| 失敗時挙動 | リトライ 3 回。失敗時は `report_generation_failures` テーブルに記録し、ops_admin へ通知 |
| 実行保証 | 冪等設計（同一 work_execution_id に対して重複生成しない） |
| 処理時間目標 | 平均 3 秒以内（Step 数 50 以下の場合） |

---

**本節で確定した方針**
- **RP-001 は PDF/A-3 形式で生成し、Header/Body/Footer の 3 部構成・TPL-001 テンプレートを使用する。worker_id はハッシュのみ帳票に出力し、氏名・平文 ID は一切含めない。**
- **Body の各 Step 行は work_events テーブルから直接マッピングし、XES 変換（RP-002）の前処理データとして兼用できる属性構成を保持する。**
- **BAT-006 は work_executions.status = 'COMPLETED' を LISTEN/NOTIFY で検知して起動し、全 Step 完了・電子サイン確認の 2 条件を満たした場合のみ生成する。生成後の帳票は document_hash により改ざん検知を保証する。**
