# 14 RP-010 リワーク・廃却記録設計

本章は帳票 RP-010「リワーク・廃却記録」のレイアウト・データフィールド・生成規則を確定する。RP-010 は不適合発生品のディスポジション判定から修正作業・廃却・返却の全プロセスを ALCOA+ 準拠の単一帳票として記録する。

---

## 1. 帳票概要

| 項目 | 内容 |
|---|---|
| 帳票 ID | RP-010 |
| 帳票名 | リワーク・廃却記録 |
| 準拠規格 | ISO 9001:2015 §8.7（不適合なアウトプットの管理）/ ISO 13485 §8.5.2 是正処置 |
| 主目的 | リワーク実施証拠（元 WorkExecution との不可侵関係・新 case_id・前後写真・再検査結果） |
| 副目的 | 廃却/返却の法的記録（廃棄物処理票・立会者サイン・追跡番号） |
| 出力形式 | PDF/A-3 |
| テンプレート ID | TPL-010 |
| 生成単位 | reworks 1 レコードごとに 1 帳票（CLOSED_* 状態遷移時に生成） |
| 保存先 | NAS（`/reports/rp010/{year}/{month}/`） |
| 保存期間 | 7 年以上（NFR-DQ-010） |
| 担当要件 | FR-ST-013, FR-ST-014, FR-EV-014, FR-EV-015, FR-MA-017, FR-MA-018, BR-BUS-040〜044 |

---

## 2. レイアウト構造（4 部構成）

### 2-1. Section A: ディスポジション判定

| フィールド名 | 表示ラベル | ALCOA+ |
|---|---|---|
| disposition_id | ディスポジション ID | O |
| nonconformity_id | 不適合 ID（RP-003 リンク） | A |
| parent_case_id | 元作業実行 ID（不変参照） | O |
| decision | 判定区分（REWORK / SCRAP / RETURN） | AC |
| decision_reason | 判定理由 | C |
| quality_admin_sign_hash | 品質担当電子サイン（ハッシュ）| A |
| supervisor_sign_hash | 現場監督電子サイン（ハッシュ）| A |
| decided_at | 判定日時 | C |

### 2-2. Section B: リワーク実施記録（REWORK 区分時のみ）

| フィールド名 | 表示ラベル |
|---|---|
| rework_id | リワーク ID |
| rework_case_id | リワーク作業実行 ID（新 case_id） |
| rework_type | リワーク種別（TOUCH_UP / REWORK_FULL / SORTING）|
| rework_sop_name | 適用リワーク SOP 名（版番号固定） |
| worker_id_hash | リワーク実施者 ID（ハッシュ） |
| before_evidence_hash | 作業前写真 SHA-256 |
| after_evidence_hash | 作業後写真 SHA-256 |

### 2-3. Section C: 再検査記録

| フィールド名 | 表示ラベル |
|---|---|
| verification_id | 再検査 ID |
| verifier_id_hash | 再検査者 ID（ハッシュ・リワーク実施者と異一性保証）|
| verdict | 再検査判定（OK / NG / DOWNGRADE）|
| rework_qr_label_giai | 修正品 QR ラベル GIAI |

### 2-4. Section D: 廃却/返却記録（SCRAP / RETURN 区分時のみ）

| フィールド名 | 表示ラベル |
|---|---|
| waste_manifest_no | 廃棄物処理票番号（SCRAP）|
| witness_id_hash | 立会者 ID（ハッシュ・廃却実施者と異一性保証）|
| carrier | 運送業者名（RETURN） |
| tracking_no | 返却追跡番号（RETURN） |

---

## 3. ALCOA+ Original 検証記述

Section B の `parent_case_id` は ALCOA+ Original 原則（NFR-DQ-010）に従い、参照のみで書き換え不可。RP-010 の証明として「元 WorkExecution ID: {parent_case_id} は本リワーク作業によって一切改変されていないこと」を帳票本文に明記する。
