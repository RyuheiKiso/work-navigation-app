# 07 RP-005 設備点検記録設計

本章は IPA 共通フレーム 2013「2.4 ソフトウェア方式設計プロセス」出力設計タスクに準拠し、帳票 RP-005（設備点検記録）のレイアウト・データソース・法令保管要件・生成トリガを確定する。設備点検記録は労働安全衛生法および ISO 9001/45001 に基づく法定記録であり、改ざん防止・電子サイン付与・長期保管の三要件を同時に満たす設計とする。

---

## 1. 概要

設備点検記録（RP-005）は、製造設備に対して実施される日常点検・定期点検・特別点検の実施結果を電子記録として確定するための帳票である。本帳票は法令上の義務記録であり、設備の安全性・稼働信頼性を証明する根拠文書として機能する。ISO 9001（品質マネジメントシステム）および ISO 45001（労働安全衛生マネジメントシステム）の要求事項に対応する。

### 1-1. 法令・規格根拠

| 法令・規格 | 関連条文・条項 | 本帳票での対応 |
|---|---|---|
| 労働安全衛生法 | 第 45 条（定期自主検査）| 点検記録の作成・保存義務 |
| 労働安全衛生規則 | 第 151 条の 21 等（設備別検査周期）| 点検種別と周期の設定 |
| ISO 9001:2015 | 7.1.3（インフラストラクチャ）・8.5.1（生産・サービス提供の管理）| 設備管理記録の維持 |
| ISO 45001:2018 | 8.1.2（リスク除去・低減）・10.2（事故・不適合・是正処置）| 安全点検記録の保持 |

### 1-2. 適用範囲と帳票識別

| 項目 | 内容 |
|---|---|
| 帳票 ID | RP-005 |
| 帳票名 | 設備点検記録 |
| 目的 | 設備の安全性・稼働信頼性の証明と法令記録の担保 |
| 適用設備 | 本システムで管理する全登録設備 |
| 点検種別 | 日常点検 / 定期点検 / 特別点検 |
| 主利用者 | 設備管理担当・安全管理担当・品質保証担当・監査機関 |
| テンプレート ID | TPL-005 |

## 2. レイアウト設計

帳票のビジュアルレイアウトは `img/` 配下の drawio/svg ファイルを参照すること。

> **図参照**: fig_des_report_rp005（img/ 配下）を参照

帳票は論理的に 5 つのセクションで構成される。

### 2-1. ヘッダーセクション

| フィールド名 | データ型 | 必須 | 説明 |
|---|---|---|---|
| 設備 ID | VARCHAR(30) | 必須 | システム内設備識別子（EQ-NNNN 形式） |
| 設備名 | VARCHAR(200) | 必須 | 設備の正式名称 |
| 点検種別 | ENUM | 必須 | 日常点検 / 定期点検 / 特別点検 |
| 点検日時 | TIMESTAMPTZ | 必須 | ISO 8601 形式（UTC+9 表示） |
| 点検者 ID（ハッシュ）| VARCHAR(64) | 必須 | SHA-256(worker_id \|\| salt)。平文の個人 ID は非表示 |
| 根拠 SOP ID | VARCHAR(20) | 必須 | 点検手順を定義する SOP の識別子 |
| SOP バージョン | VARCHAR(20) | 必須 | 実施時点の SOP バージョン |

### 2-2. ボディセクション（点検項目テーブル）

各設備に定義された点検項目を 1 行 1 項目で表示する。

| フィールド名 | データ型 | 必須 | 説明 |
|---|---|---|---|
| 点検項目名 | VARCHAR(200) | 必須 | 例: 油圧圧力・絶縁抵抗値 |
| 基準値 | NUMERIC + 単位 | 必須 | 許容範囲（例: 4.5〜5.5 MPa） |
| 測定値 | NUMERIC | 必須 | 点検時の実測値 |
| 単位（UCUM） | VARCHAR(30) | 必須 | UCUM 準拠の単位記号（例: MPa, Ω, °C） |
| 合否 | ENUM | 必須 | 合格 / 要注意 / 不合格 |
| 写真エビデンス有無 | BOOLEAN | 必須 | 写真が登録されている場合 TRUE |
| 写真エビデンス参照 | UUID | 条件付き | `evidence_files.id` への参照（有の場合のみ）|

### 2-3. 異常・所見セクション（異常がある場合のみ表示）

合否が「要注意」または「不合格」の項目が 1 件以上存在する場合にのみ表示する。

| フィールド名 | データ型 | 必須 | 説明 |
|---|---|---|---|
| 異常項目名 | VARCHAR(200) | 必須 | 異常が検出された点検項目名 |
| 異常内容 | TEXT | 必須 | 異常の具体的記述 |
| 処置内容 | TEXT | 条件付き | 即時処置を実施した場合の内容 |
| 是正要否 | ENUM | 必須 | 即時是正 / 計画的是正 / 経過観察 / 不要 |
| CAPA 紐付け ID | VARCHAR(20) | 条件付き | 是正が必要な場合の CAPA レコード参照 |

### 2-4. フッターセクション

| フィールド名 | データ型 | 必須 | 説明 |
|---|---|---|---|
| 点検者電子サイン | BYTEA | 必須 | 点検実施者の電子サインデータ（BR-BUS-015 準拠）|
| ドキュメントハッシュ | VARCHAR(64) | 必須 | SHA-256（全セクション連結）|
| 出力日時 | TIMESTAMPTZ | 必須 | ISO 8601 形式（UTC+9 表示）|
| 次回点検予定日 | DATE | 必須 | 点検種別・設備マスタの周期設定から算出 |

## 3. データソース

RP-005 のデータソースは以下のテーブルおよび条件を結合して生成する。

### 3-1. 主要テーブル構成

| テーブル名 | 用途 | 結合条件 / フィルタ |
|---|---|---|
| `work_executions` | 点検作業の実行記録 | `sop.category = 'inspection'` でフィルタ |
| `sops` | 点検 SOP の定義・カテゴリ判定 | `work_executions.sop_id = sops.id` |
| `equipment` | 設備マスタ（設備 ID・設備名・点検周期）| `work_executions.equipment_id = equipment.id` |
| `measurements` | 各点検項目の測定値・単位 | `measurements.execution_id = work_executions.id` |
| `inspection_items` | 点検項目定義（基準値・UCUM 単位）| `measurements.item_id = inspection_items.id` |
| `evidence_files` | 写真エビデンスファイル参照 | `evidence_files.execution_id = work_executions.id` |
| `user_signatures` | 点検者電子サイン | `user_signatures.user_id = work_executions.worker_id` |

### 3-2. SOP カテゴリによるフィルタリング

RP-005 は `sops.category = 'inspection'` に該当する作業実行記録のみを対象とする。他カテゴリ（通常作業・Kaizen・訓練）の作業実行記録は本帳票に含まない。

---

## 4. 法令保管要件

### 4-1. 法定最低保管期間と本システムの方針

| 根拠法令 | 対象記録 | 法定保管期間 | 本システム保管期間 |
|---|---|---|---|
| 労働安全衛生法施行規則 | 定期自主検査記録 | 3 年以上 | **7 年以上（統一方針）** |
| ISO 9001:2015 | 設備管理記録 | 規定なし（組織が決定）| **7 年以上（統一方針）** |
| ISO 45001:2018 | 安全点検記録 | 規定なし（組織が決定）| **7 年以上（統一方針）** |

本システムは法定最低保管期間（3 年）を上回る 7 年以上保管を統一方針（NFR-OPS-035）として採用する。これにより複数法令・規格への対応を単一方針で担保する。

### 4-2. 保管形式と改ざん防止

| 項目 | 実装 |
|---|---|
| 保管形式 | PDF/A-3（ISO 19005-3 準拠）+ JSON サイドカー |
| 改ざん防止 | SHA-256 フッター埋め込み + ハッシュチェーン連結（09_帳票改ざん防止技術設計参照）|
| 格納先 | NAS: `/reports/RP-005/{YYYY}/{equipment_id}/{execution_id}.pdf` |
| 廃棄 | 保管期間満了後は廃棄ログを記録した上で削除（10_帳票保管・廃棄・出力監査参照）|

---

## 5. 生成トリガ

| 項目 | 内容 |
|---|---|
| バッチ ID | BAT-009 |
| トリガ条件 | 点検 SOP（`sops.category = 'inspection'`）の全 Step 完了時 |
| 生成方式 | イベント駆動（SOP 完了イベント → Rust バックエンドの非同期ジョブキュー）|
| 電子サイン付与タイミング | SOP 完了時に点検者サインを自動取得し帳票に埋め込み |
| 出力形式 | PDF/A-3（埋め込みメタデータ付き）+ JSON サイドカー |
| 格納先 | NAS: `/reports/RP-005/{YYYY}/{equipment_id}/{execution_id}.pdf` |
| エラー時 | リトライ 3 回後にアラートキューへ投入（OPS-ALERT-002）|

---

**本節で確定した方針**

- **RP-005 は労働安全衛生法・ISO 9001/45001 に基づく法定記録として位置づけ、点検者電子サイン・SHA-256 フッター・PDF/A-3 保管の三要件で ALCOA+ 原則を担保する。**
- **法定保管期間（3 年）を上回る 7 年以上保管を統一方針（NFR-OPS-035）とし、複数法令・規格への対応を単一方針で一元化する。**
- **RP-005 の生成は BAT-009（点検 SOP 全 Step 完了時イベント駆動）により自動起動し、`sops.category = 'inspection'` フィルタで他カテゴリ作業と明確に分離する。**
