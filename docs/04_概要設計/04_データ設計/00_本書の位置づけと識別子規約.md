## 3. 本サブで使用する識別子

### 3-1. TBL（物理テーブル）

- 形式: `TBL-NNN`（全域 001 から）
- 現在の採番済み範囲: `TBL-001`〜`TBL-035`（付録/99 採番台帳参照）
- 本サブが一次管理する: 全 TBL
- 注意: EN と TBL は 1:1 または 1:N の対応。中間テーブルは N:M と明記

### 3-2. IDX（インデックス）

- 形式: `IDX-NNN`（全域 001 から）
- 本サブが採番する: IDX 全件
- 必須メタデータ: 対応 TBL-NNN・種別・対象列・根拠 NFR-NNN

### 3-3. VW（物理ビュー・実体化ビュー）

- 形式: `VW-NNN`（全域 001 から）
- 本サブが採番する: VW 全件
- 必須メタデータ: 対応 TBL-NNN 群・消費側 API-NNN または RP-NNN

### 3-4. PART（パーティション）

- 形式: `PART-NNN`（本サブ内 001 から）
- 採番対象: work_events の月次パーティション群

## 4. ALCOA+ × テーブル設計 対応マトリクス

各 ALCOA+ 原則がどのテーブル設計によって実現されるかを確定する。

| ALCOA+ 原則 | 実現テーブル/設計要素 | 担当章 |
|---|---|---|
| Attributable | TBL-001（worker_id・terminal_id 必須） | §04 |
| Legible | TBL-007/008（instruction_text JSONB 多言語） | §03 |
| Contemporaneous | TBL-001（timestamp_client + timestamp_server 二重） | §05 |
| Original | TBL-009（evidence_file_hash SHA-256） | §04 |
| Accurate | TBL-010（measured_value + unit_ucum） | §04 |
| Complete | TBL-001（event_type NOT NULL・スキップも記録） | §05 |
| Consistent | TBL-031（hash_chain_blocks・prev_hash 連鎖） | §05 |
| Enduring | TBL-001〜002（保存期間 7 年以上・Append-only） | §09 |
| Available | TBL-003（outbox_events 端末 SQLite 複製） | §04 |

---

**本節で確定した方針**
- **本サブは IPA 2.3/2.4 の DB 方式・DB 構造設計タスクを担当し、EN-001〜027 の全 27 論理エンティティを TBL-NNN に物理化する責務を持つ。**
- **TBL/IDX/VW 識別子の一次採番権は本サブにあり、付録/99 採番台帳を逐次更新する。**
- **ALCOA+ 9 原則のテーブル設計レベルでの担保を §05 と本章で確定し、各原則の技術的根拠を設計命題として記述する。**
