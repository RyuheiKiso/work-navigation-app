# 04_データ設計

IPA 共通フレーム 2013「2.3 データベース方式設計タスク」および「2.4 データベース構造設計タスク」に対応する。

要件定義フェーズで確定した 27 論理エンティティ（EN-001〜027）を物理テーブル（TBL-001〜）に変換し、インデックス・パーティション・暗号化・データフロー・ライフサイクルの全設計命題を確定する。

---

## 担当する上流要件

| 種別 | 範囲 | 完全カバー要件 |
|---|---|---|
| 論理エンティティ | EN-001〜027（全 27 件） | 全件が TBL にマッピングされること |
| データ品質 NFR | NFR-DQ 全件（ALCOA+ 9 原則） | ALCOA+ マトリクスで各原則の担保を確認 |
| 性能 NFR | NFR-PRF-001/003/004/010/015 | インデックス・パーティション設計で担保 |
| セキュリティ NFR | NFR-SEC-040（Append-only 物理保証）/ NFR-SEC-035（PII）| DB ロール分離・暗号化設計で担保 |
| 倫理品質 NFR | NFR-ETH（個人別集計ビュー禁止） | §12 でマスキング・禁止クエリを確定 |
| 移行 NFR | NFR-MIG（スキーマ移行 15 分以内） | §11 で sqlx migrate 運用を確定 |
