# 01 論理 ER（要件 EN-001〜027 継承）

本章の責務は、要件定義フェーズで確定した 27 論理エンティティ（EN-001〜027）の完全な一覧と、エンティティ間の関係（カーディナリティ・外部キー方向・関係種別）を確定することである。この論理 ER が以降の物理設計（TBL・IDX・VW）の基盤となる。

## 2. 主要エンティティ属性定義

### 2-1. EN-012 WorkEvent（最重要・Append-only コア）

XES（IEEE 1849）互換 8 必須属性を全て持つ。

| 属性名 | 型 | 制約 | 説明 |
|---|---|---|---|
| event_id | UUID | PK, NOT NULL | Idempotency Key を兼用（UUID v7）|
| case_id | UUID | NOT NULL, FK → work_executions | XES case 識別子（作業実行 ID）|
| activity | VARCHAR(64) | NOT NULL | XES activity（step_completed/step_skipped 等 9 種）|
| timestamp_client | TIMESTAMPTZ | NOT NULL | 端末時刻（申告時刻 = Contemporaneous 担保）|
| timestamp_server | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | サーバー受信時刻（証拠時刻）|
| resource | UUID | NOT NULL, FK → users | 実施者 user_id |
| sop_version_id | UUID | NOT NULL, FK → master_versions | 時点参照固定 |
| payload | JSONB | NOT NULL | activity 種別ごとの構造化データ |
| prev_hash | CHAR(64) | NOT NULL | SHA-256 ハッシュチェーン前ブロック |
| terminal_id | UUID | NOT NULL, FK → devices | 記録端末（Attributable）|
| is_offline | BOOLEAN | NOT NULL, DEFAULT FALSE | オフライン記録フラグ |
| sync_lag_ms | INTEGER | NULLABLE | 同期遅延（ミリ秒）|

**Activity 列挙値（9 種）**:
- `step_completed`, `step_skipped`, `step_rejected`, `step_started`
- `work_interrupted`, `work_resumed`, `andon_triggered`
- `nonconformance_filed`, `kaizen_filed`

### 2-2. EN-015 ElectronicSign（電子サイン・4 必須要素）

| 属性名 | 型 | 制約 | 説明 |
|---|---|---|---|
| sign_id | UUID | PK | — |
| signer_id | UUID | NOT NULL, FK → users | 実施者 ID |
| signed_at | TIMESTAMPTZ | NOT NULL | サイン時刻 |
| meaning | VARCHAR(128) | NOT NULL | 署名の意味（承認・確認・作業実施 等）|
| content_hash | CHAR(64) | NOT NULL | 署名対象コンテンツの SHA-256 |
| event_id | UUID | NULLABLE, FK → work_events | 関連作業イベント |
| master_version_id | UUID | NULLABLE, FK → master_versions | 関連マスタ版数 |

### 2-3. EN-021 OutboxEvent（同期・Append-only）

| 属性名 | 型 | 制約 | 説明 |
|---|---|---|---|
| outbox_id | UUID | PK | — |
| event_id | UUID | NOT NULL, UNIQUE | 参照 WorkEvent の event_id（Idempotency）|

| payload | JSONB | NOT NULL | 送信ペイロード |
| created_at | TIMESTAMPTZ | NOT NULL | — |
| sent_at | TIMESTAMPTZ | NULLABLE | — |
| retry_count | SMALLINT | NOT NULL, DEFAULT 0 | 再試行回数（上限 3 回）|
| dlq_reason | TEXT | NULLABLE | DLQ 移行理由 |

| 属性名 | 型 | 制約 | 説明 |
|---|---|---|---|

## 3. エンティティ間関係（カーディナリティ）

### 3-1. マスタ階層

```
Process (EN-005)
  └─ 1:N ─ Operation (EN-006)
              └─ 1:N ─ SOP (EN-008)
                          └─ 1:N ─ Step (EN-009)
                          └─ N:1 ─ MasterVersion (EN-010)  ← 版数管理
Product (EN-007) ─ N:M ─ SOP (EN-008)  ← 製品×SOP 対応
```

### 3-2. 実行・トランザクション

```
WorkInstance (EN-011)  ← SOP + User + Device の組み合わせ
  └─ 1:N ─ WorkEvent (EN-012, Append-only)
  └─ 1:N ─ EvidenceFile (EN-013, Append-only)
  └─ 1:N ─ Measurement (EN-014, Append-only)
  └─ 0:N ─ Suspension (EN-016, Append-only)

WorkEvent (EN-012)
  └─ N:1 ─ ElectronicSign (EN-015, Append-only)  ← 承認サイン紐付け
  └─ N:1 ─ OutboxEvent (EN-021, Append-only)  ← 同期キュー
```

### 3-3. ユーザー・権限

```
User (EN-001)
  └─ N:M ─ Role (EN-002)  → UserRole（中間テーブル）
  └─ N:M ─ Skill (EN-003)  → UserSkill（中間テーブル）
```

### 3-4. 外部連携

```
Device (EN-023) ─ 1:1 ─ DeviceSyncState (EN-024)
ExternalKeyBinding（計画 06 章定義）  ─ N:1 ─ WorkPattern
```

### 3-5. 4M+1E+1M 参照軸（6 軸トレサビ）

WorkEvent（EN-012）が以下の参照軸を保持：

| 軸 | 参照 EN | 参照方法 |
|---|---|---|
| Man（人） | EN-001 User | worker_id FK（必須）|
| Machine（設備） | EN-019 Equipment | payload.equipment_id（主使用）, payload.tool_scan_refs[]（検証スキャン履歴 JSONB）|
| Material（材料） | EN-020 Lot | payload.lot_id（JSONB）|
| Method（方法） | EN-008 SOP | sop_version_id FK（必須）|
| Environment（環境） | — | payload.env_snapshot（JSONB）|
| Measurement（測定） | EN-014 Measurement | measurement_id FK（任意）|

**※既知の不整合（将来対応）**: 要件定義/12_データ要件（論理）§1 では EN-019 = Equipment を定義するが、本書 §1 の EN リスト上では EN-019 が別エンティティを指す記載がある。本書は要件定義側の定義（EN-019 = Equipment）を採用する。HLD 側 EN 番号の再整合は別課題として記録する。

## 4. 主キー生成戦略

全エンティティの主キーは **UUID v7** を採用する。

| 選択理由 | 内容 |
|---|---|
| 時系列ソート | UUID v7 は生成時刻をビット先頭に持ち、INSERT 順にほぼ単調増加する（B-Tree インデックス効率）|
| 分散一意性 | 端末・サーバーがそれぞれ生成しても衝突しない |
| Idempotency Key 兼用 | event_id = Idempotency Key として `Idempotency-Key` ヘッダで送信 |

UUID v4 は使用しない（B-Tree のページ分散が激しいため）。

---

**本節で確定した方針**
- **EN-001〜027 の全 27 論理エンティティを確定し、12 件の Append-only エンティティ（WorkEvent/EvidenceFile/Measurement/ElectronicSign/Suspension/OutboxEvent/AuthLog/HashChainBlock/BackupLog 等）を識別した。**
- **WorkEvent は XES 8 必須属性と SHA-256 ハッシュチェーン（prev_hash）を持ち、ALCOA+ Consistent/Original/Attributable の技術的担保をテーブルレベルで確定する。**
- **全エンティティの主キーは UUID v7 に統一し、Idempotency Key との兼用と B-Tree 効率の両立を実現する。**
- **EN-019 Equipment に scan_code / tool_subtype / calibration_due_date の拡張を反映し、ポカヨケ用スキャン照合のマスタとして確定する。**
