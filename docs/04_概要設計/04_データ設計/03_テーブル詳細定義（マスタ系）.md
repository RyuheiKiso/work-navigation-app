# 03 テーブル詳細定義（マスタ系）

本章の責務は、マスタ系テーブル（更新可・版管理）の物理列定義・データ型・制約・既定値を確定することである。対象は TBL-004/007/008/016/017/018/019/020/021/022/023/024/025/026/028/029/030/033/034。

---

## 1. ユーザー・権限マスタ

### 1-1. TBL-016 users（EN-001 User）

| 列名 | PostgreSQL 型 | 制約 | 説明 |
|---|---|---|---|
| user_id | UUID | PK, NOT NULL | UUID v7 |
| login_id | VARCHAR(128) | NOT NULL, UNIQUE | ログイン識別子（LDAP 連携時は LDAP DN）|
| display_name | VARCHAR(256) | NOT NULL | 表示名 |
| is_active | BOOLEAN | NOT NULL, DEFAULT TRUE | 退職時 FALSE（物理削除しない）|
| anonymized_at | TIMESTAMPTZ | NULLABLE | PII 匿名化実施時刻（退職 60 日後）|
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |

**PII 匿名化設計**: 退職から 60 日（`CFG-010` で設定可能）後に BAT-004 が `login_id`・`display_name` を内部 UUID 文字列に置換する。`user_id` は変更しない（既存 WorkEvent の `resource` FK が壊れないため）。

### 1-2. TBL-017 roles（EN-002 Role）

固定 6 種のマスタ行。アプリ起動時 INSERT し、その後 UPDATE しない（ソース管理）。

| role_id（定数）| role_name | 説明 |
|---|---|---|
| `role-operator` | operator | 作業員 |
| `role-supervisor` | supervisor | 現場監督 |
| `role-master-admin` | master_admin | マスタ管理者 |
| `role-quality-admin` | quality_admin | 品質担当 |
| `role-system-admin` | system_admin | IT 担当 |
| `role-executive` | executive | 経営層 |

### 1-3. TBL-019 user_roles（N:M 中間）

| 列名 | 型 | 制約 |
|---|---|---|
| user_id | UUID | PK（複合）, FK → users |
| role_id | UUID | PK（複合）, FK → roles |
| granted_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() |
| granted_by | UUID | NOT NULL, FK → users |

### 1-4. TBL-018 skills / TBL-020 user_skills

skills: skill_id（PK）・skill_name（VARCHAR 128）・skill_level（SMALLINT 1〜5）・description（TEXT）・is_active
user_skills: user_id + skill_id（複合 PK）・achieved_level（SMALLINT）・certified_at（TIMESTAMPTZ）・certified_by（UUID FK）

## 2. プロセス・SOP 階層マスタ

### 2-1. TBL-021 processes（EN-005 Process）

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| process_id | UUID | PK | — |
| process_code | VARCHAR(64) | NOT NULL, UNIQUE | 業務コード（外部参照用）|
| name | JSONB | NOT NULL | 多言語名称 `{"ja": "組立", "en": "Assembly"}` |
| is_active | BOOLEAN | NOT NULL, DEFAULT TRUE | — |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |

### 2-2. TBL-022 operations（EN-006 Operation）

process_id FK に加え、operation_code（UNIQUE）・name（JSONB）・sequence_number（INTEGER）・is_active を持つ。

### 2-3. TBL-023 products（EN-007 Product）

product_id（PK）・product_code（UNIQUE）・name（JSONB）・is_active。

### 2-4. TBL-007 sops（EN-008 SOP）

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| sop_id | UUID | PK | — |
| operation_id | UUID | NOT NULL, FK → operations | 所属オペレーション |
| sop_code | VARCHAR(64) | NOT NULL, UNIQUE | — |
| current_version_id | UUID | NULLABLE, FK → master_versions | 現在有効版（PUBLISHED）|
| is_active | BOOLEAN | NOT NULL, DEFAULT TRUE | — |

### 2-5. TBL-004 master_versions（EN-010 MasterVersion）

版数管理テーブル。SOP・Step・StepTypeDefinition・StepFlowRule の各版数を管理する。

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| master_version_id | UUID | PK | — |
| master_type | VARCHAR(32) | NOT NULL, CHECK | SOP / STEP / STEP_TYPE / FLOW_RULE |
| master_id | UUID | NOT NULL | 対象マスタ ID |
| version_number | SMALLINT | NOT NULL | 1 から増分 |
| status | VARCHAR(16) | NOT NULL | DRAFT / UNDER_REVIEW / PUBLISHED / ARCHIVED |
| effective_date | DATE | NULLABLE | NULL は即時有効 |
| created_by | UUID | NOT NULL, FK → users | — |
| published_by | UUID | NULLABLE, FK → users | — |
| sign_id | UUID | NULLABLE, FK → electronic_signs | 承認電子サイン |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |
| UNIQUE | (master_type, master_id, version_number) | — | 同一マスタ × 版数の重複を禁止 |

**PUBLISHED 状態への遷移後は status 以外の列を変更しない**（DB チェック制約で強制）。

### 2-6. TBL-008 steps（EN-009 Step）

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| step_id | UUID | PK | — |
| sop_id | UUID | NOT NULL, FK → sops | 所属 SOP |
| step_number | SMALLINT | NOT NULL | SOP 内の表示順 |
| input_type | VARCHAR(32) | NOT NULL | boolean_check / numeric_input / photo_capture / text_input / slider_range 等 |
| instruction_text | JSONB | NOT NULL | `{"ja": "...", "en": "..."}` 多言語 |
| judgment_condition | JSONB | NULLABLE | `{"usl": 10.5, "lsl": 9.5, "unit": "mm"}` |
| evidence_required | BOOLEAN | NOT NULL | ゲート（BR-BUS-003）|
| fmea_rpn_flag | BOOLEAN | NOT NULL | FMEA 重点管理フラグ |
| skill_level_required | SMALLINT | NOT NULL, DEFAULT 1 | 1〜5 |
| expected_unit | VARCHAR(32) | NULLABLE | UCUM コード |
| media_refs | JSONB | NULLABLE | 参照メディア（動画・写真 URL）|
| tips_refs | JSONB | NULLABLE | コツ・注意事項 |

## 3. 設備・計測器・外部連携マスタ

### 3-1. TBL-025 equipments（EN-019 Equipment）

equipment_id（PK）・equipment_code（UNIQUE）・name（JSONB）・equipment_type（VARCHAR 64）・process_id（FK, NULLABLE）・is_active

### 3-2. TBL-026 instruments（EN-024 Instrument）

instrument_id（PK）・instrument_code（UNIQUE）・name（JSONB）・instrument_type（VARCHAR 64）・calibration_due_date（DATE）・calibration_cert_ref（TEXT）・is_active

### 3-3. TBL-027 external_key_bindings（EN-022 ExternalKeyBinding・Append-only）

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| binding_id | UUID | PK | — |
| external_system | VARCHAR(64) | NOT NULL | 親機システム名 |
| external_key | JSONB | NOT NULL | `{"lot_id": "L001", "product_code": "P100"}` |
| work_pattern_id | UUID | NOT NULL, FK → work_patterns | マッピング先 WorkPattern |
| valid_from | DATE | NOT NULL | 有効開始日 |
| valid_to | DATE | NULLABLE | 有効終了日（NULL = 無期限）|
| sync_status | VARCHAR(16) | NOT NULL | ACTIVE / CONFLICT / DEPRECATED |
| factory_id | UUID | NOT NULL | （ver1.0.0 では定数値）|

変更時: 旧レコードの `valid_to` を設定 + 新レコード INSERT（完全 Append-only）。

## 4. デバイス・同期管理マスタ

### 4-1. TBL-033 devices（EN-023 Device）

device_id（PK）・serial_number（UNIQUE）・device_type（android/ios/windows）・is_active・registered_at

### 4-2. TBL-034 device_sync_states（EN-024 DeviceSyncState）

device_id（PK FK）・last_sync_at（TIMESTAMPTZ）・last_master_version_id（UUID FK, NULLABLE）・sync_status（SYNCED / PENDING / CONFLICT）

---

**本節で確定した方針**
- **マスタ系テーブルの物理列・型・制約を確定し、全マスタで `is_active` フラグによる論理削除を採用し物理 DELETE を禁止する。**
- **SOP/Step/StepTypeDefinition の版数管理は TBL-004（master_versions）で一元管理し、PUBLISHED 後は内容変更を DB 制約で禁止する。**
- **external_key_bindings（TBL-027）は変更を Append-only で管理し、`valid_to` 設定 + 新 INSERT の 2 件操作で安全に更新する。**
