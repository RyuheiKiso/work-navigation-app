# 04 テーブル詳細定義（トランザクション系）

本章の責務は、トランザクション系テーブルの物理列定義を確定することである。対象は TBL-001/003/005/009/010/011/012/013/014/015/021/031/032/035。

---

## 1. 作業実行・イベント系

### 1-1. TBL-005 work_executions（EN-011 WorkInstance）

作業セッションの管理テーブル（更新可）。

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| work_execution_id | UUID | PK | — |
| sop_id | UUID | NOT NULL, FK → sops | 実行 SOP |
| sop_version_id | UUID | NOT NULL, FK → master_versions | 時点参照固定 |
| primary_worker_id | UUID | NOT NULL, FK → users | 主担当作業員 |
| device_id | UUID | NOT NULL, FK → devices | 記録端末 |
| work_order_id | UUID | NULLABLE, FK → work_orders | ワークオーダー参照 |
| status | VARCHAR(16) | NOT NULL | NOT_STARTED / IN_PROGRESS / SUSPENDED / COMPLETED / CANCELLED |
| started_at | TIMESTAMPTZ | NULLABLE | — |
| completed_at | TIMESTAMPTZ | NULLABLE | — |
| current_step_index | SMALLINT | NOT NULL, DEFAULT 0 | プレースキーパー（再開時に使用）|
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |

### 1-2. TBL-001 work_events（EN-012 WorkEvent・Append-only）

イベントストアのコアテーブル（詳細は §05 参照）。

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| event_id | UUID | PK | UUID v7 = Idempotency Key |
| case_id | UUID | NOT NULL, FK → work_executions | XES Case ID |
| activity | VARCHAR(64) | NOT NULL | 9 種の列挙（§05 参照）|
| timestamp_client | TIMESTAMPTZ | NOT NULL | 端末時刻（申告時刻）|
| timestamp_server | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | サーバー受信時刻（証拠時刻）|
| resource | UUID | NOT NULL, FK → users | 実施者 user_id |
| sop_version_id | UUID | NOT NULL, FK → master_versions | 時点参照固定 |
| step_id | UUID | NULLABLE, FK → steps | 対象ステップ（activity 種別による）|
| payload | JSONB | NOT NULL | activity 固有データ |
| prev_hash | CHAR(64) | NOT NULL | SHA-256 チェーン前ブロック |
| content_hash | CHAR(64) | NOT NULL | 本レコードの SHA-256 |
| terminal_id | UUID | NOT NULL, FK → devices | 記録端末 |
| is_offline | BOOLEAN | NOT NULL, DEFAULT FALSE | オフライン記録フラグ |
| sync_lag_ms | INTEGER | NULLABLE | 同期遅延 |

## 3. 同期・制御系

### 3-1. TBL-003 outbox_events（EN-021 OutboxEvent・Append-only + status 更新）

| 列名 | 型 | 制約 | 説明 |
|---|---|---|---|
| outbox_id | UUID | PK | — |
| event_id | UUID | NOT NULL, UNIQUE | 参照 WorkEvent.event_id |
| event_type | VARCHAR(64) | NOT NULL | work_event / electronic_sign 等 |
| status | VARCHAR(16) | NOT NULL | PENDING / SENDING / SENT / FAILED / DLQ |
| payload | JSONB | NOT NULL | 送信ペイロード（圧縮推奨）|
| idempotency_key | UUID | NOT NULL, UNIQUE | API-sync-002 の Idempotency-Key ヘッダ |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | — |
| sent_at | TIMESTAMPTZ | NULLABLE | — |
| next_retry_at | TIMESTAMPTZ | NULLABLE | — |
| retry_count | SMALLINT | NOT NULL, DEFAULT 0 | 上限 CFG-002（デフォルト 5）|
| dlq_reason | TEXT | NULLABLE | DLQ 移行理由 |

status の UPDATE は `app_read_write` ロールに許可する（PENDING → SENDING → SENT 等）。

### 3-2. TBL-031 hash_chain_blocks（EN-025 HashChainBlock・Append-only）

（§05 参照）

### 3-3. TBL-032 auth_logs（EN-022 AuthLog・Append-only）

log_id（PK）・user_id（UUID, NULLABLE FK）・action（VARCHAR 64: LOGIN_SUCCESS/LOGIN_FAILURE/LOGOUT/TOKEN_REFRESH）・device_id（UUID FK, NULLABLE）・ip_address（INET, PII マスク 24 bit）・created_at（TIMESTAMPTZ, DEFAULT NOW()）

### 3-4. TBL-035 idempotency_keys（制御テーブル）

idempotency_key（UUID, PK）・endpoint（VARCHAR 128）・response_status（SMALLINT）・response_body（JSONB）・created_at（DEFAULT NOW()）

24 時間後に `pg_cron` 等で自動 DELETE（唯一 DELETE が許可される制御テーブル）。

---

**本節で確定した方針**
- **トランザクション系テーブルの物理列・型・制約を確定し、WorkEvent（TBL-001）の XES 互換 8 属性 + ハッシュチェーン列を設計命題として固定した。**
- **OutboxEvent（TBL-003）の status 更新は `UPDATE（status 列のみ）` を許可し、Outbox Consumer（BAT-002）が送信状態を管理できる設計とした。**
- **idempotency_keys（TBL-035）は TTL 24 時間の制御テーブルとし、全体方針上禁止した DELETE を唯一許可する例外テーブルとして明示した。**
