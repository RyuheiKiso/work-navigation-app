## 4. 識別子規約

### 4-1. JOB-NNN 識別子（スケジュールジョブバリアント）

JOB-NNN は BAT-NNN から派生する実行バリアント識別子である。1 BAT = 複数環境での実行スケジュール差分（本番/ステージング・週次/月次等）がある場合に JOB-NNN を付与する。BAT-NNN 自体はジョブの論理定義であり、JOB-NNN はその物理的スケジュール構成を識別する。

**採番規約**:
- 形式: `JOB-NNN`（3 桁ゼロ埋め）
- 採番開始: JOB-001
- 必須メタデータ: 対応 BAT-NNN・実行環境（prod/staging）・cron 式・タイムゾーン（JST 固定）・実行ホスト（Windows タスクスケジューラ / Docker Compose）
- 欠番禁止・再使用禁止・廃止メタデータ保持は BAT と同一規約

**採番開始番号**: JOB-001（本サブで採番を開始する）

| JOB-ID | 対応 BAT | cron 式 | 実行環境 | 実行ホスト |
|---|---|---|---|---|
| JOB-001 | BAT-001（hash_chain_verifier） | `0 2 * * 0` (日曜 02:00 JST) | prod | Docker Compose |
| JOB-002 | BAT-004（pii_anonymizer） | `0 3 * * *` (毎日 03:00 JST) | prod | Docker Compose |
| JOB-003 | BAT-005（backup_pg_dump） | `0 1 * * *` (毎日 01:00 JST) | prod | Docker Compose |
| JOB-004 | BAT-009（cert_expiry_notifier） | `0 9 * * *` (毎日 09:00 JST) | prod | Docker Compose |
| JOB-005 | BAT-010（jwt_key_rotator） | `0 3 1 */3 *` (90 日相当・四半期 1 日) | prod | Docker Compose |
| JOB-006 | BAT-001（hash_chain_verifier） | `0 2 * * 0` (日曜 02:00 JST) | staging | Docker Compose |

### 4-2. OPS-PROC-NNN 識別子（運用手順）

OPS-PROC 識別子は、IT 担当（system_admin）が手動で実施する運用手順書の論理識別子である。手順 1 件 = OPS-PROC 1 件に対応する。

**採番規約**:
- 形式: `OPS-PROC-NNN`（3 桁ゼロ埋め）
- 採番開始: OPS-PROC-001
- 必須メタデータ: 手順名・実行者ロール（system_admin / quality_admin）・実行頻度・推定所要時間（分）・関連 NFR-NNN・関連 BAT-NNN（自動ジョブに対応する場合）
- 欠番禁止・再使用禁止・廃止メタデータ保持

**採番開始番号**: OPS-PROC-001（本サブで採番を開始する）

| OPS-PROC-ID | 手順名 | 実行者 | 頻度 | 推定所要時間 | 関連 NFR |
|---|---|---|---|---|---|
| OPS-PROC-001 | 週次ヘルスチェック | system_admin | 週次（毎週月曜朝） | 30 分以内 | NFR-OPS-002, NFR-OPS-052 |
