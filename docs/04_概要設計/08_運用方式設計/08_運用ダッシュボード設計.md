# 運用ダッシュボード設計

本節では、マスタメンテナンス画面（React Web アプリ）内の運用ダッシュボード画面 SCR-MC-001 のウィジェット構成・メトリクス表示マッピング・アラートリスト仕様・自動リフレッシュ設定を定義する。単一管理者（system_admin）が日常的に参照する「システムの健全性を一覧で把握する」画面であり、個人の生産性指標は一切表示しない（BR-BUS-029 準拠）。ダッシュボードは閾値を超えた際のアラート起点となり、詳細調査画面（SCR-MC-007/008 等）へのドリルダウンリンクを提供する。

---

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面識別子 | SCR-MC-001 |
| 画面名 | 運用ダッシュボード |
| 対象ロール | system_admin |
| アクセス方法 | マスタメンテナンス UI トップメニュー > 「運用」 |
| 自動リフレッシュ | **30秒**（CFG キー: `dashboard.refresh_interval_sec`、管理画面から変更可能） |
| レイアウト | 3カラムグリッド（1200px 以上）、1カラム（768px 未満） |

## 2. メトリクス定義（MET-001〜010）

| 識別子 | メトリクス名 | 収集元 | 単位 | 正常範囲 | 警告閾値 | 危険閾値 |
|--------|-------------|--------|------|----------|----------|----------|
| MET-001 | API P95 レイテンシ | Rust API メトリクスエンドポイント | ms | <500ms | 500〜2000ms | ≥2000ms |
| MET-002 | API エラー率 | Rust API メトリクスエンドポイント | % | <1% | 1〜5% | ≥5% |
| MET-003 | API 可用性（稼働率） | ヘルスチェック結果（直近24h） | % | ≥99.5% | 99.0〜99.5% | <99.0% |
| MET-004 | オフライン同期率 | `sync_events` テーブル（成功/全件） | % | ≥99% | 95〜99% | <95% |
| MET-005 | Outbox DLQ 件数 | `outbox_dlq` テーブル COUNT | 件 | 0 | 1〜10 | ≥11 |
| MET-006 | ハッシュチェーン最終検証経過時間 | `job_executions` BAT-003 最終成功 | h | <168h (7日) | 168〜336h | ≥336h (14日) |
| MET-007 | バックアップ最終成功経過時間 | `job_executions` BAT-001 最終成功 | h | <25h | 25〜48h | ≥48h |
| MET-008 | PostgreSQL 接続数 | pg_stat_activity COUNT | 件 | <50 | 50〜80 | ≥80 |
| MET-009 | ディスク使用率 | Docker ホスト `/backup` パーティション | % | <70% | 70〜85% | ≥85% |
| MET-010 | アクティブアラート件数 | `alerts` テーブル（status='active'） | 件 | 0 | 1〜5 | ≥6 |

### 3.1 SliGauge コンポーネント（CMP-MC-001）× 3個

SliGauge は半円形ゲージ（Slider + Gauge の合成）コンポーネントで、SLI（Service Level Indicator）を視覚的に表示する。色は正常=緑 / 警告=黄 / 危険=赤。

| ウィジェット番号 | 表示メトリクス | コンポーネント識別子 | 配置位置 |
|-----------------|---------------|---------------------|----------|
| SLI Gauge 1 | MET-001: API P95 レイテンシ | CMP-MC-001-A | 上段左 |
| SLI Gauge 2 | MET-003: API 可用性 | CMP-MC-001-B | 上段中央 |
| SLI Gauge 3 | MET-004: オフライン同期率 | CMP-MC-001-C | 上段右 |

#### CMP-MC-001 プロパティ仕様

```typescript
interface SliGaugeProps {
  metricId: string;          // 'MET-001' 等
  label: string;             // 表示ラベル
  value: number;             // 現在値
  unit: string;              // 単位文字列
  warningThreshold: number;  // 警告閾値
  criticalThreshold: number; // 危険閾値
  higherIsBetter: boolean;   // true=高い値が良い（可用性等）, false=低い値が良い（レイテンシ等）
  size: 'sm' | 'md' | 'lg'; // ゲージサイズ
}
```

### 3.2 メトリクスカードグリッド（MET-005〜009）

上段 SliGauge の下に、MET-005〜009 を数値カード（小）として横並びで表示する。各カードは現在値・正常/警告/危険ステータスインジケータ・最終更新時刻を表示する。

| カード | メトリクス | クリック先 |
|--------|-----------|-----------|
| Outbox DLQ | MET-005 | SCR-MC-007（DLQ 管理画面） |
| ハッシュチェーン年齢 | MET-006 | SCR-MC-008（ハッシュチェーン管理画面） |
| バックアップ年齢 | MET-007 | SCR-MC-009（バックアップ状態画面） |
| DB 接続数 | MET-008 | SCR-MC-010（DB 詳細画面） |
| ディスク使用率 | MET-009 | SCR-MC-011（ストレージ詳細画面） |

### 3.3 アクティブアラートリスト

| 項目 | 仕様 |
|------|------|
| 表示件数 | **上位5件**（アラート発生日時の新しい順） |
| 表示順 | 発生日時降順（新しいアラートが上） |
| 表示項目 | アラート ID / 重要度（P1/P2/P3）/ 発生日時 / メッセージ概要 / ステータス（active/acknowledged） |
| ソート変更 | 重要度順への切り替えボタンを提供 |
| 全件表示 | 「すべて表示」リンクで SCR-MC-012（アラート一覧画面）へ遷移 |
| スタイル | P1=赤背景 / P2=橙背景 / P3=黄背景 |

```
[ P1 ] ERR-SYS-005: Outbox DLQ に 3件滞留           2026-05-17 14:23  active
[ P1 ] ERR-SEC-002: ハッシュチェーン破断を検出        2026-05-16 03:15  acknowledged
[ P2 ] BAT-001: バックアップ失敗（2回目）             2026-05-17 02:10  active
[ P2 ] MET-009: ディスク使用率 78%                   2026-05-17 12:00  active
[ P3 ] BAT-004: アーカイブ遅延                       2026-05-16 04:05  active
                                         [すべて表示 →]
```

### 3.4 BAT ジョブ最終実行ステータス

BAT-001〜010 の最終実行結果を表形式で一覧表示する（`job_executions` テーブルを参照）。

| 表示項目 | 内容 |
|----------|------|
| ジョブ ID | BAT-001 等 |
| 最終実行日時 | `completed_at` |
| ステータス | success / failed / skipped（色付きバッジ） |
| 次回予定 | cron 式から算出 |

## 4. 自動リフレッシュ設計

```
デフォルト: 30秒（CFG キー: dashboard.refresh_interval_sec = 30）
設定可能範囲: 10〜300秒（管理画面 SCR-MC-013 から変更）
実装: React useEffect + setInterval（バックグラウンドタブでは一時停止）
API エンドポイント: GET /api/v1/admin/dashboard/metrics（キャッシュ: 10秒 TTL）
```

自動リフレッシュ中はスピナーを表示せず、前回データを保持したまま差分更新する（UX 優先）。ネットワークエラー時はトーストで通知し、次のサイクルで再取得する。

## 6. 個人生産性指標の非表示方針

BR-BUS-029「個人の生産性指標はダッシュボードに表示しない」に基づき、以下の情報は **SCR-MC-001 に表示しない**。

- 作業者別の作業完了件数・速度
- 個人別エラー率・不適合発生率
- 氏名を含むランキング形式の表示

これらのデータを必要とする場合は、RP-006（集計レポート）の組織単位集計を参照する。

## 7. ダッシュボード API 設計（概要）

```
GET /api/v1/admin/dashboard/metrics
Authorization: Bearer <JWT> (system_admin ロール必須)

Response:
{
  "generated_at": "2026-05-17T14:00:00Z",
  "metrics": {
    "MET-001": { "value": 245, "unit": "ms", "status": "ok" },
    "MET-002": { "value": 0.8, "unit": "%", "status": "ok" },
    ...
  },
  "active_alerts": [...上位5件...],
  "job_last_executions": {...BAT-001〜010...}
}
```

---

**本節で確定した方針**

- SCR-MC-001 はシステム全体の健全性を俯瞰する単一の起点画面とし、MET-001〜010 をウィジェットに配置する。SliGauge（CMP-MC-001）3個で API P95 レイテンシ・可用性・オフライン同期率を視覚化し、30秒自動リフレッシュで最新状態を維持する。
- アクティブアラートは発生日時降順の上位5件を表示し、P1/P2/P3 で色分けすることで優先度を即時把握可能とする。DLQ 件数およびハッシュチェーン経過時間からはワンクリックで詳細画面へ遷移できる。
- BR-BUS-029 に基づき個人の生産性指標は一切表示しない。集計データは組織単位の RP-006 帳票を参照先とする。
