# 07 セキュリティ方式設計

本サブ（07_セキュリティ方式設計）は IPA 共通フレーム 2013「2.3 システム方式設計プロセス」のセキュリティ方式設計タスクに対応する。上流の NFR-SEC-001〜015・FR-AU-001〜006・BR-BUS-028/029/035/040〜045 を受けて、認証・認可・暗号化・脅威対策・監査証跡・電子サインの設計命題を確定する。

---

## 章構成

| ファイル名 | タイトル | 一行説明 |
|---|---|---|
| `00_本書の位置づけと識別子規約.md` | 本書の位置づけと識別子規約 | IPA 2.3 カバレッジ・対象要件範囲（NFR-SEC 全件・FR-AU 全件・BR-BUS 9 件）・KEY/PRM 識別子採番規約・他サブとの責任分界を確定する |
| `01_脅威モデル（STRIDE_LINDDUN要素抽出）.md` | 脅威モデル（STRIDE/LINDDUN 要素抽出） | システムの 4 トラストバウンダリを対象に STRIDE 25 件・LINDDUN 18 件の脅威を抽出し、高リスク 6 件の対策骨格を確定する |
| `02_RBAC設計（6ロール×35画面×86FR）.md` | RBAC 設計（6 ロール × 35 画面 × 86 FR） | 6 ロールの責務定義・BR-BUS-040〜045 の実装方式・JWT ロールクレーム・axum ミドルウェアによる権限強制・PRM-001〜010 を確定する |
| `03_認証方式（JWT・PIN・電子サイン）設計.md` | 認証方式（JWT・PIN・電子サイン）設計 | JWT RS256（RSA 4096bit・90 日鍵ローテーション）・PIN（bcrypt 12・3 回ロック）・電子サイン（UUID v7・HMAC-SHA256）・LDAPS 連携の 4 方式を確定する |
| `04_暗号化方式（保存時・通信時・端末ローカル）.md` | 暗号化方式（保存時・通信時・端末ローカル） | 通信時（TLS 1.3）・保存時（AES-256-GCM・pgcrypto）・端末ローカル（SQLCipher v4）の 3 文脈暗号化方式と KEY 管理台帳を確定する |
| `05_監査証跡（AuditTrail）方式.md` | 監査証跡（AuditTrail）方式 | SHA-256 ハッシュチェーン・TBL-032（audit_logs）Append-only 設計・XES エクスポート・週次整合性バッチ・7 年保全方針を確定する |
| `06_キー・秘密情報管理.md` | キー・秘密情報管理 | KEY-001〜009 のライフサイクル全体（生成・配布・ローテーション・廃棄）・Docker Secret 保管・OT 環境対応の自己完結型鍵管理を確定する |
| `07_脆弱性管理・依存バージョン管理.md` | 脆弱性管理・依存バージョン管理 | cargo audit / npm audit 週次実行・CVSS スコア別対応期限・SBOM 生成・OWASP Top 10（2021）A01/A02/A03 優先対応を確定する |
