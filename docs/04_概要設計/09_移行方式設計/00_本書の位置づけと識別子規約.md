# 00 本書の位置づけと識別子規約

本章の責務は、移行方式設計サブ（09）全体の IPA 共通フレーム 2013 上の位置を確定し、カバーする要件範囲・他サブとの責任分界・DES-MIG 識別子の採番規約を権威的に定めることである。本章に記載した規約は移行方式設計の全章に適用される権威規約であり、下位章での再解釈を禁止する。

## 4. 識別子規約（DES-MIG 識別子）

### 4-1. 移行方式設計識別子の体系

本サブで確定する設計命題には、以下の識別子体系を適用する。

| 識別子プレフィックス | 意味 | 採番範囲 | 例 |
|---|---|---|---|
| `DES-MIG-NNN` | 移行方式設計命題（本サブ固有の設計決定） | DES-MIG-001〜DES-MIG-099 | DES-MIG-001 |
| `MIG-NNN` | 上流移行要件 ID（引用時） | MIG-001〜MIG-074 | MIG-005 |
| `D-MIG-NNN` | 移行設計の対象外宣言（Disclaimer） | D-MIG-001〜D-MIG-009 | D-MIG-001 |

### 4-2. DES-MIG 識別子の章別割り当て

| DES-MIG ID 範囲 | 担当章 |
|---|---|
| DES-MIG-001〜DES-MIG-020 | 01_移行戦略の概要設計版 |
| DES-MIG-021〜DES-MIG-040 | 02_マスタ初期投入方式（GUIウィザード） |
| DES-MIG-041〜DES-MIG-050 | 03_ロット履歴・過去記録引継ぎ方式 |
| DES-MIG-051〜DES-MIG-065 | 04_カットオーバー手順とリハーサル設計 |
| DES-MIG-066〜DES-MIG-075 | 05_移行データ品質チェック方式 |
| DES-MIG-076〜DES-MIG-090 | 06_移行ツール構成（CSV_Excel双方向） |

### 4-3. 識別子の記述ルール

- DES-MIG 識別子は設計命題の先頭に `**DES-MIG-NNN**:` 形式で付与する。
- 上流要件 MIG-NNN を根拠として示す場合は「（MIG-NNN 対応）」を行末に付記する。
- 同一設計命題に複数の上流要件が対応する場合は「（MIG-NNN / MIG-NNN 対応）」と列挙する。

---

**本節で確定した方針**
- 本サブは IPA 共通フレーム 2013「2.3 移行方式設計タスク」の全タスクをカバーし、ERP 自動移行・移行コンサルは対象外として確定する。
- 上流要件 MIG-001〜074・FR-MA-001〜003・FR-MA-006 を本サブの設計対象として確定し、他サブとの責任分界を章レベルで明示する。
- 設計命題には DES-MIG-001〜099 の識別子を付与し、D-MIG-001〜009 で対象外宣言を管理する。
