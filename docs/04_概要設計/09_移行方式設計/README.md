# 09 移行方式設計

本サブ（09_移行方式設計）は IPA 共通フレーム 2013「2.3 システム方式設計プロセス」の移行方式設計タスクに対応する。上流の移行要件（MIG-001〜MIG-074）を受けて、紙・Excel ベースの作業指示運用から本システムへの移行を安全かつ確実に実施するための設計命題を確定する。

---

## 章構成

| ファイル名 | タイトル | 一行説明 |
|---|---|---|
| `00_本書の位置づけと識別子規約.md` | 本書の位置づけと識別子規約 | IPA 2.3 移行方式設計タスクのカバレッジ・対象要件範囲（MIG-001〜074）・DES-MIG 識別子採番規約・他サブとの責任分界を確定する |
| `01_移行戦略の概要設計版.md` | 移行戦略の概要設計版 | 並行運用（2〜4 週間）→ カットオーバー方式の採用根拠・移行フェーズ（M-1/M-2/M-3）・リスク対策・DES-MIG-001〜020 を確定する |
| `02_マスタ初期投入方式（GUIウィザード）.md` | マスタ初期投入方式（GUI ウィザード） | GUI 初期構築ウィザードによるマスタ初期投入方式・投入順序・CSV/Excel テンプレート形式・バリデーション規則・トランザクション設計を確定する |
| `03_ロット履歴・過去記録引継ぎ方式.md` | ロット履歴・過去記録引継ぎ方式 | 過去作業記録の任意移行方針・LEGACY_IMPORT イベント設計・ハッシュチェーンでの LEGACY ブロック管理・ALCOA+ 準拠確保を確定する |
| `04_カットオーバー手順とリハーサル設計.md` | カットオーバー手順とリハーサル設計 | カットオーバーチェックリスト（10 項目）・Go/No-Go 判定（T-30 分）・本番切替所要時間（2〜4 時間）・リハーサル実施計画を確定する |
| `05_移行データ品質チェック方式.md` | 移行データ品質チェック方式 | 投入前・投入後の品質チェック項目・ALCOA+ コンプライアンス確認・自動チェックツール構成・品質ゲート基準（クリティカルエラー 0 件）を確定する |
| `06_移行ツール構成（CSV_Excel双方向）.md` | 移行ツール構成（CSV/Excel 双方向） | CSV/Excel 双方向インポート・エクスポートツール構成・チャンク処理（1000 行/トランザクション）・エラーレポート CSV 生成方式を確定する |
| `99_前提制約と本書が約束しないこと.md` | 前提制約と本書が約束しないこと | D-MIG-001〜004（ERP 自動移行・移行コンサル・旧システム廃棄・データクレンジング詳細手順）を対象外として確定する |
