# 09 カメラ・ファイル取込 IF（IF-007）

本章の責務は、端末内蔵カメラ（IF-007）を使用した写真証拠取得と SHA-256 計算タイミングを確定することである。

---

## 1. カメラ IF の設計方針

### 1-1. 技術選定

| プラットフォーム | 使用 API | 実装 |
|---|---|---|
| React Native（Android/iOS）| `react-native-camera` / Expo Camera | MOD-FE-HA-004 |
| Windows | `react-native-windows` Camera API | MOD-FE-HA-004（プラットフォーム分岐）|

### 1-2. 写真品質基準（BR-BUS-033）

```typescript
// 撮影後の品質チェック
function validatePhoto(photo: CapturedPhoto): ValidationResult {
  // 最低解像度チェック（1280×720）
  if (photo.width < 1280 || photo.height < 720) {
    return { valid: false, error: 'ERR-VAL-003', message: '解像度が不足しています' };
  }
  return { valid: true };
}
```

### 1-3. SHA-256 計算タイミング

```
撮影完了（端末）
  → Exif 削除（プライバシー保護・NFR-SEC）
  → JPEG 圧縮（品質 85%）
  → SHA-256 計算（file_hash 生成）  ← ここで計算（改ざん前のオリジナル）
  → NAS への送信（API-evidences-001 multipart）
  → バックエンドも受信後 SHA-256 を再計算し、端末送信値と突合
  → 一致した場合のみ TBL-009 に INSERT
```

### 1-4. 動画・音声証拠

- 動画: React Native の Video Recorder（任意・FR-EV-010 の音声メモ相当）
- ファイルサイズ上限: 100MB（設定）
- 動画も SHA-256 を計算して NAS 保存・メタデータのみ DB

---

**本節で確定した方針**
- **SHA-256 は Exif 削除・JPEG 圧縮後の端末で計算し、バックエンドが再計算して突合することで ALCOA+ Original（オリジナルデータの完全性）を二重に担保する。**
- **写真品質（最低 1280×720・BR-BUS-033）をアプリ層で事前チェックし、品質基準未達の場合は ERR-VAL-003 でブロックする。**
