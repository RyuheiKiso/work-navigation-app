# 12 外部 IF テスト方式と契約テスト

本章の責務は、外部インタフェースのテスト方式と、スキーマ駆動の Contract Test 設計を確定することである。

---

## 1. API テスト戦略

### 1-1. テストピラミッドでの IF テスト位置付け

```
E2E（少）
  ↑ IF-001/002 の統合テスト（モック親機使用）
結合（中）
  ↑ 各 API エンドポイントの Handler テスト（sqlx testcontainers）
単体（多）
  ↑ Domain サービスのロジックテスト（モック Repository）
```

### 1-2. Contract Test（OpenAPI スキーマ駆動）

```rust
// Rust 側: テストで実際の axum ルーターを起動
#[tokio::test]
async fn test_post_work_event_contract() {
    let app = create_test_app().await;
    
    let response = app
        .post("/api/v1/work-executions/{id}/events")
        .header("Idempotency-Key", Uuid::now_v7().to_string())
        .header("Authorization", "Bearer {test_jwt}")
        .json(&test_payload())
        .await;
    
    assert_eq!(response.status(), 200);
    
    // OpenAPI スキーマとのレスポンス突合
    let schema = load_openapi_schema();
    assert!(schema.validate_response("postWorkEvent", &response.body()));
}
```

### 1-3. モック親機（IF-001/002 テスト用）

```rust
// テスト用のモック親機サーバー（wiremock を使用）
let mock_server = MockServer::start().await;

Mock::given(method("POST"))
    .and(path("/api/v1/sync/outbox/inbound"))
    .respond_with(ResponseTemplate::new(200))
    .mount(&mock_server)
    .await;
```

## 2. セキュリティテスト方針

| テスト | ツール | 頻度 |
|---|---|---|
| OWASP A01（認証欠如）| 手動 + curl | リリース前 |
| OWASP A02（JWT 検証欠如）| jose tool | CI |
| OWASP A03（インジェクション）| sqlmap（読み取りのみ）| リリース前 |
| 証明書期限検証 | openssl s_client | 日次（BAT-009）|
| JWT ブラックリスト検証 | 自動テスト | CI |

---

**本節で確定した方針**
- **OpenAPI 3.1 スキーマを Contract として使用し、Handler テストでレスポンスを自動スキーマ検証することで API 後退を防止する。**
- **IF-001/002 テストには wiremock による モック親機を使用し、実際の親機に依存しないテストを可能にする。**
