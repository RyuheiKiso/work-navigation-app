## 4. API 設計の不変制約

| 制約 | 内容 |
|---|---|
| DELETE 禁止 | Append-only 原則により全テーブルで物理削除禁止。廃止は `POST /{id}/deactivate` |
| Idempotency Key 必須 | 全書き込み API に `Idempotency-Key: {UUID v7}` ヘッダ必須 |
| TLS 1.3 必須 | HTTP 平文通信禁止 |
| RFC 9457 エラー | 全エラーは Problem Details 形式 |
| OpenAPI 3.1 | 全エンドポイントを `openapi.yaml` で仕様化 |

---

**本節で確定した方針**
- **本サブは IPA 2.3/2.4 の IF 方式設計タスクを担当し、IF-001〜007 全 7 件を担当章で完全にカバーする。**
- **API 識別子の一次採番権は本サブにあり、付録/99 採番台帳の API セクションで管理する。**
- **DELETE 禁止・Idempotency Key 必須・TLS 1.3 必須・RFC 9457 の 4 不変制約を全 API に適用する。**
