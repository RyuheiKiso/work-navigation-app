# 99 前提制約と本書が約束しないこと

本章の責務は、外部インターフェース設計サブが「対象外と判断する」事項を宣言することである。

---

## 1. OT 直結（永続的非提供）

**PLC・SCADA・OPC-UA・MQTT による OT との制御信号接続は対象外と判断する。**

- 理由: P4 アーキテクチャ原則・IT 領域のみを設計範囲とする
- 担保: IF 一覧（IF-001〜007）に OT 系が含まれないことを §01 で確認

---

## 2. クラウド SaaS 連携（ver1.0.0 対象外）

**AWS / Azure / GCP の各種 SaaS（S3・Azure Blob・BigQuery 等）への直接 IF は対象外と判断する。**

- 理由: オンプレミス前提・IT 担当 1 名・ライセンスコスト
- 将来: Enterprise 版でのクラウドバックアップ検討余地あり

---

## 3. GraphQL / gRPC / WebSocket（対象外）

**GraphQL・gRPC・WebSocket による API 提供は対象外と判断する。**

- 理由: REST + OpenAPI 3.1 で全要件を満たす。追加プロトコルは IT 担当 1 名の運用負荷を増大させる
- 代替: Server-Sent Events（SSE）は将来検討余地あり（SCR-MC-001 ダッシュボードの自動更新）

---

## 4. 公開 REST API 提供（対象外）

**本アプリ自体が外部パートナーへの公開 API を提供することは対象外と判断する。**

- 理由: 導入先組織が「子機」として親機に接続する方向のみ（IF-001/002）
- 将来: Enterprise でのマルチサイト API を検討

---

**本節で確定した方針**
- **OT 直結・クラウド SaaS・GraphQL/gRPC/WS・公開 REST API の 4 事項を外部 IF 設計の対象外と判断し、IF-001〜007 の外に追加 IF を作成しないことを確定した。**
