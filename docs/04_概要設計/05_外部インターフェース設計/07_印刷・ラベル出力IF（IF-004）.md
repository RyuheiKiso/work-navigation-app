# 07 印刷・ラベル出力 IF（IF-004）

本章の責務は、帳票印刷（RP-001〜006）の出力 IF（IF-004）の設計を確定することである。

---

## 1. 印刷方式の確定

### 1-1. PDF 生成 → IPP 送信

```
バックエンド（Rust）
  → PDF 生成（headless Chrome / Rust PDF crate）
  → IPP（Internet Printing Protocol）経由で共有プリンタに送信
  → または NAS に保存して手動印刷
```

### 1-2. 帳票形式の優先順位

| 形式 | 用途 | 生成方法 |
|---|---|---|
| PDF/A-3（ISO 32000-3）| 顧客提出・規制対応・長期保存 | Rust crate（printpdf / lopdf）|
| XES 1.0（IEEE 1849）| プロセスマイニング・監査 | カスタム XML 生成 |
| CSV | 集計データのエクスポート | Rust の `csv` crate |

### 1-3. IF-004 の接続仕様

| 項目 | 内容 |
|---|---|
| プロトコル | IPP（TCP/631）|
| 認証 | プリンタ側の設定（本アプリは認証情報を保持しない）|
| 構成 | CFG（プリンタの IP アドレス・キュー名）で設定 |
| 対応 OS | Windows Server 2022 の IPP クライアント |

印刷機能は「任意」（MoSCoW: Should）。IPP が利用不可の環境では NAS 保存 + 手動印刷で代替する。

---

**本節で確定した方針**
- **IF-004 は PDF/A-3 生成 + IPP 送信を主方式とし、IPP 不可時は NAS 保存 + 手動印刷でフォールバックする設計を確定した。**
- **帳票生成は Rust crate で PDF/A-3 準拠の生成を行い、XES/CSV も同一パイプラインで処理する。**
