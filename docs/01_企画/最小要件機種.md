# 最小要件機種

> 対応 §: ロードマップ §6.3 §6.4 §11.7.1 §13.1 §22.1
> 対象読者: 導入企業の調達／IT 部門、メンテナ
> 改訂サイクル: §22.1 半期サイクル

§6.4 受入観点「最小要件代表機が Android／Windows 各 1 機種以上指定され、その機種で §5.2 の性能目標が CI（または半期実機検証）で達成される」を満たすため、本書で代表機種を版管理する。

## 1. 代表機種（初期決定 2026-05）

| OS 系統 | 機種 | 区分 | 採用理由 |
| --- | --- | --- | --- |
| Android | Zebra ET60（10 インチ／IP65） | 最小要件代表機 | 産業用堅牢タブレットの普及機。手袋対応／屋内輝度 1,000nits／MIL-STD-810H／Android 13 標準 |
| Android（推奨） | Panasonic TOUGHBOOK A3（10 インチ／IP65） | 推奨代表機 | ホットスワップ電池／高耐熱／製造現場での導入実績多数 |
| Windows | Panasonic TOUGHBOOK FZ-G2（10 インチ／IP65） | 最小要件代表機 | Windows 11 標準／粉塵対応／屋外併用可（1,000nits） |

## 3. CI／半期実機検証

§6.4 受入観点を満たすため、次のいずれかで §5.2 性能目標達成を確認する。

| 方式 | 概要 | 周期 |
| --- | --- | --- |
| エミュレータ CI | Android Emulator（API 33／ハード仕様模倣）／Windows 11 仮想機 | プッシュごと |
| 半期実機検証 | 代表機 4 機種に対し §5.2 全指標と §11.7.1 バッテリー寿命を実測 | §22.1 半期サイクル |

実機検証の結果は `docs/04_運用/環境別検証.md` に記録する（§9.5.5）。

## 4. 機種追加・撤退規約

- **追加**: §22.4 是正フロー or §22.2 即時トリガー（製造業向け新機種の登場、現行機種の生産停止）。
- **撤退**: 同 §。撤退理由は §30 Type 2（撤退コスト低）として PR ログに記録するが、§24.2 への追加は不要。
- **新規 OS バージョン下限変更**（§6.1）: Android 12／Windows 11 を下限とするが、Microsoft／Google のサポート終了（EOL）に追従して下限を上げる場合は ADR で記録する（§30 Type 1 候補）。

## 5. 代表機種で達成すべき指標（再掲）

| 指標 | 目標 | 出典 |
| --- | --- | --- |
| 画面遷移応答（中央値／p95） | ≤ 200ms ／ ≤ 500ms | §5.2 |
| タッチ操作フィードバック | ≤ 100ms | §5.2 |
| アプリ起動（コールド／ウォーム） | ≤ 3s ／ ≤ 1s | §5.2 |
| 連続稼働 24h でメモリ漸増なし | 達成 | §5.2／§13.1 長時間連続運転 |
| 8 時間連続運用時のバッテリー消費 | ≤ 80% | §11.7.1 |

## 6. 受入観点（§6.4 整合）

- Android／Windows それぞれで最小要件代表機が指定されていること（Zebra ET60／TOUGHBOOK FZ-G2）。
- §22.1 半期サイクルで本書が更新されていること。
- 機種で §5.2 性能目標が CI または半期実機検証で達成されていることが §22.1 集計で確認可能であること。
- 機種追加・撤退の履歴が PR ログで追跡可能であること。
