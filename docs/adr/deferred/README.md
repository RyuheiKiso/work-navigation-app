# 遅延決定（DEFER-）

> 対応 §: ロードマップ §33 §2.2 §22.1
> 対象読者: メンテナ、§22.1 改訂サイクルレビュア
> 改訂サイクル: 期日到来時に再評価／§22.1 半期サイクルで集計

§33「意思決定の遅延戦略」に従い、**明示的に保留した決定** を本ディレクトリに保管する。沈黙の妥協（§2.2）と区別するための手続きである。

## 1. 保留 ID 規約（§33.2）

```
DEFER-<YYYYMMDD>-<short>
```

例: `DEFER-20260509-cncf-graduation.md`

## 2. 保留対象の判定（§33.1）

次の 3 条件をすべて満たす決定のみ遅延可能。

1. **可逆性**: §30 Type 2 に分類できる（撤退コスト低）。
2. **未確定の入力**: 決定根拠となる情報が公開資料・OSS／規格動向で 12 ヶ月以内に明確化する見込み。
3. **下流非ブロック**: 当該決定を保留しても下流タスク（§23 依存関係）が止まらない。

## 3. 必須記載事項（§33.2）

各 `DEFER-` ファイルには次を記載する。

| 項目 | 規定 |
| --- | --- |
| 保留 ID | `DEFER-<YYYYMMDD>-<short>` |
| 保留対象 | 決定の主題（1 行） |
| 保留理由 | §33.1 のどの条件で遅延可能と判定したか |
| 既定の暫定値 | 保留中に採用する安全側の暫定値（§9.2.3 デフォルト効果） |
| 再評価トリガー | 期日・入力到来・上流の §22.2 トリガー等 |
| 影響評価 | 保留により発生する技術的負債・受入観点の暫定 |

## 4. 暫定値の安全側既定（§33.3）

| 領域 | 安全側暫定の例 |
| --- | --- |
| しきい値（性能・容量） | §5.2 中規模目標値を採用 |
| 既定権限 | deny（§17.4） |
| 既定保持期間 | 365 日（§20.3） |
| 既定ロケール | 日本語＋英語（§11.3） |
| 既定認証 | ID＋パスワード（§10.5） |

## 5. 再評価の強制（§33.4）

- 保留 ID の再評価期日が CI（`scripts/lint-deferred.sh`、未整備）で監視され、期日超過の保留があれば fail。
- 保留期間は **最長 12 ヶ月**。延長は ADR（§9.4）に理由を残し、§22.1 で承認。
- 期日到来時に「再決定」「延長」「Type 1 への昇格」のいずれかを選び、結果を ADR 化。

## 7. 受入観点（§33.6）

- 本ディレクトリの保留 ID が CI で再評価期日を監視されること（CI 整備後）。
- §2.2 沈黙の妥協ラベル（`silent-compromise`）と `DEFER-` の Issue 件数が独立に集計されていること。
- 保留期間 12 ヶ月超過の延長件数が §22.1 で公開されていること。
- 主要 ADR（§30.2）から「遅延候補としての検討と却下／採用」の経路がたどれること。
- 暫定値が §33.3 表のいずれかに合致しているか、または ADR で個別の安全側根拠が示されていること。
