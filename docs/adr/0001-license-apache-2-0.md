# ADR-0001: ライセンス: Apache-2.0

> 提案日: 2026-05-09
> 採用日: 2026-05-09
> 提案者: @RyuheiKiso
> 関連 §: ロードマップ §19.1 §17 §24.2 §26 §29 R-012

## Status

Accepted

## Type

Type 1（不可逆。再ライセンスは §19.5.3 で禁止）

## Context

- 本プロジェクトはメモ L5「OSS として公開予定」、L65「導入企業のシステム開発部門がアドオン機能を開発できる」に従い、商用アドオン許容のライセンスが必要。
- メモ L59「妥協禁止」、§4.4「ライセンス・TCO ◎」を裏付けるため、ライセンス費ゼロかつ特許リスクの低い OSS が要件。
- ライセンス選定は **不可逆**（より制限的なライセンスへの後退は §19.5.3 で禁止、より寛容な側への移行は実質的にユーザー混乱・契約上のフォーク）であり、慎重な選定が必要。

## Decision

**Apache License 2.0** を採用する。

- 寛容ライセンス。
- 明示的な特許許諾を含む（特許リスク回避）。
- 商用アドオン許容（§17／§19.4.3）。
- OSS の TCO 差を構造的優位として §4.9「完全 OSS／Apache-2.0／個人開発の透明性」に反映。

## Alternatives

| 代替案 | 概要 | 却下理由 |
| --- | --- | --- |
| MIT | 寛容だが特許許諾なし | 特許リスクを残す。長期的な企業導入時のリスク |
| MPL-2.0 | ファイル単位の弱コピーレフト | アドオン開発に対するライセンス整合の説明コスト増 |
| GPLv3 | 強コピーレフト | 企業内プロプライエタリアドオン困難（メモ L65 と不整合） |
| AGPLv3 | ネットワーク利用にも copyleft | SaaS 非提供方針（§1）でも企業導入時の解釈リスク |
| BSL（Business Source License） | 時限式の商用制限 | OSS 認定外。§19.4 中立化と整合せず |

## Consequences

- **正の帰結**: 商用アドオン互換、特許リスク低、エコシステムが OSS 標準と整合、OpenSSF／Linux Foundation 寄贈時に互換問題なし（§19.4／§19.5.3）。
- **負の帰結**: コピーレフトを期待する貢献者（GPL 文化圏）から「保護が弱い」と評される可能性。
- **影響範囲**: §19.1（ライセンス）、§17.6（アドオン署名・配布）、§19.5.3（再ライセンス禁止）、§26（GPL／非寛容ライセンス採用しない）、`LICENSE`／`CONTRIBUTING.md`／`MAINTAINERS.md`。

## Type 1 撤退条件

- §19.5.3「より制限的なライセンスへの後退は禁止」に従い、後退は採用しない。
- フォーク発生時のみ協調可能性を Discussions で対話する（§19.5.3）。
- より寛容なライセンス（MIT 等）への移行は理論上可能だが、特許許諾保護の喪失と引き換えとなり、現時点で採用判断材料がない。
