# ADR-0007: 既定認証: ID＋パスワード（高度認証はアドオン化）

> 提案日: 2026-05-09
> 採用日: 2026-05-09
> 提案者: @RyuheiKiso
> 関連 §: ロードマップ §10.5 §10.5.0 §10.5.1 §17 §27 F-006 §29

## Status

Accepted

## Type

Type 1（既定認証の選択は永続データ・端末配布形態と密接に結びつくため、変更は移行コスト大）

## Context

- メモ L54「アプリへのログインはユーザー ID とパスワードにする」が一次入力。
- §29 R-006（個人情報保護法／GDPR 改定）／§27 F-006（認証共有による監査証跡信頼性喪失）への対応は本体保守範囲を最小化する戦略が必要。
- 大企業・防衛・医療等の高度認証要件は導入企業ごとに異なる。

## Decision

- 既定認証は **ユーザー ID＋パスワード** とする（§10.5.0）。
  - パスワードは Argon2id 派生鍵を OS Keystore（Android）／DPAPI（Windows）に保護保存（§10.5.1／ADR-0004）。
  - オフライン継続ログインは最大 7 日（§10.5.1 ／§21 注 9）。
- 高度認証は **アドオン化** する（§17）。
  - SAML / OIDC SSO: capability `auth.extend:saml`／`auth.extend:oidc`
  - Passkey / WebAuthn: capability `auth.extend:webauthn`
  - スマートカード: capability `auth.extend:piv`
  - 顔認証: capability `auth.extend:face`

## Alternatives

| 代替案 | 概要 | 却下理由 |
| --- | --- | --- |
| Passkey / WebAuthn を既定 | 漏洩耐性が高い | 産業現場の共有端末文化／パスワード覚え書き運用と不整合 |
| SAML / OIDC SSO を既定 | 大企業の既存 ID 体系活用 | 個人開発の本体保守範囲が拡大／中小製造業で要件外 |
| 顔認証を既定 | 手袋環境で利便性高い | 偽造耐性・公平性検証が個人開発の責務範囲を超える |
| パスワードレス（メールリンク等） | OTP | 企業内ネットワーク（§1）でメール基盤前提が崩れる |

## Consequences

- **正の帰結**:
  - 中小製造業の現場文化（共有端末・パスワード覚え書き）に整合。
  - 本体保守範囲を最小化（§19.5 個人開発の維持コスト軽減）。
  - 高度認証は導入企業要件に応じてアドオン選択可能。
- **負の帰結**:
  - 共有 ID 文化下で監査証跡の信頼性が下がる（§27 F-006 AP=H）。
  - パスワードリセット運用が現場負担となる場合がある。
- **影響範囲**: §10.5（認証）、§10.5.0（拡張認証アドオン化）、§10.5.1（オフライン継続ログイン）、§17（アドオン）、§27 F-006、§29 R-006。

## Type 1 撤退条件

- 産業現場の主流が Passkey／顔認証へ移行し、ID＋パスワードの利便性が逆転した時点（§22.1 半期で判定）。
- パスワード認証が規制遵守上不可となる時代（NIST SP 800-63B 等の改定）。
- 撤退時の代替: ADR で再評価し、Passkey 既定＋ ID＋パスワードのアドオン化を再判断。
