# ADR-0008: iOS／iPadOS 非対応（閲覧用途を除く）

> 提案日: 2026-05-09
> 採用日: 2026-05-09
> 提案者: @RyuheiKiso
> 関連 §: ロードマップ §6.1 §6.2 §26 §27 F-016

## Status

Accepted

## Type

Type 1（プラットフォームサポートは配布チャネル・テスト基盤・ライセンス契約と結びつき、後追い拡張のコストが大）

## Context

- メモ L13 が Android／Windows と明記している。
- 産業用堅牢タブレット市場の中心は Android／Windows（Panasonic Toughbook、Zebra ET、Getac 等）。iPad は IP 等級・MIL-STD・ホットスワップ電池などの産業要件で不利。
- 個人開発の維持コスト（§19.5）として 3 OS 並行は過大。
- iOS／iPadOS の Apple Developer Program 加入・配布審査は OSS の透明性原則と部分的に矛盾する場面がある。

## Decision

- **iOS／iPadOS は対象外** とする（§6.2 §26）。
- iPad ユーザーは設定 Web UI（§7.2）の **閲覧用途** で対応可能（ブラウザでアクセス）。
- 端末作業ナビアプリは Tauri により Android／Windows をサポートする。
- iOS 対応の要望は §22.4 是正フローで起票されても §17 アドオン化の対象とせず、原則 `anti-goal` ラベルでクローズする（§26.2）。

## Alternatives

| 代替案 | 概要 | 却下理由 |
| --- | --- | --- |
| iOS 対応（PWA） | Web ベース | オフライン・カメラ／ハプティック制約／§10 の機能制約が大きい |
| iOS 対応（Tauri Mobile） | Tauri 公式の iOS サポート | 個人開発の維持コスト／産業要件（IP 等級・MIL-STD）で機種選択肢が限られる |
| iOS 対応（React Native 別実装） | フルネイティブ | 並行スタック保守不可／§19.5 コスト超過 |

## Consequences

- **正の帰結**:
  - 個人開発の維持コスト最小化（§19.5）。
  - 産業用堅牢タブレットの主流（Android／Windows）に集中。
  - §4.4「対応端末 ◎」の戦略がブレない。
- **負の帰結**:
  - iPad を業務用途に標準化している企業からは選定外となる。
  - 設定 UI の閲覧用途で iPad を使う場合、表示崩れの監視が必要（§27 F-016 AP=L）。
- **影響範囲**: §6.1（動作前提）、§6.2（iOS 不採用の戦略的理由）、§26（アンチゴール）、§27 F-016。

## Type 1 撤退条件

- 産業用堅牢 iPad（IP 等級・MIL-STD 対応モデル）が普及し、競合が iOS で先行差別化を達成した場合（§22.3 撤退条件）。
- 個人開発から複数メンテナ体制（§19.4 +12 ヶ月）へ移行し、iOS 担当オーナーが任命された場合。
- 撤退時の代替: ADR で再評価し、iOS 対応を §17 アドオン or 公式サブシステムとして起票。
