# ADR-0009: 初期投入における 1 PR ≤ 500 行差分の例外

> 提案日: 2026-05-10
> 採用日: 2026-05-10
> 提案者: @RyuheiKiso
> 関連 §: ロードマップ §9.4 §22.4 §32

## Status

Accepted

## Type

Type 2（可逆。本セッション以降は通常の §9.4 規律に戻す）

## Context

- §9.4「1 PR ≤ 500 行差分」はレビュアビリティ研究（Bacchelli & Bird 2013）に基づく規律であり、通常の運用では遵守すべき。
- 一方、本リポジトリはロードマップ確定（2026-05-09）からドキュメント／コード骨格を **連続 3 セッション** で一括投入する局面にある。
  - Session 1: ドキュメント 63 ファイル（§34 派生資料／LICENSE／ADR 8 件 ほか）
  - Session 2: コード骨格 71 ファイル（バックエンド 5 crate／端末 Tauri+React／設定 UI／アドオン SDK／scripts）
  - Session 3: 機能拡充（auth／idempotency／追加アドオン 2／業界テンプレ／追加 scripts）
- いずれのセッションも、**初期投入の性質上 500 行以下の PR への分割が困難**:
  - クリーンアーキテクチャ層構造を 5 crate に分割しているため、`cargo check` が通る最小単位でも 1,000 行を超える。
  - ドキュメント間相互参照（§24 追跡可能性）を満たすには、複数派生ファイルを **同時に** 整合させる必要がある。
  - 部分投入では `scripts/glossary-lint.sh`／`scripts/lint-deferred.sh` 等の自動チェックが偽陰性／偽陽性を出す。

## Decision

- 本初期投入（3 セッション分の差分）は **§9.4 1 PR ≤ 500 行差分の例外** とし、一括コミット／PR を許容する。
- 例外の根拠と範囲を `CHANGELOG.md` の `[Unreleased]` セクションに「規律例外」として記録する（既に Session 2 完了時点で記録済）。
- 例外は **本初期投入限定** であり、次回以降の PR は通常の §9.4 規律に戻る（再発防止策は不要、初期投入の特殊性として受容）。

## Alternatives

| 代替案 | 概要 | 却下理由 |
| --- | --- | --- |
| 厳格に 500 行以下の PR に分割 | §9.4 規律完全遵守 | クリーンアーキ層構造／§24 追跡可能性の整合検証が PR 単位で完結せず、レビュアビリティが逆に低下 |
| 500 行毎に複数の WIP PR を起こし最後にまとめてマージ | 形式上の遵守 | 「形式遵守の沈黙の妥協（§2.2）」に該当。実態は同じ |
| 例外を ADR 化せず暗黙に運用 | 軽量 | §2.2 沈黙の妥協検出条件に違反（決定の出所が記録されない） |
| 永久例外（500 行制限を撤廃） | 自由度最大 | §9.4 規律の趣旨（レビュアビリティ）に反する |

## Consequences

- **正の帰結**: 初期投入が一括で整合した状態で完結し、レビュアビリティが「初期コミットを 1 度通読する」形になる。
- **負の帰結**: 初期 PR のレビュー負荷は通常より大きい。本初期投入のレビュー所要時間は §19.4 PR レビュー SLA（7 日）を超える可能性があり、本 ADR と CHANGELOG の参照で誘導する。
- **影響範囲**: 本初期投入のみ。次回以降の PR には影響しない。

## §22.3 撤退条件（Type 2 のため不要）

本 ADR は Type 2（可逆）。初期投入完了後、自動的に「適用なし」となる。
