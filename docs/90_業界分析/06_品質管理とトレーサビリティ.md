# 品質管理とトレーサビリティ

## 1. 品質管理の歴史的系譜

### 1.1 検査中心時代 (〜1940 年代)

初期の品質保証は「作ってから検査する (Inspection)」モデルに依拠した。不良品を後工程・市場に流さないことを目的とするが、製造コストの削減には繋がらない受動的手法である。

### 1.2 統計的品質管理 — W. A. Shewhart & W. E. Deming

Walter A. Shewhart (1891–1967) は 1924 年に管理図 (Control Chart) を考案し、工程変動を「偶然原因 (Common Cause)」と「異常原因 (Special Cause)」に分けて管理する統計的工程管理 (SPC) の基礎を築いた。

W. Edwards Deming (1900–1993) は第二次世界大戦後の日本産業再建期 (1950 年代〜) に SPC を講義し、PDCA サイクル (Plan-Do-Check-Act) の普及に寄与した。Deming の「14 の経営原則」は品質を経営戦略の核に置くことを説く。

J. M. Juran (1904–2008) は「品質は設計される (Quality is Planned)」という視点から品質計画・品質管理・品質改善の 3 要素を体系化 (Juran Trilogy)。コストに占める品質の重要性を経営層に訴えた。

### 1.3 TQC / TQM

日本では 1960〜70 年代に「全社的品質管理 (Total Quality Control: TQC)」が展開され、品質管理を一部門の専門業務から全員参加型の経営活動へと転換した。国際的には TQM (Total Quality Management) として再解釈され、ISO 9001 の哲学的基盤をなす。

## 2. QC の主要手法

### 2.1 QC 七つ道具

データの視覚化・分析に用いられる 7 種の基本手法。製造現場の QC サークル活動で広く使用される。

| 道具 | 用途 |
|---|---|
| 特性要因図 (魚の骨図) | 問題の原因を 4M 軸で体系化 |
| パレート図 | 問題の重要度を累積割合で視覚化 (80:20 則) |
| ヒストグラム | データの分布形状を把握 |
| 管理図 | 工程変動の時系列監視 |
| 散布図 | 二変量間の相関関係の可視化 |
| チェックシート | データ収集・記録の標準化 |
| 層別 | データを変動要因別に分類分析 |

### 2.2 新 QC 七つ道具

言語データ・非定量的情報の整理に適した手法群 (1977 年、日本科学技術連盟)。

| 道具 | 用途 |
|---|---|
| 親和図 | 言語情報をグループ化し、混沌から体系を見出す |
| 連関図 | 複雑な因果関係のネットワークを可視化 |
| 系統図 | 目的達成のための手段を階層展開 |
| マトリックス図 | 複数要因間の関連の強さを格子表示 |
| マトリックス・データ解析 | 数量化されたマトリックスの主成分分析 |
| PDPC (過程決定計画図) | 計画の障害と対策を事前に網羅 |
| アロー・ダイアグラム | 日程計画の依存関係を有向グラフで表現 |

### 2.3 FMEA (Failure Mode and Effects Analysis)

設計・工程の潜在的故障モードを事前に洗い出し、影響の深刻度 (Severity)・発生頻度 (Occurrence)・検出困難度 (Detection) の積で危険優先度 (RPN: Risk Priority Number) を算出し、対策優先順位をつける手法。

```
RPN = S × O × D    (各項 1〜10 のスコア)
```

自動車産業では AIAG & VDA の共同 FMEA ハンドブックが標準として採用される。

## 3. ISO 9001 品質マネジメントシステム

ISO 9001 は品質マネジメントシステム (QMS) の国際規格であり、2015 年版 (ISO 9001:2015) で「リスクベースの考え方」が強化された。認証取得により顧客・規制機関への品質保証能力を対外的に証明できる。

主要要求事項 (箇条 8「運用」より):

- 製品・サービスに関するプロセスの計画・管理
- 設計・開発の管理 (設計変更の文書化含む)
- **外部から提供されるプロセス・製品・サービスの管理**
- 生産・サービス提供の管理 (**識別とトレーサビリティの要求含む**)
- 不適合のアウトプットの管理

## 4. トレーサビリティの定義と構造

### 4.1 ISO 定義

ISO 9000:2015 は**トレーサビリティ (traceability)** を次のように定義する。

> 「対象の履歴、適用、又は所在を追跡できる能力。」

製造業においては、製品・部品の「どこから来たか」「どこへ行ったか」「いつ・誰が・何を用いて製造したか」の記録連鎖を意味する。

### 4.2 追跡方向による分類

| 方向 | 内容 | 用途 |
|---|---|---|
| 順方向 (Forward Traceability) | 原材料→完成品→出荷先の追跡 | 流通経路の把握、リコール範囲の特定 |
| 逆方向 (Backward Traceability) | 完成品→部品→原材料ロットへの遡及 | 不具合原因の特定、サプライヤ追跡 |

### 4.3 トレーサビリティの記録要素

製造現場でのトレーサビリティ確保に必要な最小限の記録情報:

| カテゴリ | 記録項目の例 |
|---|---|
| 製品識別 | 品番、シリアル番号、ロット番号 |
| 工程情報 | 作業日時、設備 ID、工程番号 |
| 人員情報 | 作業者 ID、資格・技能レベル |
| 材料情報 | 原材料ロット、部品ロット、仕入先 |
| 検査情報 | 測定値、判定結果、測定器識別 |
| 設備情報 | 設備点検記録、段取り条件 |

## 5. 業界別規制とトレーサビリティ

| 業界 | 規制・規格 | 主なトレーサビリティ要件 |
|---|---|---|
| 食品 | HACCP (Codex 委員会)、食品衛生法 (日本) | 原材料のロット追跡、温度管理記録 |
| 医薬品 | GMP (Good Manufacturing Practice), ICH Q10 | バッチ記録、製造工程の完全な文書化 |
| 自動車 | IATF 16949, AIAG CQI シリーズ | 部品のシリアル管理、変更点管理 |
| 航空宇宙 | AS9100 | 形状・適合性・機能の完全記録保持 |
| 医療機器 | ISO 13485, MDR (EU) | UDI (Unique Device Identification)、PMS (市販後安全活動) |

### 5.1 リコール対応とトレーサビリティ

2000 年代以降の大規模製品リコールは、不完全なトレーサビリティが被害拡大と対応コスト増大を招くことを示した。効率的なトレーサビリティシステムは:

- 影響範囲の迅速な特定 (顧客への連絡対象を最小化)
- 原因工程・材料ロットへの即時遡及
- 再発防止のための工程改善に直結する

## 6. 批判と限界

### 6.1 TQM の「感情労働」無視批判

TQM および ISO 9001 が要求する「全員参加型の品質活動」は、製造現場における作業者の心理的・情動的コストを組織的に不可視化するという批判がある。Hochschild が *The Managed Heart* (1983) で概念化した「感情労働 (Emotional Labour)」の視点から見ると、品質問題を報告・提案する行為は単なる情報提供ではなく、管理者・同僚との関係調整を含む複合的な社会行為である。

QC サークル活動が形骸化する背景には、問題提起が「ネガティブな烙印」につながりかねないという心理的安全性の欠如がある。Edmondson (*Teaming*, 2012) の研究は、心理的安全性が低い職場では改善提案行動が抑制されることを示しており、TQM の「全員参加」という建前と「問題を起こした部署・人物への批判」という組織力学の矛盾が現実に発生している。

### 6.2 ISO 9001 認証の形式主義化問題

ISO 9001 認証が「顧客への品質保証」から「認証維持のための文書整備」に目的がすり替わる「認証の形式主義化」は国際的に広く報告されている。Boiral & Roy (2007) は ISO 9001 認証企業の調査から、多くの組織が認証要件を「日常業務とは別のペーパーワーク」として取り扱っており、マネジメントシステムが実際の品質改善と連動していないことを明らかにした。

認証審査の限界として、外部審査員がサンプリングで行う文書確認・インタビューでは、現場の実態との乖離を捉えることが困難である。審査の「事前通告」慣行が常態化している環境では、審査時だけ整備された状態が演出される「審査準備モード」が生じる。この問題は ISO 9001 が「品質」を体系化する力を持ちながらも、形式的遵守と実質的改善の間のギャップを制度的に生みやすい構造を持つことを示している。

### 6.3 トレーサビリティ = 品質の代理指標問題

「記録が取れている = 品質が保証されている」という等式は成立しない。トレーサビリティの確保は問題発生後の遡及調査を可能にするが、予防的品質保証とは本質的に異なる。Juran が指摘したように、品質の要は「工程の設計」にあり、記録の蓄積はそれを代替するものではない。

特に電子トレーサビリティシステムの普及により、「記録入力の完了 = 作業の正確な実施」という信念が現場に定着しやすい。しかし入力者が誤った値を記録した場合、あるいは実際の作業と異なるパラメータを入力した場合、トレーサビリティシステムは虚偽の証拠を生産することになる。このトレーサビリティの「証拠品質」問題は、システム設計における二重確認・センサ自動入力・作業実績との照合機能の重要性を示す。

### 6.4 本アプリへの含意

作業ナビゲーション＆トレサビ記録アプリを設計する際、「記録の完全性」と「品質の実質的保証」を明確に区別するシステム思想が必要である。単に全ステップに完了チェックを求めるのではなく、測定値の外れ値検出・前工程記録との整合性確認・作業者識別の多要素認証等の機能が「証拠品質」を高める。また TQM の感情労働問題への対策として、不適合報告・問題提起が匿名または心理的安全な形でできる仕組みの設計も検討すべきである。
