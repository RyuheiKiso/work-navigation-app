# 製造コストと価値工学

## 1. 製造原価の構造

製品の製造原価は以下の 3 要素に分解される。

```
製造原価
├── 直接材料費  (Direct Material Cost)   ← 製品に直接紐づく材料
├── 直接労働費  (Direct Labor Cost)      ← 製品に直接関与する作業者の賃金
└── 製造間接費  (Manufacturing Overhead) ← 工場家賃・設備減価償却・間接労務費等
```

**変動費 (Variable Cost)** は生産量に比例して増減し、**固定費 (Fixed Cost)** は生産量に関わらず一定に発生する。この区別は CVP 分析 (Cost-Volume-Profit Analysis) の基礎となり、損益分岐点 (BEP: Break-Even Point) を算出できる。

```
BEP 生産量 = 固定費 / (単位販売価格 − 単位変動費)
```

## 2. 標準原価計算 (Standard Costing)

**標準原価** は科学的または統計的に設定された「あるべき原価」であり、IE が設定した標準時間 × 標準賃率 = 標準直接労働費 として作業コストを事前に決定する。

### 2.1 標準時間の役割

標準原価計算において、[02_作業研究の系譜とIE](./02_作業研究の系譜とIE.md) で解説した標準時間 (PTS 法や直接時間研究で設定) が原価計算の入力値として直接機能する。

### 2.2 差異分析 (Variance Analysis)

実際原価と標準原価の差異を原因別に分解し、管理に活かす。

| 差異の種類 | 計算式 | 原因 |
|---|---|---|
| 能率差異 | (実際時間 − 標準時間) × 標準賃率 | 作業効率の良否 |
| 賃率差異 | (実際賃率 − 標準賃率) × 実際時間 | 賃金変動・人員構成 |
| 予算差異 | 実際固定費 − 予算固定費 | 設備・維持費の超過 |
| 能力差異 | (正常生産量 − 実際生産量) × 固定費/単位 | 生産量の未達 |

差異分析は月次で現場管理者へフィードバックされ、改善活動の根拠となる。

### 2.3 標準原価計算への批判

- **リードタイムへの貢献が不明**: 仕掛品コストの配賦が実態を反映しない。
- **間接費の恣意的配賦**: 伝統的な「直接労働時間比例配賦」が間接費の増大・直接労働費の縮小した現代工場では実態から乖離する。
→ この批判から **ABC (活動基準原価計算)** が生まれた。

## 3. 活動基準原価計算 (ABC: Activity-Based Costing)

Robin Cooper と Robert Kaplan が 1988 年に提唱。製造間接費を「活動 (Activity)」単位に割り付け、製品・顧客単位の真のコストを可視化する。

### 3.1 ABC の 2 段階配賦

```
第 1 段階: 資源 → 活動
  製造間接費を原因に応じた活動に紐づける
  例: 機械段取活動に「機械オペレーター給与・段取工具費」を集計

第 2 段階: 活動 → 製品
  各活動のコストを「コストドライバー (活動量)」で製品に配賦
  例: 段取費は「段取回数」に比例して各製品に配賦
```

| 活動の種類 (Cooper の 4 段階) | 内容 | コストドライバー例 |
|---|---|---|
| 単位レベル (Unit-Level) | 1 単位生産のたびに発生 | 機械時間、直接労働時間 |
| バッチレベル (Batch-Level) | ロットのたびに発生 | 段取回数、検査回数 |
| 製品レベル (Product-Level) | 品番ごとに発生 | 設計変更件数、部品点数 |
| 施設レベル (Facility-Level) | 工場全体の維持 | 床面積 |

### 3.2 ABC の適用効果

多品種・多工程環境において伝統的配賦法では「少量複雑品のコスト過小計上、大量単純品のコスト過大計上」という歪みが生じやすい。ABC はこれを是正し、製品別の収益性評価・価格設定・品種削除判断を精緻化する。

## 4. 原価企画と目標原価 (Target Costing)

**原価企画 (Genka Kikaku)** は 1960 年代からトヨタが開発した、**製品の企画・設計段階** でコストを作り込む手法。田中雅康が体系化し (1985)、後に国際的に "Target Costing" として普及した。

```
目標原価 = 目標販売価格 − 目標利益

目標販売価格: 市場分析により顧客が許容する価格を設定
目標利益:    事業計画で設定された必要利益
```

**コスト削減活動の方向性**:
- 設計の VE (Value Engineering) で機能とコストのバランスを最適化
- サプライヤと協働した部品コストの低減 (VA/VE 提案)
- 素材・製法・加工精度の見直し

原価企画の特徴は、製造段階に比べて設計段階のほうがコスト低減の自由度が大きい (一般に 70〜80% のコストが設計段階で決まる) という「コミット曲線」の認識に基づく。

## 5. 価値工学 (VE: Value Engineering / VA: Value Analysis)

Lawrence Miles が 1947 年に GE の購買部門で開発。VA は既存製品への遡及適用、VE は設計段階への適用を指すが、現在はほぼ同義で使われる。

### 5.1 価値の定義

```
価値 (Value) = 機能 (Function) / コスト (Cost)
```

価値向上のアプローチ:
- コストを同じに保って機能を高める
- 機能を同じに保ってコストを下げる (主要アプローチ)
- 機能をやや下げてコストを大幅に下げる
- コストをやや上げて機能を大幅に高める

### 5.2 機能定義の原則

VE では製品・部品の機能を「**動詞 + 名詞**」で表現する。

| 製品/部品 | 機能の表現 |
|---|---|
| ボルト | 「部品を結合する」「力を伝達する」 |
| 包装材 | 「製品を保護する」「情報を伝達する」 |
| 電源スイッチ | 「回路を開閉する」 |

この機能思考により、同じ機能を提供する代替手段への視野が広がる。

### 5.3 FAST ダイアグラム

FAST (Function Analysis System Technique) は機能を「なぜ?→より高位機能」「どのように?→より低位機能」という問いで連鎖させて階層化する図。基本機能 (必須) と二次機能 (補助) を分離し、二次機能のコスト削減ポイントを特定する。

### 5.4 VE ジョブプランの 5 フェーズ

| フェーズ | 内容 |
|---|---|
| 情報収集フェーズ | コスト・機能・品質・数量のデータ収集 |
| 機能定義フェーズ | FAST ダイアグラムで機能を体系化 |
| 創造フェーズ | ブレインストーミングで代替手段を発想 |
| 評価フェーズ | コスト・実現可能性・リスクで代替案を評価 |
| 提案フェーズ | 最良案を実行計画とともに提案 |

## 6. 批判と限界

### 6.1 ABC の「活動」定義の恣意性

ABC は製造間接費を「活動 (Activity)」に割り付けることで伝統的配賦法の歪みを是正しようとするが、**どの活動を定義するかに恣意性が残る**という批判がある。活動の粒度や数は設計者の判断に依存し、異なる設計者が同じ工場に ABC を適用しても異なる原価が算出されることがある。Cooper & Kaplan 自身も後に Time-Driven ABC (TDABC) を提唱し、従来 ABC の実装負担 (活動のインタビュー・維持コストの高さ) を認めている (Kaplan & Anderson, 2007)。製造ナビゲーションアプリが生成するステップ別作業時間データは TDABC の入力値として活用できるが、時間コストドライバーへの過度な単純化もまた現場の複雑性を捨象するリスクを持つ。

### 6.2 原価企画目標達成圧力と品質トレードオフ

原価企画では「目標原価 = 目標販売価格 − 目標利益」として設計段階で目標コストを設定するが、目標原価の達成を優先するあまり **部品品質・材料スペック・工程信頼性を過度に削減するリスク** がある。Cooper & Slagmulder (1997) もサプライヤへの原価削減圧力が関係の短期化・品質劣化を招く「ダークサイド」を指摘している。設計段階でのコスト削減が後工程での品質不具合コスト増加 (手直し・廃棄・リコール) として現れる場合、トータルコストでは目標原価達成の便益が消える。

### 6.3 VE/VA のコスト削減至上主義批判

VE は「価値 = 機能/コスト」として機能を維持しながらコストを下げることを目的とするが、実務では **「コストを下げる」圧力が「機能を保つ」条件を形骸化させる** ことがある。機能の定義が「動詞 + 名詞」という記述的な方法で行われるため、機能の「質」や「信頼性の余裕」が評価から抜け落ちやすい。また、顧客が明示的に要求していない「潜在的品質 (Kano の魅力品質)」は FAST ダイアグラムで捕捉されにくく、VE の徹底がブランド価値や使用経験の劣化につながる場合がある。

### 6.4 標準原価の分散分析が生む短期思考

標準原価計算の能率差異・能力差異は月次フィードバックとして機能するが、**短期的な分散の改善に注力するあまり長期的な能力投資を妨げる** という批判がある (Johnson & Kaplan, 1987)。例えば設備の予防保全コストは「実際コストが標準を超える」として能率差異としてネガティブに評価されるが、長期的な設備稼働率向上をもたらす。また、稼働率最大化 (能力差異の最小化) への圧力が不要な仕掛品を生み出し、TPS のフロー思想と矛盾する。

### 6.5 本アプリへの含意

作業ナビゲーションアプリが記録するステップ別作業時間・作業者 ID・ステップ完了率は、標準原価計算の実際工数データ収集コストを大幅に削減できる。ただし、記録されたデータが短期の能率管理にのみ使われると、作業者への過度な効率プレッシャーとなる ([24_作業者プライバシー・データ倫理と労務監視](./24_作業者プライバシー・データ倫理と労務監視.md) 参照)。データは中長期的な工程改善・訓練設計の目的に優先的に活用する設計方針が望ましい。
