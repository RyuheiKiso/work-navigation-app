# サプライチェーンと作業依存性

## 1. サプライチェーンにおける製造「作業」の位置づけ

製造工程における「作業」は、上流 (調達・入庫) と下流 (出荷・配送・顧客) の文脈に不可分に埋め込まれている。材料が届かなければ作業は始められず、完成品の出荷速度は下流の物流能力に制約される。製造作業は「孤立した変換工程」ではなく、**サプライチェーン全体の流れの一部** として設計・管理される必要がある。

## 2. SCOR モデル (Supply Chain Operations Reference)

Supply Chain Council (現 ASCM) が 1996 年に開発したサプライチェーン管理の参照フレームワーク。製造作業は "Make" プロセスとして位置づけられる。

### 2.1 6 つのプロセスタイプ

| プロセス | 内容 |
|---|---|
| Plan | 需要予測・在庫計画・生産計画・調達計画の策定と同期 |
| Source | 仕入先評価・発注・入荷・支払 |
| **Make** | 生産指示・作業実行・品質検査・包装・出荷準備 |
| Deliver | 受注処理・出荷・配送・顧客への引渡し |
| Return | 不良品返品・修理・廃棄処理 |
| Enable | サプライチェーン管理の基盤 (データ管理・契約・コンプライアンス) |

### 2.2 Make の 3 タイプ

| タイプ | 内容 | 工場作業の特性 |
|---|---|---|
| MTS (Make-to-Stock) | 見込生産。在庫に向けて製造する。 | 生産計画精度が在庫水準を左右 |
| MTO (Make-to-Order) | 受注生産。注文確定後に製造開始。 | リードタイム短縮が競争要因 |
| ETO (Engineer-to-Order) | 受注設計生産。設計から作業が始まる。 | 設計変更・仕様確定の遅延が作業に直結 |

## 3. ブルウィップ効果 (Bullwhip Effect)

Lee, Padmanabhan & Whang (1997) が命名。川下 (小売) の需要変動が川上 (メーカー・部品供給者) に向かって増幅されて伝わる現象。玩具の鞭 (bullwhip) のように先端の小さな動きが末端では大きな振れになることに由来する。

### 3.1 ブルウィップ効果の発生原因

| 原因 | 内容 |
|---|---|
| 需要予測の更新 | 各段階が独自に予測し、不確実性を上乗せする |
| 過剰なロット発注 | 経済的発注量 (EOQ) を前提に需要を「丸める」 |
| 価格変動 | セール時の買い溜めが需要のスパイクを作る |
| 供給不足への対応 | 不足が予想されると過剰発注し、実需の数倍を発注する |

### 3.2 製造作業への影響

- 生産計画の変更頻度が高くなり、段取り回数が増加する。
- 生産量が週単位・月単位で大きく振れるため、稼働率が不安定になる。
- 残業・一時的な派遣増員など労務管理が複雑化する。
- 過剰在庫と欠品が交互に発生し、作業環境が悪化する。

### 3.3 緩和策

- **情報共有 (CPFR: Collaborative Planning, Forecasting, and Replenishment)**: 需要情報をサプライチェーン全体で共有する。
- **平準化 (Heijunka)**: 生産量・品種を日々均等にならす ([03_作業標準化と生産方式](./03_作業標準化と生産方式.md) 参照)。
- **VMI (Vendor Managed Inventory)**: サプライヤが在庫水準を管理し、補充責任を持つ。

## 4. JIT 調達とサプライヤ関係

トヨタ生産方式の JIT は工程内だけでなく、外部サプライヤからの調達にも適用される。

### 4.1 ミルクラン (Milk Run)

トラック 1 台が複数のサプライヤを巡回して部品を収集し、組立工場へ届ける物流方式。

利点:
- サプライヤごとの小ロット頻繁納入が可能
- 在庫削減 (工場内の部品バッファが最小化)
- 環境負荷の低減 (車両台数削減)

課題:
- サプライヤの地理的集中が前提 (遠距離サプライヤには不向き)
- 一社の供給遅延がすべての後続作業に波及

### 4.2 サプライヤ育成と協働品質管理

日本の系列取引 (Keiretsu) は単なる売買関係を超えた長期的パートナーシップを特徴とし、発注元がサプライヤへ品質改善の技術指導・人員派遣を行う協働モデルを生んだ。

## 5. 製造実行における作業と在庫の相互作用

### 5.1 MRP (Material Requirements Planning)

**MRP** は BOM (Bill of Materials: 部品表) と在庫データから、品番ごとの必要量と発注タイミングを計算する需要従属型在庫管理システム (Orlicky, 1975)。

```
正味必要量 = 所要量 − 手持在庫 − 発注残
発注日 = 必要日 − リードタイム
```

MRP は部品の必要タイミングを正確に計算できる反面、リードタイムの精度・在庫データの正確性に強く依存し、現場の実際の状況変化への追従が課題となる。

### 5.2 MRP II と ERP

MRP に生産能力計画 (CRP: Capacity Requirements Planning)・購買・財務を統合したのが **MRP II (Manufacturing Resource Planning)**。さらに会計・販売・人事を統合した企業全体のシステムが **ERP (Enterprise Resource Planning)** である。SAP, Oracle, Infor がグローバルで主要ベンダー。

### 5.3 待ち行列理論と工場内仕掛品

工場内の仕掛品 (WIP: Work-in-Process) の蓄積は、待ち行列理論によって分析できる。

**M/M/1 モデル** (到着・サービスともに指数分布、サーバー 1 台) の基本公式:

```
λ: 到着率 (生産投入速度)
μ: サービス率 (工程処理速度)
ρ = λ/μ: 利用率 (ρ < 1 で安定)

平均仕掛品数: L = ρ / (1 - ρ)
平均待ち時間: W = L / λ
```

**リトルの法則 (Little's Law)**: *L = λW* — 平均仕掛品数 = 到着率 × 平均滞在時間。この式は分布の仮定に依らず成立し、リードタイム短縮策の評価に有用。

利用率が 1 に近づくほど仕掛品と待ち時間は急増する (非線形)。この特性は、設備の高稼働率追求がリードタイム延長・フロー阻害を生むという直感と合致する。
