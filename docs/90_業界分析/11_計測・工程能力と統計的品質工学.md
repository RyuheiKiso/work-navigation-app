# 計測・工程能力と統計的品質工学

## 1. 工程能力指数

**工程能力 (Process Capability)** は、工程が顧客規格に対してどれほど安定的に製品を生産できるかを定量化する指標。前提条件: 工程が統計的管理状態 (In-Control) にあること。

### 1.1 Cp (Process Capability Index)

ばらつきのみを評価するインデックス (中心偏りを考慮しない)。

```
Cp = (USL - LSL) / (6σ)

USL: 上方規格限界 (Upper Specification Limit)
LSL: 下方規格限界 (Lower Specification Limit)
σ:   工程の標準偏差 (管理図の R̄/d₂ や s̄/c₄ で推定)
```

### 1.2 Cpk (Process Capability Index considering Shift)

中心偏りを考慮した実用的なインデックス。

```
Cpk = min[(USL - μ) / (3σ), (μ - LSL) / (3σ)]

μ: 工程平均
```

Cp ≧ Cpk の関係が常に成立する。Cp と Cpk が等しい場合、工程中心が規格中央に位置する。

### 1.3 Pp / Ppk (Process Performance Index)

短期ばらつき (サブグループ内) ではなく、長期の全サンプルの標準偏差 (s) を使用。
Pp・Ppk は「実際の性能」、Cp・Cpk は「ポテンシャル」を表す。

### 1.4 工程能力の判断基準

| Cpk 値 | 判定 | 対処 |
|---|---|---|
| ≧ 1.67 | 非常に優れている | 工程監視コスト削減を検討 |
| 1.33〜1.67 | 十分な能力 | 現状維持 |
| 1.00〜1.33 | 辛うじて合格 | 要改善 |
| < 1.00 | 不合格 | 直ちに改善が必要 |

自動車 IATF 16949 では Cpk ≧ 1.67 を特殊特性への要求とする。

## 2. シックスシグマ (Six Sigma)

### 2.1 起源と背景

シックスシグマは 1986 年に Motorola の品質エンジニア Bill Smith が開発し、Bob Galvin CEO のもとで全社展開された品質改善方法論。**工程中心が ±1.5 σ シフトする前提** で計算すると、6σ 水準は **3.4 DPMO (Defects Per Million Opportunities)** に相当する。

1990 年代に GE の Jack Welch が積極的に導入し、世界的に普及した。

### 2.2 σ レベルと不良率の対応

| シグマ水準 | DPMO (1.5σ シフト考慮) | 合格率 |
|---|---|---|
| 3σ | 66,807 | 93.32% |
| 4σ | 6,210 | 99.379% |
| 5σ | 233 | 99.977% |
| 6σ | 3.4 | 99.9997% |

### 2.3 DMAIC プロセス

既存工程の改善に使用する標準的なロードマップ。

| フェーズ | 内容 |
|---|---|
| D (Define) | 問題範囲・顧客要求・プロジェクト憲章の定義 |
| M (Measure) | 現状測定・データ収集・プロセスマップ作成 |
| A (Analyze) | 根本原因分析 (特性要因図、回帰、Anova 等) |
| I (Improve) | 解決策の設計・実験・検証 (DOE 等) |
| C (Control) | 改善後の管理計画、管理図による維持 |

### 2.4 DMADV / DFSS

新規設計プロセスに使用する手法 (Design For Six Sigma: DFSS)。

| フェーズ | 内容 |
|---|---|
| D (Define) | 設計目標・顧客要求の定義 |
| M (Measure) | 顧客ニーズの定量化 (VOC, QFD) |
| A (Analyze) | 設計コンセプトの評価・選択 |
| D (Design) | 詳細設計・最適化 (DOE, シミュレーション) |
| V (Verify) | プロトタイプ検証・量産移行 |

### 2.5 役職体系

| 役職 | 役割 |
|---|---|
| Champion | 経営幹部; プロジェクトのスポンサーと障壁除去 |
| Master Black Belt (MBB) | 社内の方法論専門家; BB の指導・教育 |
| Black Belt (BB) | 専任でプロジェクトリーダー; 改善活動の中核 |
| Green Belt (GB) | 通常業務と並行してプロジェクトに参加 |

## 3. 測定システム解析 (MSA)

**MSA (Measurement System Analysis)** は測定システムの変動を評価し、観測されたばらつきが「真の工程ばらつき」か「測定誤差」かを分離する手法。AIAG MSA リファレンスマニュアルが自動車産業の標準。

### 3.1 Gauge R&R

測定の変動を繰り返し性 (Repeatability) と再現性 (Reproducibility) に分解する。

| 変動要因 | 内容 |
|---|---|
| 繰り返し性 (Repeatability) | 同一測定者が同一部品を繰り返し測定したときの変動 (Equipment variation) |
| 再現性 (Reproducibility) | 異なる測定者間の平均値の変動 (Appraiser variation) |

```
%Gauge R&R = √(σ²ゲージ) / σ総合 × 100

< 10%: 良好
10〜30%: 条件付き許容
> 30%: 不可 (測定系の改善が必要)
```

### 3.2 その他の MSA 特性

| 特性 | 内容 |
|---|---|
| 偏り (Bias) | 基準値と観測値の差の平均 |
| 線形性 (Linearity) | 測定範囲全体での偏りの一定性 |
| 安定性 (Stability) | 経時的な測定値の変動 |

## 4. 測定の不確かさ (GUM)

**GUM (Guide to the Expression of Uncertainty in Measurement; ISO/IEC Guide 98-3:2008)** は測定結果に付随する不確かさの評価・表現方法を規定する国際文書。

| 不確かさの種類 | 評価方法 |
|---|---|
| タイプ A | 統計的解析 (反復測定による標準偏差) |
| タイプ B | 非統計的方法 (仕様書・校正証明書・経験則に基づく推定) |

**合成標準不確かさ (uc)**: 各不確かさ要因を二乗和の平方根で合成。
**拡張不確かさ (U)**: uc × 包含係数 k (通常 k=2 で信頼水準約 95%)。

## 5. タグチメソッド (品質工学)

田口玄一が 1950〜60 年代に開発した**頑健設計 (Robust Design)** の方法論。設計パラメータの最適化により、使用条件のばらつき (ノイズ) に対して特性値が安定する設計を求める。

### 5.1 損失関数

品質特性が目標値 m からの偏差 y の二乗に比例して損失が発生するという概念。

```
L(y) = k(y - m)²

k: 損失係数 (経済的単位で算出)
```

「規格内は損失ゼロ、規格外は損失無限大」という従来の二段階品質観を批判し、目標値からの偏差そのものをコストとして捉える。

### 5.2 SN 比 (Signal-to-Noise Ratio)

信号 (目的機能) に対するノイズ (誤差) の割合を dB 表示で評価する指標。SN 比が大きいほど望ましい。

| 特性 | SN 比の定義 |
|---|---|
| 望目特性 | η = -10 log(σ²/μ²) |
| 望小特性 | η = -10 log(Σy²/n) |
| 望大特性 | η = -10 log(Σ(1/y²)/n) |

### 5.3 パラメータ設計と直交表

制御因子 (設計で変えられる変数) とノイズ因子 (制御困難な誤差) を分けて実験を計画し、ノイズに対して最も頑健な制御因子水準の組み合わせを効率的に探索する。

直交表 (L₈, L₁₈, L₂₇ 等) を用いることで全数実験の一部の実験数でパラメータ推定が可能。

## 6. 実験計画法 (DOE)

**DOE (Design of Experiments)** は R. A. Fisher (1935) に始まる、因子と応答の関係を体系的に実験で探索する手法の体系。

| 手法 | 用途 |
|---|---|
| 完全要因実験 (Full Factorial) | 全水準組み合わせを実験; 主効果・交互作用を完全に推定 |
| 一部実施要因実験 (Fractional Factorial) | 主効果・一部の交互作用に絞って実験数を削減 |
| 応答曲面法 (RSM) | 連続因子の最適値を曲面フィッティングで探索 |
| Box-Behnken 計画 | 中心複合計画の変形; 角点実験を省略 |

## 7. 抜取検査と AQL

**抜取検査 (Acceptance Sampling)** は全数検査の代替として、ロットから一部を抜き取って合否を判定する手法。

- **AQL (Acceptable Quality Level)**: プロセス平均として許容できる不良率の上限。
- **JIS Z 9015-1** (= ISO 2859-1): 計数抜取検査の標準手順。米 MIL-STD-105E を基にした。
- **OC 曲線 (Operating Characteristic Curve)**: ロットの真の不良率に対する合格確率を示す曲線。生産者危険 (α) と消費者危険 (β) のトレードオフを可視化する。

## 8. 批判と限界

### 8.1 Cp/Cpk の正規分布仮定と非正規分布の隠蔽

Cp・Cpk は工程データが正規分布に従うという前提を持つ。しかし製造現場のデータには、ツール摩耗による一方向のドリフト・複数の設備混在によるバイモーダル分布・加工限界による切断分布など、正規分布から大きく逸脱するケースが頻繁に存在する。Kotz & Lovelace (*Process Capability Indices in Theory and Practice*, 1998) は正規性仮定の違反が Cp/Cpk を著しく過大評価または過小評価させることを体系的に示した。

正規性の確認なしに Cp/Cpk を算出・報告する慣行は、「数値上は能力十分に見えるが実際には規格外品が多い」という危険な状況を生む可能性がある。特に自動車 IATF 16949 での Cpk ≧ 1.67 要件を「取引条件」として使用する場合、サプライヤが正規性を検証せずに要件を「クリア」する書面を提出するリスクがある。

### 8.2 タグチメソッドへの統計学的異論

田口玄一のタグチメソッドは「頑健設計」という概念と SN 比・直交表の実践的な組み合わせで製造業に大きな影響を与えたが、統計学的観点からの批判も存在する。Box, Hunter & Hunter は直交表実験を用いたタグチ法に対し、(1) 交互作用効果の推定が不完全になる可能性、(2) SN 比という合成指標が統計的に最適な分析ではないこと、(3) 応答曲面法 (RSM) に比べて少ない実験点でのパラメータ推定の不安定性を批判した (Box, 1988; "Signal-to-Noise Ratios, Performance Criteria, and Transformations")。

この論争は「実践的な工学手法」と「統計理論の厳密性」のトレードオフを示す典型例として継続的に引用されている。タグチメソッドの産業普及は否定できないが、特に複雑な交互作用が予想される実験設計では古典的 DOE との使い分けが推奨される。

### 8.3 シックスシグマの財務成果誇張と組織的限界

シックスシグマは GE・Motorola 等での成功事例を背景に大企業に広まったが、その財務成果の主張については批判的検証が行われている。Schroeder et al. (2008) はシックスシグマ導入企業の財務パフォーマンスに対する因果効果を測定することが困難であると指摘し、「シックスシグマによる成果」と称される数値の多くが厳密な帰属分析なしに算出されていることを示した。

また Black Belt 育成コスト・プロジェクト管理工数・組織の注意資源の独占といった機会費用は、公表される「改善による財務成果」から控除されない傾向がある。シックスシグマは「品質改善手法」から「ビジネスパフォーマンス管理の戦略」へと過剰に拡張された結果、統計的品質工学の本来の問題解決力が組織政治的な「BB プロジェクト数の確保」競争に希釈されるという弊害も報告されている。

### 8.4 本アプリへの含意

作業ナビゲーションアプリが工程データ・検査測定値を記録する場合、それらのデータを統計的品質分析に活用するには正規性の事前検証・測定システムの信頼性確認 (MSA) が前提条件となる。「記録があれば Cpk が計算できる」という単純化を避け、データの統計的仮定を表示・警告する設計が品質工学ツールとしての信頼性を支える。また工程能力の目標値 (Cpk ≧ 1.33 等) が目的化することで生じる測定操作リスクへの対策として、原データの改竄防止・タイムスタンプ保護・電子署名が重要な設計要件となる。
