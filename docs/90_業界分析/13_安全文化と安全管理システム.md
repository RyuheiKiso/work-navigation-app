# 安全文化と安全管理システム

## 1. 「安全文化」の誕生

**安全文化 (Safety Culture)** という語は、1986 年のチェルノブイリ原子力発電所事故の事後調査報告書 *INSAG-7* (国際原子力安全諮問委員会、1991) において「それぞれの組織および個人の安全問題を最優先する姿勢と行動の集合体」として初めて公式に使用された。その後、航空・化学・医療・製造など幅広い産業で安全管理の中核概念となった。

## 2. Reason の安全文化モデル

James Reason は *Managing the Risks of Organizational Accidents* (1997) において、事故防止に有効な安全文化を 4 つのサブカルチャーの統合として定義した。

### 2.1 4 つのサブカルチャー

| サブカルチャー | 内容 |
|---|---|
| 報告文化 (Reporting Culture) | インシデント・ヒヤリハットを恐れずに報告できる環境。報告数は「安全への関心」の指標。 |
| 公正文化 (Just Culture) | 過失と違反を区別し、誠実な誤りは責めないが故意の違反は問責する。「ブレームフリー」とは異なる。 |
| 柔軟文化 (Flexible Culture) | 通常の階層組織から、緊急時に専門家ベースのフラット組織に素早く切り替えられる。 |
| 学習文化 (Learning Culture) | 安全情報から適切な結論を引き出し、重要な変革を実行する意志と能力を持つ。 |

これら 4 つが統合された文化が「情報に基づいた文化 (Informed Culture)」= 実質的な安全文化である。

### 2.2 公正文化の境界線

報告を促進するには「責めない文化」が必要に見えるが、完全なブレームフリーは危険な違反者を保護してしまう。Reason は行動を以下に分類して公正な処遇の基準を示した。

```
行動の意図性・危険性によって分類:
  無意識のスリップ・ラプス → 非難しない
  注意を怠った行動 → 状況に応じて判断
  工具箱作業違反・手順省略 → コーチング・再訓練
  故意の危険行為・業務妨害 → 厳正な処遇
```

## 3. Safety-I vs Safety-II

### 3.1 Safety-I (従来型安全観)

**Safety-I** は「できるだけ悪いことが起きないようにする」アプローチ。事故・インシデントの分析と原因除去を中心に据える。

- 安全の定義: 「悪いことが起きていない状態」
- 前提: 「正常」と「異常」は截然と区別できる
- 方法: 失敗から学ぶ、バリアを追加する、エラー源を排除する

### 3.2 Safety-II (レジリエンス工学)

Erik Hollnagel *Safety-I and Safety-II: The Past and Future of Safety Management* (2014) が提唱。

**Safety-II** は「物事がうまくいくよう確保する」アプローチ。日常業務の適応的な変動を研究し、その成功要因を理解・強化する。

- 安全の定義: 「状況変化にかかわらず機能する能力」
- 前提: 正常業務でも常にアドホックな調整が行われている (Work-as-Done vs Work-as-Imagined)
- 方法: 成功事例も含めた業務の全体像を理解する

### 3.3 Resilience Engineering の 4 能力

| 能力 | 内容 |
|---|---|
| 対応 (Respond) | 予測または予測外の事象に対処する |
| 監視 (Monitor) | 通常業務・環境において重要な状況を観察・追跡する |
| 予期 (Anticipate) | 将来の状況変化・機会・脅威を予見する |
| 学習 (Learn) | 過去の経験から教訓を引き出す |

## 4. 正常事故論 (Normal Accident Theory)

Charles Perrow は *Normal Accidents* (1984) において、原子力・化学・航空等の「高危険テクノロジー」では事故は「正常 (normal)」= 避けられないと主張した。

高リスクシステムの特性:

| 次元 | 低リスク | 高リスク |
|---|---|---|
| 相互作用の複雑さ | 線形 | 複雑 (non-linear) |
| 結合の緊密さ | 疎結合 | 密結合 (tight coupling) |

**複雑・密結合** のシステムでは、予期しない小さな障害が連鎖反応を起こし、設計段階で想定されなかった経路で壊滅的な事故が発生する。Perrow はそのような技術の使用自体を疑問視した。

製造業への含意: 高度に自動化され相互接続された生産システムは「密結合」の性質を帯びやすく、連鎖停止リスクへの注意が必要。

## 5. 高信頼組織 (HRO: High Reliability Organizations)

Weick, Sutcliffe & Obstfeld が原子力・航空母艦・消防などを研究し、高危険環境でも卓越した安全記録を保つ組織の特性を抽出した (*Managing the Unexpected*, 2001)。

### 5.1 HRO の 5 原則

| 原則 | 内容 |
|---|---|
| 失敗への先見 (Preoccupation with Failure) | 小さな失敗・近事故を「次の大事故の前兆」として徹底的に分析する |
| 単純化への抵抗 (Reluctance to Simplify) | 複雑な状況を安易に単純化せず、多様な視点を維持する |
| 作業への感受性 (Sensitivity to Operations) | 現場の実際の状況に常に注意を払う (現地現物) |
| 回復へのコミットメント (Commitment to Resilience) | 問題が発生した後でも被害を最小化し回復する能力を持つ |
| 専門性への尊重 (Deference to Expertise) | 緊急時は階層ではなく知識・経験のある人間が意思決定する |

## 6. 安全マネジメントシステム (SMS)

### 6.1 OHSAS 18001 から ISO 45001 へ

- **OHSAS 18001** (1999): 英国規格協会主導の労働安全衛生マネジメントシステム認証規格。ISO 9001 と整合した PDCA 構造。
- **ISO 45001:2018**: OHSAS 18001 を置き換えた国際規格。労働者の参加・協議の強化、リスクおよび機会の考慮、リーダーシップの役割が拡充された。

### 6.2 ISO 45001 の主要要求事項

| 箇条 | 内容 |
|---|---|
| 5 リーダーシップ | トップマネジメントの OH&S への公約・参加促進 |
| 6 計画 | ハザードの特定、リスク評価と機会の特定、目標設定 |
| 7 支援 | 資源・力量・文書化情報の管理 |
| 8 運用 | プロセス管理、緊急時準備・対応、請負業者管理 |
| 9 パフォーマンス評価 | モニタリング・監査・マネジメントレビュー |
| 10 改善 | インシデント調査、是正処置、継続的改善 |

### 6.3 労働安全衛生法 (日本)

1972 年制定。第 3 条で事業者の「安全で健康的な職場環境の実現」義務を規定。主な規制:
- 安全管理者・衛生管理者の選任義務
- 作業環境測定・健康診断の実施
- 特定化学物質・有機溶剤等の特別規制
- 危険・有害業務の就業制限
