# 規制別トレーサビリティ要件詳論

## 1. 業界横断的なトレーサビリティの位置づけ

[06_品質管理とトレーサビリティ](./06_品質管理とトレーサビリティ.md) が ISO 9001 を中心とした一般的品質管理を扱うのに対し、本章は **業界別規制** が課すトレーサビリティ要件を横断的に整理する。業界によって要求される記録の粒度・保管期間・電子記録の法的要件は大きく異なり、アプリ導入時の設計判断に直結する。

**ALCOA+ 原則** (FDA が電子記録評価に用いるデータ完全性の原則):

```
A — Attributable   (誰が記録したか)
L — Legible        (読み取れる)
C — Contemporaneous (リアルタイムに記録)
O — Original       (原本であり改ざんされていない)
A — Accurate       (正確)

+ Complete  (完全)
+ Consistent (一貫)
+ Enduring  (耐久性がある)
+ Available (アクセス可能)
```

ALCOA+ はすべての規制業界で事実上の標準フレームワークとなっており、電子記録システムが備えるべき要件を定義する。

## 2. 医薬品・医療機器

### 2.1 GMP (Good Manufacturing Practice)

**GMP** は医薬品・医療機器・食品の製造品質保証の規範。日本では薬機法 (旧薬事法) に基づく **CGMP** (Current GMP)、国際的には **PIC/S GMP** が主流。

主なトレーサビリティ要件:
- 原材料ロット番号から最終製品ロットまでの **順方向・逆方向トレーサビリティ**
- 製造バッチ記録 (Batch Record) の完全保管 (製品の有効期限 + 1 年以上)
- すべての製造操作の実施者 ID・日時・測定値の記録

### 2.2 FDA 21 CFR Part 11 — 電子記録と電子署名

米国 FDA が 1997 年に制定した電子記録規制。紙の記録と同等の法的効力を電子記録に認めるための条件を定める。

| 要件 | 内容 |
|---|---|
| アクセス制御 | 権限のある者のみシステムにアクセスできること |
| 監査証跡 (Audit Trail) | 記録の作成・変更・削除履歴を自動記録し改ざん不可能とすること |
| 電子署名の紐付け | 署名者の氏名・日時・意味 (承認/レビュー等) を記録に紐付け |
| システム検証 | ソフトウェアが意図した機能を正確に実行することを文書化して検証 |
| 記録の保護 | バックアップ、アーカイブ、物理的・論理的改ざん防止 |

### 2.3 MDR / IVDR (EU 医療機器規制)

EU では 2017/2017 Medical Device Regulation が施行され、**UDI (Unique Device Identification)** による個体レベルの追跡が義務化された。製造者は EUDAMED データベースへのデータ登録も求められる。

## 3. 自動車

### 3.1 IATF 16949

**IATF 16949:2016** は ISO 9001 をベースに自動車業界固有の要求を追加した品質マネジメントシステム規格。VDA/AIAG の共同策定。主要なトレーサビリティ要求:

- **PPAP (Production Part Approval Process)**: 新部品・変更部品の量産移行前の承認プロセス。設計記録・材料証明書・工程フロー図・管理計画書・初回サンプル検査報告書を一式提出
- **APQP (Advanced Product Quality Planning)**: 設計〜量産の品質計画フレームワーク。5 フェーズで各段階の記録を管理
- 不具合品の隔離と **8D レポート** (根本原因分析・再発防止) の記録保管

### 3.2 リコール対応のトレーサビリティ

日本の **自動車リコール制度** (道路運送車両法第 63 条) は、対象車両の特定のために製造ロット〜シリアル番号レベルのトレーサビリティを事実上要求する。電動パワートレイン・安全装置に関わる部品では **10 年以上** の記録保管が実務的に必要とされる。

## 4. 食品

### 4.1 HACCP (Hazard Analysis and Critical Control Points)

**HACCP** はコーデックス委員会 (国際食品規格委員会) が策定した食品安全管理システム。日本では 2021 年に食品衛生法改正により原則義務化。

7 原則の中の **モニタリング記録** と **是正措置記録** がトレーサビリティの核となる。特に CCP (重要管理点) での温度・時間・pH 等の測定値をリアルタイム記録する要件は、製造ナビゲーションアプリの測定値入力機能と直結する。

### 4.2 ISO 22000 / FSSC 22000

**ISO 22000:2018** は HACCP を組織マネジメントシステムに統合した国際規格。**FSSC 22000** はこれに追加要求事項を加えた民間認証スキームで、大手食品メーカーの調達要件として広く使われる。

### 4.3 米国 FSMA 204 (食品安全強化法 Section 204)

2023 年に施行された FDA の **FSMA Rule 204** (Requirements for Additional Traceability Records for Certain Foods) は、高リスク食品 (葉物野菜、貝類、軟質チーズ等) に対して農場〜消費者まで **24 時間以内** のトレースバックを可能とする電子記録を義務付けた。

## 5. 航空宇宙

### 5.1 AS9100

**AS9100 Rev D (2016)** は航空・宇宙・防衛分野の品質マネジメントシステム規格 (IAQG 策定)。ISO 9001 に加えて以下を要求:

- **コンフィギュレーション管理 (Configuration Management)**: 製品の設計状態・変更履歴を完全に管理し、実機と設計書の整合性を保証
- **リスクマネジメント**: 製品安全リスクの特定・評価・対策の記録
- **First Article Inspection (FAI)**: 新規・変更部品の初号品検査記録の詳細な文書化

### 5.2 Nadcap (National Aerospace and Defense Contractors Accreditation Program)

特殊工程 (熱処理、溶接、NDT、化学処理等) を対象とした業界認定プログラム。認定取得には工程パラメータ (温度・時間・電流等) の完全な記録と監査対応が必須であり、電子記録システムの整備が実質的に要求される。

## 6. 共通要件の整理

業界横断で要求されるトレーサビリティ機能をまとめると以下となる:

| 要件 | 内容 | 関連業界 |
|---|---|---|
| ロット追跡 | 原材料ロット → 仕掛品 → 製品ロットの系統管理 | 全業界 |
| 個体シリアル管理 | 製品 1 個ごとに固有 ID を付与し記録と紐付け | 医療機器、自動車、航空 |
| データ完全性 | 改ざん検知・監査証跡・バックアップ | FDA 規制業界 |
| 保管期間 | 製品ライフサイクル + α (業界・製品ごとに異なる) | 全業界 |
| 電子署名 | 承認行為の記録、否認防止 | FDA 21 CFR Part 11 対象 |
| アクセス制御 | ロール別権限管理、操作ログ | 全業界 |

### 6.1 記録保管期間の比較

| 業界 | 対象 | 最低保管期間 |
|---|---|---|
| 医薬品 | 製造バッチ記録 | 有効期限 + 1 年 (または 3 年以上) |
| 医療機器 | 製造記録 | 製品寿命 + 2 年 (EU MDR) |
| 自動車 | 製品品質記録 | 顧客要求 (通常 15 年以上) |
| 食品 | HACCP 記録 | 2〜5 年 (製品・規制による) |
| 航空宇宙 | 製造・検査記録 | 機体の就役期間中 (数十年) |

## 7. アプリ設計への含意

- **記録の不変性**: 一度記録したステップ完了データは上書き不可とし、修正は差分レコードで管理する (監査証跡)
- **タイムスタンプの信頼性**: サーバー時刻を使用し、端末側の時刻操作を排除する
- **電子署名機能**: 高規制業界向けには FDA 21 CFR Part 11 準拠の電子署名フローを実装できる拡張性を設計段階で確保する
- **エクスポート**: 規制当局の監査に対応するため、構造化されたフォーマット (CSV/XML/PDF) でのデータエクスポート機能を提供する
- **保管期間の設定**: 業界ごとに異なる保管期間に対応できる設定項目を設ける

## 8. 批判と限界

### 8.1 規制の断片化とコンプライアンスコストの非対称性

業界横断で規制が乱立しており、グローバルに展開する製造企業は **FDA 21 CFR Part 11・EU MDR・IATF 16949・ISO 22000・AS9100 を同時に遵守** しなければならない場面が増えている。各規制の要件は重複しながらも細部が異なるため、コンプライアンス対応コストは規模の小さな企業に不均衡に重くかかる。中小製造業では「規制対応のための記録」が目的化し、記録の質や業務改善への活用が二次的になるという逆説が生じやすい。

### 8.2 ALCOA+ 原則の解釈の幅

ALCOA+ は電子記録の完全性評価のフレームワークとして広く参照されるが、各原則の具体的な実装方法は規制当局・監査機関によって解釈が異なる。特に **「Contemporaneous (同時性)」** の定義 — 「作業完了直後」とはどのくらいの時間差まで許容されるか — は明確に規定されておらず、監査での指摘事項になりやすい。オフライン動作を含むシステム ([27_オフライン同期とデータ整合性](./27_オフライン同期とデータ整合性.md) 参照) では「ローカルに記録されたがサーバに未同期」の状態をどう扱うかが設計上の重要論点となる。

### 8.3 UDI・シリアル管理のリソース要件

EU MDR の UDI 義務化や自動車業界の個体シリアル管理は理論的には完全なトレーサビリティを実現するが、**生産ライン全体での個体識別インフラの整備コストが高い**。特に多品種少量生産の工場では、ロット管理から個体管理へのシフトは管理工数の飛躍的増加を意味する。また、個体シリアルの誤刻印・読み取りエラー・ラベルはがれ等の現場問題が蓄積すると、トレーサビリティデータの信頼性が根本から損なわれる。完璧なシリアル管理を目指すより、**エラー発生時の修正フローと監査証跡の整備** を優先する実用的な設計判断が求められる。

### 8.4 規制変化への対応コスト

食品分野の FSMA 204 (2023) のように規制要件は頻繁に追加・改訂される。システムが特定規制の要件に硬直化して実装されると、規制変更のたびにシステム全体の改修が必要となる。**規制に中立なコアデータモデル** (ALCOA+ を満たす汎用的なイベントログ設計) を維持しつつ、業界固有の要件はアドオン設定で対応するアーキテクチャが長期維持コストを抑える。

### 8.5 本アプリへの含意

規制対応の設計原則として、**最も厳しい規制要件 (FDA 21 CFR Part 11) を基準としたコアを設計し、緩い要件の業界ではオプションをオフにする** アプローチが複数業界対応の実用的な方針となる。ただし、全機能を最高水準で実装することによるシステムの複雑性増加・導入コスト増も無視できないため、ターゲットとする規制業界のセグメントを初期設計段階で明確化することが重要である。
