# 作業中断・割込み・再開の認知科学

## 1. 中断研究の背景と重要性

製造現場の作業者は連続する手順を遂行中に、頻繁な中断 (Interruption) を経験する。呼び出し・機械アラーム・管理者からの確認要請・部品不足などである。中断後の再開時に正確に作業を再開できるかどうかは、品質・安全の重要な変数となる。

**中断に関する研究の系譜:**

```
1968  Miller — 記憶の検索と中断の関係 (初期認知心理学研究)
1990s Gillie & Broadbent — 中断が作業遂行に与える影響の実証
2004  Czerwinski, Horvitz, Wilhite — 知識作業者の中断調査 (MS Research)
2007  Trafton & Monk — Resumption Lag の認知モデル
2010s Li, Blandford ら — 医療現場・HCI での中断研究の拡張
```

## 2. 中断の分類

### 2.1 発生源による分類

| 種別 | 発生源 | 例 |
|---|---|---|
| 外部中断 | 他者・システムからの割込み | 電話、アラーム、呼び出し |
| 自己中断 | 作業者自身の注意転換 | 材料確認、思い出し作業 |
| システム中断 | デバイス・アプリの割込み | 通知、強制ログアウト、バッテリー警告 |

### 2.2 中断の深さ (Disruption Depth)

**Monk et al. (2008)** は中断の深さを以下の 4 段階で定義した:

```
レベル 0: 注意のブレ (Attention capture) — 視線が一瞬それる
レベル 1: 短期中断 (Brief disruption) — 数秒の作業停止
レベル 2: 中断 (Interruption) — 別タスクへの切替え
レベル 3: 没入 (Task switching) — 中断元タスクの一時放棄
```

製造現場では緊急対応が発生した場合、レベル 2〜3 の深い中断が連続し、作業再開時のエラーリスクが著しく高まる。

## 3. Resumption Lag と Memory for Goals

**Resumption Lag (再開遅延)** は中断前のタスクに戻った後、次の行動を想起するまでに要する時間のことである。Altmann & Trafton (2002) は **Memory for Goals モデル** でこの現象を説明した。

### 3.1 Memory for Goals モデル

```
目標の活性化 (Goal Activation)
   ↓
注意リソース配分 (Attentional Focus)
   ← 中断によって活性化が減衰 →
   ↓
再開時に目標を再活性化するのに時間がかかる = Resumption Lag
```

Resumption Lag の実測値:
- 軽度の中断 (5〜10 秒): 約 3〜5 秒の再開遅延
- 中程度 (2〜3 分の別タスク): 約 10〜30 秒の再開遅延
- 長時間中断 (数時間以上): 作業状態の再構成に数分を要する場合がある

Trafton & Monk (2007) は Resumption Lag の低減のために **サブゴールの外部化** (目に見えるプレースキーパーの配置) が有効であることを示した。

## 4. Post-Completion Error

**Post-Completion Error (PCE)** は Byrne & Bovair (1997) が定義した、主目的達成直後に付随するサブゴールを忘れるエラーパターンである。

最も有名な日常例:
- ATM で現金を取った後にカードを取り忘れる
- コピー機で原稿を置き忘れる
- レジで袋を忘れて帰る

製造現場での対応:
- 部品の供給ジグを取り去ることで次の作業が物理的にブロックされる (ポカヨケ設計)
- 電子手順書では「最終確認ステップ」を主目的の後に必ず挿入し、完了前のシステムロックで強制する
- 作業者に「これは後処理ステップです」と明示的に注意を引く (Signaling, Leung et al., 2011)

## 5. 中断後の再開を支援する設計

### 5.1 プレースキーパー (Placekeeper)

作業中断時に「どこまでやったか」を視覚的・物理的に示す仕掛け。

```
物理的プレースキーパー: 治具のピン・マーカーテープ・ステップカード
電子的プレースキーパー: 現在ステップのハイライト表示・
                        「ここまで完了」のステータスアイコン・
                        再開ウェルカム画面での状態サマリ
```

### 5.2 再開キュー (Resumption Cue)

中断前の状態に注意を向け直す情報提示。

| キューの種類 | 内容 | 効果 |
|---|---|---|
| 視覚的キュー | 前ステップのサムネイル・スクリーンショット | 記憶の再活性化 |
| テキストサマリ | 「直前の操作: ○○ を締め付け完了」 | 状態の再確認 |
| 警告ダイアログ | 「中断から○○分経過しています。再確認してください」 | 注意喚起 |
| 手順の巻き戻し | 直前 n ステップを再表示して確認させる | 状態再構築 |

### 5.3 ブレークポイント (Breakpoint) の設計

**Iqbal & Bailey (2007)** は知識作業において、中断を受け入れやすい **サブタスクの自然な区切り点** (Breakpoint) が存在することを示した。中断されにくいタイミング (アクション実行中) と比較して、中断を受け入れやすいタイミング (サブゴール完了直後) では再開時のエラーが有意に少なかった。

製造ナビゲーションへの含意:
- アラームや呼び出しが来た場合、「次のブレークポイントまで待つ」機能 (Snooze/Defer) を提供する
- 緊急中断が避けられない場合は現在ステップの「中断マーク」をログに自動記録する

## 6. 警報疲労 (Alarm Fatigue)

製造現場では多数のセンサー・システムからアラームが発生するが、過剰なアラームは **警報疲労 (Alarm Fatigue)** を引き起こし、重要な警報の見落としを招く。

ANSI/ISA-18.2 (製造プロセスアラーム管理規格) は以下を定義:
- 平均アラーム頻度: **10 件/時間** 以下を「許容可能」とし、30 件/時間以上を「過負荷」と判定

ナビゲーションアプリの設計では、アプリ内通知が外部アラームと競合しないよう通知の **重要度階層** と **モード分離** が必要である。

## 7. 製造現場での中断管理の実践的介入

| 介入 | 内容 | 効果の根拠 |
|---|---|---|
| ロックステップ式手順 | 確認操作なしに次ステップへ進めないUI | Post-Completion Error の構造的排除 |
| 中断ログの自動記録 | 中断発生・再開時刻と対象ステップをDB記録 | 事後分析・品質トレーサビリティ |
| 状態復元表示 | 再ログイン時に中断前の状態をサマリ表示 | Resumption Lag の短縮 |
| クリティカルステップの保護 | 特定ステップ中は通知をミュートし割込みを防ぐ | 集中保護 (Protection Window) |
| 引継ぎ記録の構造化 | 交代時に残作業状態をデジタル引継ぎ票に出力 | シフト交代時の再開エラー防止 |

## 8. 批判と限界

中断研究の多くはオフィスワーカー (知識労働者) を対象としており、製造現場の **身体作業と認知作業の並行** という特殊性を十分に扱っていない。また、Altmann & Trafton のモデルは単一タスクの実験から導かれており、複数の並行タスクが常態化する製造現場への外挿には注意が必要である。
