# 作業の定義と分類

## 1. 「作業」の語義

英語圏では製造現場の文脈において "work"・"operation"・"task" の三語が使い分けられる。

| 英語 | 典型的な粒度 | 例 |
|---|---|---|
| work | 広義の労働・仕事一般 | 作業員の日常的な労務 |
| operation | 生産工程上の処理単位 | 「溶接工程」「組立工程」 |
| task | 人間が遂行する目標指向的な行為単位 | 「ボルト締め付け」「外観検査」 |

日本語「作業」は operation と task の両義を含む。文脈に応じた粒度の明示が重要である。

## 2. JIS Z 8141 による定義

JIS Z 8141:2001「生産管理用語」は生産管理の標準用語を定める。関連する定義を以下に示す。

- **工程 (process)**: 原材料を製品に変換するための系統的な一連の活動。
- **作業 (operation)**: ある目的のために、人・機械などが行う一定のまとまりのある活動。
- **要素作業 (work element)**: 作業を構成する最小の単位であり、これ以上分割すると意味をなさなくなる作業の構成要素。

標準における「作業」はあくまで「目的をもつ活動の単位」として中立的に定義されており、人間が行うか機械が行うかを問わない。

## 3. IE における階層的分類

Industrial Engineering (生産工学) では作業を次の階層構造で捉える。

```
工程 (Process)
  └─ 単位作業 (Unit Operation)
        └─ 要素作業 (Work Element)
              └─ 動素 (Therblig)
```

| 階層 | 内容 | 分析ツール |
|---|---|---|
| 工程 | 素材が機能的に変化する段階 (加工・検査・搬送・保管) | 工程分析、工程図 |
| 単位作業 | 作業者・機械単体で完結する一連の動作 | 作業分析 |
| 要素作業 | 計測可能な最小の有意義な動作単位 | 時間研究 (stopwatch study) |
| 動素 (サーブリッグ) | F.B. Gilbreth が定義した 18 種の基本動作要素 | 動作研究、micromotion study |

### 18 動素 (Therblig)

Gilbreth 夫妻が提唱した基本動作要素であり、どの製造工程においても人手作業はこれらの組み合わせとして分解できるとされる。代表的なものを示す。

| 記号 | 名称 | 概要 |
|---|---|---|
| Te | 手を伸ばす (Transport Empty) | 物体に向けて空手で手を動かす |
| G | つかむ (Grasp) | 対象物を把持する |
| TL | 運ぶ (Transport Loaded) | 物体を持ちながら移動する |
| P | 位置決め (Position) | 使用・組付けに合わせて位置を定める |
| RL | 離す (Release Load) | 対象物を放す |
| I | 検査する (Inspect) | 品質・数量を確認する |
| U | 使用する (Use) | 工具・機械を操作する |
| D | 遅延 (Delay) | 不可避の待機 |

## 4. 作業時間の分類

IE では作業に費やす時間を以下の区分で分析する。

```
総時間
├── 正味作業時間
│   ├── 主作業 (Main Operation)        ← 製品価値を直接生む活動
│   └── 付随作業 (Incidental Operation) ← 主作業に付随する補助的活動
└── 余裕時間
    ├── 管理余裕 (Management Allowance)
    ├── 疲労余裕 (Fatigue Allowance)
    └── 個人余裕 (Personal Allowance)
```

さらに工場全体での稼働分析では「付帯作業」(段取り、清掃等) を別途計上するケースもある。

## 5. 4M / 5M+1E フレーム

製造工程における変動要因を体系化した管理フレームワーク。

| 要素 | 内容 |
|---|---|
| Man (人) | 作業者のスキル・経験・状態 |
| Machine (機械) | 設備の種類・状態・精度 |
| Material (材料) | 原材料・部品の品質・ロット |
| Method (方法) | 作業手順・標準・条件 |
| (Measurement) | 計測・検査方法 |
| (Environment) | 温度・湿度・照明・騒音 |

異常・品質問題の原因分析 (特性要因図) の大骨として広く用いられる。

## 6. 批判と限界

### 6.1 JIS 定義の硬直性と現場実態とのズレ

JIS Z 8141 における「作業」定義は「目的をもつ活動の単位」という中立的な記述に留まり、現代の製造現場で発生する複合的・創発的な活動を適切に捉えられないという批判がある。Zuboff (*In the Age of the Smart Machine*, 1988) が指摘したように、情報化環境での「作業」には知識処理・モニタリング・判断という認知的側面が不可分に含まれるが、JIS フレームはこれらを「主作業」「付随作業」の二分法に押し込めてしまう。

製造現場での実態調査 (Strauss & Corbin, 1990; situational analysis 系統の研究) では、作業者は公式手順書が想定しない「段取り替えの調整」「隣工程との非公式連絡」「工具の即興的修正」といった活動を常時行っており、これらは作業時間分析では「余裕」あるいは「無価値な動作」として扱われる。すなわち現場の価値創出に不可欠な活動が定義上の「正味作業」から系統的に排除される構造的問題がある。

### 6.2 4M フレームにおける「人」への過度の責任転嫁

4M / 5M+1E フレームは問題原因の可視化に有効だが、その運用実態では「Man (人)」起因の割り当てが過度に多くなる傾向が報告されている (Dekker, *The Field Guide to Understanding Human Error*, 2006)。特性要因図を用いた原因分析において、機械・材料・方法の原因が特定困難な場合に「人的要因」という大骨に問題が集約される「責任転嫁バイアス」が観察される。

このことは組織レベルの問題 (設備の保全不足、不適切な作業設計、訓練資源の不足) が個人の技能不足・注意力不足として診断され、再発防止策が「作業者への注意喚起」に留まるという悪循環を生む。「人」を原因として特定することの簡便さが、システム的な問題発見を阻害する点は管理実務上の重大な限界である。

### 6.3 「標準」作業定義の普遍化不可能性

IE 的な階層分類 (工程→単位作業→要素作業→動素) は観察者の理論的枠組みと文化的前提を反映しており、客観的・普遍的な作業分解は存在しないという構成主義的批判がある。Engeström の活動理論 (Activity Theory, 1987) は、作業 (activity) を個人の行為 (action) レベルではなく、道具・コミュニティ・ルール・分業を含む社会的活動システムとして分析する必要性を説く。

18 動素 (Therblig) は西洋・工業化社会の作業様式を基盤として導出されており、手工芸・伝統技能・非規格化作業への適用には限界がある。また「最善の方法 (One Best Way)」という前提自体が、作業者の身体的多様性 (左利き・障害・体格差) を無視した均質化の論理として批判されてきた。

### 6.4 本アプリへの含意

作業ナビゲーションアプリにおいて「作業」をシステムの基本データ構造とする際には、JIS・IE 的な粒度定義の恣意性を認識する必要がある。作業分解粒度は業界・工程・作業者習熟度によって異なり、単一の汎用スキーマで全業種をカバーすることは困難である。

アプリ設計では作業単位を固定的に定義するのではなく、現場ごとのカスタマイズを可能にする柔軟な粒度設定機能と、「非標準的な調整作業」を記録できる自由記述フィールドを設けることが望ましい。また 4M の「人」起因の原因分類に偏らない、システム要因・環境要因を積極的に記録できるトレサビ設計が重要である。
