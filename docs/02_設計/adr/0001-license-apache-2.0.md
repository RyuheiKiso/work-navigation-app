# ADR-0001. ライセンス: Apache License 2.0

| 項目 | 内容 |
|---|---|
| Status | Accepted |
| Date | 2026-05-09 |
| Deciders | RyuheiKiso |
| 関連 | 企画書 §14.1（v0.7.0 で本決定）、上流制約 C-11 |

## Context and Problem Statement

work-navigation-app は OSS として公開し、中小製造業のライン生産現場（楔: 直接作業者×ライン生産×段取り替え×単位作業）に導入されることを目指す。OSS ライセンスは派生物の挙動・特許条項・SaaS 提供時の挙動・依存ライブラリとの互換性まで縛る最上流の意思決定であり、後から動かすと既存導入の法務確認をやり直すことになる。本 ADR は本企画書 v0.7.0 で確定した選定の機械的記録である。

## Decision Drivers

- **特許権の取扱いを明示する必要**: 製造業ドメインは特許密度が高く、特許トロール起因の差別圧力リスクを構造的に下げる必要がある。
- **派生クローズドの許容**: 企業の社内ツールへの組込みを阻害しないこと（GPL 系は阻害する）。
- **依存ライブラリとの互換性**: 主要 OSS 依存（PostgreSQL／React／Tauri／Keycloak 等）と互換であること。
- **企業導入時の法務確認の簡略化**: 著作権表示・NOTICE の要件が明示されており、企業法務が判断しやすいこと。
- **単一メンテナでの運用負荷**: ライセンス順守の運用負荷が単一メンテナで吸収可能なこと（NOTICE ファイル保守程度）。
- **収益化余地は要求しない**: 本プロジェクトは個人開発の OSS であり、デュアルライセンスによる収益化を主眼としない（企画書 Q15-A）。

## Considered Options

1. **Apache License 2.0**
2. MIT License
3. MPL-2.0（弱コピーレフト）
4. LGPL-3.0（弱コピーレフト）
5. GPL-3.0（強コピーレフト）
6. AGPL-3.0（ネットワーク・コピーレフト）
7. デュアルライセンス（OSS ＋ 商用）

## Decision Outcome

**選定: Apache License 2.0**

明示的な特許権付与（§3）と特許訴訟への報復条項（§3 後段）を備え、製造業の既存特許との衝突リスクを構造的に下げる。NOTICE ファイル要件は単一メンテナでも保守可能。著作権表示要件は MIT より厳格だが、企業導入時の法務確認を簡略化する効果がある。

### Consequences

- **Good**: 特許権付与により導入企業の法務確認が短縮される。
- **Good**: 派生クローズド可能であり、社内ツールへの組込みが阻害されない。
- **Good**: PostgreSQL／React／Tauri／Keycloak 等の主要依存と互換。
- **Good**: fork 自由により、メンテナ離脱時のユーザー離脱経路が保証される（企画書 §16.1 のリスク緩和）。
- **Bad**: NOTICE ファイル保守と再配布時の表示要件が MIT より厳格。
- **Bad**: SaaS 派生の場合、ソース開示義務がない（フォーク SaaS による Moat 侵食リスク）。本 ADR は §16.4 で AGPL 化を再評価対象として保持することで緩和する。

### 依存ライセンスの制約

依存範囲を **Apache-2.0 / MIT / BSD-2/3 / ISC / MPL-2.0** 互換に限定する（企画書 §14.1）。AGPL の依存（DB／検索エンジン／LLM 関連）は採用しない（企画書 Q39-Q）。

## Pros and Cons of the Options

### Apache License 2.0（採用）

- **Good**: 特許権付与・特許訴訟報復条項あり。
- **Good**: 派生クローズド可。
- **Good**: 企業法務での前例豊富。
- **Bad**: NOTICE 保守。

### MIT License（却下）

- **Good**: 最も短く、利用者が読みやすい。
- **Bad**: 特許権について沈黙する。製造業向け OSS では特許トロール起因の差別圧力リスクを残す。

### MPL-2.0 / LGPL-3.0（却下）

- **Bad**: 改変還元の縛りが商用組込みを遠ざける（企業の社内ツール統合の阻害要因）。

### GPL-3.0（却下）

- **Bad**: 派生 OSS 化強制が、製造業の既存資産（クローズドな社内ツール）への組込みを構造的に阻害する。

### AGPL-3.0（却下、再評価対象として保持）

- **Bad**: SaaS 提供を行わない（企画書 NS-12）ため主たる動機が成立しない。
- **再評価の条件**: 企画書 §16.1 シナリオ 1 (S) ／ §16.4 シナリオ 4 で「フォーク SaaS による Moat 侵食」が観測された場合に限り、本 ADR を Superseded として再評価する。

### デュアルライセンス（OSS ＋ 商用、却下）

- **Bad**: 単一メンテナ（企画書 Q15-A）の運用負荷に対し収益化余地は小さい。
- **Bad**: コントリビュータ受け入れ時に CLA が必須となり、参入障壁を上げる。
