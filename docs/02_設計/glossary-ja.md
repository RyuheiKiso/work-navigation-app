# 用語集（日本語）

> 対応 §: ロードマップ §28 §3 §9.4 §11.3.1 §22.1
> 対象読者: メンテナ、貢献者、ドキュメント・コード・UI・API スキーマの作成者
> 改訂サイクル: §22.1 半期サイクル／用語追加・削除は ADR（§9.4）と §24.2 出所表に同期

§9.1 採用の DDD はユビキタス言語の確立を要求する。本書は §3 ドメインモデリングを補強する **辞書本体** であり、UI 表記・コード命名・API スキーマ・ドキュメント記述すべての参照源となる。同義語・類義語の併存は禁止する（§9.4／§28）。

## 9. 用語の追加・削除規約（§28.3）

- 追加: PR 必須項目「ユビキタス語彙の更新が必要か Yes/No」を Yes にした PR で本章を更新。
- 削除: 廃止用語は ADR（§9.4）に理由を残し、CHANGELOG で 2 リリース分のグレース期間を設ける。
- 翻訳: §11.3.1 各言語版で本章の翻訳辞書を維持（[`glossary-en.md`](./glossary-en.md) ほか）。
- 略語・短縮形は **コード上のみ** 許容（命名規則）。UI・ドキュメント・API スキーマでは正式表記を強制。

## 10. 受入観点（§28.4）

- 主要用語が UI／コード／API／ドキュメントで一貫使用されていることを `scripts/glossary-lint.sh` が CI 検査（スクリプト未整備期は手動 grep）。
- 同義語混入は CI fail。
- 各言語版の対訳が同期している（§11.3.1 と整合）。
- §3 ／§9.3 ／§10 各章が本辞書を参照する形に保たれる。
- 新規用語追加が ADR（§9.4）と §24.2 出所表に同期して記録される。
