## 3. ADR 採番規約

### 3-1. 採番ルール

- `ADR-IMPL-001` から 3 桁連番で採番する。
- 欠番は **禁止** する。廃止した ADR には `[廃止]` タグを付与し、番号は再利用しない。
- 採番は `付録/99_実装識別子採番台帳.md` §6 と同期して行う。

### 3-2. ADR 起票の判断基準

以下の 3 条件を **全て** 満たす場合に ADR-IMPL-NNN を起票する。

1. **技術的に不可逆な決定**: 後から変更すると大きなコストを伴う判断（例: ライブラリ選定・ストレージ形式・通信プロトコル）
2. **有力な代替案が存在した**: 少なくとも 2 つの選択肢を比較検討した
3. **選択理由が自明でない**: 数ヶ月後に見直した際に理由が不明瞭になる可能性がある

上記条件を 1 つでも満たさない場合は ADR 起票を **対象外と判断する**（コメントまたは実装メモで十分）。

### 3-3. ADR の状態遷移

| 状態 | 定義 |
|---|---|
| `提案中` | 検討中。まだ確定していない |
| `採択` | 決定済み。実装に反映された |
| `廃止` | 後続 ADR により置き換えられた |
| `非推奨` | 技術的負債として認識されているが即時変更しない |

### 3-4. 詳細設計逸脱 ADR の必須記録

`付録/03_05詳細設計との対応一覧.md` §3 に記載の通り、05_詳細設計 からの逸脱を伴う実装変更は ADR-IMPL-NNN の起票が **必須** である（任意判断ではない）。

---

**本節で確定した方針**

- **ADR-IMPL-NNN は ADR-IMPL-001 から 3 桁連番で採番し、欠番禁止とする。**
- **ADR 起票基準の 3 条件（不可逆・代替案あり・理由非自明）を満たす場合のみ起票し、基準を下回る判断は実装メモ（実装メモ/README.md 参照）で記録する。**
- **詳細設計逸脱は条件に関わらず ADR-IMPL-NNN の起票を必須とする。**
