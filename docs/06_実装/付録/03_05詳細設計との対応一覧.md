# 03 05_詳細設計との対応一覧

本章は `05_詳細設計` の各サブディレクトリ・全章が `06_実装` のどの章・ファイルで具現化されているかを一覧化する。`05_詳細設計/付録/03_04概要設計との対応一覧.md` が概要設計→詳細設計の引き継ぎを担ったのと同様に、本章は詳細設計→実装の引き継ぎを監査可能にする。

---

## 1. 対応の目的

- 詳細設計の設計成果物（DDL・OpenAPI・型定義・アルゴリズム仕様）が漏れなく実装されていることを追跡する。
- 逸脱・変更が生じた場合に ADR-IMPL-NNN を起票し、詳細設計との差異を明示的に記録する。
- 上流仕様（05_詳細設計）の変更が実装に与える影響範囲を即座に特定できるようにする。

## 3. 変更・逸脱の記録方針

### 3-1. 逸脱の定義

詳細設計で確定した以下の成果物から実装が乖離する場合を「逸脱」と定義する。

- DDL の列名・型・制約の変更
- OpenAPI スキーマのフィールド追加・削除・型変更
- アルゴリズム仕様の変更（ハッシュ計算式・JSON Logic 評価順序等）
- テストケース仕様の変更（入力・期待出力の変更）

### 3-2. ADR-IMPL-NNN 連携

逸脱が発生した場合は以下のプロセスを踏む。

1. `付録/05_実装ADR索引.md` に ADR-IMPL-NNN を採番する（付録/99 の ADR-IMPL 台帳に登録）。
2. `docs/06_実装/18_実装判断ADR索引.md` に ADR 本文を記述する。
3. 本対応一覧の該当行を「完全実装」→「部分実装（ADR-IMPL-NNN 参照）」に更新する。
4. 05_詳細設計 の当該章に変更影響の注記を追記する（ドキュメント先行原則）。

### 3-3. 変更不可の識別子

MOD-001〜031・TBL-001〜035・API-001〜039・FR-NNN・NFR-NNN は上流確定済みであり、実装フェーズでの変更は **対象外と判断する**。変更が必要な場合は設計変更プロセス（PRJ-037）を経由する。

**本節で確定した方針**

- **05_詳細設計の 8 サブ + 付録から 06_実装 への引き継ぎ対応を全量確定し、詳細設計成果物の取りこぼしがないことを本対応一覧で監査可能にした。**
- **実装が詳細設計から逸脱する場合は必ず ADR-IMPL-NNN を起票し、本対応一覧の継承種別を更新する（逸脱の隠蔽禁止）。**
- **本章は付録/01（ITM）・付録/99（採番台帳）と三点セットで参照し、05→06 の引き継ぎ完全性の根拠文書とする。**
