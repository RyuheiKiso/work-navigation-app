## 5. 文書スタイル規約

本書（`08_移行/`）の全 md ファイルに以下のスタイル規約を適用する。

### 5-1. H1 命名規約

- 形式: `# NN タイトル`（NN はゼロ埋め 2 桁）
- `00`: 本書の位置づけ・識別子規約
- `01〜`: 本論章（通し番号）
- `07a / 07b / 07c`: サブ章（サフィックスで OS 区別）
- `99`: 前提制約と本書が約束しないこと
- `README`: 索引（H1 規約の対象外）

### 5-2. 本章責務段落

各章（README・採番台帳を除く）の冒頭に「本章の責務は〜することである。」または「本書の責務は〜することである。」で始まる段落を必ず置く。

### 5-3. 「本節で確定した方針」

各 H2 節の末尾に次の形式で配置する（README・付録/採番台帳・付録/README を除く）:

```
---

**本節で確定した方針**
- <断定文 1（「確定する」「準拠する」「対応する」「対象外と判断する」のいずれかで終止）>
- <断定文 2>
- <断定文 3>

---
```

### 5-4. 規制表記三段階

| 表記 | 意味 | 使用場面 |
|---|---|---|
| 確定する | 変更不可の設計決定 | 方式選定・スコープ宣言 |
| 準拠する | 標準・規格への完全適合 | IPA・ALCOA+ への準拠 |
| 対応する | 上流要件・設計への対応確認 | IPA タスクカバレッジ表 |
| 対象外と判断する | スコープ除外の明示 | D-MIG-X 宣言 |

**禁止表現**: 「考慮する」「検討する」「望ましい」「べき」「〜したい」「〜可能性がある」「〜場合がある」「適宜」「なるべく」

### 5-5. 章末「参照業界分析」

全章（README・付録/採番台帳を除く）の末尾に配置する:

```markdown
## 参照業界分析

### 必須

- [`90_業界分析/XX_*.md`](../../90_業界分析/XX_*.md) — <参照理由>

### 関連

- [`90_業界分析/YY_*.md`](../../90_業界分析/YY_*.md) — <参照理由>（最大 4 件）
```

**本節で確定した方針**
- §5-1〜§5-5 のスタイル規約を `08_移行/` の全 md に強制適用することを確定する。
