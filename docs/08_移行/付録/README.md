# 付録

本付録は `docs/08_移行/` の補完資料群である。採番台帳・トレーサビリティ正典（MTM）・IPA 共通フレーム 2013 カバレッジ表・事後レビュー報告書テンプレート・ADR 索引の 6 ファイルで構成される。

---

## §1 本付録の責務

本付録の責務は次の 5 点を提供することである。

1. **採番台帳**（`02_移行識別子採番台帳.md`）: `MIG-NNN`・`DES-MIG-NNN`・`MIG-X-NNN` 等 7 種の識別子について、採番状況・権威ソース・衝突防止規則を一元管理する。
2. **トレーサビリティ正典（MTM）**（`01_移行トレーサビリティマトリクス（MTM）.md`）: 移行要件 MIG-001〜074 を起点に、設計命題 DES-MIG → 実施命題 MIG-X/RH/RB → 検証 MIG-CK → 報告 MIG-REP までの双方向リンクを管理する。
3. **IPA カバレッジ表**（`03_IPA共通フレーム2013移行プロセスカバレッジ表.md`）: IPA 共通フレーム 2013 INST-A0〜A6 全タスクと 08 章の対応関係を正典化し、IPA 準拠を宣言する。
4. **事後レビュー報告書テンプレート**（`04_事後レビュー報告書テンプレ.md`）: IPA 6.5.5 文書化 + INST-A5-d 90 日事後レビューのテンプレートを提供する。
5. **ADR 索引**（`05_移行ADR索引.md`）: 移行段階の意思決定記録（ADR-MIG-NNN）と移行問題記録（PROB-MIG-NNN）を管理する。

## §3 付録の利用方法

### 採番台帳（`02_移行識別子採番台帳.md`）の利用

採番台帳は本文との整合確認に使用する。新規識別子を発番する際は台帳の「次採番」欄を参照し、衝突防止規則に従って登録する。本文ファイルに識別子を記述した後、台帳の「状態」列を「予約済み」から「確定済み」に更新する。

### MTM（`01_移行トレーサビリティマトリクス（MTM）.md`）の利用

MTM は要件追跡に使用する。MIG 要件が設計・実施・検証・報告のすべての階層で担当章を持つことを確認する双方向トレースに使用する。レビュー時は「状態」列が全行「確定済み」であることを確認する。

### IPA カバレッジ表（`03_IPA共通フレーム2013移行プロセスカバレッジ表.md`）の利用

IPA カバレッジ表は IPA 監査・社内品質監査に使用する。INST-A0〜A6 の全タスクについて担当ファイルが割り当てられていることを確認する。カバレッジ判定が「○（対応済み）」以外の行は、対応章を特定して担当ファイル列を更新する。

### 事後レビューテンプレート（`04_事後レビュー報告書テンプレ.md`）の利用

移行完了宣言後 90 日以内に quality_admin と system_admin が共同でテンプレートを埋め、GitHub Issue に登録する。記録は 7 年以上保存する。

### ADR 索引（`05_移行ADR索引.md`）の利用

移行計画の変更・手順解釈の確定・トレードオフ判断が発生した都度、ADR-MIG-NNN を追記する。廃止された決定は削除せず `[廃止]` ノートを付与して履歴を保持する。
