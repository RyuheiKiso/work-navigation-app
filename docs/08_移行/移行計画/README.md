# README 移行計画サブ

## 1. 本サブの責務

本サブ（`08_移行/移行計画/`）の責務は、IPA 共通フレーム 2013「INST-A1 移行プロセス開始準備」から「INST-A6 旧運用の凍結・撤去・記録保管」までの計画書群を提供することである。移行の「何を・いつ・誰が・どのように実施するか」を文書化し、quality_admin・system_admin・master_admin・現場監督・作業員の全ステークホルダーが移行判断の根拠として参照できる計画書体系を構成する。

実施手順（「どのように実行するか」の詳細操作）は `08_移行/導入手順/` サブが担当し、本サブは計画・判断基準・評価基準を確定することを責務とする。

## 5. 疑念軸別読み順

本サブに対してよく提起される懸念に直接応答する章の順序を示す。

| 疑念 | 応答する章の順序 |
|---|---|
| 「移行でデータが壊れるのでは」 | 03（LEGACY_IMPORT の取込方式）→ 08（品質チェック計画・クリティカルエラー定義） |
| 「カットオーバーに失敗したら」 | 07（ロールバック計画・発動条件）→ 06（カットオーバー中断判断基準） |
| 「訓練が間に合わないのでは」 | 09（T-2 週〜T+0 の 3 段階訓練スケジュール）→ 05（並行運用開始前の訓練完了確認） |
| 「受入評価の基準が曖昧では」 | 10（MIG-CK-051〜068 の定量基準・合格判定手順） |
| 「旧紙記録はどうなるのか」 | 11（フォールバック転換・保管台帳・廃棄判断はスコープ外） |
| 「移行判断は誰がするのか」 | 01（判定権限者・RACI）→ 06（Go/No-Go 判断手順） |
