# 00 本書の位置づけと識別子規約

本章の責務は、移行計画サブ（`08_移行/移行計画/`）の IPA 共通フレーム 2013 上の位置・担当する計画フェーズ（INST-A1+A2〜A6）の範囲・他サブとの責任分界・完成判定基準・識別子規約のすべてを権威的に確定することである。本章の規約は移行計画サブの全 md ファイルに適用され、下位章での再解釈を禁止する。

---

## 1. 本サブの責務と IPA タスク対応

### 1-1. IPA 共通フレーム 2013 上の位置

移行計画サブは、`08_移行/00_本書の位置づけと識別子規約.md`（以下「ルート 00 章」）が定めた IPA 共通フレーム 2013「INST-A0〜A6」全体のうち、計画フェーズ（INST-A1+A2〜A3・A5・A6）を担当する計画書群である。

INST-A4（移行作業の実施）および INST-A4.5（並行運用フェーズ管理）の実施手順は `08_移行/導入手順/` サブが担当する。ただし INST-A4.5 の管理計画（並行運用管理計画）は移行計画/05 章が担当する。

| IPA 活動 ID | 正規名称 | 担当サブ |
|---|---|---|
| INST-A0 | 移行プロセス計画の枠組み確定 | `08_移行/00_*.md`（ルート 00 章） |
| INST-A1 | 移行プロセス開始準備 | 移行計画/01 |
| INST-A2 | 移行計画書の作成・承認 | 移行計画/01〜08 |
| INST-A2.5 | 移行リハーサル計画と実施 | 移行計画/04 |
| INST-A3 | 移行通知・利用者教育・業務移行計画の周知 | 移行計画/09 |
| INST-A4 | 移行作業の実施 | 導入手順/01〜08 |
| INST-A4.5 | 並行運用フェーズ管理 | 移行計画/05 + 導入手順/09 |
| INST-A5 | 移行結果の評価・レビュー・完了承認 | 移行計画/10 |
| INST-A6 | 旧運用の凍結・撤去・記録保管 | 移行計画/11 |

### 1-2. 計画書群としての責務範囲

移行計画サブは「何を・いつ・誰が・どのような基準で判断するか」を確定する計画書群である。「どのように実行するか」の詳細操作手順は導入手順サブの責務である。

| 移行計画サブが確定する内容 | 導入手順サブが確定する内容 |
|---|---|
| 移行フェーズの定義（Phase M-1〜M-3） | 各 Phase の具体的な操作コマンド・実行手順 |
| 移行タスクの識別（MIG-T-001〜020） | 各タスクの step-by-step 操作手順書 |
| 合格基準・判定基準の確定 | 合格確認のための具体的な検証コマンド |
| リスクと対策方針の確定 | 対策実行の具体的な設定手順 |
| ロールバック発動条件の確定 | ロールバックの実行コマンド・操作手順 |

---

**本節で確定した方針**
- 移行計画サブが INST-A1〜A3・A5・A6 の計画フェーズを担当し、INST-A4 の実施手順は導入手順サブが担当することを確定する。
- 移行計画サブの責務を「計画・基準・判断」の確定に限定し、「実行手順の詳細」を導入手順サブに分界することを確定する。
- IPA 活動と担当サブの対応表（§1-1）を本サブの公式カバレッジ表として確定する。

### 2-2. スタイル規約の継承

## 4. 他サブ（導入手順・付録）との責任分界

### 4-1. 計画書（本サブ）vs 実施手順書（導入手順サブ）の境界

移行計画サブと導入手順サブの責任分界を以下のとおり確定する。

| 責任分界の観点 | 移行計画サブ（本サブ）の責務 | 導入手順サブの責務 |
|---|---|---|
| 文書の性質 | 計画書・基準書・評価書 | 実施手順書・操作ガイド・確認チェックリスト（実行時） |
| Phase M-1 への対応 | MIG-T-001〜010 の計画・担当者・合格基準 | MIG-T-001〜010 の実施コマンド・操作手順 |
| Phase M-2 への対応 | 並行運用管理計画（05 章）・完了条件の定義 | 日次ログ確認手順・エラー集計操作手順（導入手順/09） |
| Phase M-3 への対応 | カットオーバー計画（06 章）・ロールバック計画（07 章） | カットオーバー実行手順（導入手順/08）・ロールバック実行（導入手順/11） |
| 受入レビューへの対応 | 受入レビューのタイミング・チェックリスト・合格判定（10 章） | 受入チェックリストの実行手順（導入手順/10） |
| 旧システム廃止 | 廃止後の管理状態の計画（11 章） | 廃止作業の実施手順（廃棄はスコープ外） |

### 4-2. 移行計画サブ vs 付録サブの境界

| 付録ファイル | 付録の役割 | 本サブとの関係 |
|---|---|---|
| `付録/01_移行トレーサビリティマトリクス（MTM）.md` | MIG → DES-MIG → MIG-X の双方向トレーサビリティの正典 | 本サブの章別採番結果を MTM に反映する |
| `付録/02_移行識別子採番台帳.md` | MIG-X / MIG-CK / MIG-REP 等の採番台帳の正典 | 本サブで採番した識別子を台帳に転記する |
| `付録/03_IPA共通フレーム2013移行プロセスカバレッジ表.md` | INST-A0〜A6 の全件カバレッジの正典 | 本サブの §1-1 の IPA 対応表と整合する |

**本節で確定した方針**
- 移行計画サブの責務を「計画・基準・評価」の確定に限定し、「実行手順の詳細操作」を導入手順サブに分界することを確定する。
- 付録サブとの関係を §4-2 で明示し、採番後の台帳転記を義務として確定する。
- 移行計画サブが確定した MIG-X-NNN・MIG-CK-NNN・MIG-REP-NNN を `付録/02_移行識別子採番台帳.md` に転記することを完成条件として確定する。

## 5. 完成判定基準（本サブ固有）

### 5-1. 章完成の判定基準

移行計画サブの各章が「完成」と判定されるための条件を以下のとおり確定する。

| 判定基準 | 確認方法 |
|---|---|
| 章冒頭に「本章の責務は〜することである。」が存在する | 目視確認 |
| 全 H2 末尾に「本節で確定した方針」（3 行の断定文）が存在する | 目視確認 |
| 章末に「参照業界分析」節（必須・関連区分あり）が存在する | 目視確認 |
| 禁止表現（9 語）が 0 件である | grep 等による機械的確認 |
| 採番した識別子に欠番・重複が存在しない | `付録/02_移行識別子採番台帳.md` との照合 |
| ASCII 罫線文字（─ │ ┼ 等）が 0 件である | grep 等による機械的確認 |
| drawio 図の参照は Markdown 画像構文 `![図 N キャプション](img/fig_mig_xxx.svg)` を使用する | 目視確認 |
| 章内の行数が 80 行以上である | wc -l 等による確認 |

### 5-2. サブ全体の完成判定基準

| 判定基準 | 確認方法 |
|---|---|
| 00〜11 + 99 の全 13 章が §5-1 の章完成基準を充足 | 各章の完成確認後に横断確認 |
| 移行計画サブ担当の識別子（MIG-X-001〜105 等）が全件 `付録/02_移行識別子採番台帳.md` に転記済み | 台帳との照合 |
| `README.md` の章構成一覧と実際のファイル構成が一致 | 目視確認 |
| INST-A1〜A3・A5・A6 の全件に担当章が割り当てられている | §1-1 の IPA 対応表で確認 |

---

**本節で確定した方針**
- §5-1 の 8 項目を章完成の必要十分条件として確定し、1 項目でも未充足の章を「未完成」として扱うことを確定する。
- 禁止表現・ASCII 罫線文字の検出は機械的確認（grep 等）を完成判定の前提として確定する。
- 識別子台帳への転記を完成条件に含めることで、採番の一元管理を制度的に担保することを確定する。
