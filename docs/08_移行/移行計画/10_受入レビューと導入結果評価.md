本章の責務は、IPA 共通フレーム 2013「INST-A5 移行結果の評価・レビュー・完了承認」に対応し、カットオーバー後の受入レビュー手順・チェックリスト（MIG-CK-051〜068）・導入結果評価レポート（MIG-REP-016〜020）・受入合格判定の基準を確定することである。移行要件 MIG-060〜068（移行判定基準・移行完了定義）に対応し、quality_admin による電子署名での正式合格宣言から 90 日事後レビューの日程確定まで、移行完了の法的・組織的証跡を確定する。

## 2. 受入レビューの実施タイミング

### 2-1. カットオーバー後 3 回の受入レビュー

**MIG-X-090**: カットオーバー後の受入レビューをカットオーバー後 24 時間・72 時間・1 週間（168 時間）の 3 時点で実施することを確定する。各回の実施目的・主な確認範囲・判定権限者を以下のとおり定める。

| 受入レビュー回 | 実施タイミング | 実施目的 | 確認範囲 |
|---|---|---|---|
| 第 1 回（速報レビュー） | カットオーバー後 24 時間以内 | 重大障害・データ整合性異常の早期検出 | 機能受入（操作性・エラー率）の軽量確認 + 性能受入（レスポンス）の実測 |
| 第 2 回（中間レビュー） | カットオーバー後 72 時間以内 | 1 作業日を超えた運用安定性の確認 | 機能受入・性能受入・運用受入の 3 軸全量確認 |
| 第 3 回（最終レビュー） | カットオーバー後 1 週間以内 | 移行完了宣言の前提確認 | 18 項目全量確認 + 訓練効果測定（エラー率推移）+ 是正処置の完了確認 |

### 2-2. 各回の実施者・目的・判定基準

**MIG-X-091**: 各受入レビュー回の実施者・判定基準を以下のとおり確定する。

| レビュー回 | 実施者 | 判定基準 | 結果の扱い |
|---|---|---|---|
| 第 1 回 | system_admin（主）・quality_admin（確認） | 重大障害 0 件・データ欠損 0 件 | 合格→第 2 回へ / 不合格→是正処置後に再確認 |
| 第 2 回 | quality_admin（主）・system_admin（技術確認） | §3 の MIG-CK-051〜068 のうち 16 項目以上合格 | 合格→第 3 回へ / 不合格→是正処置計画の作成 |
| 第 3 回 | quality_admin（主・最終判定）・system_admin（確認） | §3 の MIG-CK-051〜068 全 18 項目合格 | 合格→移行完了宣言書の作成（§5-3）/ 不合格→§5-2 の是正処置 |

**MIG-X-092**: 受入レビューの実施記録（実施日時・実施者・確認項目・判定結果・是正処置の有無）を quality_admin が管理する移行完了記録に保管し、移行完了宣言書に添付することを確定する。

---

**本節で確定した方針**
- カットオーバー後 24 時間・72 時間・1 週間の 3 回のレビューを確定し（MIG-X-090）、各回の実施目的・確認範囲を段階的に拡大する構成を確定する。
- 第 3 回レビューで MIG-CK-051〜068 全 18 項目の合格を移行完了宣言の前提条件として確定する（MIG-X-091）。
- 受入レビューの実施記録保管を quality_admin の義務として確定し（MIG-X-092）、移行完了宣言書への添付を確定する。

## 3. 受入レビューチェックリスト（MIG-CK-051〜068）

### 3-1. チェックリスト構成概要

受入レビューチェックリストは以下の 3 軸 18 項目で構成する。各項目の判定は「合格 / 条件付き合格 / 不合格」の 3 段階とし、「不合格」が 1 件でもある場合は総合判定を不合格とする。「条件付き合格」は是正処置の期限（翌営業日以内）を明記した場合に限り認める。

| 軸 | 担当 ID 範囲 | 項目数 |
|---|---|---|
| 機能受入（操作性・エラー率） | MIG-CK-051〜056 | 6 項目 |
| 性能受入（レスポンス・スループット） | MIG-CK-057〜062 | 6 項目 |
| 運用受入（バックアップ・監視・引継ぎ） | MIG-CK-063〜068 | 6 項目 |

### 3-2. 機能受入（操作性・エラー率）軸

**MIG-CK-051**: 全作業員が補助なしでハンディ端末によるナビゲーション開始〜完了（クリティカルステップ電子サイン含む）を実施できることを確認する。合格基準: 抽出サンプル 10 件中 10 件が正常完了。

**MIG-CK-052**: 作業員によるスキップ・強制終了（操作エラー）の発生率がカットオーバー後 24 時間の全作業件数の 2% 未満であることを確認する。合格基準: エラー率 < 2%（DES-MIG-003 準拠）。

**MIG-CK-053**: オフライン状態での作業実施後、Wi-Fi 復旧時に SQLite ローカル記録が PostgreSQL に自動同期され、work_events テーブルの欠損が 0 件であることを確認する。合格基準: 同期欠損 0 件。

**MIG-CK-054**: quality_admin が SOP の電子承認フロー（ドラフト作成→承認付与→有効化）を補助なしで実施できることを確認する。合格基準: フロー完了まで補助なし。

**MIG-CK-055**: master_admin がマスタ管理 GUI（React Web）で工程・作業・製品マスタの新規登録・更新・非活性化を補助なしで実施できることを確認する。合格基準: 3 操作全件を補助なしで完了。

**MIG-CK-056**: 現場監督がアンドン発報受信・エスカレーション登録を補助なしで実施できることを確認する。合格基準: 発報から登録まで 5 分以内で完了。

### 3-3. 性能受入（レスポンス・スループット）軸

**MIG-CK-057**: ハンディ端末での作業ナビゲーション画面初期表示（SOP 取得・Step 一覧描画）が 2 秒以内であることを実測で確認する。合格基準: 10 回計測の中央値 ≤ 2.0 秒。

**MIG-CK-058**: 作業ステップの完了登録（work_events テーブルへの INSERT）が 1 秒以内であることを実測で確認する。合格基準: 10 回計測の中央値 ≤ 1.0 秒。

**MIG-CK-059**: マスタ管理 React Web アプリのページ遷移が 3 秒以内であることを実測で確認する。合格基準: 10 回計測の中央値 ≤ 3.0 秒。

**MIG-CK-060**: 並行運用中の最大同時接続数（想定: 接続ハンディ台数 × 2）における PostgreSQL の CPU 使用率が 80% 以下であることを確認する。合格基準: 並行運用ピーク時の CPU 使用率 ≤ 80%。

**MIG-CK-061**: 作業証跡のハッシュチェーン検証（work_events テーブル全件の SHA-256 整合性確認）が 30 分以内で完了することを確認する。合格基準: 検証完了時間 ≤ 30 分。

**MIG-CK-062**: Rust バックエンド（tokio + axum）のリクエスト処理において 500 系エラーが発生していないことをカットオーバー後 72 時間のログで確認する。合格基準: 500 系エラー 0 件。

### 3-4. 運用受入（バックアップ・監視・引継ぎ）軸

**MIG-CK-063**: PostgreSQL の自動バックアップが計画どおりのスケジュール（`04_概要設計/08_運用方式設計/` 準拠）で実行され、バックアップファイルの整合性検証が正常完了していることを確認する。合格基準: バックアップジョブ成功・整合性検証 OK。

**MIG-CK-064**: システム監視アラート（バックエンドプロセス停止・DB 接続エラー・ディスク容量 90% 超）が system_admin に通知されることを確認する。合格基準: テスト通知の受信確認。

**MIG-CK-065**: ロールバック手順書（移行計画/07 章）に記載の手順を system_admin が 4 時間以内に実行できることをドライラン（本番環境に影響しない範囲）で確認する。合格基準: ドライラン完了時間 ≤ 4 時間。

**MIG-CK-066**: 移行完了後の運用引継ぎ情報（運用手順書・連絡先・エスカレーション経路）が system_admin に引き渡されていることを確認する。合格基準: 引継ぎ文書の受領署名取得。

**MIG-CK-067**: LEGACY_IMPORT で投入した過去記録の source_document フィールドに旧記録の識別子が保持されており、work_events テーブルからの参照が正常であることを抽出サンプル 20 件で確認する。合格基準: 20 件中 20 件で参照正常。

**MIG-CK-068**: カットオーバー時点の PostgreSQL フルバックアップが取得済みであり、バックアップファイルのハッシュ値が記録されていることを確認する。合格基準: バックアップファイル存在 + ハッシュ記録 OK。

---

**本節で確定した方針**
- 受入チェックリストを機能受入・性能受入・運用受入の 3 軸 18 項目（MIG-CK-051〜068）で構成することを確定する。
- 各チェック項目に定量的合格基準を設け、定性的判断のみによる合格を認めないことを確定する。
- 「不合格」が 1 件でも存在する状態での総合合格を認めないことを確定し、「条件付き合格」は翌営業日以内の是正処置完了を条件として確定する。

## 4. 導入結果評価レポート（MIG-REP-016〜020）

### 4-1. 投入件数・移行所要時間・問題発生件数の記録

**MIG-REP-016**: 移行データ投入実績レポートとして以下の項目を記録する。記録者は system_admin とし、第 3 回受入レビュー前日までに quality_admin に提出することを確定する。

| 記録項目 | 内容 |
|---|---|
| マスタデータ投入件数 | 工程・作業・製品・SOP・ユーザーの各テーブルの最終投入件数 |
| LEGACY_IMPORT 投入件数 | 過去記録の取込件数（work_events テーブルへの登録件数） |
| 投入時の検証エラー件数 | クリティカルエラー（FK 違反・NULL・重複）の発生件数と解消件数 |
| 投入作業開始〜完了日時 | Phase M-1 の MIG-T-005 開始時刻〜MIG-T-006 完了時刻 |

**MIG-REP-017**: 移行所要時間実績レポートとして以下の項目を記録する。

| 記録項目 | 内容 |
|---|---|
| Phase M-1 所要日数 | MIG-T-001 開始日〜MIG-T-010 完了日の実績日数 |
| Phase M-2 所要日数 | MIG-T-011 開始日〜MIG-T-015 完了日の実績日数（計画比較付き） |
| Phase M-3 所要時間 | MIG-T-016 開始時刻〜MIG-T-020 完了時刻の実績時間（計画: 2〜4 時間との比較） |
| カットオーバー延期有無 | 当初計画日からの変更有無と理由 |

**MIG-REP-018**: 問題発生・解消実績レポートとして以下の項目を記録する。

| 記録項目 | 内容 |
|---|---|
| 移行期間中の問題件数 | 重大度別（重大 / 軽微 / 情報）の発生件数 |
| 解消済み件数 | 第 3 回受入レビュー時点の解消件数 |
| 未解消件数 | 第 3 回受入レビュー時点の未解消件数（0 件が合格基準） |
| 平均解消所要時間 | 問題報告受付〜解消完了の平均時間（時間単位） |
| ロールバック発動有無 | Phase M-2 中のロールバック発動の有無と理由 |

### 4-2. 訓練効果測定（エラー率推移）

**MIG-REP-019**: 訓練効果測定レポートとして、並行運用開始〜カットオーバー後 1 週間のエラー率推移を記録する。記録対象のエラーは「作業員による操作エラー（スキップ・強制終了・誤操作）」であり、全作業件数に対する割合で記録することを確定する。

| 計測期間 | 計測基点 | 記録内容 |
|---|---|---|
| Phase M-2 開始〜1 週目 | MIG-T-011 実施日 | 1 週目のエラー率（%）・エラー件数 |
| Phase M-2 2 週目〜最終週 | 各週の初日 | 週次エラー率の推移・最大値・最小値 |
| カットオーバー後 24 時間 | MIG-T-020 完了時刻 | カットオーバー後最初の 24 時間のエラー率 |
| カットオーバー後 72 時間〜1 週間 | カットオーバー後 72 時間時点 | エラー率の安定傾向（増加 / 横ばい / 減少） |

**MIG-REP-020**: 訓練効果測定の評価基準を以下のとおり確定する。並行運用中に 2% 以上のエラー率が 3 週間以上継続した場合は、再訓練（09 章の§3 に従う）を実施してからカットオーバー判断を行うことを確定する。

| 評価基準 | 判定 |
|---|---|
| カットオーバー後 1 週間のエラー率が 2% 未満で安定 | 訓練効果あり（合格） |
| カットオーバー後 1 週間のエラー率が 2% 以上で継続 | 訓練効果不足（是正処置対象） |
| 並行運用中のエラー率が一貫して減少トレンド | 訓練の習熟効果を確認（合格補足根拠） |

---

**本節で確定した方針**
- 導入結果評価レポートを投入件数実績（MIG-REP-016）・移行所要時間実績（MIG-REP-017）・問題発生件数（MIG-REP-018）・訓練効果測定（MIG-REP-019〜020）の 5 種で構成することを確定する。
- 全レポートの提出期限を第 3 回受入レビュー前日と確定し、system_admin が作成・quality_admin に提出する義務を確定する。
- エラー率 2% が継続する場合の再訓練実施を義務として確定し（MIG-REP-020）、カットオーバー判断の前提条件に組み込むことを確定する。

## 5. 受入合格判定と移行完了宣言

### 5-1. 3 軸 18 項目すべて合格 + quality_admin 電子署名で正式合格

**MIG-X-093**: 受入合格判定の基準を以下のとおり確定する。3 軸 18 項目すべてが合格（または期限内是正処置完了）であり、かつ quality_admin が電子署名または書面署名で正式合格を宣言した場合に、移行完了の合格判定とする。（MIG-064〜067 対応）

| 合格条件 | 判定方法 |
|---|---|
| MIG-CK-051〜068 全 18 項目が「合格」または「是正処置完了」 | 第 3 回受入レビューのチェックリスト記録で確認 |
| 重大障害（500 系エラー・データ欠損）が 0 件 | カットオーバー後 72 時間のログ確認で確認 |
| 未解消問題が 0 件 | MIG-REP-018 の未解消件数が 0 件であることを確認 |
| quality_admin による電子署名または書面署名 | 移行完了宣言書（§5-3）への署名取得 |

**MIG-X-094**: 合格条件のいずれか 1 つでも未充足の場合は不合格と判定し、§5-2 の是正処置計画書の作成に移行することを確定する。部分合格・暫定合格の扱いは認めない。

**MIG-X-095**: 合格判定の結果は、合否を問わず quality_admin が移行完了記録に記録し、第 3 回受入レビューの実施日から 5 営業日以内に記録を確定することを確定する。

### 5-2. 不合格時の是正処置計画書の作成

**MIG-X-096**: 不合格判定となった場合、system_admin は不合格通知受領後 2 営業日以内に以下の項目を含む是正処置計画書を作成し、quality_admin の承認を得ることを確定する。（MIG-067 対応）

| 是正処置計画書の必須項目 | 内容 |
|---|---|
| 不合格項目の特定 | 不合格となった MIG-CK 番号・不合格理由 |
| 根本原因 | 技術的原因または運用的原因の特定 |
| 是正処置内容 | 具体的な修正・設定変更・追加訓練の内容 |
| 是正処置完了期限 | 最大 5 営業日以内 |
| 再レビュー実施日 | 是正処置完了後 1 営業日以内 |
| 担当者 | 是正処置の実施責任者（氏名） |

**MIG-X-097**: 是正処置完了後の再レビューは、不合格となった項目のみを対象とした簡易レビューとする。再レビューの合格判定は quality_admin が行い、合格を確認した後に移行完了宣言の手続きに移行することを確定する。

### 5-3. 移行完了宣言書の作成と保存

**MIG-X-098**: 第 3 回受入レビューが合格と判定された後、quality_admin は以下の内容を含む移行完了宣言書を作成し、電子記録として保管することを確定する。保管期間は最低 7 年とする。（MIG-066 対応）

| 移行完了宣言書の必須記載項目 | 内容 |
|---|---|
| 宣言日時 | ISO 8601 形式（YYYY-MM-DDTHH:MM:SS+09:00） |
| カットオーバー完了日時 | MIG-T-020 の完了日時 |
| 移行実施者 | quality_admin・system_admin の氏名 |
| 3 軸 18 項目の合格確認 | MIG-CK-051〜068 の合格記録への参照 |
| 是正処置の完了確認 | 実施した是正処置の一覧（是正なしの場合は「なし」と記載） |
| 移行完了の宣言文 | 「本システムの移行が完了し、本番稼働を正式に認定する。」の文言を含む |
| quality_admin の署名 | 電子署名または自筆署名（スキャン保管） |
| 添付資料 | 受入レビュー記録・MIG-REP-016〜020 の各レポート |

### 5-4. 90 日事後レビュー予定の確定

移行完了宣言後 90 日時点での事後レビューは、移行の最終評価として MIG-068 が要求するものである。

移行完了宣言書の発行と同時に、90 日後の事後レビュー実施日を quality_admin が確定し、事後レビューの予定日を移行完了記録に記録することを確定する。（MIG-068 対応）

事後レビューで確認する項目:
- エラー率の長期推移（移行完了後 90 日間の実績）
- 未解消問題のその後の解消状況
- 訓練効果の持続（作業員の操作習熟度の維持）
- 運用引継ぎ後の運用品質（バックアップ・監視の継続状態）
- 移行計画との差異分析と次回移行への教訓

事後レビューの実施記録は quality_admin が作成し、移行完了宣言書と同じ保管場所に 7 年以上保管することを確定する。

---

**本節で確定した方針**
- 受入合格判定の条件を「MIG-CK-051〜068 全 18 項目合格 + quality_admin 電子署名」の両方充足に確定し（MIG-X-093）、部分合格・暫定合格を認めないことを確定する。
- 不合格時の是正処置計画書を 2 営業日以内に作成する義務（MIG-X-096）を確定し、最大 5 営業日以内の是正完了を確定する。
- 移行完了宣言書を 7 年以上保管する義務（MIG-X-098）と、90 日事後レビュー予定の宣言書発行当日確定を義務として確定する。
