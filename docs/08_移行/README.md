# 08_移行

`docs/08_移行/` は製造業向け作業ナビゲーション・トレサビシステムの初回リリース（ver1.0.0）における導入プロセスの成果物群である。IPA 共通フレーム 2013「3.1.2 システム導入プロセス（旧 1.5.6 移行プロセス）」の主活動 INST-A0〜A6 全件をカバーし、製造業固有の ALCOA+ 9 要素および GAMP 5 IQ/OQ/PQ バリデーション要件との統合を実現する。

---

## §1 本サブの責務

本サブディレクトリの責務は以下の 3 点である。

1. **IPA 共通フレーム 2013 INST-A0〜A6 全件カバレッジ**: 「3.1.2 システム導入プロセス（旧 1.5.6 移行プロセス）」に定義される主活動 INST-A0（導入計画の確立）・INST-A1（導入環境の準備）・INST-A2（データ移行の実施）・INST-A3（システム動作検証）・INST-A4（移行・カットオーバーの実施）・INST-A5（受入と評価）・INST-A6（旧システムの廃止）の全 35 タスクを担当章でカバーすることを宣言する。カバレッジの正典は `付録/03_IPA共通フレーム2013移行プロセスカバレッジ表.md` が管理する。

2. **製造業 ALCOA+ 9 要素 + GAMP 5 IQ/OQ/PQ の統合**: ALCOA+（Attributable・Legible・Contemporaneous・Original・Accurate・Complete・Consistent・Enduring・Available）9 要素と GAMP 5 IQ/OQ/PQ の 3 段階バリデーション要件を移行手順書に統合する。各対応の詳細は `付録/03_IPA共通フレーム2013移行プロセスカバレッジ表.md` §4 に記載する。

3. **移行要件 MIG-001〜074 の全件充足**: `docs/03_要件定義/移行要件/` に定義された移行要件 MIG-001〜074（全 74 件）を実施・検証・報告の階層で完全に充足することを宣言する。要件充足の正典は `付録/01_移行トレーサビリティマトリクス（MTM）.md` が管理する。

| 番号 | 基準 | 確認方法 |
