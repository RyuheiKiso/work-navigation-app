# 10 カットオーバー後 72 時間ハイパーケア手順

本章の責務は、INST-A4-l（72h ハイパーケア）に対応し、DES-MIG-059〜060 に定めた監視方針を具体的な手順に落とし込むことで、カットオーバー後 72 時間の集中監視・縮退検出・移行完了宣言の前提充足確認を確定することである。GAMP 5 PQ（Performance Qualification）の実施記録としても機能する。

---

## 1 本章の責務（INST-A4-l / DES-MIG-059〜060 対応）と GAMP 5 PQ の開始

### 1-1. INST-A4-l との対応関係

本章は IPA 共通フレーム 2013 の「3.2 導入プロセス」タスク INST-A4-l（72h ハイパーケア）に対応する実施手順書である。カットオーバー本番移行宣言（`08_カットオーバー手順書.md` §6-3 MIG-X-183）の完了をもって本章の手順を開始する。

本章は GAMP 5 の PQ（Performance Qualification）に対応し、本番稼働環境における実稼働条件下でのシステムパフォーマンスおよびデータ整合性を 72 時間にわたり実証する。

| 対応区分 | 内容 |
|---|---|
| DES-MIG-059 | カットオーバー後 72 時間の監視方針の実施 |
| DES-MIG-060 | 移行完了条件 4 項目の充足確認 |
| GAMP 5 PQ | 本番稼働条件下でのシステムパフォーマンス実証 |

---

**本節で確定した方針**
- 本章の手順は `08_カットオーバー手順書.md` §6-3 の本番移行宣言完了直後に開始することを確定する。
- GAMP 5 PQ の実施記録として本章の監視記録を管理することを確定する。
- 72 時間監視完了後の移行完了宣言を DES-MIG-060 の 4 条件全充足により判定することを確定する。

---

## 2 72 時間監視の対象

### 2-1. 監視対象と目標値

**MIG-X-191**: 72 時間ハイパーケア期間中の監視対象と目標値を以下に定める。

| 監視対象 | 監視内容 | 目標値 | 参照要件 |
|---|---|---|---|
| 証跡記録の正常生成率 | 全作業件数に対する `work_events` 生成率 | 100% | DES-MIG-059 |
| ハッシュチェーン整合性 | 整合性チェックバッチの結果 | エラー 0 件 | DES-MIG-060 |
| API エラーレート | API 総リクエストに対するエラー（4xx/5xx）の割合 | 5% 未満 | LEVEL-1 縮退基準 |
| API P95 応答時間 | API 応答時間の 95 パーセンタイル | 2 秒未満 | LEVEL-1 縮退基準 |
| DB 接続状態 | PostgreSQL への接続が維持されていること | 接続 100% | LEVEL-3 縮退基準 |
| エラーログ異常検出 | バックエンドコンテナの ERROR/FATAL ログ検出 | 0 件（FATAL）/ 5% 未満（ERROR） | 縮退LEVEL 判定 |

---

**本節で確定した方針**
- 証跡記録の正常生成率 100% を 72 時間監視の主要目標とすることを確定する。
- ハッシュチェーン整合性確認を 2 時間ごとに実施することを確定する。
- API エラーレート・P95 応答時間・DB 接続状態を継続監視の必須対象とすることを確定する。

## 4 ALCOA+ 追加チェック

### 4-1. 記録完全性率の自動計測と後入力確認

**MIG-X-193**: 72 時間ハイパーケア期間中は、ALCOA+（Attributable, Legible, Contemporaneous, Original, Accurate + Complete, Consistent, Enduring, Available）の要件に対応するため、以下の追加チェックを 12 時間ごとに実施する。

**MIG-CK-101〜110（ALCOA+ チェック項目）:**

| チェック番号 | チェック内容 | 確認クエリ・手順 | 合格基準 |
|---|---|---|---|
| MIG-CK-101 | 記録完全性率 100% 確認 | `SELECT COUNT(*) FROM work_events WHERE work_id IN (SELECT id FROM works WHERE completed_at IS NOT NULL AND completed_at >= NOW() - INTERVAL '12h') AND is_complete = true` | 完了作業に対するレコード生成率 100% |
| MIG-CK-102 | is_retroactive=false 確認（後入力ゼロ） | `SELECT COUNT(*) FROM work_events WHERE is_retroactive = true AND created_at >= NOW() - INTERVAL '12h'` | 件数が 0 であること |
| MIG-CK-103 | ハッシュチェーン連結確認（自動バッチ） | `docker exec terminal-api /usr/local/bin/verify_hash_chain --since 12h` | 検証エラー 0 件 |
| MIG-CK-104 | タイムスタンプの連続性確認 | 連続する `work_events` の `created_at` が単調増加であること | 逆順タイムスタンプ 0 件 |
| MIG-CK-105 | 記録者の一致確認 | `work_events.operator_id` と `works.assigned_operator_id` が一致すること | 不一致件数 0 件 |
| MIG-CK-106 | 必須フィールド NULL 確認 | `work_events` の必須フィールド（operator_id, work_id, step_id, created_at）に NULL がないこと | NULL 件数 0 件 |
| MIG-CK-107 | 外部キー整合性確認 | FK 違反の検出（`work_events.work_id` が `works` テーブルに存在すること） | FK 違反 0 件 |
| MIG-CK-108 | 二重記録確認 | 同一 `(work_id, step_id, operator_id)` の重複レコードがないこと | 重複 0 件 |
| MIG-CK-109 | 記録の可読性確認 | 文字化けや異常値を含むレコードがないこと | 目視確認で異常なし |
| MIG-CK-110 | バックアップ最新確認 | 直近 24 時間以内にバックアップが取得されていること | バックアップファイルの更新時刻確認 |

---

**本節で確定した方針**
- ALCOA+ チェック（MIG-CK-101〜110）を 12 時間ごとに実施することを確定する。
- is_retroactive=false の確認（MIG-CK-102）を後入力発生検出の必須手順とすることを確定する。
- ハッシュチェーン連結の自動バッチ確認（MIG-CK-103）を整合性監視の主要手段とすることを確定する。

## 5 縮退検出基準と対応

### 5-1. 縮退レベル定義と対応手順

**MIG-X-194**: ハイパーケア期間中に以下の縮退レベルを検出した場合、対応する手順を即時実行する。

**MIG-CK-111〜120（縮退検出チェック項目）:**

| チェック番号 | 縮退レベル | 検出条件 | 即時対応 | 通報先 |
|---|---|---|---|---|
| MIG-CK-111 | LEVEL-1（API 縮退） | API エラーレート ≥ 5% | エラーログの即時確認・原因特定 | system_admin 自身が即時通報 |
| MIG-CK-112 | LEVEL-1（API 縮退） | API P95 応答時間 ≥ 2 秒 | DB クエリの最適化・接続プール確認 | system_admin → quality_admin |
| MIG-CK-113 | LEVEL-2（緊急オフライン） | サーバー 60 秒以上到達不可 | 紙運用再開の判断・全作業員への通知 | quality_admin に即時通報 |
| MIG-CK-114 | LEVEL-2（緊急オフライン） | Docker コンテナの予期しない停止 | `docker compose up -d terminal-api master-api` で再起動試行 | quality_admin に即時通報 |
| MIG-CK-115 | LEVEL-3（サーバー停止） | PostgreSQL 停止（接続不可） | Docker 再起動・DB 復旧手順実行 | 即時ロールバック判断（11 章参照） |
| MIG-CK-116 | LEVEL-3（サーバー停止） | IIS サービス停止 | IIS 再起動試行 | 即時ロールバック判断（11 章参照） |
| MIG-CK-117 | 縮退継続（4h） | LEVEL-1 が 72h 内で 4 時間以上継続 | MIG-054 によるロールバック候補として quality_admin が評価 | quality_admin + system_admin 緊急協議 |
| MIG-CK-118 | 縮退継続（4h） | LEVEL-2 が 72h 内で 4 時間以上継続 | ロールバック実行（11 章）を強く推奨 | quality_admin 即時判断 |
| MIG-CK-119 | データ整合性異常 | ハッシュチェーン検証エラーが 1 件以上 | 異常レコードの特定・手動修正または再投入 | quality_admin + system_admin |
| MIG-CK-120 | データ完全性欠損 | 証跡記録生成率が 100% 未満 | 欠損レコードの特定・is_retroactive=true での後入力 | quality_admin 判断 |

### 5-2. 縮退継続（72h 内 4h 以上）時のロールバック判断

LEVEL-1 の縮退が 72 時間以内に累計 4 時間以上継続した場合、MIG-054 に基づき quality_admin がロールバック実施の要否を判断する。ロールバックを実施する場合は `11_ロールバック実行手順.md` の手順に従う。

---

**本節で確定した方針**
- LEVEL-3（サーバー停止）検出時は即時ロールバック判断（11 章参照）に対応することを確定する。
- 縮退継続 4 時間以上（LEVEL-1）は MIG-054 によるロールバック候補として評価することを確定する。
- ハッシュチェーン検証エラー発生時は quality_admin・system_admin の即時協議に対応することを確定する。

## 6 移行完了宣言の前提充足確認

### 6-1. 4 完了条件（DES-MIG-060 対応）の確認

**MIG-X-195**: 72 時間ハイパーケア完了時点で、以下の移行完了条件（DES-MIG-060）の全充足を確認する。

| 完了条件 | 確認クエリ・手順 | 合格基準 |
|---|---|---|
| 証跡記録の完全性（DES-MIG-060 条件 1） | `SELECT ROUND(COUNT(*) FILTER (WHERE is_complete=true) * 100.0 / COUNT(*), 2) FROM work_events WHERE created_at >= <cutover_time>` | 100% |
| ハッシュチェーン整合性（DES-MIG-060 条件 2） | `docker exec terminal-api /usr/local/bin/verify_hash_chain --full` | エラー 0 件 |
| 旧運用の完全停止（DES-MIG-060 条件 3） | 現場監督による紙記録使用ゼロの確認（目視調査） | 紙記録使用件数 0 件 |
| 重大バグ発生件数（DES-MIG-060 条件 4） | 課題管理システムで 72h 以内の Critical/High バグを確認 | 0 件 |

4 条件全充足の場合、quality_admin は移行完了宣言を実施する。1 条件でも未充足の場合は未充足条件を解消するまでハイパーケアを延長する。

### 6-2. 72h ハイパーケア完了記録テンプレート（MIG-REP-021〜022）

**MIG-REP-021**: 72 時間ハイパーケア完了時に作成する「72h ハイパーケア完了記録」のテンプレートを以下に定める。

| 記録区分 | 記録項目 |
|---|---|
| 基本情報 | ハイパーケア開始日時・終了日時・実施期間 |
| 36 スロット監視結果 | MIG-CK-083〜100 の全スロット確認結果（合格/不合格/スロット番号） |
| ALCOA+ チェック結果 | MIG-CK-101〜110 の確認結果（確認時刻・合格/不合格） |
| 縮退検出記録 | 縮退検出時刻・レベル・対応内容・復旧時刻 |
| 移行完了条件確認結果 | DES-MIG-060 の 4 条件の確認結果（確認時刻・合格/不合格） |
| 移行完了宣言 | 宣言時刻・quality_admin 電子署名 |

**MIG-REP-022**: 72h ハイパーケア完了記録は quality_admin が最終確認の上、電子署名を付与して移行記録文書として 3 年間保存する。以降の運用は `12_運用引継ぎ会議手順.md` の手順に移行する。

---

**本節で確定した方針**
- DES-MIG-060 の 4 完了条件の全充足を移行完了宣言の唯一の条件とすることを確定する。
- 72h ハイパーケア完了記録（MIG-REP-021）への quality_admin 電子署名を完了記録の必須要件とすることを確定する。
- 完了記録を移行記録文書として 3 年間保存することを確定する。
