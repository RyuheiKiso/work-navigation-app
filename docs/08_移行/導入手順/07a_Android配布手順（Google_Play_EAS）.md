## 2 EAS Build の実行

### 2-1. `eas build --platform android --profile production` の実行

**MIG-X-161**: Android 本番ビルドは以下のコマンドで実行する。実行前に統合書 §2 で確認した `eas.json` の `production` プロファイルが有効であることを確認する。

```bash
# EAS Build の実行（Android production ビルド）
eas build --platform android --profile production

# ビルド進捗確認（Expo ダッシュボードの URL が出力される）
# 例: https://expo.dev/accounts/<account>/projects/<project>/builds/<build-id>
```

ビルド実行後、Expo ダッシュボードでビルドのステータスが `FINISHED` になるまで待機する。ビルド所要時間は環境によって異なるが、通常 10〜30 分を要する。

### 2-2. ビルド完了確認と AAB/APK のダウンロード

**MIG-X-162**: EAS Build 完了後、以下の確認とダウンロードを実施する。

| 確認事項 | 確認方法 |
|---|---|
| ビルドステータスが `FINISHED` であること | Expo ダッシュボードで確認 |
| ビルドアーティファクトが AAB（Google Play 配布用）または APK（直接配布用）であること | Expo ダッシュボードのアーティファクト種別で確認 |
| アプリバージョンが `1.0.0`、versionCode が `1` であること | ビルド詳細画面で確認 |
| 署名済みであること | ビルド詳細画面の「Signed」表示で確認 |

ダウンロード手順を以下に定める。

```bash
# EAS CLI によるダウンロード
eas build:download --id <build-id>

# または Expo ダッシュボードからブラウザで直接ダウンロード
```

ダウンロードした AAB/APK ファイルは `releases/android/v1.0.0/` ディレクトリに保存し、ファイルのハッシュ値（SHA256）を記録する。

---

**本節で確定した方針**
- `eas build --platform android --profile production` を Android 本番ビルドの唯一の実行コマンドとすることを確定する。
- ビルドアーティファクトの SHA256 ハッシュ値を記録することを確定する。
- ビルドステータス `FINISHED` の確認をダウンロード開始条件とすることを確定する。

## 3 配布方法の選択と実施

### 3-1. 社内配布（APK 直接配布）

**MIG-X-163**: Google Play を経由しない社内配布の場合、以下の手順で APK を配布する。

社内 APK 配布手順を以下に定める。

| ステップ | 手順 | 担当者 |
|---|---|---|
| 1 | ビルドプロファイルを `staging`（APK 生成）に変更して EAS Build を実行する | system_admin |
| 2 | 生成された APK ファイルを社内ファイルサーバーの配布用ディレクトリにアップロードする | system_admin |
| 3 | 配布用ディレクトリのアクセス権限を確認する（工場内 LAN のみアクセス可能） | system_admin |
| 4 | ダウンロード URL を全 Android デバイスの担当者に共有する | system_admin |
| 5 | Android デバイスの「不明なアプリのインストール」設定を有効にする | 各デバイス担当者 |
| 6 | ブラウザからダウンロード URL にアクセスし APK をインストールする | 各デバイス担当者 |

APK 配布時には Android デバイスのセキュリティ設定で「不明なアプリのインストール」を許可する必要がある。インストール完了後は当該設定を無効に戻す。

### 3-2. Google Play 内部テスト配布

**MIG-X-164**: Google Play Console を使用した内部テスト配布の場合、以下の手順を実施する。

| ステップ | 手順 | 担当者 |
|---|---|---|
| 1 | Google Play Console（https://play.google.com/console）にログインする | system_admin |
| 2 | 対象アプリを選択し「内部テスト」トラックに移動する | system_admin |
| 3 | 「新しいリリースを作成」ボタンをクリックする | system_admin |
| 4 | MIG-X-162 でダウンロードした AAB ファイルをアップロードする | system_admin |
| 5 | リリースノート（日本語）を入力し「内部テストに保存」をクリックする | system_admin |
| 6 | テスターとして登録された Google アカウントを確認する（全端末担当者が登録済みであること） | system_admin |
| 7 | 内部テストトラックのダウンロード URL を全テスターに共有する | system_admin |

内部テスト配布はストア審査を必要とせず、アップロード後数分でテスターのデバイスへの配布が可能になる。

---

**本節で確定した方針**
- 社内配布（APK）とGoogle Play 内部テスト配布の両方の手順を準備し、統合書 §3 の判定表に基づいて選択することを確定する。
- APK 直接配布時のセキュリティ設定変更は対象デバイスのみに限定し、インストール後に無効化することを確定する。
- Google Play 内部テスト配布はトラック審査なしで実施することを確定する。

## 4 Android デバイスへのインストール確認

### 4-1. インストール完了確認と本番 URL 接続確認

**MIG-X-165**: 全 Android デバイスへのインストール完了後、以下のチェックリストで確認を実施する。全項目の確認完了をもって Android 配布完了と判定する。

| チェック番号 | 確認項目 | 確認方法 | 担当者 |
|---|---|---|---|
| MIG-CK-Android-01 | ハンディ APP がデバイスにインストール済みであること | アプリ一覧で確認 | system_admin |
| MIG-CK-Android-02 | アプリバージョンが `1.0.0` であること | アプリの「設定」→「バージョン情報」で確認 | system_admin |
| MIG-CK-Android-03 | 本番サーバー URL が正しく設定されていること | アプリの設定画面で URL を確認 | system_admin |
| MIG-CK-Android-04 | 本番 API エンドポイントへの接続が成功すること | アプリ内「接続確認」機能を実行 | system_admin |
| MIG-CK-Android-05 | テストアカウントでのログインが成功すること | worker_test アカウントでログイン実行 | system_admin |

疎通確認の結果は統合書 §5 のデバイス登録チェックリストに記録する。全 Android デバイスで MIG-CK-Android-01〜05 の全項目が合格になった時点で Android 配布手順を完了する。

---

**本節で確定した方針**
- MIG-CK-Android-01〜05 の全確認を Android 配布完了条件とすることを確定する。
- 疎通確認結果の統合書 §5 チェックリストへの記録を必須とすることを確定する。
- 全デバイス確認完了をもって Android 固有手順の完了と判定することを確定する。
