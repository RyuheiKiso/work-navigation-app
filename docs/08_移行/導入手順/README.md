# README — 導入手順サブ

本サブの責務は、IPA 共通フレーム 2013 の「3.2 導入プロセス」タスク INST-A4 全体の実施手順を提供することである。本番環境構築から本番リリース・ハンディ APP 配布・カットオーバー・並行運用・ハイパーケア・ロールバック・運用引継ぎ・確認チェックリストまでの全ステップを網羅する。

---

## 1 本サブの責務（INST-A4 全体の実施手順を提供する）

本サブは `08_移行/移行計画/` サブで確定した計画書群の実行手順を提供する。計画書が「方針・基準・判定条件」を定めるのに対し、本サブは「誰が・何を・どの順番で・どのように実施するか」を具体的に定める。

## 3 ステークホルダー別読み順

| ステークホルダー | 推奨読み順 | 読む目的 |
|---|---|---|
| system_admin | 00 → 01 → 02 → 03 → 04 → 08 → 10 | 本番環境構築からカットオーバー実行・ハイパーケア監視まで、技術実施の全体像を把握する |
| quality_admin | 00 → 05 → 06 → 07（統合）→ 08 → 10 → 12 | リハーサル・リリース判定・カットオーバー宣言・運用引継ぎまで、品質責任者の実施範囲を把握する |
| 現場監督 | 99 → 09 → 08 | 前提制約の確認・並行運用中の日次フロー・カットオーバー当日の役割を把握する |
