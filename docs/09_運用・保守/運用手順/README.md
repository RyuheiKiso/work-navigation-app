# 運用手順 — README

本ディレクトリは製造業向け作業ナビゲーションシステムの運用手順書群を収める。
IPA 共通フレーム 2013 SLCP-JCF2013「4.2.1 運用プロセス」および「4.2.1.c 業務及びシステムの運用」に直接対応する成果物である。


## 3. ステークホルダー別読み順

### IT 担当（system_admin）が初めて読む場合

1. `00_運用手順の位置づけと採番.md` — 採番・改訂管理を把握
2. `99_前提制約と運用手順サブが約束しないこと.md` — スコープ外を確認
3. `13_運用テストと初期稼働判定.md` — 初稼働判定フローを理解
4. `01_週次ヘルスチェック手順.md` — 日常運用の基本を習得
5. `02_バックアップ運用と完了確認.md` → `03_月次リストア検証手順.md`

### 障害対応中（インシデント発生時）

1. `docs/09_運用・保守/障害対応/` の該当インシデント手順を最優先で参照
2. フェイルオーバーが必要な場合: `10_ActiveStandby切替手順.md`
3. 緊急パッチが必要な場合: `09_緊急パッチ適用手順.md`
4. 計画停止が必要な場合: `11_計画停止と再開手順.md`

### 月次定期作業担当

1. 月初: `03_月次リストア検証手順.md`
2. 月末: `04_月次SLOレポート作成手順.md`
3. 入退職発生時: `05_アカウント入退職管理手順.md`

### 四半期・年次作業担当

1. 四半期: `06_JWT鍵ローテーション確認手順.md` + `07_四半期依存更新レビュー手順.md`
2. 年次: `08_年次自己監査手順.md`

## 6. 規制表記

本ディレクトリ内のすべての手順書は以下の表記規則に従う。

- 許容: 「準拠する」「対応する」「対象外と判断する」「確定する」
- 禁止: 「考慮する」「検討する」「努める」「目指す」「参考にする」「可能性がある」「場合がある」

## 8. 約束しないこと

`99_前提制約と運用手順サブが約束しないこと.md` に詳細を記述する。
主要事項:

- 24 時間有人監視（1 名体制のため夜間は無人）
- 自動障害復旧スクリプトによる無人復旧
- PagerDuty 等の外部アラート SaaS との連携
- 地理冗長切替（単一工場構成）
