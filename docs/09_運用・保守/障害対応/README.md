最終更新: 2026-05-18 | 管理者: system_admin

## § 1 位置づけ

本ディレクトリは IPA「システム管理基準」4.2.1.c（インシデント管理）および 4.2.2.b（問題管理）に対応する障害対応ドキュメント一式を格納する。

作業ナビゲーションシステムは製造業の作業記録・トレサビを担うため、障害時の適切な対応が ALCOA+（Attributable / Legible / Contemporaneous / Original / Accurate）原則の維持に直結する。障害対応手順はシステム保全と同等の優先度で整備する。

## § 3 ステークホルダー別読み順

### 3.1 障害発生中（今まさに対応中）

```
01 → 02 → 03 → 04 → 05
           ↓
        06（縮退判断）
           ↓
        07（BCP 判断）
```

**最短ルート（P1 時）**: 01 → 04（RUN-010〜019 該当節）→ 05（RUN-020〜029 該当節）→ 10（該当アラート）

### 3.2 事後分析（インシデントクローズ後）

```
11（ポストモーテム手順）→ docs/ポストモーテム/ に個別ファイル作成
→ 12（INC 索引に登録）
```

### 3.3 定期確認（週次・月次・四半期）

```
00（採番状況確認）→ 12（障害記録台帳）→ 99（前提制約の変化確認）
```

| 疑念（「これは何の障害か？」） | 参照先 |

| PostgreSQL が落ちた | 04（RUN-011）→ 05（RUN-022/023） |
