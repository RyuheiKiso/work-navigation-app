# 07 BCP発動と紙フォールバック

最終更新: 2026-05-18 | 管理者: system_admin | 根拠要件: OPS-062

---

## 1 概要

図: fig_inc_paper_fallback（img/ 配下）を参照

本章は LEVEL-3（サーバー完全停止）時の事業継続計画（BCP）を定義する。システムが完全停止した状態でも製造ラインの作業記録・品質管理が継続できるよう紙フォールバック手順を確定する。

**BCP 発動トリガー**

- 06 章 LEVEL-3 が発動してから 6 時間以上経過、かつ復旧見込みなし
- または system_admin と quality_admin が協議の上で即時 BCP 発動を決定

---

## 2 紙記録フォーム一覧

| フォーム ID | 対象業務 | 格納場所（紙の場所） |
|---|---|---|
| FORM-BCP-001 | 作業実績記録（工程別）| 各工程の監督ボックス |
| FORM-BCP-002 | IQC（受入検査）記録 | 受入検査エリア |
| FORM-BCP-003 | 検査結果記録 | 各検査ステーション |
| FORM-BCP-004 | 不適合記録 | 品質管理部門 |
| FORM-BCP-005 | リワーク指示票 | 品質管理部門 |

**FORM-BCP-001（作業実績記録）必須記入項目**

```
+----------------------------------------------------------+
| 作業実績記録（BCP 紙フォーム）                FORM-BCP-001 |
|                                                           |
| 記録日時（JST）: ___年___月___日 ___時___分              |
| 工程名:      ____________________________                 |
| 作業者 ID:   ____________________________                 |
| 作業指示番号: ____________________________                 |
| 製品ロット番号: __________________________                |
| 作業内容:    ____________________________                 |
| 実施結果:    良 / 不良（理由: _____________）             |
| 監督者確認:  ___________________________（署名）          |
+----------------------------------------------------------+
```

すべてのフォームに以下を必ず記入する: 記録日時（JST）・作業者 ID（個人名ではなく社員番号）・監督者確認署名。

**本節で確定した方針**
- **紙フォームは各工程の現場監督ボックスに 30 枚以上の在庫を常時確保する。**
- **在庫確認は四半期点検（06 章 BCP 訓練と同時）に実施する。**
- **BCP 発動時には現場監督が担当工程の FORM-BCP-001 配布を 15 分以内に完了させる。**

## 3 現場監督への連絡内容

BCP 発動時に system_admin が現場監督に伝達する内容:

```
連絡テンプレート（BCP 発動）:
「システムが完全停止しました。ただいまより紙による作業記録に切り替えます。
INC 番号: INC-YYYY-NNN
BCP 発動時刻: YYYY-MM-DD HH:MM（JST）
推定復旧時刻: HH:MM（※未確定の場合は「未定」と伝える）

対応事項:
1. 担当工程の FORM-BCP-001 を作業者に配布してください
2. 不適合が発生した場合は FORM-BCP-004 も使用してください
3. 記入済みの紙フォームは現場監督が回収・保管してください
4. 復旧後に system_admin から遡及入力の指示を行います

作業自体は継続可能です。SOP 参照は紙の作業標準書を使用してください。」
```

**本節で確定した方針**
- **連絡は電話（口頭）+ INC 通知文書（書面）の両方で行う。**
- **現場監督は BCP 連絡を受けた時刻と対応内容を FORM-BCP-001 の備考欄に記録する。**
- **system_admin は各工程への連絡完了を INC 記録に「連絡先・時刻」として記録する。**

## 4 紙フォームの管理・保管

| 項目 | 内容 |
|---|---|
| 回収担当 | 各工程の現場監督 |
| 一時保管場所 | 各工程の監督ボックス（施錠可能） |
| 長期保管場所 | 品質管理部門書庫（施錠・アクセス制限） |
| 保管期間 | 7 年（ALCOA+ Enduring 要件） |
| 遡及入力期限 | システム復旧後 24 時間以内 |

## 5 復旧後の遡及入力手順

システムが復旧した後、紙フォームの内容をシステムに遡及入力する。

### 5.1 遡及入力の前提条件

```
CHK-007: 遡及入力前確認
□ システム health check が全件 OK であることを確認した
□ ハッシュチェーン整合性（BAT-003）が OK であることを確認した
□ quality_admin が遡及入力の承認を行った
□ 対象の紙フォームが全件回収・確認済みである
□ 遡及入力に使用する日時は紙フォームの記録日時を使用することを確認した
```

### 5.2 遡及入力手順

```bash
# Step 1: 遡及入力用 API エンドポイント確認
curl -fsS http://localhost:8080/api/v1/ops/backfill/status \
  -H "Authorization: Bearer ${SYSTEM_ADMIN_JWT}" | jq .

# Step 2: 遡及入力（1 件ずつ実施）
# POST /api/v1/step-results （backfill=true パラメータ付き）
curl -fsX POST http://localhost:8080/api/v1/step-results \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${SYSTEM_ADMIN_JWT}" \
  -d '{
    "backfill": true,
    "recorded_at": "2026-05-18T14:30:00+09:00",
    "operator_id": "{OPERATOR_ID}",
    "step_id": "{STEP_ID}",
    "result": "pass",
    "backfill_reason": "BCP紙記録 INC-2026-NNN"
  }' | jq .
```

### 5.3 遡及入力後の確認

```sql
-- 遡及入力されたレコードの確認
SELECT id, operator_id, recorded_at, created_at, backfill_reason
FROM step_results
WHERE backfill_reason LIKE 'BCP%'
  AND created_at > NOW() - INTERVAL '1 day'
ORDER BY recorded_at;

-- ハッシュチェーン検証
SELECT count(*) FROM hash_chain_errors WHERE resolved_at IS NULL;
-- 期待値: 0
```

**本節で確定した方針**
- **遡及入力の recorded_at は紙フォームに記載された実際の作業時刻を使用する（システム復旧時刻は禁止）。**
- **遡及入力完了後に BAT-003（ハッシュチェーン検証）を実行し、整合性を確認する。**
- **遡及入力の全件完了を quality_admin が確認した後に LEVEL-3 解除を宣言する。**

## 6 BCP 訓練

| 項目 | 内容 |
|---|---|
| 訓練頻度 | 6 ヶ月毎（1 月・7 月を目安） |
| 訓練内容 | LEVEL-3 発動〜紙フォーム配布〜遡及入力〜解除の全フロー実施 |
| 訓練記録 | `docs/09_運用・保守/保守記録/` に BCP 訓練記録として保管 |
| 訓練後のアクション | 手順の不備・紙フォーム在庫不足を改善し、本章を更新する |

**訓練チェックリスト**

```
CHK-008: BCP 訓練
□ 紙フォームの在庫確認（各工程 30 枚以上）
□ LEVEL-3 発動手順の確認（06 章の通り）
□ 現場監督への連絡訓練（本章 §3 の手順で）
□ 紙フォームへの記録・回収シミュレーション
□ 遡及入力手順の確認（本章 §5 の手順で）
□ 訓練所要時間の記録（目標: 30 分以内で全工程に連絡完了）
```

**本節で確定した方針**
- **BCP 訓練は 6 ヶ月毎に実施し、訓練記録を保守記録に保管する。**
- **訓練で発見された手順の不備は 2 週間以内に本章を更新して解消する。**
- **紙フォームの在庫が 20 枚を下回った場合は即時補充を system_admin が手配する。**
