# 01 インシデント分類と判定基準（P1〜P4）

最終更新: 2026-05-18 | 管理者: system_admin | 根拠要件: OPS-054

---

## 1 概要

本章はインシデント優先度 P1〜P4 の定義・判定基準・具体例・判定権限を確定する。判定は製造ラインへの実際の影響を基準とする。システム内部状態（CPU 使用率・ログエラー数）のみで判定することを禁止する。

---

## 2 優先度定義（OPS-054）

### 2.1 P1（緊急）— 完全停止

| 項目 | 内容 |
|---|---|
| 定義 | システム全体が利用不可、または製造ラインの作業記録が完全に不能 |
| 影響 | 全工程で作業指示参照・作業実績記録・検査記録のいずれも不可 |
| RTO | 1 時間以内（OPS-057） |
| 検知 SLO | 5 分以内 |
| 通知 SLO | 5 分以内 |
| 切り分け SLO | 15 分以内 |
| 事後分析 SLO | 48 時間以内 |

**P1 具体例**

| シナリオ | 判定根拠 |
|---|---|
| PostgreSQL サービス停止 | 全 API が 503 を返し作業記録が不可能 |
| Docker network 断（API コンテナ分離） | 全ハンディ端末から API への疎通が断絶 |
| WAL アーカイブ失敗 + DB 破損 | データ不可逆損失のリスクで即 P1 |
| IIS リバースプロキシ設定誤り（全 5xx） | 全クライアントから完全に到達不可 |
| ハッシュチェーン破断（ERR-DB-003）検知 | ALCOA+ Original 不変性への直接脅威 |

**本節で確定した方針**
- **P1 は「製造ラインで作業記録が完全に不能」という事実に基づき即時判定する。**
- **内部監視指標のみで判定せず、現場からの確認を優先する。**
- **P1 は system_admin またはラインに到達した現場監督が確認した時点で即座に判定する。**

---

### 2.2 P2（高）— 機能縮退

| 項目 | 内容 |
|---|---|
| 定義 | 一部機能停止、または一部作業記録に影響する状態 |
| 影響 | 特定工程・特定機能が利用不可だが、他の工程は継続可能 |
| RTO | 4 時間以内（OPS-057） |
| 検知 SLO | 15 分以内 |
| 通知 SLO | 15 分以内 |
| 切り分け SLO | 30 分以内 |
| 事後分析 SLO | 72 時間以内 |

**P2 具体例**

| シナリオ | 判定根拠 |
|---|---|
| 帳票生成機能（ERR-SYS-003）のみ停止 | 作業記録は継続可能だが出荷書類作成不可 |
| LDAP 連携断（ERR-EXT-002） | 新規ログイン不可。既存セッションは継続 |
| Outbox DLQ 滞留（ALERT-001） | 一部イベントの非同期処理が停止 |
| 特定ハンディ端末グループの接続不可 | 当該工程の記録に影響するが他工程は継続 |
| pg_dump バックアップ失敗（ALERT-004） | 稼働継続中だがリカバリポイントに影響 |

**本節で確定した方針**
- **P2 は「一部影響あり・完全停止ではない」状態を指し、縮退運用（06 章）の発動を判断する。**
- **P2 であっても ALCOA+ に影響が波及する場合は P1 に再分類する。**
- **P2 のポストモーテムは system_admin が要否を判断し、原則実施する（OPS-063）。**

---

### 2.3 P3（中）— パフォーマンス劣化

| 項目 | 内容 |
|---|---|
| 定義 | システムは稼働しているが応答時間が SLO を超過している状態 |
| 影響 | 作業記録は継続可能だが応答遅延により生産性が低下 |
| RTO | 翌営業日以内 |
| 対応 | 通常業務時間内での対応。夜間対応は発動しない |

**P3 具体例**

| シナリオ | SLO 基準 |
|---|---|
| API P95 レイテンシ > 5,000ms（ALERT-003） | api_step_event_post_duration_p95_ms |
| PostgreSQL slow query（実行時間 > 10s） | pg_stat_statements 参照 |
| ディスク使用率 > 85%（ALERT-002） | disk_usage_percent（遅延原因になる前に対処） |
| tokio スレッドプール枯渇（スレッド待ち増加） | tokio_workers_busy 指標 |

**本節で確定した方針**
- **P3 は翌営業日対応とし、SLO 超過が継続する場合は P2 に昇格させる。**
- **P3 のポストモーテムは不要だが、INC 記録は作成する。**
- **P3 時に縮退運用（06 章 LEVEL-1）の予防的発動を判断する。**

---

### 2.4 P4（低）— 軽微

| 項目 | 内容 |
|---|---|
| 定義 | 表示不具合・誤字等、作業記録継続に影響しない軽微な問題 |
| 影響 | 作業記録・検査記録に影響なし |
| RTO | 次回定期メンテナンス時 |
| 対応 | チケットに記録し定期バッチで対応 |

**P4 具体例**

| シナリオ |
|---|
| 画面上の項目ラベルの誤字（機能影響なし） |
| ログに INFO レベルの不要メッセージが出力される |
| Grafana ダッシュボードの表示項目の配置ずれ |

**本節で確定した方針**
- **P4 は INC 記録を作成するが、緊急対応は発動しない。**
- **P4 が複数同時発生する場合は P3 への昇格を評価する。**
- **ポストモーテムは不要。**

## 3 判定権限と判定原則

### 3.1 判定権限

| 役割 | 権限 |
|---|---|
| system_admin | P1〜P4 すべての判定権限を持つ |
| 現場監督 | P1 の初期判定権限を持つ（作業記録完全停止の確認が根拠） |

判定は確認時点で即座に行う。「しばらく様子を見る」という保留状態は P3 以上では禁止する。

### 3.2 製造ライン影響を基準とした判定原則

```
判定フロー:
1. 製造ラインで作業記録が完全に不可能か？ → YES: P1
2. 一部の工程・機能で記録・参照に影響があるか？ → YES: P2
3. 稼働中だが応答が遅い・SLO を超えているか？ → YES: P3
4. 上記いずれにも該当しない → P4
```

**昇格ルール**

| 条件 | 昇格 |
|---|---|
| P3 が 30 分以上継続 | P2 に昇格 |
| P2 が 2 時間以内に復旧見込みなし | P1 に昇格 |
| P4 が ALCOA+ に影響する可能性が判明 | P2 に昇格 |

**本節で確定した方針**
- **判定は製造ラインへの実際の影響事実に基づき行い、内部指標のみでは判定しない。**
- **判定後に優先度が変化した場合は INC 記録に昇格・降格の理由を記録する。**
- **current_P + 昇格条件 の組み合わせで判定できるよう上記フローを使用する。**
