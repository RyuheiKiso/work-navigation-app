# 00 障害対応の位置づけと採番

最終更新: 2026-05-18 | 管理者: system_admin

---

## 1 本章の目的

本章は障害対応ドキュメント群が体系上どこに位置するかを明示し、障害記録・ポストモーテム・ランブックの採番規約を確定する。すべての後続ファイル（01〜99）はここで確定した識別子体系に従う。

---

## 2 IPA 基準とのタスク対応

| IPA 4.2.1.c（インシデント管理） | 対応ファイル |
|---|---|
| インシデントの識別と記録 | 01、12 |
| 優先度の決定と拡大 | 01、02 |
| 診断と解決 | 04、05 |
| クローズと確認 | 02（F5）|
| 報告 | 11 |

| IPA 4.2.2.b（問題管理） | 対応ファイル |
|---|---|
| 問題の特定と記録 | 12 |
| 根本原因分析（RCA） | 11 |
| 既知エラーデータベース管理 | 13 |
| 恒久対策の追跡 | 11（アクション項目）|

**本節で確定した方針**
- **IPA 4.2.1.c（インシデント管理）は 01/02/04/05/12 が対応し、4.2.2.b（問題管理）は 11/13 が対応する。**
- **識別子は本章で採番体系を一元管理し、後続ファイルは参照のみ行う。**
- **タスク対応の過不足は四半期レビューで 00 章を更新して確認する。**

## 3 新規採番規約

### 3.1 INC-YYYY-NNN（インシデント記録）

| 項目 | 規約 |
|---|---|
| 形式 | `INC-YYYY-NNN`（YYYY=西暦4桁、NNN=001始まり3桁） |
| 採番者 | 対応開始時に system_admin が発番 |
| 発番タイミング | P1/P2: F2（通知）完了時。P3/P4: クローズ確認時 |
| 格納先 | 12 章の台帳に 1 行追加 |
| 例 | `INC-2026-001` |

```
INC 採番手順:
1. 12 章ファイルを開く
2. 現在の最大連番を確認
3. 連番 +1 で新規行を追加
4. 発番日時・P 分類・概要を記入
5. 対応完了後にステータスを更新
```

### 3.2 PM-YYYY-NNN（ポストモーテム）

| 項目 | 規約 |
|---|---|
| 形式 | `PM-YYYY-NNN`（NNN は INC 番号と一致させる） |
| 物理配置 | `docs/ポストモーテム/YYYY-MM-DD_PX_<slug>.md`（OPS-064 準拠） |
| 対象 | P1 は必須。P2 は system_admin が要否を判断し、原則実施 |
| 期限 | P1: 48 時間以内（OPS-057）。P2: 72 時間以内 |
| 例 | `PM-2026-001` → `docs/ポストモーテム/2026-05-20_P1_api-timeout.md` |

### 3.3 RUN-NNN（ランブック）

| 項目 | 規約 |
|---|---|
| 形式 | `RUN-NNN`（NNN=3桁、001始まり） |
| 割り当てブロック | RUN-001〜003 検知通知 / RUN-010〜019 切り分け / RUN-020〜029 復旧 |
| 追加規約 | 030 以降は空きを順次使用 |
| 変更管理 | RUN 内容変更時はファイルヘッダーの最終更新日と変更概要を更新 |

**本節で確定した方針**
- **INC は P1/P2 では通知完了時に即時発番し、P3/P4 はクローズ時に発番する。**
- **PM は INC 番号と一致させ、物理ファイルは `docs/ポストモーテム/` に格納する（OPS-064）。**
- **RUN は 001/010/020 ブロックを固定し、追加は 030 以降を順次使用する。**

### 4.2 ALERT 識別子

| ID | 条件 | 重大度 |
|---|---|---|
| ALERT-001 | `outbox_dlq_total > 100` | CRITICAL |
| ALERT-002 | `disk_usage_percent > 85%` | HIGH |
| ALERT-003 | `api_step_event_post_duration_p95_ms > 5000` | WARN |
| ALERT-004 | `backup_age_hours > 25` | HIGH |

### 4.3 ERR 識別子（全 22 件）

ERR-AUTH-001〜004 / ERR-VAL-001〜004 / ERR-BIZ-001〜005 / ERR-DB-001〜003 / ERR-EXT-001〜002 / ERR-SYS-001/003〜005 の全 22 件は 13 章に詳細を記載する。本章は識別子体系の確定のみ担い、内容変更は 13 章を正として管理する。

**本節で確定した方針**
- **OPS-054〜066 はすべて 03_要件定義 から継承し、本ディレクトリでは参照扱いとする。**
- **ALERT / ERR の追加・変更は 10 章・13 章を正として管理し、本章はカタログ確認用にとどめる。**
- **既存識別子の再採番は禁止する。**

## 5 ポストモーテム物理配置規約（OPS-064 詳細）

```
docs/ポストモーテム/
└── YYYY-MM-DD_PX_<slug>.md
    例: 2026-05-20_P1_api-db-connection-timeout.md
```

| 要素 | 規約 |
|---|---|
| `YYYY-MM-DD` | インシデント発生日（JST） |
| `PX` | 優先度（P1 または P2） |
| `<slug>` | 内容を表す英語ケバブケース（最大 5 単語） |
| 文字コード | UTF-8 |
| 改行 | LF |
