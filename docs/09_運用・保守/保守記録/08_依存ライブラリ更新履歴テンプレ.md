# 08 依存ライブラリ更新履歴テンプレ

本章は依存ライブラリの四半期更新台帳テンプレートを提供する。MNT-038・NFR-OPS-042 に基づく四半期更新義務の記録・CVE 対処の台帳管理を実体化する。

---

## 1. 台帳の目的と更新タイミング

| 目的 | 内容 |
|---|---|
| 四半期更新の記録 | MNT-038 の更新義務を記録で証明する（ALCOA+ Accurate / Enduring 原則）|
| CVE 対処の追跡 | セキュリティパッチの採否・対処内容を追跡する |
| 依存 EOL の管理 | 依存ライブラリの EOL 情報を記録して廃棄判断（06 章）に活用する |

**更新タイミング**:
- 四半期定期更新実施時（Q1: 1〜3 月末・Q2: 4〜6 月末・Q3: 7〜9 月末・Q4: 10〜12 月末）
- CVE 対処時（緊急の場合は当日中に追記）
- EOL 情報更新時（ベンダー発表後 7 日以内に追記）

**本節で確定した方針**
- **四半期更新の実施結果は本台帳に記録し、REC-NNN（record_type: dependency_update）と紐付ける。**
- **CVE 対処は `cargo audit` / `pnpm audit` が CRITICAL/HIGH 0 件になるまで対処を続け、全 CVE に対して採否決定（対処 / 受け入れ / 例外適用）を記録する。**
- **依存 EOL 情報は発表後 7 日以内に本台帳へ記録し、廃棄評価（06 章 §2）の起点とする。**

## 3. CVE 対処記録

| CVE ID | 影響クレート / パッケージ | CVSS スコア | 発見日 | 採否 | 対処内容 | ロールバック条件 | REC-NNN |
|---|---|---|---|---|---|---|---|
| CVE-YYYY-NNNNN | <!-- 名前 @version --> | <!-- CVSS v3 スコア --> | YYYY-MM-DD | 対処 / 受入 / 例外 | <!-- 対処内容 --> | <!-- ロールバック基準 --> | REC-NNN |

### 3-1. 採否の定義

| 採否 | 定義 | 条件 |
|---|---|---|
| **対処** | 脆弱性のあるバージョンを更新または削除した | CVSS ≥ 7.0（HIGH / CRITICAL）は原則対処 |
| **受入** | リスクを評価した上で現状維持とした | CVSS < 4.0 かつ攻撃ベクタがローカルのみ |
| **例外適用** | 代替手段で脆弱性の影響を無効化した | WAF / 設定変更等で攻撃経路を遮断 |

「受入」および「例外適用」の採否は ADR-OPS-NNN に根拠を記録する。

### 3-2. CVE 対処の緊急手順

P1 CVE（CVSS ≥ 9.0 または ALCOA+ 影響あり）は以下の手順で対処する。

```bash
# 1. 影響クレートの特定
cargo audit --json | jq '.vulnerabilities.list[].advisory.id'

# 2. 緊急更新
cargo update -p <vulnerable-crate>

# 3. テスト
cargo test --release
cargo audit

# 4. STG 確認
# 03 章 §4 の STG 5 項目確認を実施

# 5. 緊急本番移行
# 05 章の GO/NOGO 判定（P1 のためメンテナンス窓外移行を許可）
```

**本節で確定した方針**
- **CVE 対処記録は発見日から 24 時間以内に採否を決定し、本台帳に記録する（ALCOA+ Contemporaneous 原則）。**
- **「受入」「例外適用」の採否は ADR-OPS-NNN に根拠を記録してから本台帳に採否を記入する。根拠なしの「受入」は不正とする。**
- **CRITICAL CVE（CVSS ≥ 9.0）は発見当日中に P1 REC-NNN を起票して緊急対処を開始する。**
