# 00 保守記録の位置づけと採番

本章は `保守記録/` サブディレクトリの位置づけ・IPA 4.2.2.a〜f タスクカバレッジ・新規識別子採番規約・上流継承識別子の一覧を確定する権威文書である。保守記録サブで使用する全識別子の採番ルールを制定し、上流継承識別子の再採番を禁止する。

---

## 1. 保守記録サブの位置づけ

`保守記録/` サブは IPA SLCP-JCF2013「**4.2.2 保守プロセス**」全 6 タスクに準拠した記録・手順・テンプレートを提供する。個人開発 1 名体制（system_admin）の現実制約を前提とし、quality_admin（製品品質責任者）との協働が必要な箇所を明示する。

| 軸 | 内容 |
|---|---|
| **準拠プロセス** | SLCP-JCF2013 4.2.2.a〜f（全 6 タスク）・4.2.1.e（運用評価）|
| **権威** | 実行可能な保守手順・記録テンプレートの確定 |
| **非権威** | 設計変更・SLO 値変更・上流識別子の再採番（上流フェーズが権威）|

---

## 2. IPA 4.2.2.a〜f タスクカバレッジ表

| IPA タスク | タスク内容 | 担当ファイル | 準拠宣言 |
|---|---|---|---|
| 4.2.2.a | 保守プロセス開始準備（保守体制・受入れ条件の確立）| `01_保守プロセス開始準備.md` | 準拠する |
| 4.2.2.b | 問題把握及び修正分析（問題の識別・分類・原因分析）| `02_問題把握分析手順.md` | 準拠する |
| 4.2.2.c | 修正の実施（コード修正・パッチ適用・テスト確認）| `03_修正実施手順.md`・`08`〜`11` | 準拠する |
| 4.2.2.d | 保守レビュー及び/又は受入れ（修正後の評価・承認）| `04_保守レビュー受入れ.md` | 準拠する |
| 4.2.2.e | 移行（修正版の本番適用）| `05_本番移行.md` | 準拠する |
| 4.2.2.f | ソフトウェア廃棄（システム廃棄・データ消去・アーカイブ）| `06_ソフトウェア廃棄.md` | 準拠する |
| 4.2.1.e | 運用評価（年次評価・改善計画策定）| `12_運用評価レポートテンプレ.md` | 準拠する |

**本節で確定した方針**
- **IPA 4.2.2 の全 6 タスクは保守記録サブの 01〜06 で対応し、カバレッジに欠番を設けない。**
- **4.2.1.e（運用評価）は 12 章で対応し、年次サイクルで実施する。**
- **タスクカバレッジの追加・変更は CR-OPS-NNN を起票して本章を改訂する。**

## 3. 新規採番識別子

保守記録サブで新規に採番する識別子は以下の 3 種とする。権威台帳は `../付録/99_運用保守識別子採番台帳.md` §5〜§7 である。

### 3-1. REC-NNN（保守記録）

| 項目 | 内容 |
|---|---|
| **用途** | 保守作業の実施記録（バグ修正・依存更新・鍵ローテーション・DB マイグレーション等）|
| **採番方式** | 3 桁ゼロ埋め連番（REC-001 から開始）、欠番禁止 |
| **記録単位** | 1 つの保守作業 = 1 つの REC-NNN（複数修正を 1 回の CR で実施した場合も 1 つ）|
| **テンプレート** | `07_保守記録テンプレ（REC-NNN）.md` |
| **権威台帳** | `../付録/99_運用保守識別子採番台帳.md` §5 |

```
REC-001  # 最初の保守記録
REC-002  # 2 件目の保守記録
REC-999  # 上限（999 件超の場合は CR-OPS-NNN で採番方式改訂を申請）
```

### 3-2. CR-OPS-NNN（運用フェーズ変更要求）

| 項目 | 内容 |
|---|---|
| **用途** | 手順書・設定・SOP の変更要求。上流フェーズへの変更依頼にも使用 |
| **採番方式** | 3 桁ゼロ埋め連番（CR-OPS-001 から開始）、欠番禁止 |
| **起票条件** | 手順の変更・追加・廃止の全て。口頭での変更合意は不正とする |
| **権威台帳** | `../付録/99_運用保守識別子採番台帳.md` §6 |

### 3-3. ADR-OPS-NNN（運用判断 ADR）

| 項目 | 内容 |
|---|---|
| **用途** | 運用フェーズ固有の Architecture Decision Record（保守戦略・廃棄判断等）|
| **採番方式** | 3 桁ゼロ埋め連番（ADR-OPS-001 から開始）、欠番禁止 |
| **記録項目** | 背景・決定内容・却下した選択肢・影響・ステータス |
| **権威台帳** | `../付録/99_運用保守識別子採番台帳.md` §7 |

**本節で確定した方針**
- **保守記録サブの新規識別子は REC-NNN・CR-OPS-NNN・ADR-OPS-NNN の 3 種とし、全て欠番禁止とする。**
- **識別子は台帳（付録/99）登録前に使用しない。未登録識別子の参照は不正とする。**
- **採番方式の変更は CR-OPS-NNN を起票し、本章と付録/99 を同時改訂する。**

## 4. 上流継承識別子

### 4-1. MNT-036〜043（保守要件）

`03_要件定義/非機能要件/` から継承する保守要件 ID。再採番禁止。

| 識別子 | 要件概要 | 保守記録サブでの利用箇所 |
|---|---|---|
| MNT-036 | 保守プロセスの開始条件（インシデント / 定期 / 要求）| `01` §2 |
| MNT-037 | 修正時のバックアップ取得義務 | `05` §2 |
| MNT-038 | 依存ライブラリの四半期更新 | `08` §1 |
| MNT-039 | 鍵ローテーション周期の遵守 | `09` §1 |
| MNT-040 | SOP 改訂の quality_admin 承認 | `10` §3 |
| MNT-041 | 廃棄時のデータ 7 年保全 | `06` §4 |
| MNT-042 | ハンディ端末廃棄時の鍵廃棄 | `06` §5 |
| MNT-043 | 年次運用評価の実施 | `12` §1 |

### 4-2. NFR-OPS-040〜042（保守 SLO）

| 識別子 | SLO 内容 | 保守記録サブでの利用箇所 |
|---|---|---|
| NFR-OPS-040 | P1 修正完了 ≤ 24h（発見から本番適用まで）| `02` §4・`05` §1 |
| NFR-OPS-041 | P2 修正完了 ≤ 72h | `02` §4 |
| NFR-OPS-042 | 定期依存更新の年 4 回（四半期）完了 | `08` §1 |

### 4-3. PROB-NNN（06_実装/17_実装問題管理ログ）

`06_実装/17_実装問題管理ログ.md` で採番済みの問題 ID を継承する。保守フェーズで新たに発見した問題も同形式で起票し、当該ログに追記する。

```
継承形式: PROB-NNN（NNN は 06 章台帳の連番を継続使用）
新規問題の起票先: 06_実装/17_実装問題管理ログ.md（権威台帳）
保守記録サブでの参照: REC-NNN の related_prob フィールドに記載
```

**本節で確定した方針**
- **MNT-036〜043・NFR-OPS-040〜042・PROB-NNN は上流確定識別子であり、本章では参照のみ行う。**
- **継承識別子の再採番・値変更は対象外と判断する。変更が必要な場合は CR-OPS-NNN で上流改訂を依頼する。**
- **PROB-NNN の採番台帳は 06_実装/17_実装問題管理ログ.md が権威であり、保守記録サブはその権威を変更しない。**

## 5. 保守記録の物理的配置方針

| ディレクトリ | 配置内容 | 命名規約 |
|---|---|---|
| `保守記録/` | 手順書・テンプレート（本ディレクトリ）| `NN_タイトル.md` |
| `保守記録/img/` | 図ファイル（.drawio + .svg ペア）| `fig_mnt_<意味語>.drawio` / `.svg` |
| `../付録/99_...` | REC/CR-OPS/ADR-OPS 採番台帳 | — |
| `../../ポストモーテム/` | ポストモーテム個別記録 | `YYYY-MM-DD_PX_<slug>.md` |

個別の保守作業記録（REC-NNN の実体）は本ディレクトリに新規ファイルとして追加しない。REC-NNN の実体は `07_保守記録テンプレ（REC-NNN）.md` のコピーに記入した文書として管理し、命名は `REC-NNN_YYYY-MM-DD_<概要スラグ>.md` とする。

**本節で確定した方針**
- **保守記録テンプレートは `07` を使用し、記録実体は `REC-NNN_YYYY-MM-DD_<slug>.md` で本ディレクトリに追加する。**
- **手順書（01〜12・99）は本ディレクトリに配置し、図は `img/` に配置する。**
- **採番台帳は `../付録/99` が権威であり、本ディレクトリには台帳を複製しない。**
