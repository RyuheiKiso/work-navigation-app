# 99 前提制約と本書が約束しないこと

本章は `09_運用・保守` 全体の前提・制約・適用範囲外の事項を明記する。本書を利用する際に誤解を避けるための境界を定義し、本書が責任を負わない領域を宣言する。本章は AI/ML に関する方針を言及できる唯一の節でもある。

---

## 1. 本書の適用範囲外

以下の事項は本書（09_運用・保守）の対象外である。

| 対象外の事項 | 権威ドキュメント | 判断理由 |
|---|---|---|
| 地理冗長 DR（Disaster Recovery）| 対象外と判断する | 単一工場・単一 IIS + WSL2 構成の物理制約。複数データセンターは現スコープ外 |
| 自動アラート SaaS 連携（PagerDuty 等）| 対象外と判断する | クローズド LAN 環境・個人開発スコープ。外部 SaaS への接続は不可 |
| 24 時間有人監視 | 対象外と判断する | 個人開発・1 名体制のため 24h 有人監視は不可能。夜間対応は `03_運用体制` §4 の方針に準拠する |
| AI/ML 自動診断・予知保全 | 対象外と判断する | 本章 §3 に詳述 |
| 多拠点クラスタ構成 | 対象外と判断する | 単一拠点の IIS + Windows Server 2022 + WSL2 + Docker 構成 |
| 設計変更の権限 | `04_概要設計`・`05_詳細設計` の責務 | 本書は実行可能な手順のみを定める |
| テスト仕様書・品質保証記録 | `07_テスト/` の責務 | テスト実施記録は別フェーズ |
| 要件定義・設計の変更 | `03〜05 章` の責務 | SLO 値・OPS-PROC 定義は上流が権威 |

**本節で確定した方針**
- **地理冗長 DR・24h 有人監視・SaaS アラートは対象外と判断する。根拠は単一拠点・個人開発制約である。**
- **設計変更の権限は 04〜05 章にあり、本書は手順・記録のみを担当する。**
- **対象外事項は省略ではなく明示的な判断であり、この節が権威的な宣言である。**

## 3. AI/ML に関する方針

本節はこのプロジェクト（09_運用・保守 フェーズ）において AI/ML を言及できる唯一の節である。

| AI/ML 機能 | 判断 | 理由 |
|---|---|---|
| AI による障害自動診断 | **対象外と判断する** | Just Culture・5 Whys に基づく人間主導のポストモーテムを採用する。決定論的手順が監査対応性を確保する |
| ML による予知保全 | **対象外と判断する** | クローズド LAN・個人開発スコープで学習データの収集・モデル管理が不可能 |
| 自動復旧スクリプト（AI 制御）| **対象外と判断する** | RUN-NNN の手順は人間が実行する。AI による自動復旧は製造データの整合性保証ができない |
| ChatOps（AI チャットボット）| **対象外と判断する** | クローズド LAN 環境での外部 AI サービス接続は不可 |

運用保守フェーズへの AI/ML 機能追加は ver 2.0 以降に ADR-OPS-NNN を経て正式に判断する。現在の手順はこれらの機能を前提としない。

**本節で確定した方針**
- **AI/ML を使用した障害診断・予知保全・自動復旧は ver 1.0.0 フェーズでは対象外と判断する。**
- **AI/ML 言及は本節（99 章）のみとする。他の章での言及を禁止する。**
- **ver 2.0 以降の AI/ML 導入は ADR-OPS-NNN を経て判断する。**

## 4. 本書が約束しないこと

以下の事項は本書（09_運用・保守）が約束しない。

- **障害の完全回避**: SLO（稼働率 99.5%）は目標値であり、すべての障害を防止することを保証しない
- **RTO/RPO の無条件達成**: NFR-OPS-002/003 の値は目標であり、重大なインフラ障害時の絶対保証ではない
- **夜間自動対応**: 夜間・休日の障害は system_admin の自発的な対応に依存する（`03_運用体制` §4）
- **第三者によるレビュー・監査**: 個人開発スコープのため、外部監査は対象外と判断する
- **ポストモーテムの外部公開**: PM-YYYY-NNN は内部文書として管理し、外部公開義務を負わない
- **セキュリティ侵害の完全防止**: セキュリティ要件（NFR-SEC）への対応は設計・実装フェーズで確保する。運用フェーズは KEY ローテーション・パッチ適用を担当する

**本節で確定した方針**
- **本書が約束しないことを明示する。過度な期待と誤解を防止する。**
- **SLO/RTO/RPO は目標値であり、保証値ではない。**
- **約束しない領域が問題になる場合は CR-OPS-NNN を起票して対策を確定する。**
