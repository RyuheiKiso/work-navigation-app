## 3. 09_運用・保守 新規プレフィックス定義

09 フェーズで新規に採番する識別子は以下の 7 種とする。

| プレフィックス | 対象 | 採番方式 | 欠番 | 権威台帳 |
|---|---|---|---|---|
| `CHK-NNN` | 定期確認チェックリスト項目（週次・月次・年次）| 3 桁ゼロ埋め連番（001 開始）| 禁止 | 付録/99 §1 |
| `RUN-NNN` | 障害対応ランブック（切り分け・復旧手順の実行単位）| 3 桁ゼロ埋め連番（001 開始）| 禁止 | 付録/99 §2 |
| `INC-YYYY-NNN` | 個別障害記録（年度 + 3 桁連番）| 年度ごとに 001 から開始 | 禁止 | 付録/99 §3 |
| `PM-YYYY-NNN` | ポストモーテム記録（年度 + 3 桁連番）| 年度ごとに 001 から開始 | 禁止 | 付録/99 §4 |
| `REC-NNN` | 保守記録（定期保守・修正・依存更新の記録単位）| 3 桁ゼロ埋め連番（001 開始）| 禁止 | 付録/99 §5 |
| `CR-OPS-NNN` | 運用フェーズ変更要求（手順・設定・上流文書の変更要求）| 3 桁ゼロ埋め連番（001 開始）| 禁止 | 付録/99 §6 |
| `ADR-OPS-NNN` | 運用判断 ADR（運用フェーズ固有の不可逆判断記録）| 3 桁ゼロ埋め連番（001 開始）| 禁止 | 付録/99 §7 |

## 4. 識別子の命名規則

### 4-1. 一般規則

- `NNN` は **3 桁ゼロ埋め**（001〜999）とする。
- 欠番は **禁止** する。廃止した識別子には `[廃止: YYYY-MM-DD 理由]` を付与し、番号を再利用しない。
- 大文字・ハイフン区切りを厳守する（例: `CHK-001`、`RUN-023`、`ADR-OPS-007`）。
- 識別子は台帳（付録/99）への登録前に使用しない（未登録識別子の参照は不正）。

### 4-2. INC/PM の年度付き採番

```
INC-2026-001  # 2026 年度 1 件目
INC-2026-002  # 2026 年度 2 件目
INC-2027-001  # 2027 年度は 001 から再開始（前年度識別子は維持）
PM-2026-001   # P1 障害 INC-2026-001 に対応するポストモーテム
```

年度をまたいで継続するインシデントは **発生年度の識別子** を維持する。

### 4-3. CHK の分類規則

```
CHK-001〜006   : 週次チェックリスト
CHK-007〜014   : 月次チェックリスト
CHK-015〜028   : 年次チェックリスト
CHK-029〜      : 臨時チェックリスト（CR-OPS-NNN での追加時）
```

## 5. 識別子ライフサイクル

| 状態 | 定義 | 次状態 |
|---|---|---|
| **予約済み** | 台帳に登録したが未実施 | 確定済み |
| **確定済み** | 手順が実施済み・記録が存在する | 廃止 / (維持) |
| **廃止** | 対応する手順・記録が削除または統合された | 欠番 |
| **欠番** | 廃止後の識別子。台帳の行は残し `[廃止]` タグを付与する | 終端状態 |

ライフサイクルの遷移は **台帳更新 → コミット** の順で行う（ドキュメント先行原則）。

---

## 6. 採番台帳との関係

- `付録/99_運用保守識別子採番台帳.md` が **7 種 prefix 全ての権威台帳** である。
- 本章（00）は規約の定義を行い、具体的な採番値・状態は 99 が管理する。
- 99 と本章との不整合が生じた場合は **台帳（99）を正** とし、本章を修正する。

---

## 7. 上流識別子との対応関係

```
NFR-OPS-001 ← OPS-PROC-002 ← CHK-001 ← REC-NNN（N:M 対応）
                    ↑
              RUN-001（障害時分岐）
                    ↓
              INC-YYYY-NNN → PM-YYYY-NNN（P1 障害時）
```

- 1 つの NFR に複数の OPS-PROC・RUN・CHK が対応することを許容する（M:N 対応）。
- 対応関係の全件は `付録/01 OTM` で追跡する。
