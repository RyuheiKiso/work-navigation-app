# 04 変更管理と版数規約

本章の責務は、`07_テスト` 配下の文書・識別子体系・品質ゲート基準に対する変更の手続き・版数規約・CR-TEST-NNN の起票プロセスを確定することである。

---

## 1. 版数規約

### 1-1. SemVer 採用

本書のすべての md ファイルはセマンティックバージョニング（SemVer: `MAJOR.MINOR.PATCH`）で版数を管理する。

| 版の区分 | 変更の種類 | 例 |
|---|---|---|
| MAJOR（X.0.0）| テスト戦略・ゲート基準の根本的変更、識別子体系の再構築 | `1.0.0`（初版完成）|
| MINOR（x.Y.0）| テストケース追加・実施手順の大幅変更・付録マトリクスの行追加 | `0.2.0` |
| PATCH（x.y.Z）| 誤記修正・参照パス修正・軽微な文言修正 | `0.1.1` |

初版は全ファイル `0.1.0`。ver1.0.0 は全テスト実施完了・全 RGATE 合格・サインオフ完了後に `1.0.0` へ昇格する。

### 1-2. 版数履歴テーブルの形式

全 md 末尾に以下の形式で配置する（省略禁止）。

```markdown
## 版数履歴

| 版 | 日付 | 変更者 | 変更内容 |
|---|---|---|---|
| 0.1.0 | 2026-05-18 | RyuheiKiso | 初版 |
```

## 2. 変更管理プロセス

### 2-1. 変更の種類と手続き

| 変更の種類 | 手続き | 記録 |
|---|---|---|
| **軽微な修正**（誤記・リンク切れ等）| 直接修正・版数 PATCH 更新 | 版数履歴に記載 |
| **通常変更**（実施手順追加・テンプレート改訂）| CR-TEST-NNN 起票 → 7 日クーリング → 適用 | CR-TEST-NNN + 版数履歴 |
| **重要変更**（ゲート基準変更・識別子体系変更）| CR-TEST-NNN 起票 → ADR-TEST-NNN 記録 → 7 日クーリング → 適用 | CR-TEST-NNN + ADR-TEST-NNN + 版数履歴 |
| **緊急変更**（P1 欠陥による即時対応）| 即時適用 → 事後 CR-TEST-NNN 起票（3 営業日以内）| CR-TEST-NNN（事後）+ 版数履歴 |

### 2-2. 個人開発のクーリング期間

個人開発のため、通常変更・重要変更とも 7 日間の自己再考クーリング期間を設ける。クーリング中に重大な問題を発見した場合は ADR-TEST-NNN で記録して撤回する。

## 3. CR-TEST-NNN テンプレート

```markdown
## CR-TEST-NNN 変更要求

| フィールド | 値 |
|---|---|
| CR-ID | CR-TEST-NNN |
| 起票日 | YYYY-MM-DD |
| 変更区分 | 軽微修正 / 通常変更 / 重要変更 / 緊急変更 |
| 対象ファイル | `docs/07_テスト/...` |
| 変更理由 | ... |
| 変更内容 | ... |
| 影響範囲（上流）| FR-NNN / TST-NNN / TENV-NNN への影響（あれば記載）|
| 影響範囲（下流）| なし（07_テスト が最下流）|
| 適用予定日 | YYYY-MM-DD（クーリング 7 日後）|
| 適用日 | YYYY-MM-DD |
| 状態 | PROPOSED → ACCEPTED → APPLIED → REJECTED |
```

## 4. 変更禁止事項

以下は CR-TEST-NNN だけでは変更できない。上流文書の改訂が必要である。

| 変更禁止項目 | 変更が必要な場合の手続き |
|---|---|
| 上流継承 TST-NNN（116 件）のテストケース内容 | `05_詳細設計/08_` の改訂 CR を起票 |
| TENV-001〜008 の環境定義 | `04_概要設計/10_テスト方式設計/02_` の改訂 CR を起票 |
| FR-NNN / UC-NNN / BR-BUS-NNN | `03_要件定義/` の改訂 CR を起票 |
| RGATE-001〜006 の合格基準数値 | `04_概要設計/10_テスト方式設計/04_` の改訂 + CR-TEST-NNN + ADR-TEST-NNN を起票 |

---

**本節で確定した方針**
- **SemVer 管理を全 md に適用し、初版 `0.1.0`・リリース完成 `1.0.0` の昇格基準を確定した。**
- **通常変更は CR-TEST-NNN 起票・7 日クーリング後に適用し、緊急変更のみ即時適用（事後起票 3 営業日以内）を許可することを確定した。**
- **上流継承識別子（TST-NNN / TENV-NNN / RGATE 基準数値）の変更は本書単独では行わず、必ず上流文書改訂 CR を起票することを確定した。**
