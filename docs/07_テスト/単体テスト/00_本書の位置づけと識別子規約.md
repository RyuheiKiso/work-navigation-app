# 00 本書の位置づけと識別子規約

本章は `07_テスト/単体テスト/` サブの IPA 対応関係・識別子継承宣言・責任範囲境界を確定する。

---

## 1. IPA §6.3.6.2「ソフトウェアユニットの検証」との対応

SLCP-JCF2013 §6.3.6.2 は「ソフトウェアユニットを設計要求事項に対して検証する」ことを要求するプロセスタスクである。本サブディレクトリは同タスクの実施活動に相当し、以下の対応関係を確定する。

| §6.3.6.2 要求タスク | 対応成果物 | 対応ファイル |
|---|---|---|
| テスト要件の特定 | テストケース詳細設計（権威ソース）| `05_詳細設計/08_テストケース詳細設計/01_`・`02_` |
| テスト手順の確立 | 実施手順書 | `01_`・`02_`（本サブ）|
| テストの実施 | CI 自動実行 + 手動実行手順 | `03_単体テスト実施計画.md`（本サブ）|
| テスト結果の評価 | カバレッジ計測・合否判定 | `04_`・`05_`（本サブ）|
| 欠陥の記録 | 欠陥管理テンプレート | `06_`（本サブ）|

対象テストケース: TST-unit-BE-001〜030（30 件）+ TST-unit-FE-001〜020（20 件）= 合計 50 件

**本節で確定した方針**
- **SLCP-JCF2013 §6.3.6.2 の全要求タスクを本サブの 5 ファイル（01〜06）に割付け、IPA 準拠を宣言する。**
- **テストケース定義（TST-ID・入力・期待出力・対応 FR/NFR）は `05_詳細設計/08_テストケース詳細設計/` が権威ソースであり、本サブはそれを変更しない。**

## 2. TST-unit-BE / TST-unit-FE 識別子の継承宣言

### 2-1. 識別子の権威ソース

TST-unit-BE-NNN および TST-unit-FE-NNN 識別子は以下のファイルで確定済みであり、本サブはその識別子を参照専用で引用する。

| 識別子プレフィクス | 権威ソースファイル | テストケース件数 |
|---|---|---|
| TST-unit-BE-001〜030 | `05_詳細設計/08_テストケース詳細設計/01_単体テストケース（バックエンド）.md` | 30 件 |
| TST-unit-FE-001〜020 | `05_詳細設計/08_テストケース詳細設計/02_単体テストケース（フロントエンド）.md` | 20 件 |

### 2-2. 本サブの識別子（新規採番）

本サブが新たに採番する識別子は以下の 2 種類のみとする。

| 識別子 | 採番規則 | 採番主体 |
|---|---|---|
| `TST-EXEC-YYYYMMDD-NNN` | 実施日 + 3 桁連番（当日分は 001 から開始）| テスト実施者 |
| `DEFECT-NNN` | プロジェクト横断 3 桁連番（全テストレベル共通・001 から開始）| 欠陥発見者 |

### 2-3. 変更禁止事項

TST-unit-BE-NNN および TST-unit-FE-NNN に対して、本サブは以下の変更を行わない。

- テストケースの追加・削除・ID 変更
- テスト対象・入力・期待出力・対応 FR/NFR の変更

上流変更が必要な場合は `05_詳細設計/08_テストケース詳細設計/` を改訂し、変更後の識別子を本サブが参照する。

**本節で確定した方針**
- **TST-unit-BE-001〜030 および TST-unit-FE-001〜020 は `05_詳細設計/08_` が唯一の権威ソースであり、本サブはその識別子を変更しないことを確定した。**
- **本サブが採番する新規識別子は TST-EXEC-YYYYMMDD-NNN と DEFECT-NNN の 2 種のみとすることを確定した。**

## 3. 本サブの責任範囲

### 3-1. 責任範囲

| 責任 | 内容 |
|---|---|
| 実施手順 | `cargo test` / `pnpm test` の具体的な実行コマンドと前提条件を確定する |
| 実施計画 | CI ジョブ設計・実施タイミング・失敗時対応手順を確定する |
| カバレッジ計測 | 計測ツール・目標値・除外対象・CI ゲート条件を確定する |
| 実施結果記録 | TST-EXEC 形式で実施事実を記録する空テンプレートを提供する |
| 欠陥管理 | DEFECT-NNN 採番・修正追跡・回帰テスト範囲を確定する |

### 3-2. 責任範囲外

| 対象外 | 理由 |
|---|---|
| テストコードの実装 | ソースコード管理（`src/` 配下）の責任範囲である |
| CI YAML の実装詳細 | インフラ設定（`.github/workflows/` 等）の責任範囲である |
| テストケースの設計変更 | `05_詳細設計/08_` の責任範囲である |
| 統合・E2E テスト | `07_テスト/結合テスト/` および `07_テスト/適格性確認テスト/` の責任範囲である |

**本節で確定した方針**
- **本サブの責任範囲は「実施手順・実施計画・カバレッジ計測・実施結果記録・欠陥管理」の 5 点であり、テストコード実装と CI YAML 詳細は対象外と判断する。**
- **統合・E2E テスト成果物は本サブの対象外と判断し、それぞれ専用サブディレクトリで管理することを確定した。**
