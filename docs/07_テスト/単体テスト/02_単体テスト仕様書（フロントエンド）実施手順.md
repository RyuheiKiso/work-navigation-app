# 02 単体テスト仕様書（フロントエンド）実施手順

本章は `05_詳細設計/08_テストケース詳細設計/02_単体テストケース（フロントエンド）.md` で確定済みの TST-unit-FE-001〜020 を引き継ぎ、SLCP-JCF2013 §6.3.6.2 に準拠した実施手順を確定する。テストケース自体（入力・期待出力・対応 FR/NFR）は上流文書が権威ソースであり、本章は変更しない。

---

## 1. 前提条件

本実施手順を実行するには、以下の環境が全て満たされていることを確認する。

| 前提項目 | 必要値 | 確認コマンド |
|---|---|---|
| Node.js | 20 LTS 以上 | `node --version` |
| pnpm | 8 以上 | `pnpm --version` |
| Jest | `package.json` に `^29` 以上で定義済み | `pnpm list jest` |
| @testing-library/react-native | `package.json` に定義済み | `pnpm list @testing-library/react-native` |
| jest-expo または jest-react-native | `jest.config.js` に preset 設定済み | `cat jest.config.js` |
| ネイティブモジュールモック | `__mocks__/` ディレクトリに設定済み | `ls src/__mocks__/` |

フロントエンドテストは外部 API・DB・ネイティブデバイス機能を一切使用しない。全テストケースはモック環境内で完結する。

**本節で確定した方針**
- **TST-unit-FE-001〜020 は全て外部依存なし（モック環境）で実行可能とすることを確定した。**
- **ネイティブモジュール（SQLite・カメラ等）は `__mocks__/` ディレクトリのモックで差し替えることを確定した。**

## 2. 実行コマンド一覧

### 2-1. 全件実行（推奨）

```sh
# 全件実行（CI 向け・ウォッチなし）
pnpm test --watchAll=false

# カバレッジ付き全件実行
pnpm test --watchAll=false --coverage
```

### 2-2. グループ単位の実行

```sh
# FE-001〜005: StepEngine.canAdvanceToStep
pnpm test --testPathPattern="StepEngine" --watchAll=false

# FE-006〜010: JsonLogicEvaluator
pnpm test --testPathPattern="JsonLogicEvaluator" --watchAll=false

# FE-011〜015: OutboxWorker バックオフ
pnpm test --testPathPattern="OutboxWorker" --watchAll=false

# FE-016〜020: 入力バリデーション
pnpm test --testPathPattern="validation" --watchAll=false
```

### 2-3. 特定ケースの単独実行

```sh
# TST-unit-FE-006 を単独実行
pnpm test --testNamePattern="TST-unit-FE-006" --watchAll=false

# TST-unit-FE-011〜015 をまとめて実行
pnpm test --testNamePattern="OutboxWorker" --watchAll=false
```

**本節で確定した方針**
- **CI での全件実行には `--watchAll=false` を必須オプションとして使用することを確定した。**
- **カバレッジ計測は `--coverage` オプションを付加した `pnpm test` コマンドで実施することを確定した。**

## 3. テストグループ別実施順序

テストグループは以下の順序で実施する。各グループは独立して実行可能であり、依存関係はない。

| 順序 | グループ | TST-ID 範囲 | テスト対象 | 外部依存 |
|---|---|---|---|---|
| 1 | StepEngine | FE-001〜005 | `StepEngine.canAdvanceToStep` | なし |
| 2 | JsonLogicEvaluator | FE-006〜010 | `JsonLogicEvaluator.evaluate` | なし |
| 3 | OutboxWorker バックオフ | FE-011〜015 | `OutboxWorker.flush` | API モック |
| 4 | 入力バリデーション | FE-016〜020 | 各バリデーション関数 | なし |

全グループは並列実行可能である（Jest のデフォルト並列実行を活用する）。ただし CI のリソース制約がある場合はグループ 1→2→3→4 の順に逐次実行する。

**本節で確定した方針**
- **FE-001〜020 の全グループは外部 API 依存なし（API モック使用）で実行可能とすることを確定した。**
- **Jest のデフォルト並列実行を許可し、CI リソース制約時のみ逐次実行に切り替えることを確定した。**

## 4. React Native テスト固有の注意点

### 4-1. ネイティブモジュールのモック設定

React Native 環境では、以下のネイティブモジュールを Jest モックで差し替える。

| ネイティブモジュール | モック方針 |
|---|---|
| SQLite（`react-native-sqlite-storage`）| インメモリ実装モックに差し替える |
| カメラ（証拠写真取得）| `jest.mock()` でスタブ化する |
| ネットワーク状態（`@react-native-community/netinfo`）| `NetInfo.fetch` を手動モックで制御する |
| AsyncStorage | `@react-native-async-storage/async-storage/jest/async-storage-mock` を使用する |

### 4-2. @testing-library/react-native 設定

| 設定項目 | 方針 |
|---|---|
| レンダリング | `render()` 関数でコンポーネントを VDOM にレンダリングする |
| ユーザーインタラクション | `fireEvent` を使用する（`userEvent` は React Native 19 以降で対応済みであれば使用可）|
| 非同期処理の待機 | `waitFor()` を使用する（`setTimeout` のモックは `jest.useFakeTimers()` で管理する）|

### 4-3. バックオフタイマーテスト（FE-011〜015）

OutboxWorker のバックオフ遅延テスト（1000/4000/16000ms）は実時間待機を行わない。`jest.useFakeTimers()` を使用して時間を手動進行させ、API 呼び出し回数と遅延間隔を検証する。

**本節で確定した方針**
- **全ネイティブモジュールは `__mocks__/` ディレクトリのモックで差し替え、実デバイス不要でテストを実行することを確定した。**
- **バックオフタイマーテストは `jest.useFakeTimers()` を使用し、実時間待機を排除することを確定した。**

## 5. PASS/FAIL 判定基準

| 判定 | 条件 |
|---|---|
| 個別 PASS | Jest が当該テストに対して `✓` を出力し、期待出力と一致する |
| 個別 FAIL | Jest が当該テストに対して `✗` を出力する、またはタイムアウトが発生する |
| グループ PASS | グループ内の全テストケースが PASS |
| 全件 PASS | TST-unit-FE-001〜020 の全 20 件が PASS かつ行カバレッジ（React Native ≥ 80%）を達成する |

FAIL が発生した場合は直ちに DEFECT-NNN を起票し、マージをブロックする。
