# 単体テスト

本サブディレクトリは SLCP-JCF2013 §6.3.6.2「ソフトウェアユニットの検証」に対応する単体テスト成果物を管理する。
テストケース定義は `05_詳細設計/08_テストケース詳細設計/` が権威ソースであり、本サブは実施手順・実施計画・カバレッジ計測方式・実施結果記録・欠陥管理の 5 責務を担う。

---

## 責務

| 責務 | 説明 |
|---|---|
| 実施手順の確定 | 上流で設計済みのテストケースを確実に実行するための手順を明示する |
| 実施計画の管理 | CI ジョブ設計・実施タイミング・失敗時対応手順を規定する |
| カバレッジ計測 | 計測ツール・目標値・除外対象・ゲート条件を確定する |
| 実施結果の記録 | TST-EXEC-YYYYMMDD-NNN 形式の記録テンプレートを提供する |
| 欠陥管理 | DEFECT-NNN 採番規約と修正追跡テンプレートを提供する |

## ステークホルダー別読み順

| ステークホルダー | 推奨読み順 |
|---|---|
| 開発者（初回参照）| `00_` → `01_` または `02_` → `03_` → `04_` |
| 開発者（テスト実行時）| `01_` または `02_` → `05_`（結果記録）→ `06_`（欠陥発生時）|
| レビュアー・品質確認者 | `00_` → `03_` → `04_` → `05_` → `06_` |
| 本書の維持管理者 | `99_` → `00_` → 全ファイル順に確認 |
