# 03 単体テスト実施計画

本章は SLCP-JCF2013 §6.3.6.2 に基づく単体テストの実施計画を確定する。CI ジョブ構成・実施タイミング・ローカル実行手順・失敗時対応・TST-EXEC 記録粒度を規定する。

---

## 1. 実施タイミング

単体テストは以下のタイミングで実施する。

| タイミング | 対象 | 実施主体 | 必須/任意 |
|---|---|---|---|
| コード変更のたびに（コミット前）| 変更対象ファイルに関連するテストグループ | 開発者（ローカル）| 必須 |
| Pull Request 作成・更新時 | TST-unit-BE-001〜030 + TST-unit-FE-001〜020 全件 | CI（自動）| 必須 |
| M3（ドメイン層実装完了）時 | TST-unit-BE-001〜030 全件 | CI（自動）+ 開発者確認 | 必須 |
| M4（フロントエンド実装完了）時 | TST-unit-FE-001〜020 全件 | CI（自動）+ 開発者確認 | 必須 |
| リリース前最終確認時 | 全 50 件 + カバレッジ達成確認 | 開発者（手動確認）| 必須 |

「コード変更のたびに CI で自動実行する」とは、Pull Request のコミット push をトリガーとして GitHub Actions が自動起動することを指す。

**本節で確定した方針**
- **単体テストは Pull Request の全コミット push を自動実行トリガーとすることを確定した。**
- **M3・M4 マイルストーン完了時に全件 PASS + カバレッジ達成を確認し、次工程に移行することを確定した。**

---

## 2. CI ジョブ構成

### 2-1. ジョブ一覧

| ジョブ名 | 対象 | 実行コマンド | 並列度 |
|---|---|---|---|
| `rust-unit` | TST-unit-BE-001〜030 | `cargo test --all --lib` | 1（逐次）|
| `rust-unit-coverage` | カバレッジ計測 | `cargo llvm-cov --all --lib --lcov --output-path lcov.info` | `rust-unit` 後に実行 |
| `react-unit` | TST-unit-FE-001〜020 | `pnpm test --watchAll=false` | 1（逐次）|
| `react-unit-coverage` | カバレッジ計測 | `pnpm test --watchAll=false --coverage` | `react-unit` 後に実行 |

### 2-2. ジョブ依存関係

```
[push to PR] → rust-unit → rust-unit-coverage
             → react-unit → react-unit-coverage
```

`rust-unit` と `react-unit` は並列実行する。各ジョブのカバレッジ計測は本体テストが PASS した後にのみ実行する。

### 2-3. CI ブロック条件

| 条件 | CI の動作 |
|---|---|
| `rust-unit` が 1 件以上 FAIL | マージをブロックする |
| `react-unit` が 1 件以上 FAIL | マージをブロックする |
| Rust 行カバレッジ < 80% | マージをブロックする |
| React Native 行カバレッジ < 80% | マージをブロックする |

**本節で確定した方針**
- **CI ジョブは `rust-unit` と `react-unit` を並列起動し、本体テスト PASS 後にカバレッジ計測を実行する 2 段構成とすることを確定した。**
- **テスト失敗またはカバレッジ目標未達のいずれの場合もマージをブロックすることを確定した。**

## 3. ローカル実行手順（開発者向け）

### 3-1. バックエンド単体テスト

```sh
# 前提確認
rustup show
cargo llvm-cov --version

# 全件実行
cargo test --all --lib

# カバレッジ計測（lcov.info + HTML レポート生成）
cargo llvm-cov --all --lib --lcov --output-path lcov.info
cargo llvm-cov --all --lib --html --output-dir reports/coverage/$(date +%Y%m%d)/rust
```

### 3-2. フロントエンド単体テスト

```sh
# 前提確認
node --version
pnpm --version

# 全件実行
pnpm test --watchAll=false

# カバレッジ計測
pnpm test --watchAll=false --coverage --coverageReporters=lcov,text-summary
```

### 3-3. 統合実行スクリプト（任意）

`scripts/test-unit.sh` が存在する場合は以下で全テストを一括実行できる。

```sh
bash scripts/test-unit.sh
```

スクリプトは上記コマンドを順次実行し、全件 PASS かつカバレッジ目標達成時に終了コード 0 を返す。いずれかが失敗した場合は終了コード 1 を返す。

---

## 4. 失敗時の対応手順

### 4-1. CI でテスト失敗が発生した場合

1. CI ログを確認し、FAIL したテスト関数名と失敗メッセージを特定する。
2. 対応する TST-ID を特定する（例: `test_tst_unit_be_003` → TST-unit-BE-003）。
3. DEFECT-NNN を起票する（`06_欠陥管理と修正記録テンプレート.md` のテンプレートを使用する）。
4. ブランチのマージを行わない（CI がブロックしている状態を維持する）。
5. 原因を修正してコミットし、再度 CI を実行する。
6. 全件 PASS を確認してから DEFECT-NNN の状態を VERIFIED/CLOSED に更新する。

### 4-2. ローカルでテスト失敗が発生した場合

1. ローカルで上記ステップ 1〜3 を実施する。
2. 修正後、ローカルで全件実行して PASS を確認してからコミットする。
3. コミット後の CI 結果で全件 PASS を再確認する。

**本節で確定した方針**
- **CI テスト失敗時はマージを行わず即時原因調査と DEFECT-NNN 起票を実施することを確定した。**
- **修正コミット後は必ず CI で全件 PASS を再確認してからマージすることを確定した。**

## 5. TST-EXEC 記録の粒度

| 記録事項 | 根拠 | 保管場所 |
|---|---|---|
| CI ビルド番号・実行時刻 | GitHub Actions の run_id と created_at | CI ログ（永続化）|
| 全件 Pass/Fail 件数 | `cargo test` / `jest` のサマリ出力 | TST-EXEC 記録テンプレート（`05_`）|
| 行カバレッジ（Rust/TS）| lcov.info サマリ・Jest --coverage 出力 | TST-EXEC 記録テンプレート（`05_`）|
| DEFECT-NNN 発生 | 欠陥管理テンプレート（`06_`）への起票 | `06_欠陥管理と修正記録テンプレート.md` |

CI ビルドログ 1 件 = TST-EXEC 記録 1 件の対応関係とする。ローカル実行の記録は開発者の任意とし、M3・M4 マイルストーン確認時のみ TST-EXEC 記録への転記を必須とする。

**本節で確定した方針**
- **CI ビルド 1 件を TST-EXEC-YYYYMMDD-NNN の 1 件として記録し、CI ログを根拠ドキュメントとすることを確定した。**
- **マイルストーン確認時の TST-EXEC 記録は `05_単体テスト実施結果テンプレート.md` のテンプレートに転記することを確定した。**
