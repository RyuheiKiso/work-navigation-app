# 04 カバレッジ計測方式と目標

本章はバックエンド（Rust）とフロントエンド（TypeScript）のカバレッジ計測ツール・設定・目標値・除外対象・CI ゲート条件を確定する。カバレッジ目標値は `04_概要設計` で確定済みであり、本章はその運用実装を規定する。

---

## 1. Rust バックエンドのカバレッジ計測

### 1-1. 計測ツール

| 項目 | 値 |
|---|---|
| ツール | `cargo-llvm-cov`（LLVM ソースベースカバレッジ）|
| インストール | `cargo install cargo-llvm-cov` |
| バージョン要件 | 0.6.0 以上 |
| 使用 LLVM 機能 | `-Z instrument-coverage`（nightly 不要・stable 対応）|

`grcov` は対象外と判断する。`cargo-llvm-cov` の方が計測精度が高く、分岐カバレッジにも対応しているためである。

### 1-2. 実行コマンド

```sh
# lcov.info 形式で出力（CI アーティファクト向け）
cargo llvm-cov --all --lib --lcov --output-path lcov.info

# HTML レポート生成（開発者向け目視確認）
cargo llvm-cov --all --lib --html --output-dir reports/coverage/$(date +%Y%m%d)/rust

# カバレッジ閾値チェック（CI ゲート）
cargo llvm-cov --all --lib --fail-under-lines 80
```

### 1-3. Cargo.toml への設定

```toml
[profile.coverage]
inherits = "test"
debug = 1
```

**本節で確定した方針**
- **Rust カバレッジ計測には `cargo-llvm-cov` を使用し、`grcov` は対象外と判断する。**
- **CI ゲートには `--fail-under-lines 80` オプションを使用して閾値未達を自動検出することを確定した。**

## 2. TypeScript フロントエンドのカバレッジ計測

### 2-1. 計測ツール

| 項目 | 値 |
|---|---|
| ツール | Jest 組み込みカバレッジ（Istanbul）|
| フレームワーク | Jest 29 以上 |
| 出力形式 | lcov（CI 向け）+ text-summary（コンソール向け）|

### 2-2. Jest 設定（`jest.config.js` または `package.json` の `jest` セクション）

```js
module.exports = {
  // ... 既存設定 ...
  coverageThreshold: {
    global: {
      lines: 80,
      branches: 80,
    },
  },
  coverageReporters: ['lcov', 'text-summary'],
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/*.stories.tsx',
    '!src/**/index.ts',   // バレルファイル除外
  ],
};
```

### 2-3. 実行コマンド

```sh
# カバレッジ計測（CI 向け）
pnpm test --watchAll=false --coverage --coverageReporters=lcov,text-summary

# カバレッジレポート HTML 生成（開発者向け）
pnpm test --watchAll=false --coverage --coverageReporters=html
```

**本節で確定した方針**
- **TypeScript カバレッジ計測は Jest 組み込みの Istanbul を使用し、別途ツール追加は行わないことを確定した。**
- **`coverageThreshold` を `jest.config.js` に記述し、閾値未達時に Jest が終了コード 1 を返すことで CI ゲートとすることを確定した。**

## 3. カバレッジ目標表

| 対象 | 指標 | 目標値 | 根拠 | 測定ツール |
|---|---|---|---|---|
| Rust バックエンド全体 | 行カバレッジ | ≥ 80% | 04_概要設計 確定値 | cargo-llvm-cov |
| Rust ドメイン層（`wnav_domain` クレート）| 行カバレッジ | ≥ 90% | 04_概要設計 確定値（ドメイン層は最高品質を要求）| cargo-llvm-cov |
| React Web（マスタメンテナンス）| 行カバレッジ | ≥ 80% | 04_概要設計 確定値 | Jest（Istanbul）|
| React Native（ハンディ APP）| 行カバレッジ | ≥ 80% | 04_概要設計 確定値 | Jest（Istanbul）|
| ERR ハンドリングコード全体 | 行カバレッジ | 100% | 04_概要設計 確定値（エラー処理は全経路検証必須）| cargo-llvm-cov / Jest |

目標値の緩和は対象外と判断する。目標値の引き上げは上流文書（04_概要設計）の改訂を経てから本章に反映する。

**本節で確定した方針**
- **カバレッジ目標値は 04_概要設計 で確定済みであり、本章はその運用実装を規定するのみで目標値を変更しないことを確定した。**
- **ERR ハンドリングコードのカバレッジは 100% を目標とし、除外対象から外すことを確定した。**

## 4. カバレッジ計測の除外対象

以下のコードはカバレッジ計測から除外する。

| 除外対象 | 理由 | 除外方法 |
|---|---|---|
| `main.rs` エントリポイント | バイナリ起動コードであり単体テストの対象外と判断する | `cargo llvm-cov --all --lib`（`--lib` でバイナリ除外）|
| `build.rs` | ビルドスクリプトであり実行時コードではない | 同上 |
| PoC コード（`poc/` 配下）| 試作コードであり製品コードではない | `.cargo/config.toml` で除外 |
| コード生成ファイル（`*.generated.rs`）| 生成コードの責任は生成ツール側にある | `#[cfg(not(coverage))]` または `cargo-llvm-cov` 除外設定 |
| `*.d.ts` ファイル | 型定義ファイルで実行コードを含まない | `collectCoverageFrom` の `!` パターン |
| `*.stories.tsx` ファイル | Storybook 専用であり本番実行コードではない | 同上 |
| バレルファイル（`index.ts`）| re-export のみで実ロジックを含まない | 同上 |

**本節で確定した方針**
- **PoC コード・生成コード・エントリポイントはカバレッジ除外対象とし、製品ロジックのみをカバレッジ対象とすることを確定した。**
- **除外設定は `jest.config.js` および `cargo-llvm-cov` の設定ファイルで明示的に記述することを確定した。**

## 5. CI でのカバレッジゲート

### 5-1. Rust バックエンド

```sh
# 閾値 80% 未達で CI fail（終了コード 1）
cargo llvm-cov --all --lib --fail-under-lines 80
```

### 5-2. TypeScript フロントエンド

`jest.config.js` の `coverageThreshold` 設定により、閾値未達時に Jest が終了コード 1 を返す。CI はこの終了コードを検出してジョブを FAIL とする。

### 5-3. ゲート動作まとめ

| ゲート | 閾値 | 判定失敗時の動作 |
|---|---|---|
| Rust 全体行カバレッジ | 80% | CI ジョブ FAIL → マージブロック |
| Rust ドメイン層行カバレッジ | 90% | CI ジョブ FAIL → マージブロック |
| React Web 行カバレッジ | 80% | CI ジョブ FAIL → マージブロック |
| React Native 行カバレッジ | 80% | CI ジョブ FAIL → マージブロック |

## 6. カバレッジレポートの保管

| 保管場所 | 形式 | 保管期間 | 用途 |
|---|---|---|---|
| CI アーティファクト | lcov.info | ビルド保持ポリシーに従う（最低 90 日）| CI 間の比較・回帰検出 |
| `reports/coverage/YYYYMMDD/rust/` | HTML | ローカル開発者確認用（Git 管理外）| 開発者の目視確認 |
| `reports/coverage/YYYYMMDD/ts/` | HTML | ローカル開発者確認用（Git 管理外）| 開発者の目視確認 |

`reports/coverage/` ディレクトリは `.gitignore` に追加し、Git リポジトリには含めない。
