# 06 欠陥管理と修正記録テンプレート

本章は SLCP-JCF2013 §6.3.6.2 に基づく欠陥管理の識別子規約・記録テンプレート・優先度別 SLA・修正後の回帰テスト範囲を確定する。DEFECT-NNN 識別子はプロジェクト全テストレベル横断で使用する。

---

## 1. DEFECT-NNN 採番規約

| 規則 | 内容 |
|---|---|
| フォーマット | `DEFECT-NNN`（例: DEFECT-001）|
| 連番範囲 | DEFECT-001 から開始し、上限なし |
| 連番単位 | プロジェクト横断（単体・結合・E2E テストを問わず通算採番する）|
| 欠番禁止 | クローズした欠陥番号は再利用しない |
| 採番主体 | 欠陥発見者（RyuheiKiso）|

DEFECT 台帳は本章の §4 に蓄積する。初版時点では空欄とする。

**本節で確定した方針**
- **DEFECT-NNN はテストレベル横断で通算採番し、欠番・番号再利用を禁止することを確定した。**
- **採番主体は欠陥発見者であり、発見と同時に起票することを確定した。**

---

| フィールド | 値 |

## 3. 重大度定義（P1/P2/P3）

| 重大度 | 定義 | 例 |
|---|---|---|
| P1（即時対応）| セキュリティ脆弱性・データ整合性破壊・システム停止を引き起こす欠陥 | RBAC バイパス・ハッシュチェーン改ざん・DB トリガ無効 |
| P2（影響 UC 再テスト）| 対応する FR/UC の正常動作を阻害するが P1 には該当しない欠陥 | ステートマシン遷移エラー・バックオフ計算誤り |
| P3（バックログ）| 機能に影響しないコード品質・パフォーマンス・ログ等の問題 | ログ出力フォーマット誤り・不要な警告 |

**本節で確定した方針**
- **重大度 P1 は発見直後にマージをブロックし、72 時間以内の修正を必須とすることを確定した。**
- **重大度 P2 は影響する UC の結合テスト再実施を修正確認の条件とすることを確定した。**

---

## 4. 優先度別対応 SLA

| 重大度 | CI ブロック | 修正期限 | 確認方法 |
|---|---|---|---|
| P1 | 即時（発見と同時にブロック）| 発見から 72 時間以内 | CI 全件 PASS + レビュー承認 |
| P2 | 即時（発見と同時にブロック）| 次の作業セッション内（最長 1 週間）| CI 全件 PASS |
| P3 | ブロックしない | バックログ優先度に従う | CI 全件 PASS（修正時）|

SLA の延長は対象外と判断する。P1 欠陥は 72 時間以内に修正するか、修正不可能な場合は当該機能の無効化を実施する。

**本節で確定した方針**
- **P1 欠陥の修正 SLA は 72 時間以内とし、延長を禁止することを確定した。**
- **P3 欠陥はマージをブロックせず、バックログとして管理することを確定した。**

---

## 5. 修正後の回帰テスト範囲

| 修正種別 | 回帰テスト範囲 |
|---|---|
| 関数・メソッドの実装変更 | L1 単体テスト全件（TST-unit-BE-001〜030 + TST-unit-FE-001〜020）を実行する |
| API エンドポイントの変更 | L1 単体テスト全件 + L2 統合テスト（TST-intg 対応範囲）を実行する |
| データベーススキーマ変更 | L1 単体テスト全件 + L2 統合テスト全件を実行する |
| 設定ファイル・環境変数の変更 | L1 単体テスト全件を実行する |
| テストコードのみの変更 | 変更したテストファイルに対応する TST-ID 範囲のみを実行する |

「L1 単体テスト全件を実行する」とは `cargo test --all --lib` と `pnpm test --watchAll=false` の両方を実行することを指す。

---

## 6. 状態遷移

欠陥の状態は以下の遷移のみを許可する。

```
OPEN → IN_PROGRESS → FIXED → VERIFIED → CLOSED
OPEN → VERIFIED → CLOSED  （軽微で IN_PROGRESS をスキップ可）
FIXED → OPEN  （修正確認で再現した場合）
```

状態の逆行（CLOSED → OPEN 等）は新規 DEFECT-NNN を起票することで表現する。既存の DEFECT-NNN の状態を CLOSED から戻すことは禁止する。

**本節で確定した方針**
- **欠陥状態の CLOSED からの逆行は禁止し、再現した場合は新規 DEFECT-NNN を起票することを確定した。**
- **関数・メソッドの変更を伴う修正後は L1 単体テスト全件を回帰実行することを確定した。**
