# 99 前提制約と本書が約束しないこと

本章は `07_テスト/単体テスト/` サブ全体の前提条件・適用範囲外事項・個人開発スコープの制約・AI/ML 方針・規制との関係・本書が約束しないこと・自己監査チェックリストを確定する。

---

## §1 範囲外

本サブが対象外と判断する事項は以下のとおりである。

| 対象外事項 | 理由 |
|---|---|
| テストコードの実装 | ソースコード（`src/` 配下）の責任範囲であり、本サブは文書成果物のみを管理する |
| CI YAML の実装詳細（`.github/workflows/` 等）| インフラ設定の責任範囲であり、本サブは実施計画のみを規定する |
| テストケースの新規設計・変更 | `05_詳細設計/08_テストケース詳細設計/` の責任範囲である |
| 結合テスト・E2E テストの実施手順 | `07_テスト/結合テスト/` および `07_テスト/適格性確認テスト/` の責任範囲である |
| テスト環境の構築手順詳細 | インフラ文書（`08_移行/` 等）の責任範囲である |
| パフォーマンステスト・セキュリティテスト | それぞれ専用テストケース（TST-perf / TST-sec）の責任範囲である |

---

## §2 個人開発スコープ

本プロジェクトは個人開発であり、以下の制約が存在する。

| 制約事項 | 内容 |
|---|---|
| 担当者 | テスト実施者は RyuheiKiso の 1 名である。担当者不在時のカバー体制は存在しない |
| レビュープロセス | 開発者自身によるセルフレビューを採用する。第三者レビューは行わない |
| DEFECT 修正 SLA | P1 は 72 時間以内・P2 は 1 週間以内であるが、個人スケジュールに依存する |
| テスト実施頻度 | CI による自動実行を基本とし、手動での強制実施を義務付けるプロセスは存在しない |

---

## §3 AI/ML 方針

本プロジェクトのテスト対象コードは AI/ML モデルを含まない。以下の点を確認する。

- テスト対象（WorkExecution ステートマシン・HashChainService・RBAC・JSON Logic・OutboxConsumer・Append-only 強制）はいずれも決定論的アルゴリズムであり、確率的出力を含まない。
- JSON Logic DSL の評価結果は入力が同一であれば常に同一の出力を返す（決定論性をテストケース BE-016〜020 / FE-006〜010 で検証する）。
- AI 支援ツールによって生成されたコードも本テスト仕様の対象とし、生成コードという理由でテスト除外を認めない。

## §4 規制との関係

| 規制 | 本書の立場 |
|---|---|
| ISO 9001:2015（品質マネジメント）| 個人開発のため認証取得は対象外と判断する。ただし品質マネジメントの考え方を本書の設計原則として準拠する |
| IEC 62443（産業用制御システムセキュリティ）| 本システムは産業用制御システムではないため対象外と判断する |
| FDA 21 CFR Part 11（電子記録・電子署名）| 製薬業界向け規制であり、本システムは対象外と判断する |
| SLCP-JCF2013 §6.3.6.2 | 本サブのプロセス設計の準拠基準として対応する。認証取得は行わない |

---

## §5 約束しないこと

本サブおよびその成果物が約束しない事項は以下のとおりである。

- **テストの網羅性保証**: 50 件のテストケースが全バグを発見できることを保証しない。テストケースは上流で設計された重要領域のみを対象とする。
- **テストコードの自動生成**: 本サブはテスト仕様書・手順書を作成するが、テストコードの実装は行わない。
- **カバレッジ目標の絶対安全性**: カバレッジ目標（80%/90%）の達成が製品の無欠陥性を意味しない。カバレッジは十分条件ではない。
- **継続的な文書更新**: テストケース追加・変更が発生した場合、本サブの文書が即時に反映されることを保証しない。上流文書の改訂後に本サブを追随して更新する。
- **第三者による独立検証**: 個人開発のため、独立した第三者によるテスト結果の検証は実施しない。
