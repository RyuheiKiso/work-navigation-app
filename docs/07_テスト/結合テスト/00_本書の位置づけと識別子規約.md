# 00 本書の位置づけと識別子規約

本章は `07_テスト/結合テスト` サブの IPA 共通フレーム 2013 における位置づけ・新定義識別子の採番規約・上流文書から引き継ぐ要件範囲を確定する。

---

## 1. IPA タスク対応表

IPA 共通フレーム 2013「§6.1.1.3 ソフトウェア結合テストプロセス」および「§6.1.2.4 システム結合テストプロセス」が要求するタスクと本サブ各章の対応を以下に確定する。

| SLCP-JCF2013 タスク | 対応ファイル | 備考 |
|---|---|---|
| §6.1.1.3 ソフトウェア結合テスト仕様書（実施手順）| `01_ソフトウェア結合テスト仕様書実施手順.md` | axum_test + sqlx::test |
| §6.1.1.3 API 契約テスト仕様書（実施手順）| `02_API契約テスト仕様書実施手順.md` | Schemathesis / OpenAPI 3.1 |
| §6.1.2.4 システム結合テスト仕様書（実施手順）| `03_システム結合テスト仕様書実施手順.md` | API + DB + Outbox 三者結合 |
| §6.1.1.3／§6.1.2.4 統合シナリオの確定 | `04_結合テストシナリオ.md` | シナリオ A〜E |
| §6.1.1.3／§6.1.2.4 テスト実施計画の作成 | `05_結合テスト実施計画.md` | CI ジョブ・マイルストーン対応 |
| §6.1.1.3／§6.1.2.4 テスト結果の記録と評価 | `06_結合テスト実施結果テンプレート.md` | TST-EXEC テンプレート |
| §6.1.1.3／§6.1.2.4 欠陥の記録と管理 | `07_欠陥管理と修正記録テンプレート.md` | DEFECT-NNN テンプレート |

**本節で確定した方針**
- **IPA §6.1.1.3（L2 統合テスト）および §6.1.2.4（システム結合テスト）の全タスクを本サブ 9 ファイル（00〜07 + 99）に割付けることを確定する。**
- **本サブは上流 3 文書（要件定義・概要設計・詳細設計）を変更しない。実施手順として実行可能なプロセスのみを規定する。**
- **テストケース ID TST-intg-001〜020 は `05_詳細設計/08_テストケース詳細設計/03_統合テストケース（API）.md` が権威ソースであり、本サブはそれを継承する。**

### 2-2. 本サブで新定義する識別子

| 識別子 | フォーマット | 例 | 用途 |
|---|---|---|---|
| DEFECT-NNN | 3 桁ゼロ埋め（全テストレベル横断）| DEFECT-001 | 欠陥報告（P1/P2/P3）|
| TST-EXEC-YYYYMMDD-NNN | 日付＋3 桁 | TST-EXEC-20260518-001 | テスト実行記録 |

本サブにおける DEFECT-NNN は `07_テスト/テスト計画/00_本書の位置づけと識別子規約.md` §2-1 で定義された全レベル横断採番規約に従う。結合テストで発見した欠陥は同一採番シーケンスに収容する。

**本節で確定した方針**
- **TST-intg-001〜034 の 34 件（境界テスト 031〜034 含む）は本サブで採番変更・追加を行わない。実施手順上の参照のみ許可する。**
- **DEFECT-NNN は単体テスト・結合テスト・適格性確認テスト横断で連番管理する。本サブが独自シーケンスを設けることを禁止する。**

## 3. 上流から引き継ぐ要件範囲

### 3-1. テストケース対象 API エンドポイント（34 件）

バックエンドは `wnav_terminal_api`（8080）と `wnav_master_api`（8081）の 2 バイナリ構成である。各テストケースの接続先バイナリは以下の通り。

| グループ | TST-ID 範囲 | 接続先バイナリ | 対象エンドポイント |
|---|---|---|---|
| 認証 | TST-intg-001〜005 | terminal-api（8080）| `POST http://localhost:8080/api/v1/auth/login`, `GET http://localhost:8080/api/v1/work-executions` |
| ステップイベント記録 | TST-intg-006〜010 | terminal-api（8080）| `POST http://localhost:8080/api/v1/step-events` |
| 証拠・電子署名・アンドン | TST-intg-011〜015 | terminal-api（8080）| `POST http://localhost:8080/api/v1/evidences`, `POST http://localhost:8080/api/v1/sign-records`, `POST http://localhost:8080/api/v1/andon-alerts` |
| SOP 承認・マスタ同期・Webhook | TST-intg-016〜020 | master-api（8081）※同期プルと Outbox は 8080 | `POST http://localhost:8081/api/v1/sops`, `GET http://localhost:8080/api/v1/sync/master`, Outbox 配信, `POST http://localhost:8080/api/v1/work-executions/{id}/complete` |
| IQC | TST-intg-021〜025 | master-api（8081）※後工程ハードゲートは 8080 | `POST http://localhost:8081/api/v1/iqc/incoming-inspections`, `POST http://localhost:8080/api/v1/work-executions/{id}/events` |
| リワーク・ディスポジション | TST-intg-026〜030 | master-api（8081）| `POST http://localhost:8081/api/v1/dispositions`, `POST http://localhost:8081/api/v1/reworks` |
| バイナリ境界テスト | TST-intg-031〜034 | 両バイナリ（意図的越境）| 誤ったバイナリへの HTTP 404、誤 JWT aud への HTTP 401 |

### 3-2. 品質ゲート継承

`04_概要設計/10_テスト方式設計/04_品質ゲートとカバレッジ目標.md` で確定した RGATE-004（Schemathesis 100%）および RGATE-002（BR-BUS 100%）を本サブの合格条件として引き継ぐ。

| RGATE | 判定基準 | 本サブ対応ファイル |
|---|---|---|
| RGATE-002 | BR-BUS 全 45 件が ≥1 統合テストでカバー（DTM M6 完全性 100%）| `03_システム結合テスト仕様書実施手順.md` |
| RGATE-004 | Schemathesis スキーマ準拠率 100%・未定義エラーゼロ（NFR-QUA-032）| `02_API契約テスト仕様書実施手順.md` |

**本節で確定した方針**
- **バックエンドは `wnav_terminal_api`（8080）と `wnav_master_api`（8081）の 2 バイナリ構成であり、全テストケースの接続先 URL はバイナリ別ポートに基づく。シングルバイナリ時代の `/api/v1/...` 省略形ではなく、`http://localhost:808X/api/v1/...` の絶対 URL を使用する。**
- **TST-intg-001〜034 は `05_詳細設計/08_テストケース詳細設計/03_統合テストケース（API）.md` から変更なく継承し、本サブは実施手順・実施計画・結果記録・欠陥管理のみを担当することを確定する。**
- **RGATE-002 および RGATE-004 は本サブ結合テストの合格条件として適用する。両ゲートを満たさない限りリリースゲートを通過させない。**
