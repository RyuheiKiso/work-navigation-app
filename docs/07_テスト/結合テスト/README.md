# 結合テスト

本サブディレクトリは SLCP-JCF2013 §6.1.1.3「ソフトウェア結合テストプロセス」および §6.1.2.4「システム結合テストプロセス」に対応する結合テスト成果物を管理する。
テストケース定義は `05_詳細設計/08_テストケース詳細設計/` が権威ソースであり、本サブは実施手順・API 契約テスト・システム結合テスト・実施計画・シナリオ・実施結果記録・欠陥管理の 7 責務を担う。

---

## 責務

| 責務 | 説明 |
|---|---|
| 実施手順の確定（L2 統合テスト）| axum_test + sqlx::test による TST-intg-001〜020 の実行手順を明示する |
| 実施手順の確定（API 契約テスト）| Schemathesis による OpenAPI 3.1 仕様書準拠率検証の実行手順を明示する |
| 実施手順の確定（システム結合テスト）| API + DB + Outbox 三者結合テストの実行手順と BR-BUS カバレッジ確認手順を明示する |
| シナリオ管理 | 業務フロー全体を通した統合シナリオ 5 件（A〜E）を規定する |
| 実施計画の管理 | CI ジョブ設計・実施タイミング・失敗時対応手順を規定する |
| 実施結果の記録 | TST-EXEC-YYYYMMDD-NNN 形式の記録テンプレートを提供する |
| 欠陥管理 | DEFECT-NNN 採番規約と修正追跡テンプレートを提供する |
