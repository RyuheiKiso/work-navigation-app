## 2. CI ジョブ構成

### 2-1. rust-integration ジョブ

```yaml
# .github/workflows/integration.yml（抜粋）
name: rust-integration
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  integration:
    runs-on: ubuntu-latest
    services:
      postgres_test:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: wnav
          POSTGRES_PASSWORD: wnav
          POSTGRES_DB: wnavdb_test
        ports:
          - 5433:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
    steps:
      - uses: actions/checkout@v4
      - name: Run migrations
        run: sqlx migrate run --database-url "$TEST_DATABASE_URL"
      - name: Run integration tests
        run: cargo test --test integration -- --test-threads=1
        env:
          TEST_DATABASE_URL: postgres://wnav:wnav@localhost:5433/wnavdb_test
```

### 2-2. contract ジョブ（Schemathesis）

```yaml
# .github/workflows/contract.yml（抜粋）
name: contract
on:
  push:
    branches: [main, develop]

jobs:
  schemathesis:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Start backend
        run: cargo run --bin wnav-server -- --port 8080 &
      - name: Run Schemathesis
        run: |
          pip install schemathesis
          schemathesis run ./openapi.yaml \
            --base-url http://localhost:8080 \
            --checks all \
            --report reports/schemathesis-$(date +%Y%m%d).html
      - name: Upload report
        uses: actions/upload-artifact@v4
        with:
          name: schemathesis-report
          path: reports/schemathesis-*.html
```

## 3. 環境準備シーケンス

結合テスト実施前に以下の手順を完了する。手順に抜けがある場合はテスト実行を開始しない。

```
[1] TENV-002 コンテナ起動（postgres_test）
    └─ docker compose --profile test up -d postgres_test
    └─ 起動確認: docker inspect --format='{{.State.Health.Status}}' ...

[2] マイグレーション適用
    └─ sqlx migrate run --database-url "$TEST_DATABASE_URL"
    └─ 確認: sqlx migrate info で全マイグレーションが Applied

[3] シード SQL 投入
    └─ psql "$TEST_DATABASE_URL" -f tests/fixtures/seed_auth.sql
    └─ psql "$TEST_DATABASE_URL" -f tests/fixtures/seed_step_events.sql
    └─ psql "$TEST_DATABASE_URL" -f tests/fixtures/seed_evidences.sql
    └─ psql "$TEST_DATABASE_URL" -f tests/fixtures/seed_sop.sql

[4] テスト実行（グループ順遵守）
    └─ cargo test --test integration -- --test-threads=1

[5] 契約テスト実行（L4）
    └─ schemathesis run ./openapi.yaml --base-url http://localhost:8080

[6] システム結合テスト実行（Outbox Consumer 起動後）
    └─ docker compose --profile test up -d outbox_consumer_test
    └─ bash scripts/check-dtm-m6.sh
    └─ bash scripts/check-rbac-endpoints.sh

[7] 環境破棄
    └─ docker compose --profile test down -v
```

## 5. 失敗時対応フロー

| 失敗種別 | 対応手順 |
|---|---|
| L2 統合テスト（TST-intg）の FAIL | DEFECT-NNN を起票して `07_欠陥管理と修正記録テンプレート.md` に記録する。P1/P2 の場合は即時修正し再実行する。P3 の場合は次スプリント対応を検討する |
| Schemathesis FAILED | openapi.yaml と実装のスキーマ差異を特定する。差異が openapi.yaml 側の誤りの場合は仕様書を修正する。実装側の誤りの場合は DEFECT-NNN を起票して修正する |
| DTM M6 UNCOVERED | 未カバーの BR-BUS に対応する TST-intg-NNN または TST-unit-NNN を追加する。テストケース追加は `05_詳細設計/08_テストケース詳細設計/` を更新することで対応する |
| Append-only 制約テスト FAIL | BR-BUS-015 違反として DEFECT-NNN を起票し、P1 として即時修正する |
| ハッシュチェーン連鎖不整合 | FR-EV-001/002 違反として DEFECT-NNN を起票し、P1 として即時修正する |
| CI ジョブ環境障害（TENV-002 起動失敗など）| コンテナログを確認する。ポート競合・ディスク容量不足・Docker デーモン未起動が原因の場合は環境を修正して再実行する |

**本節で確定した方針**
- **結合テストは M3 完了後・主要 PR マージ時・M5 完了後・リリース前の 4 タイミングで実施することを確定する。タイミングを省略してのリリース判定を禁止する。**
- **CI ジョブ rust-integration および contract の両方が GREEN であることを PR マージの必要条件とする。一方でも FAILED の場合はマージを行わない。**
- **DTM M6 完全性確認（45/45 COVERED）は M5 完了後とリリース前の 2 回必須とし、スクリプト出力結果を実施結果テンプレートに添付することを求める。**
- **FAIL 発生時の欠陥起票は DEFECT-NNN 採番を必須とし、優先度（P1/P2/P3）を明示することを求める。**
