## 2. テスト環境起動手順

### 2-1. TENV-003（Playwright Web）起動

```bash
# バックエンド + DB 起動（TENV-003 前提）
docker compose -f docker-compose.test.yml up -d postgres api-server

# フロントエンド開発サーバー起動
cd apps/master-maintenance && pnpm dev &

# Playwright インストール確認
npx playwright install --with-deps

# Playwright テスト実行
npx playwright test --project=chromium --reporter=html
```

### 2-2. TENV-004（Detox RN）起動

```bash
# Android Emulator API 33 起動（要 Android Studio）
emulator -avd Pixel_6_API_33 &

# iOS Simulator 17 起動（macOS のみ）
xcrun simctl boot "iPhone 15 Pro"

# Detox ビルド（初回または変更時）
pnpm detox build --configuration android.emu.debug
pnpm detox build --configuration ios.sim.debug

# Detox テスト実行
pnpm detox test --configuration android.emu.debug --reporter spec
pnpm detox test --configuration ios.sim.debug --reporter spec
```

## 4. ゴールデンパス確認観点

各 TST-e2e の Pass 判定は以下の 3 軸で確認する。

| 確認軸 | 確認対象 | 確認方法 |
|---|---|---|
| URL 遷移 | 画面遷移シーケンスが仕様通り | Playwright: `expect(page).toHaveURL()` |
| DB 値 | work_events / audit_logs にレコード存在 | psql または Detox の DB アサーション |
| ストレージ状態 | SQLite（RN）の Outbox キュー状態 | Detox: TypeORM リポジトリ直接参照 |

---

## 5. Pass 判定基準

| 判定区分 | 基準 | 根拠 |
|---|---|---|
| Pass | UC ゴールデンパス ≥ 21 件 / 22 件（≥ 95%）| RGATE-002 E2E UC カバレッジ ≥ 95% |
| Fail | P1 欠陥（機能不全）が 1 件以上 | RGATE-002 / テスト計画 §5 合否判定 |
| 条件付 Pass | 1 件失敗かつ P2 のみ・リリース前修正条件で記録 | テスト計画 §5 条件付合格フロー |

最優先 6 件（TST-e2e-003/009/011/016/019/020）のいずれかが Fail の場合は無条件 Fail とする。

**本節で確定した方針**
- **UC ゴールデンパス ≥ 21/22 件（≥ 95%）かつ最優先 6 件全 Pass を、ソフトウェア適格性確認テストの Pass 基準として確定する。**
- **テスト結果は `07_適格性確認テスト実施結果テンプレート.md` に記録し、証跡として保存する。**
