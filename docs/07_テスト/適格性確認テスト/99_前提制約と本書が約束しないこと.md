# 99 前提制約と本書が約束しないこと

本章は本サブ（適格性確認テスト）の適用範囲外・個人開発スコープの限界・規制との関係を確定する。
本章に記載の事項は「本書が約束しないこと」として明示し、後の誤解を防ぐ。

---

## §1. 本書が扱わない事項

以下は本サブの適用範囲外であり、本書は対応を約束しない。

| 項目 | 理由 |
|---|---|
| 第三者監査（外部機関による認証・審査）| 個人開発・単一開発者スコープのため不可 |
| SLA（Service Level Agreement）の確約 | 本番運用体制・サポート体制が未確立のため不可 |
| 実 Detox/Playwright テストコードの実装 | 本書はテスト仕様・実施手順文書であり、実装コードは `06_実装/` で管理 |
| ペネトレーションテスト（外部実施）| 個人開発スコープでは対象外と判断する（後述 §4 参照）|
| 本番データを使用したテスト | `05_最終確認チェックリスト` §前提条件で明示禁止 |
| 複数開発者チームでの UAT | 本プロジェクトは個人開発であり、チーム UAT は適用範囲外 |

**本節で確定した方針**
- **上記 6 項目は本書のスコープ外として確定し、将来のチーム開発移行時に別途本書を改訂する。**
- **本書はテスト仕様・実施手順・判定基準を定める文書であり、テストコードの実装・ツール設定ファイルの作成は範囲外とする。**

---

## §2. 個人開発スコープの前提

本プロジェクトは個人開発（開発者: RyuheiKiso 1 名）であるため、以下の制約がある。

| 制約事項 | 対応方針 |
|---|---|
| UAT 参加者の独立性 | ブラインドテスト原則を維持するが、開発者が受入担当を兼任する限界がある |
| テストの客観性 | セルフレビューの限界を認識し、RGATE・ALCOA+ の自動化可能な項目はツールで判定する |
| 証跡の保管 | Git + Git 外バックアップの 2 重保管で対応するが、法的証拠能力は保証しない |
| FR-AU-001 未実装期間 | Git コミット署名を代替電子署名として使用する（`08_リリース判定とサインオフ.md` §3-2 参照）|

## §3. テスト環境の前提条件

本書が想定するテスト環境は以下の通りである。これらが満たされない場合、テスト結果の有効性は保証されない。

| 前提条件 | 詳細 |
|---|---|
| TENV-004 端末 | Android Emulator API 33 または iOS Simulator 17 が動作可能な開発マシン |
| TENV-006 シード | 1 万件のシードデータが投入済みの本番相当 Docker 環境 |
| TENV-007 ネットワーク | api-server:8080 に OWASP ZAP コンテナから到達可能なネットワーク構成 |
| veraPDF | `veraPDF` コマンドが実行環境にインストール済み |
| pdffonts | `poppler-utils` パッケージがインストール済み |
| k6 | `k6` コマンドが実行環境にインストール済み |

## §4. 規制との関係（3 段表記）

本書が扱う規制・標準との関係は「準拠する／対応する／対象外と判断する」の 3 段のみで表記する。

| 規制・標準 | 関係区分 | 根拠 |
|---|---|---|
| 21 CFR Part 11（FDA 電子記録・署名）| **準拠する** | TST-alcoa-001 Attributable 検証・FR-AU-001 電子サイン実装が対応 |
| PIC/S PE 009-16（GMP ガイドライン）| **対応する** | TST-alcoa-002 Legible 検証（veraPDF 3b）が対応。GMP 規制当局提出は範囲外 |
| ISO 19005-3（PDF/A-3b）| **準拠する** | TST-alcoa-002/008 で veraPDF 3b 適合を機械的に確認 |
| IEEE XES 2.0 | **準拠する** | TST-alcoa-006 Complete 検証（XES 必須属性 NOT NULL）が対応 |
| 外部第三者ペネトレーションテスト | **対象外と判断する** | 個人開発スコープ・外部機関費用が現時点で適用困難なため |
| ISO/IEC 27001 情報セキュリティ認証 | **対象外と判断する** | 認証機関による審査は個人開発スコープ外。OWASP ZAP による自己検証は実施する |
| 薬事法・医療機器規制（PMDA 等）| **対象外と判断する** | 本プロダクトは製造業向け作業ナビゲーションアプリであり、医療機器ではない |

## §6. 本書内の曖昧表現の禁止

本書（本サブ全体）では以下の曖昧表現を使用しない。使用を発見した場合は改訂する。

| 禁止表現 | 理由 |
|---|---|
| 「適宜確認する」| 確認タイミング・基準が不明確 |
| 「なるべく〜する」| 必須要件か任意かが不明確 |
| 「〜を目指す」| 達成基準が不明確 |
| 「準拠を目指す」| §4 の 3 段表記に置き換える |
| 「できる限り〜」| 条件・閾値が不明確 |

**本節で確定した方針**
- **21 CFR Part 11・ISO 19005-3・IEEE XES 2.0 は「準拠する」、PIC/S PE 009-16 は「対応する」、第三者ペネトレーションテスト・ISO/IEC 27001・医療機器規制は「対象外と判断する」として確定する。**
- **本書の適用範囲・限界・規制関係を本章に集約し、他章での追記を禁止する。**
