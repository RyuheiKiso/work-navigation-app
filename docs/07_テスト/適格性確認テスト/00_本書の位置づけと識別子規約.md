# 00 本書の位置づけと識別子規約

本章は `07_テスト/適格性確認テスト` サブの IPA 共通フレーム 2013 における位置づけ・テスト識別子の継承規約・個人開発での自己実施根拠を確定する。

---

## 1. IPA タスク対応表

IPA 共通フレーム 2013 が要求するタスクと本サブ各章の対応を以下に確定する。

| SLCP-JCF2013 タスク | 対応章 | 備考 |
|---|---|---|
| §6.1.1.4 ソフトウェア適格性確認テスト | `01_ソフトウェア適格性確認テスト仕様書実施手順` | E2E 22 件・TENV-003/004 |
| §6.1.2.5 システム適格性確認テスト | `02_システム適格性確認テスト仕様書実施手順` | 性能・セキュリティ・可用性 NFR |
| §6.4.8 受入支援 | `04_受入相当テストとUATシナリオ実施手順` | シナリオ A〜E・ブラインド原則 |
| リリース判定 | `08_リリース判定とサインオフ` | RGATE-001〜006・電子サインオフ |

**本節で確定した方針**
- **SLCP-JCF2013 §6.1.1.4・§6.1.2.5・§6.4.8 の 3 タスクを本サブ 10 章（00〜08 + 99）に割付けることを確定する。**
- **本サブは上流文書（要件定義・概要設計・詳細設計）の識別子を変更しない。**
- **個人開発であるため受入テスト（UAT）は開発者自身が受入担当として自己実施するが、ブラインドテスト原則（開発者はオブザーバー限定）を維持する。**

## 3. 上流から引き継ぐ要件範囲

本サブは以下の上流成果物を基礎として受け取る。変更権限を持たない。

| 上流成果物 | パス | 本サブへの影響 |
|---|---|---|
| E2E テストケース詳細設計 | `05_詳細設計/08_テストケース詳細設計/04_E2Eテストケース.md` | TST-e2e-001〜022 の全手順・期待値 |
| ALCOA+ 検証テストケース | `05_詳細設計/08_テストケース詳細設計/05_ALCOA+検証テストケース.md` | TST-alcoa-001〜009 の全手順・SQL |
| 非機能テストケース | `05_詳細設計/08_テストケース詳細設計/06_非機能テストケース.md` | TST-sec/perf の全手順・期待値 |
| デモシナリオと受入シナリオ | `03_要件定義/テスト・受入要件/05_デモシナリオと受入シナリオ.md` | UAT シナリオ A〜E |
| 非機能テスト方式 | `04_概要設計/10_テスト方式設計/07_非機能テスト方式.md` | k6・ZAP 実施要件 |

**本節で確定した方針**
- **本サブは実施手順・判定基準・証跡管理を担当し、テストケースの内容・期待値は上流成果物から変更しない。**
- **受入相当テストの自己実施は SLCP-JCF2013 §6.4.8「受入支援」として位置づけ、個人開発スコープで許容される範囲を `99_前提制約と本書が約束しないこと` に明記する。**
