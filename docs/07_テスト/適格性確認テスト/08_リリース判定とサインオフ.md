# 08 リリース判定とサインオフ

本章は ver1.0.0 リリースの最終判定フロー・FR-AU-001 準拠の電子サインオフ手順・UAT 完了記録（TST-070）の作成手順を確定する。

---

## 1. リリース判定フロー

以下フローを順序通りに完了した場合のみ ver1.0.0 をリリースする。
いずれかのステップが未完了の場合、リリースを保留しステップ先頭に戻る。

```
Step 1: RGATE-001 コード品質 → 全 Pass
Step 2: RGATE-002 テストカバレッジ → 全 Pass
Step 3: RGATE-003 セキュリティ → 全 Pass
Step 4: RGATE-004 API 準拠 → 全 Pass
Step 5: RGATE-005 性能 → 全 Pass
Step 6: RGATE-006 DTM 完全性 → 全 Pass
    ↓
Step 7: ALCOA+ 検証（TST-alcoa-001〜009）→ 9 件全 Pass
    ↓
Step 8: UAT シナリオ A〜E 実施 → 合否 4 条件全達成
    ↓
Step 9: 電子サインオフ（FR-AU-001 準拠）
    ↓
Step 10: リリースタグ付与・デプロイ
```

**本節で確定した方針**
- **Step 1〜9 を全て完了した状態でのみ Step 10（リリース）に進むことを確定する。**
- **RGATE の 1 件でも未達、または UAT 合否 4 条件のいずれか未達の場合はリリース保留とする。**

---

## 2. UAT 完了記録（TST-070）の作成手順

UAT 完了後、以下 10 フィールドを記録した UAT 完了記録を作成する。

| フィールド No. | フィールド名 | 記載内容 |
|---|---|---|
| 1 | 記録 ID | UAT-YYYYMMDD-Vx（例: UAT-20260518-V1）|
| 2 | 実施期間 | YYYY-MM-DD 〜 YYYY-MM-DD |
| 3 | 実施環境 | TENV-008（ステージング）バージョン・シード日時 |
| 4 | 参加者 | ロール・氏名・所属（4 名以上）|
| 5 | 実施シナリオ | シナリオ A〜E 全件実施 確認 |
| 6 | 欠陥サマリ | P1: N 件 / P2: N 件 / P3: N 件 |
| 7 | 合否 4 条件判定 | 各条件の Pass/Fail |
| 8 | 倫理品質チェック | NFR-QUA-003 全 PASS 確認 |
| 9 | サインオフ署名 | FR-AU-001 準拠電子サイン（後述）|
| 10 | 保存場所 | `証跡/UAT-YYYYMMDD.md` + PDF + Git 外バックアップ |

## 3. FR-AU-001 準拠の電子サインオフ

### 3-1. 正規実装時（FR-AU-001 実装後）

FR-AU-001 が実装済みの場合、以下 4 要素を含む電子サインレコードを `sign_offs` テーブルに INSERT する。

```sql
-- 電子サインオフレコード（概念）
INSERT INTO sign_offs (id, signer_id, target_record, signature, timestamp_server)
VALUES (
  gen_random_uuid_v7(),      -- UUID v7
  'ryuhei_kiso',             -- signer_id
  'UAT-20260518-V1',         -- 対象記録 ID
  hmac_sha256(payload),      -- HMAC-SHA256 署名
  NOW() AT TIME ZONE 'UTC'   -- timestamp_server（サーバー側タイムスタンプ）
);
```

4 要素確認:
- [ ] UUID v7 が発行されていること
- [ ] HMAC-SHA256 署名が記録されていること
- [ ] signer_id が記録されていること
- [ ] timestamp_server（サーバー側）が記録されていること

### 3-2. FR-AU-001 未実装期間の代替電子署名

FR-AU-001 が未実装の期間は、以下の Git コミット署名を代替電子署名として使用する。

```bash
# UAT 完了記録ファイルを Git にコミット
git add docs/07_テスト/適格性確認テスト/証跡/UAT-YYYYMMDD.md
git commit -m "chore: UAT-YYYYMMDD-V1 完了記録 / ver1.0.0 リリース判定

Signed-off-by: RyuheiKiso <uw2374@icloud.com>"

# コミットハッシュを記録
git log -1 --format="%H"
```

代替電子署名の記録形式:

```
Signed-off-by: RyuheiKiso <uw2374@icloud.com>
Git-Commit: <40 文字 SHA-1 ハッシュ>
Timestamp: YYYY-MM-DDTHH:MM:SSZ（UTC）
UAT-Record-ID: UAT-YYYYMMDD-V1
```

## 4. 証跡の保存手順

UAT 完了記録と証跡ファイルは以下の 3 箇所に保存する。

| 保存場所 | ファイル形式 | 保存内容 |
|---|---|---|
| `docs/07_テスト/適格性確認テスト/証跡/UAT-YYYYMMDD.md` | Markdown | UAT 完了記録 10 フィールド（本文）|
| `docs/07_テスト/適格性確認テスト/証跡/UAT-YYYYMMDD.pdf` | PDF/A-3b | Markdown を veraPDF 3b 適合 PDF に変換した版 |
| Git 外バックアップ（外部ストレージ or クラウド）| ZIP | 証跡フォルダ全体のアーカイブ |

```bash
# PDF 変換（pandoc + Noto フォント）
pandoc docs/07_テスト/適格性確認テスト/証跡/UAT-YYYYMMDD.md \
  -o docs/07_テスト/適格性確認テスト/証跡/UAT-YYYYMMDD.pdf \
  --pdf-engine=xelatex \
  -V mainfont="Noto Sans CJK JP"

# PDF/A-3b 適合確認
veraPDF --flavour 3b docs/07_テスト/適格性確認テスト/証跡/UAT-YYYYMMDD.pdf
```

## 5. リリースタグ付与

電子サインオフ完了後、以下の手順で ver1.0.0 リリースタグを付与する。

```bash
# リリースタグ付与
git tag -a v1.0.0 -m "ver1.0.0 release

UAT-YYYYMMDD-V1 合格
RGATE-001〜006 全 PASS
ALCOA+ 9 属性全 Pass
Signed-off-by: RyuheiKiso <uw2374@icloud.com>"

git push origin v1.0.0
```

**本節で確定した方針**
- **FR-AU-001 実装前期間は `Signed-off-by:` + Git コミットハッシュを代替電子署名として使用し、FR-AU-001 実装後は 4 要素（UUID v7・HMAC-SHA256・signer_id・timestamp_server）に移行する。**
- **UAT 完了記録は Markdown・PDF/A-3b・Git 外バックアップの 3 形式で保存し、5 年間保管する。**
