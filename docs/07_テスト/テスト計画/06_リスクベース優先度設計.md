# 06 リスクベース優先度設計

本章は業務上リスクが最大となる UC を特定し、テスト実施順序・回帰テスト範囲・P1 欠陥検出時の対応手順をリスクベースで確定する。

---

## 1. UC 優先度分類

### 1-1. 最優先 UC（6 件）

以下の 6 UC は業務停止・データ完全性違反・規制違反に直結するため、最優先で E2E テストを実施し、変更のたびに回帰テストを実施する。

| UC-ID | UC 名 | 最優先理由 |
|---|---|---|
| UC-003 | ステップ実行（測定値入力）| 測定値の誤記録は ALCOA+ Accurate 原則違反・製品品質直結 |
| UC-009 | 作業完了・電子サイン | 電子サインなし完了は規制違反・Attributable 原則違反 |
| UC-011 | アンドン発報 | 発報遅延・失敗は現場安全に直結 |
| UC-016 | SOP 承認ワークフロー | 未承認 SOP での作業開始は規制違反 |
| UC-019 | 監査ログ閲覧 | 監査ログ不閲覧・破損は規制機関対応不可 |
| UC-020 | 監査ログ XES エクスポート | エクスポート失敗はトレーサビリティ証跡の損失 |

**本節で確定した方針**
- **最優先 6 UC は E2E テスト実施順序の先頭に配置し、これら 6 件が Fail の場合は残余テストを中断することを確定する。**
- **最優先 6 UC への変更はコード変更種別に関わらず L3 E2E 再実施を必須とすることを確定する。**

---

## 2. 高優先 UC・通常 UC

| 優先度 | UC-ID | 備考 |
|---|---|---|
| 高優先 | UC-001〜009 全件 | 作業実施フロー中核。業務停止リスク大 |
| 高優先 | UC-014（CAPA 登録・承認）| 不適合対処の証跡。監査対象 |
| 通常 | UC-010（オフライン継続）| 機能は重要だが障害頻度は低い |
| 通常 | UC-015（Kaizen 提案）| 安全に関わらない改善提案 |
| 通常 | UC-017（SOP 改訂版作成）| UC-016 と組み合わせて使用。単独リスク低 |
| 通常 | UC-018（ユーザー・スキル管理）| 管理操作。業務停止とは非連動 |
| 通常 | UC-021（ハッシュチェーン検証）| 定期実行。即時業務影響なし |
| 通常 | UC-022（多言語切替）| UI 機能。業務継続への影響は低い |

**本節で確定した方針**
- **高優先 UC（UC-001〜009 + UC-014）は E2E テストを最優先 6 件の次に実施し、通常 UC はその後に実施することを確定する。**

## 3. 回帰テスト範囲（TST-030 準拠）

変更種別ごとの回帰テスト実施範囲を確定する。

| 変更種別 | 実施範囲 |
|---|---|
| 関数・モジュール単体 | L1（単体テスト）|
| API エンドポイント変更 | L2（統合テスト）＋ L3（E2E）|
| DB スキーマ変更 | L2（統合）＋ L3（E2E）＋ TST-006（移行テスト）|
| 認証・認可ロジック変更 | TST-004（セキュリティ）＋ L2（統合）|
| マスタデータ変更 | L2（統合）＋ TST-005（ALCOA+）|
| Outbox / 同期ロジック変更 | L2（統合）＋ TST-007（Outbox 検証）|
| 最優先 UC に関連する変更 | L3 E2E（該当 UC）＋上記 |

**本節で確定した方針**
- **変更種別に対する回帰テスト範囲を本章の表で確定し、範囲の縮小を禁止する。拡大は ADR-TEST-NNN で許可する。**

## 4. P1 欠陥検出時の即時対応手順

P1 欠陥（データ破損・セキュリティ侵害・ALCOA+ 原則違反）を検出した場合、以下の手順を即時実施する。

| ステップ | 内容 | 期限 |
|---|---|---|
| 1. 全 UAT 中断 | 進行中の UAT を即時中断し、参加者に通知する | 検出から 30 分以内 |
| 2. DEFECT-NNN 起票 | P1 として DEFECT-NNN を採番し、再現手順・期待動作・実動作を記録する | 検出から 1 時間以内 |
| 3. 影響範囲特定 | 欠陥が影響する FR/NFR/BR-BUS を RTM で特定する | 検出から 4 時間以内 |
| 4. 修正 | 修正コミットを作成し、CI パイプラインを全件通過させる | 修正完了まで継続 |
| 5. 全 UAT 再実施 | TST-001〜007 全件を最初から再実施する | 修正完了後 |

**本節で確定した方針**
- **P1 欠陥検出時は全 UAT 中断・全件再実施を必須とし、部分再実施で代替することを禁止する。**
- **DEFECT-NNN の起票を検出から 1 時間以内に完了させることを確定する。**
