# 00 本書の位置づけと識別子規約

本章は `07_テスト/テスト計画` サブの IPA 共通フレーム 2013 における位置づけ・新定義識別子の採番規約・上流文書から引き継ぐ要件範囲を確定する。

---

## 1. IPA タスク対応表

IPA 共通フレーム 2013「6.4.1 テスト計画プロセス（SLCP-JCF2013 §6.4.1）」が要求するタスクと本サブ各章の対応を以下に確定する。

| SLCP-JCF2013 タスク | 対応章 | 備考 |
|---|---|---|
| §6.4.1.1 テスト計画の作成 | `01_全体テスト計画と方針` | 戦略・ピラミッド・実行シーケンス |
| §6.4.1.2 テスト環境の識別 | `03_テスト環境定義（TENV）` | TENV-001〜008 |
| §6.4.1.3 テスト体制・役割の定義 | `02_テスト体制と責任マトリクス` | RACI |
| §6.4.1.4 合格基準の定義 | `05_合否判定基準と完了条件` | RGATE-001〜006 |
| §6.4.1.5 テストデータの識別 | `04_テストデータ管理計画` | ファクトリ・シード SQL |
| §6.4.1.6 テストスケジュールの確定 | `08_スケジュールとマイルストーン` | M1〜M6 |
| §6.4.1.7 リスクおよびリスク低減策 | `06_リスクベース優先度設計` | 優先 UC・回帰範囲 |
| §6.4.1.8 進捗監視・制御計画 | `07_進捗管理と欠陥管理` | DEFECT-NNN ライフサイクル |

**本節で確定した方針**
- **IPA 6.4.1 の全 8 タスクを本サブ 9 章（00〜08 + 99）に割付けることを確定する。**
- **本サブは上流 3 文書（要件定義・概要設計・詳細設計）を変更しない。計画文書として実行可能なプロセスのみを規定する。**
