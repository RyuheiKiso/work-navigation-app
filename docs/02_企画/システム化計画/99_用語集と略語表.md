# 用語集と略語表

本文書群に登場する業務用語・技術用語・規制略語を多ステークホルダー間の語彙統一のために整理する。  
**(参照用。本章は受入基準 KPI の対象外)**

---

## 99.1 業務用語

| 用語 | 読み | 意味 |
|---|---|---|
| 4M | フォーエム | Man（人）/ Machine（設備）/ Material（材料）/ Method（方法）。作業の構成要素を整理するフレームワーク |
| SOP | エスオーピー | Standard Operating Procedure（標準作業手順書）。作業手順を文書化したもの |
| CAPA | カパ | Corrective and Preventive Action（是正処置・予防処置）。不適合の根本原因を排除し再発を防ぐ活動 |
| Yokoten | ヨコテン | 水平展開。改善事例を他のラインや工場に横展開すること |
| 8D | エイトD | 8 Disciplines。不適合の根本原因分析と是正措置を 8 ステップで記録するフォーマット（IATF 16949 で要求） |
| TWI | ティーダブリューアイ | Training Within Industry。作業指導（JI）・作業改善（JM）・人間関係（JR）の 3 コースからなる現場訓練手法 |
| 5S | ゴーエス | 整理・整頓・清掃・清潔・躾。職場環境を整える活動 |
| KPI | ケーピーアイ | Key Performance Indicator（主要業績評価指標）。目標達成度を測定するための指標 |
| 星取り表 | ほしどりひょう | 多能工化の進捗を「◎○△×」で表示するスキルマトリクス表（TWI での呼称） |
| Kaizen Teian | カイゼンていあん | ボトムアップの現場改善提案制度。QC サークル・提案制度の総称 |
| ポカヨケ | ポカヨケ | Poka-yoke。人が間違いにくい設備・作業設計（Shingo 重夫が体系化）。Fool-proof / Error-proof の日本語 |
| アンドン | アンドン | Toyota Production System の生産ライン状態可視化システム。異常発生時に光で知らせる |
| 不適合 | ふてきごう | 規定された要求事項を満たさない品質上の問題。Non-conformance |
| ロット | ロット | 同一条件で製造した製品の一集団。トレーサビリティの追跡単位 |

## 99.2 技術用語

| 用語 | 意味 |
|---|---|
| Outbox Pattern | マイクロサービス等で DB とメッセージキューの整合性を保つ設計パターン。操作をローカル DB に書いてから非同期送信 |
| Append-only | 追記のみ（上書き・削除なし）のデータ記録方式。履歴保全と監査証跡に適する |
| Idempotency Key | 同一操作の重複送信を防ぐための一意な識別子（UUID v7）。リトライが安全になる |
| CRDT | Conflict-free Replicated Data Type。複数ノードが独立に更新しても矛盾なく合成できるデータ型設計 |
| Read-Your-Writes | 「自分が書いたデータは次の読み込みで必ず見える」一貫性保証。オフライン設計で重要 |
| Eventual Consistency | 結果整合性。一定時間後に全ノードのデータが収束する分散システムの整合性モデル |
| Optimistic UI | ユーザ操作を即時反映し、バックグラウンドでサーバ確定を待つ UI パターン |
| WAL | Write-Ahead Log（書き先行ログ）。PostgreSQL の変更を事前にログに記録し、障害時の復旧を可能にする |
| Scaffolding | 認知的徒弟制の一戦略。初心者向けに詳細なサポートを提供し、習熟とともに徐々に外していく（Fading） |
| Fading | Scaffolding から徐々にサポートを取り除き、作業者の自律性を高めるプロセス |
| Glanceable UI | 1〜2 秒で重要情報を把握できる UI 設計（Endsley SA モデル Level 1） |
| Resumption Lag | 作業中断後の再開時に発生する記憶再構築の遅延。Post-Completion Error の主要原因 |
| SA | Situation Awareness（状況認識）。Endsley の 3 層モデル（知覚・理解・予測） |
| JWT | JSON Web Token。認証・認可に使う自己完結型のトークン形式。RS256 署名 |
| UUID v7 | Universally Unique Identifier バージョン 7。タイムスタンプを含む単調増加する UUID |
| CAP 定理 | Consistency / Availability / Partition tolerance の 3 特性は同時に全て満たせないという分散システム理論 |
| XES | eXtensible Event Stream。IEEE Process Mining Manifesto のイベントログ標準フォーマット |

## 99.3 規制・標準略語

| 略語 | 正式名称 | 概要 |
|---|---|---|
| ALCOA+ | — | FDA が電子記録評価に使う 9 原則（Attributable/Legible/Contemporaneous/Original/Accurate + Complete/Consistent/Enduring/Available） |
| ISO 9001 | — | 品質マネジメントシステム規格（2015 年版）。リスクベース思考・トレーサビリティ箇条 8 |
| IATF 16949 | — | 自動車業界品質マネジメントシステム規格。PPAP/APQP/8D を要求 |
| PPAP | Production Part Approval Process | 自動車業界の量産移行前の部品承認プロセス |
| APQP | Advanced Product Quality Planning | 自動車業界の開発〜量産の品質計画フレームワーク |
| GMP | Good Manufacturing Practice | 医薬品・医療機器・食品の製造品質保証規範 |
| PIC/S | Pharmaceutical Inspection Co-operation Scheme | GMP の国際標準化スキーム |
| 21 CFR Part 11 | Code of Federal Regulations Title 21 Part 11 | FDA の電子記録・電子署名規制 |
| MDR | Medical Device Regulation | EU 医療機器規制（2017/745）。UDI 義務化 |
| HACCP | Hazard Analysis and Critical Control Points | コーデックス委員会による食品安全管理手法 |
| AS9100 | — | 航空・宇宙・防衛分野の品質マネジメントシステム規格 |
| Nadcap | National Aerospace and Defense Contractors Accreditation Program | 航空宇宙特殊工程の業界認定プログラム |
| JCSS | Japan Calibration Service System | 計量法に基づく校正事業者登録制度 |
| UCUM | Unified Code for Units of Measure | 医療・工学向けの単位コード標準 |
| UDI | Unique Device Identification | 医療機器個体識別コード体系（FDA/EU MDR） |
| EPCIS | Electronic Product Code Information Services | GS1 のトレーサビリティデータ交換標準 |
| GS1 | — | バーコード・RFID・EPCIS を管理する国際非営利組織 |
| WCAG 2.1 | Web Content Accessibility Guidelines 2.1 | Web コンテンツのアクセシビリティ国際標準 |
| MES | Manufacturing Execution System | 製造実行システム。ISA-95 Level 3 |
| ISA-95 | — | 製造エンタープライズシステム統合の国際標準（ANSI/IEC 62264） |
| eBR | Electronic Batch Record | 電子バッチ記録。GMP / FDA 規制業界での電子製造記録 |
| **RTO** | Recovery Time Objective | **目標復旧時間**。障害発生からシステム再稼働までの目標時間。本システムは 1 時間 |
| **RPO** | Recovery Point Objective | **目標復旧時点**。どの時点のデータまで復旧するかの目標。本システムは 15 分 |
| **SLA** | Service Level Agreement | サービス品質の水準合意。可用性・応答時間等の数値目標を記述 |
| **RLS** | Row Level Security | **行レベルセキュリティ**。PostgreSQL の機能で、テーブル行へのアクセスをロール・条件で制限する |
| **SHA-256** | Secure Hash Algorithm 256-bit | 256 ビットのハッシュ値を生成する暗号学的ハッシュ関数（NIST FIPS 180-4）。本システムでは写真の改ざん検知に使用 |
| **RFC 3161** | — | 信頼できる第三者機関（TSA: Time-Stamp Authority）によるタイムスタンプ付与の国際標準 |
| **TSA** | Time-Stamp Authority | タイムスタンプ認証局。RFC 3161 に基づき電子記録に法的効力のあるタイムスタンプを付与する機関 |
| **TDABC** | Time-Driven Activity-Based Costing | 時間基準活動原価計算。各活動の時間コストからプロセスコストを算出する手法（第 10 章の効果試算枠組み） |
| **MIL-STD-810H** | Military Standard 810H | 米国国防総省の環境試験規格。落下耐性・振動・温度・湿度等の試験方法を規定（Method 516.8: 1.2m 落下試験） |
| **IP 等級** | Ingress Protection Rating | 防水・防塵性能の国際規格（IEC 60529）。IP65 = 防塵完全保護 + 噴流水耐性、IP68 = 水中耐性 |
| **ATEX** | ATmosphères EXplosibles | EU の防爆機器指令（2014/34/EU）。爆発危険箇所での機器使用に関する認証制度 |
| **IECEx** | IEC System for Certification to Standards Relating to Equipment for Use in Explosive Atmospheres | 防爆機器の国際相互認証制度 |
| **TIIS** | Technology Institution of Industrial Safety | 技術安全性認証機関。日本の防爆機器検定機関（産業安全技術協会） |
| **SUS** | System Usability Scale | システムユーザビリティ尺度（Brooke, 1986）。10 問・100 点満点の標準的ユーザビリティ評価ツール。スコア 68 = 平均、70 = "Good" 相当 |
| **Bus Factor** | — | プロジェクトが停止するリスクを表す指標。「何人がバスにはねられたらプロジェクトが停止するか」を示す。Bus Factor = 1 は個人開発の最大リスク |
| **Exif** | Exchangeable Image File Format | デジタルカメラ/スマートフォンの画像ファイルに付与されるメタデータ形式（CIPA DC-008）。撮影日時・機器情報を含む |
| **Laplacian 分散** | — | 画像のブレ（ピンぼけ）を検出するための画像処理指標。分散値が低いほどブレが大きい |
| **NGINX** | — | 高性能 Web サーバ・リバースプロキシ。本システムでは X-Accel-Redirect による認証済みファイル配信に使用 |
| **PDF/A** | PDF/A (ISO 19005) | 長期保管向け PDF 規格。フォント埋め込み・暗号化禁止等により長期可読性を保証 |

## 99.4 ペルソナ・役割語

| 用語 | 意味 |
|---|---|
| 熟練作業員 | 同工場在籍 10 年以上の経験豊富な作業者（Dreyfus 熟練者/達人相当） |
| 見習い作業員 | 入社 6 ヶ月未満の新人（Dreyfus 初心者/高度初心者相当） |
| 工長 | ライン管理責任者。作業者の直属上長 |
| QA | 品質保証担当。不適合管理・規制対応・手順書承認 |
| Just Culture | 過失と故意違反を区別し、過失に対して公正に対処する安全文化（Reason, 1997） |
| 外部監査人 | IATF/ISO 等の認証審査員または顧客監査担当 |
| SIer | System Integrator（システムインテグレーター）。本システムを顧客に導入・保守する業者 |
| Work-as-Imagined | 手順書・設計者が想定した作業のあるべき姿 |
| Work-as-Done | 現場で実際に実行されている作業の実態 |
