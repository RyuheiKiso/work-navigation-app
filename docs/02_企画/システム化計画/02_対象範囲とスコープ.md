# 対象範囲とスコープ

**主読者**: 経営層・調達担当・SIer パートナー  
**想定所要時間**: 15 分

---

## 2.1 In Scope（対象範囲）

本システムが提供する機能・サービスの範囲。

### 業務範囲スコープ

| 区分 | 対象 | 対象外 |
|---|---|---|
| **対象工程** | 製造（組立・加工・検査・梱包）、設備点検・保全手順 | 設計・開発、調達・購買、物流・出荷 |
| **対象部門** | 製造、品質保証、生産技術 | 人事・総務・営業・経理 |
| **対象生産方式** | 個別受注生産・繰返生産（単一工場内の全生産方式） | 複数工場統合管理（Phase 3 検討） |
| **対象製品カテゴリ** | 離散製造・プロセス製造（汎用設計） | 食品・医薬品・医療機器・航空宇宙（業界固有規制は対象外） |

### コア機能
| 機能領域 | 具体的な対象 |
|---|---|
| **作業ナビゲーション** | 手順書のステップバイステップ表示、分岐・条件付き手順、習熟度別 UI（詳細/略式切替） |
| **トレーサビリティ記録** | 作業完了記録、測定値入力、写真添付（SHA-256 ハッシュ）、QR/Data Matrix スキャン、ロット紐付け |
| **不適合・CAPA 管理** | 不適合報告、8D レポート入力補助、是正措置・有効性確認ワークフロー |
| **Kaizen Teian** | ワンタップ改善提案投稿、管理者ダッシュボード、採択/見送りフィードバック |
| **マスタ管理** | 工程・手順書の版管理、有効日設定、ユーザ・ロール・スキルマトリクス管理 |
| **監査ログ** | Append-only イベントログ、アクセス記録、証跡エクスポート（CSV/JSON） |

### 対象端末・OS
| 端末 | OS | アプリ形式 |
|---|---|---|
| タブレット | Android | React Native APK |
| タブレット・PC | Windows | React Native for Windows |
| 管理端末 | Windows / Mac / Linux（ブラウザ） | React + Vite SPA |

### 対象インフラ
- 社内 LAN 完結（インターネット接続不要）
- Windows Server 2022 / IIS + WSL2 + Docker Compose
- オンプレミス PostgreSQL（Primary + Standby）

## 2.2 Out of Scope（対象外範囲）

以下は本システムの対象外と判断する。対象外とした根拠は後節に示す。

| 対象外項目 | 理由 |
|---|---|
| PLC / SCADA 双方向制御 | 生産設備制御は安全 SIL/PL 認証が必要な別システム領域 |
| 生産計画・スケジューリング | ERP/MRP との役割分担明確化（ISA-95 Level 4） |
| ERP/MRP 機能本体 | スコープ外。ただし作業実績データをエクスポートし MRP の正確性向上を支援 |
| SaaS・クラウド展開 | データ主権・BCP 要件から社内 LAN 完結を選択（本章 2.4 参照） |
| AI/ML 機能（Phase 1） | Phase 1 では非搭載。データモデルのみ将来拡張を考慮して設計 |
| 防爆認証端末の個別認証取得 | 防爆環境は SI 案件として個別対応 |
| クラウドオフサイトバックアップ | 社内 LAN 完結方針のもと、LAN 内 Active-Standby を採用 |
| 複数工場の統合管理 | Phase 1〜2 は単一工場前提。Phase 3 で検討 |
| LMS（学習管理システム） | 訓練記録は本システムのナビゲーション内学習支援に限定 |

## 2.3 スコープ外の判断根拠

### PLC/生産計画を除外する理由

ISA-95 の製造エンタープライズ階層モデルで、本システムは Level 3（製造オペレーション管理）の「作業ナビ + 記録」に特化する。Level 2（PLC/SCADA）および Level 4（ERP）との接続は、本システムのイベントログ XES 互換エクスポートを経由した疎結合で実現する（[`90_業界分析/07_スマートファクトリーと作業のデジタル化.md`](../../90_業界分析/07_スマートファクトリーと作業のデジタル化.md) 参照）。

### SaaS を採用しない理由

競合製品（Tulip / Plex MES 等）は SaaS 型が主流だが、以下の理由から除外する（[`90_業界分析/29_競合製品と作業ナビ・MES・eBR市場.md`](../../90_業界分析/29_競合製品と作業ナビ・MES・eBR市場.md) 参照）：

1. **データ主権**: 製造機密（工程・品質データ）を社外サーバに置けない顧客方針
2. **BCP 優位性**: クラウド事業者の障害時に現場が停止するリスクを排除
3. **ネットワーク前提**: 工場内 Wi-Fi 死角・低速回線でも操作継続が必要

### 防爆・クリーンルーム専用機器を除外する理由

防爆環境（ATEX/IECEx Zone 1/2）や半導体クリーンルーム（ISO 14644 Class 3〜5）は、専用機器の認証・調達が必要で汎用システムとの組合せが困難。IP65 + MIL-STD-810H の汎用タブレット + 産業用ケースを最低基準とし、防爆要件は SI 案件として個別対応とする（[`90_業界分析/35_環境耐性と防爆・クリーンルーム設計.md`](../../90_業界分析/35_環境耐性と防爆・クリーンルーム設計.md) 参照）。

## 2.4 撤退判定基準

Phase 1 PoC（1 ライン・3 ヶ月）終了時に以下の KPI が未達の場合、Phase 1 本番展開への移行を見合わせる。

| KPI | 閾値 | 評価方法 |
|---|---|---|
| 作業員による日常的な利用率 | ≥ 80%（対象ライン） | システムログ |
| 記録の完全性（全必須フィールド入力率） | ≥ 95% | DB 集計 |
| 作業員からの受容スコア | ≥ 70（100 点満点の設問票） | アンケート |
| 重大不具合（データ消失・改ざん） | 0 件 | インシデント管理 |

撤退時は全データを CSV/JSON 形式でエクスポートし、紙記録への回帰を可能とする（ベンダーロックイン防止）。

## 2.5 市場差別化軸

本システムは「**SaaS 否定環境 × 中小製造業 × 高カスタマイズ要求**」の三重交差ニッチに位置する。

| 競合 | 弱点 | 本システムの優位性 |
|---|---|---|
| Tulip | SaaS のみ・インターネット必須 | 社内 LAN 完結・オフライン耐性 |
| Plex MES (Rockwell) | SaaS のみ・日本サポート弱 | オンプレ・日本語 UI・やさしい日本語 |
| DELMIA Apriso | 導入 18〜36 ヶ月・中小向けでない | Docker Compose 単体インストール・PoC 3 ヶ月 |
| SAP ME/MII | SAP ERP 前提・UI が旧世代 | 独立インストール・モダン UI |
| 独自 Excel 管理 | 改ざん容易・検索不能・バックアップなし | ALCOA+ 準拠電子記録 |

**ポジショニング**: 本システムは MES の全機能を代替するのではなく、「作業ナビゲーション × トレサビ記録」の**ラストワンマイル記録システム**として機能する（[`90_業界分析/29_競合製品と作業ナビ・MES・eBR市場.md`](../../90_業界分析/29_競合製品と作業ナビ・MES・eBR市場.md) 参照）。

---

> **本節で確定した方針**  
> 1. 本システムのスコープは「作業ナビゲーション・トレサビ記録・不適合 CAPA・Kaizen Teian」の 4 領域に限定する。PLC 制御・ERP・SaaS は対象外と判断する。  
> 2. SaaS 否定は意図的な設計選択であり、データ主権と BCP 優位性が根拠である。  
> 3. Phase 1 PoC 終了時に撤退 KPI を達成できない場合は本番展開を中止し、データをエクスポートして終了する。
