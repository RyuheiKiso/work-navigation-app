# システム化計画書

## 1. 本書の位置づけ

本書は IPA 共通フレーム 2013 (SLCP-JCF2013) の企画プロセスにおける「システム化計画」に相当する。
上位文書としてシステム化構想（別途作成予定）が存在し、本書はその成果物を受けて作成する。

### 上位文書（システム化構想）との分担

| 区分 | 扱う問い | 文書 |
|---|---|---|
| システム化構想（別途作成予定） | Why — なぜ作るか、解決すべき課題、実現する価値、倫理スタンス | `../システム化構想/` |
| **本書（システム化計画）** | **What — 何を作るか、対象範囲、機能スコープ** | 本書 |
| **本書（システム化計画）** | **How-Frame — 枠組み（アーキテクチャ原則・技術スタック・データモデル・運用方針・スケジュール）** | 本書 |

### 下流文書への引き渡し

本書は要件定義・概要設計の入力文書として機能する。
ver1.0.0 リリース後は要件定義書および概要設計書に権威が移行し、本書は参照資料に降格する。

## 5. 引用規約

### 業界分析参照形式

業界分析ファイルへの参照は以下の形式を使用する。

```
[`90_業界分析/XX_*.md`](../../90_業界分析/XX_*.md)
```

### 必須 / 関連の区別

各章末の「## 参照業界分析」節では以下の区分を明示する。

| 区分 | 意味 | 記載方針 |
|---|---|---|
| **必須** | その章の論理が成立するために不可欠な分析 | 省略不可。章の本文で直接引用する |
| **関連** | 理解を深めるために参照を推奨する分析 | 最大 4 件まで。章本文での引用は任意 |

### 過剰引用の禁止

「関連」は最大 4 件とする。関連性が薄い文書を羅列することは禁止する。

## 6. 規制表記の三段階定義

本書内で規制・標準への対応状況を表現する場合、以下の三段階のみを使用する。
「考慮する」「検討する」「参考にする」「可能性がある」等の曖昧表現は全章を通じて禁止する。

| 表現 | 定義 | 使用条件 |
|---|---|---|
| **準拠する** | 当該規制・標準の要求事項をすべて満たすことを設計目標として確定している | 具体的な適合基準を同一文書内で明示できる場合のみ |
| **対応する** | 当該規制・標準の関連条項を設計上考慮し、ベストエフォートで満たすことを確定している | 全条項への適合を保証しないが、対応範囲を明示できる場合 |
| **対象外と判断する** | 当該規制・標準への適合を本システムの責務と判断しない | 対象外の根拠を同一文書内で明示できる場合のみ |

## 7. 図の管理規約

本書（および全下位章）における図の扱いは以下のとおりとする。

- **作成ツール**: draw.io のみ。アスキー図・手書きスキャン・PowerPoint エクスポートは禁止
- **ファイル形式**: `.drawio` と `.svg` を対で管理する。`.svg` のみの単独配置は禁止
- **配置場所**: `img/` ディレクトリ配下に格納する（例: `img/fig_01_architecture.drawio` と `img/fig_01_architecture.svg`）
- **検証必須**: `drawio-lint` と `svg-postcheck` を両方通過したファイルのみコミットする
- **本文参照形式**: `図: fig_XX（img/ 配下）を参照` の形式で本文から参照する。`<img>` タグによる直接埋め込みは禁止

## 8. 本書が約束しないこと

本書およびそれに基づいて開発するシステムが、ver1.0.0 時点で以下を提供しないことを明示する。

- ver1.0.0 時点で複数工場・SaaS・マルチテナンシーに対応する
- GMP/FDA 21 CFR Part 11、EU MDR、IATF 16949、AS9100、HACCP への適合
- AI/ML による自動判断・異常検知・自然言語による作業指示生成
- AR・スマートグラス・生体センサ・IoT センサとの直接統合
- 任意のレガシー MES/ERP/PLM とのリアルタイム双方向連携
- 労働者の生産性個別評価のためのレポーティング機能
- 法令・規制の解釈責任の代行（運用責任は導入者に帰属する）

本宣言は `11_前提・制約・リスク.md` にも重複記載する。二重記載は意図的であり、両文書での一致が要求される（後述の自己監査チェックリスト参照）。
