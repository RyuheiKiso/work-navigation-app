# システム化計画

## 本ドキュメント群の位置づけ

IPA 共通フレーム 2013 (SLCP-JCF2013) の**企画プロセス**成果物のうち、
「システム化計画」フェーズに相当する文書セット。

本ドキュメント群は `docs/02_企画/README.md` が定める責務範囲:
**対象範囲・採用技術スタック・概略アーキテクチャ・開発スケジュール** を網羅する。

上流の「システム化構想」は未作成のため、根拠となる課題・価値・コンセプトは
`docs/90_業界分析/` (39 章) を直接引用している。

---

## 対象外事項

本ドキュメント群では以下を扱わない。これらは対象外と判断する:

- 進捗管理 (タスク・工数・バーンダウン)
- 体制管理 (RACI・組織図・担当者アサイン)
- 予算稟議・費用見積
- ヒアリング議事録・MVP 計画

## 用語定義

本ドキュメント群で使用する主要用語の統一定義。

| 用語 | 定義 |
|---|---|
| **作業員** | 製造現場でタブレットを使って作業ナビに従い実績を記録する人。「オペレータ」とは呼ばない |
| **タブレット** | 現場で使用する Android / Windows タブレット端末。「ハンディ」とは呼ばない |
| **バックエンド** | Rust (axum) API サーバ。「サーバ」と略記する場合も同義 |
| **マスタ** | 作業手順・部品・ユーザなどの基準データ。マスタデータとも言う |
| **実績** | 作業員が記録した作業結果 (完了時刻・測定値・スキャン値・写真など) |
| **トレサビ** | トレーサビリティ。製品・部品・作業の追跡可能性 |
| **単一工場** | 本システムが対象とする 1 拠点の製造工場。多工場展開はスコープ外 |
| **オフライン同期** | サーバ接続なしにローカル SQLite に記録し、復帰後に自動送信する仕組み |
| **Outbox** | タブレット内の「送信待ち記録」を管理するテーブル。同期の基盤 |
| **Audit Trail** | 全操作イベントの Append-only ログ。改ざん不可 |
| **ALCOA+** | 電子記録の完全性を評価する 9 原則 (帰属性・判読性・同時性・独自性・正確性・完全性・一貫性・永続性・利用可能性) |
| **Phase 1/2/3** | 開発フェーズの区分。詳細は `07_開発スケジュールとフェーズ計画.md` を参照 |

## 図のルール

本ドキュメント群の図はすべて draw.io で作成し、SVG 形式でエクスポートする。
ASCII 図の使用は禁止 (`docs/CLAUDE.md` に従う)。
`.drawio` ファイルと `.svg` ファイルは同じディレクトリ (`docs/02_企画/システム化計画/`) に配置する。
