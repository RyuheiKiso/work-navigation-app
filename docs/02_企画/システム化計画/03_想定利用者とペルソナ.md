# 想定利用者とペルソナ

**主読者**: 工場長・作業員・品質保証担当  
**想定所要時間**: 20 分

---

## 3.1 ペルソナ定義

本システムは多様な作業者が混在する現場を想定する。4 つのペルソナを設定し、UI・機能要件・多言語対応の設計根拠とする。

> **スコープ**: 本章は**現場利用者**（作業員・工長・QA・管理者）のペルソナを定義する。管理系ステークホルダー（経営層・外部監査人・調達担当・SIer パートナー）のニーズは [02章](./02_対象範囲とスコープ.md)・[07章](./07_規制・品質・倫理上のリスクと方針.md)・[09章](./09_開発スケジュールとフェーズ計画.md) を参照。

### ペルソナ A: 熟練作業員（コアユーザー）

| 項目 | 内容 |
|---|---|
| 年齢・経験 | 40〜55 歳、同工場在籍 10 年以上 |
| 特性 | 暗黙知が豊富、手順書なしでも作業可能、デジタル機器への抵抗感あり |
| 主な課題 | 「分かりきった操作」を強制確認させられることへのストレス |
| システムへの期待 | 自分のペースを乱さない略式モード、手順改訂をすぐ確認できること |
| UI 方針 | **Fading モード**（習熟済みステップを縮約表示）、確認タップのみで次へ進める |

Dreyfus 5 段階モデルの「熟練者/達人」段階に相当し、Scaffolding ではなく Fading が適切（[`90_業界分析/15_習熟効果と学習曲線.md`](../../90_業界分析/15_習熟効果と学習曲線.md) 参照）。

### ペルソナ B: 見習い作業員（教育対象ユーザー）

| 項目 | 内容 |
|---|---|
| 年齢・経験 | 20〜30 歳、入社 6 ヶ月未満 |
| 特性 | 手順の詳細説明が必要、失敗を恐れており確認が多い |
| 主な課題 | どのステップでどう判断すべきか分からず熟練者に頻繁に確認を求める |
| システムへの期待 | 詳細な手順、動画/写真サンプル、気軽に質問できる環境 |
| UI 方針 | **Scaffolding モード**（全ステップ詳細表示、合否基準の図解表示） |

認知的徒弟制の Scaffolding 戦略に基づき、段階的なサポートからの自立を促す（[`90_業界分析/23_作業訓練設計とインストラクショナルデザイン.md`](../../90_業界分析/23_作業訓練設計とインストラクショナルデザイン.md) 参照）。

### ペルソナ C: 高齢作業員・外国人作業員（インクルーシブユーザー）

| 項目 | 内容 |
|---|---|
| 年齢・背景 | 60 歳以上またはベトナム語/英語話者（日本語 N4〜N5 レベル） |
| 特性 | 視力低下・精細操作困難（高齢）または漢字読み取り困難（外国人） |
| 主な課題 | 通常 UI では文字が小さく読みにくい、漢字の工程名が読めない |
| システムへの期待 | 大きいタップターゲット、ふりがな、ピクトグラム、やさしい日本語 |
| UI 方針 | **シニアモード**（フォント拡大・タップターゲット 72dp 以上）＋**多言語対応**（やさしい日本語・ルビ） |

製造業の外国人労働者は 2023 年時点で約 204 万人、うち製造業が 22% を占める。育成就労制度（2027 年移行予定）への対応が急務（[`90_業界分析/34_多言語化・外国人労働者と読み書き能力差.md`](../../90_業界分析/34_多言語化・外国人労働者と読み書き能力差.md) 参照）。

### ペルソナ D: 工長・品質保証担当・管理者（管理ユーザー）

| 項目 | 内容 |
|---|---|
| 役割 | ライン管理・品質確認・手順書改訂・不適合対応承認 |
| 端末 | PC ブラウザ（管理 Web） |
| 主な課題 | 手順書の版管理・有効日管理が煩雑、不適合への対応が後手に回る |
| システムへの期待 | リアルタイムダッシュボード、Kaizen 提案への返答機能、監査証跡の検索 |

## 3.2 主要ユースケース概要

| UC | 利用者 | 概要 |
|---|---|---|
| UC01 | 作業員 | 作業手順に従って作業しナビゲーション画面で完了記録 |
| UC02 | 作業員 | QR/Data Matrix スキャンで部品ロットを記録 |
| UC03 | 作業員 | 作業中断時に再開サマリ画面で状況を確認 |

| UC09 | 工長・QA | トレーサビリティ照会（ロット → 作業記録の逆引き） |

## 3.3 多言語・やさしい日本語方針

Phase 1 は日本語のみで提供するが、I18n 基盤（`i18next` 対応・JSON 外部化）を最初から組み込む。

| Phase | 対応言語 | 内容 |
|---|---|---|
| Phase 1 | 日本語 | やさしい日本語スタイルガイド準拠、ルビ表示コンポーネント、漢字レベル切替（N3/N4/N5） |
| Phase 2 | ベトナム語・英語 | DeepL 素案 + 人手グロッサリレビュー、安全クリティカル文言は機械翻訳禁止 |
| Phase 3 | 中国語簡体・ポルトガル語 | ピクトグラム UI 強化（ISO 7010 準拠）、TTS オプション |

「やさしい日本語」（厚労省ガイドライン）はJLPT N4〜N5 レベルの語彙・文型に制限する。安全上の注意（「高圧」「回転中」等）を省略してはならない（[`90_業界分析/34_多言語化・外国人労働者と読み書き能力差.md`](../../90_業界分析/34_多言語化・外国人労働者と読み書き能力差.md) 参照）。

## 3.4 夜勤・高齢配慮方針

| 配慮事項 | 実装方針 |
|---|---|
| 夜間自動ダークモード | 22:00〜06:00 に自動切替。低輝度・コントラスト保持 |
| シニアモード | 作業者プロファイルで有効化。フォント 14pt 以上、タップターゲット 72dp 以上（WCAG 2.1 AA 準拠） |
| 疲労リスク時間帯の強化確認 | 深夜 0〜6 時のクリティカルステップで追加確認プロンプト |
| シフトメタデータ自動付与 | 作業ログに `shift_type`, `hours_into_shift` を自動記録（監査分析用） |

等価血中アルコール濃度モデル（Lamond & Dawson）では、17〜19 時間の覚醒は BAC 0.05% 相当の認知機能低下をもたらす。深夜シフトでのクリティカル作業に追加確認を設けることは認知科学的に根拠がある（[`90_業界分析/37_シフト・夜勤・疲労管理と高齢化労働力.md`](../../90_業界分析/37_シフト・夜勤・疲労管理と高齢化労働力.md) 参照）。

## 3.5 アクセシビリティ方針

WCAG 2.1 AA 準拠を UI 受入基準とする。

| 項目 | 基準 |
|---|---|
| コントラスト比 | 通常テキスト 4.5:1 以上、大テキスト 3:1 以上 |
| タップターゲット | 通常 48dp 以上、手袋着用環境 72dp 以上 |
| フォントサイズ | 最小 12pt、シニアモード 14pt 以上 |
| 色のみの情報伝達 | 色 + 形状の冗長符号化（色覚特性対応） |

手袋着用環境では Fitts の法則に基づく 72dp 最小タップターゲットが推奨される（[`90_業界分析/18_現場HCIと作業者インターフェース.md`](../../90_業界分析/18_現場HCIと作業者インターフェース.md) 参照）。

> 1. 4 ペルソナ（熟練/見習/高齢・外国人/管理）の特性に応じた UI モード（Fading/Scaffolding/シニア）を実装する。  
> 2. Phase 1 は日本語のみだが I18n 基盤を最初から組み込む。安全クリティカル文言の機械翻訳は禁止とする。  
> 3. WCAG 2.1 AA 準拠・手袋 72dp を UI 受入基準とする。
