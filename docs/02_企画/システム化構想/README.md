# システム化構想

IPA 共通フレーム 2013 (SLCP-JCF2013) に準拠した企画プロセス上流成果物。  
本ディレクトリは、システム化計画の **前提となる課題・ビジョン・コンセプト・倫理スタンス** を確定する。

---

## 本書が約束しないこと

本文書群を読む前に、以下の「約束しないこと」を確認してほしい。

1. **ROI 数値は約束しない** — Phase 1 PoC 終了後の実測値を基に初めて試算可能。Phase 1 前の段階での定量的収益約束は行わない。
2. **暗黙知の完全形式知化は約束しない** — Gourlay（2006）の批判が示す通り、暗黙知は完全には言語化できない。形式知化できる範囲を着実に記録することが目標。
3. **Kaizen 参加率の数値目標化は約束しない** — 強制されたKaizen は形骸化する。自発性を損なわないために、参加率のKPI化を行わない。

これらは「弱気な立場」ではなく、**製造業DXの68%が期待効果未達**（Panorama Consulting, 2022）という業界統計を正面から受け止めた誠実な立場である。

## 文書目的

本構想書は「何のために・誰のために・どんな考え方で作るか」を確定する。  
後続の **システム化計画**（[`../システム化計画/`](../システム化計画/)）が扱う「何を・どのように・いつまでに作るか」とは役割が異なる。

| 本構想が扱う（WHY） | システム化計画が扱う（WHAT/HOW） |
|---|---|
| 経営課題・外部環境の認識 | 対象範囲・スコープの確定 |
| 現場の本質的課題の構造分析 | KPI 閾値・定量目標 |
| ビジョン・価値命題 | 技術選定の詳細根拠 |
| 設計原則（北極星） | フェーズスケジュール |
| 倫理スタンス | リスク台帳（ID 単位） |
| 不確実性とエビデンスの正直な開示 | 規制適用判断表 |
