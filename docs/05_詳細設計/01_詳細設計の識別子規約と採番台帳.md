# 01 詳細設計の識別子規約と採番台帳

本章は `05_詳細設計` フェーズで新規採番する設計識別子の規約と採番台帳を確定する。`04_概要設計/付録/99_設計識別子採番台帳.md` で「未採番」または「採番対象外」とされた識別子を本書で正式採番する。

---

## 1. 採番規約

### 1-1. 継承規約

`03_要件定義/付録/00_要件IDルールと採番規約.md` の基本規則（3 桁ゼロ埋め・欠番禁止・廃止メタデータ必須）を完全継承する。

### 1-2. 詳細設計フェーズの新規識別子

| プレフィックス | 正式名称 | 採番単位 | 参照識別子 |
|---|---|---|---|
| `IDX-NNN` | Index（物理インデックス）| 全域 | TBL-NNN |
| `VW-NNN` | View / Materialized View | 全域 | TBL-NNN |
| `TST-{unit/intg/e2e/sec}-NNN` | Test Case | レベル別 | FR-NNN / NFR-NNN / BR-BUS-NNN |
| `FNC-{BE/FE}-NNN` | Function / Method Signature | BE/FE 別 | MOD-NNN |
| `DDL-NNN` | DDL 定義参照番号（TBL と 1:1）| 全域 | TBL-NNN |

### 1-3. 識別子フォーマット

```
IDX-NNN        例: IDX-001, IDX-042
VW-NNN         例: VW-001, VW-010
TST-unit-NNN   例: TST-unit-001
TST-intg-NNN   例: TST-intg-001
TST-e2e-NNN    例: TST-e2e-001
TST-sec-NNN    例: TST-sec-001
FNC-BE-NNN     例: FNC-BE-001
FNC-FE-NNN     例: FNC-FE-001
DDL-NNN        例: DDL-001（= TBL-001 work_events の DDL）
```

## 5. FNC 採番台帳（次採番値: FNC-BE-001 / FNC-FE-001）

重要な関数/メソッドシグネチャを `FNC-NNN` で参照する。`02_バックエンド詳細設計/` および `04_ハンディAPP詳細設計/` 内で採番する。採番は重要度の高いもの（ドメインサービス・ステートマシン遷移・ハッシュチェーン計算）に限定する。

---

**本節で確定した方針**
- **IDX-001〜016（次採番値 IDX-017）・VW-001〜008（次採番値 VW-009）を確定し、04_概要設計の未採番識別子を本書で正式採番した。**
- **テストケース識別子は TST-{unit-BE/unit-FE/intg/e2e/alcoa/sec/perf}-NNN の 7 分類とし、08_テストケース詳細設計 で全採番することを確定した。**
- **FNC-{BE/FE}-NNN は重要シグネチャの参照番号として採番し、全関数の網羅は求めない（Rust の pub fn 全件ではなくドメイン境界の主要トレイトメソッドに限定）。**
