## 2. 本書が約束しないこと（対象外と判断する）

### D-DD-001: 実装ファイルの作成

**対象外と判断する。**

`crates/`・`apps/`・`src/` 配下の実装ファイルの作成は、本詳細設計の対象ではない。本書はコーディングに利用可能な仕様を確定するにとどまる。実装は 06_実装フェーズで行う。

### D-DD-002: データの実際の投入・テスト実施

**対象外と判断する。**

マスタデータ投入・単体テスト実行・統合テスト実行は 07_テスト実施フェーズで行う。本書はテストケースの「設計」を確定するにとどまる。

### D-DD-003: OpenAPI YAML ファイルの生成

**対象外と判断する。**

`openapi.yaml` の実ファイルは utoipa による Rust コードからの自動生成で作成する（`02_バックエンド詳細設計/01_wnav_api詳細設計.md` §7 に生成方針を記載）。本書は OpenAPI スキーマの仕様をドキュメントとして確定するが、YAML ファイル生成は実装フェーズに委ねる。

### D-DD-004: クラウド・外部 SaaS 連携設計

**対象外と判断する。**

本システムは単一工場・閉域ネットワーク前提（`04_概要設計/01_システム方式設計/99_前提制約` P-01）。クラウドプラットフォーム（AWS/Azure/GCP）との連携設計は本詳細設計に含まれない。

### D-DD-005: OT 系プロトコル実装設計（PLC/SCADA/OPC-UA）

**対象外と判断する。**

P4 アーキテクチャ原則（OT-IT 境界の永続分離）により、OT 系との直接通信インターフェースは本プロジェクトのスコープ外とする。詳細設計でも当該設計は行わない。

### D-DD-006: チューニング後の最終パラメータ値の確定

**対象外と判断する。**

PostgreSQL の `work_mem`・`shared_buffers`・Rust の Tokio スレッドプール数等の最終チューニング値は、実際の負荷試験後に確定する（07_テスト実施フェーズ）。本書は概要設計で確定した初期設定値と推奨範囲を引き継ぐにとどまる。

## 3. 前提制約の継承

| 制約 | 出典 |
|---|---|
| アスキー図の使用禁止、drawio + svg での図作成が必須 | `/docs/CLAUDE.md` |
| 曖昧表現（「考慮する」「検討する」等）の使用禁止 | `04_概要設計/00_本書の位置づけ` §4 |
| 全章末に「本節で確定した方針」「## 参照業界分析」を必置 | `04_概要設計/README.md` |
| eval() および動的コード生成の禁止 | `04_概要設計/02_ソフトウェア方式設計/99_前提制約` |
| 個人別ランキング画面・機能の永続的非実装 | BR-BUS-029 |

---

**本節で確定した方針**
- **本書（05_詳細設計）が担保する 10 項目と、対象外と判断する 6 項目（D-DD-001〜006）を明示し、実装フェーズとの責務境界を確定した。**
- **実装ファイル作成（D-DD-001）・OpenAPI YAML 生成（D-DD-003）は明示的に実装フェーズに委譲し、本書はコーディング仕様の確定に集中することを確定した。**
- **OT 系連携設計（D-DD-005）は P4 原則により詳細設計でも対象外と宣言し、スコープ拡大を防止した。**
