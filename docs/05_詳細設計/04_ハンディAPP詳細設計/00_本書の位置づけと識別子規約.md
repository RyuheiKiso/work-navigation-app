| QR / バーコード | react-native-vision-camera + MLKit | FR-EV-004 |

### 2-1. TypeScript strict mode 強制ルール

```typescript
// tsconfig.json（抜粋）
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true
  }
}
```

- `any` 型は禁止。回避できない外部 API 境界では `unknown` + 型ガード関数を使用する。
- 非同期関数の戻り値は必ず `Promise<T>` を明示する。
- Optional chaining（`?.`）と Nullish coalescing（`??`）を積極的に使用し、`!` non-null assertion は禁止。

## 3. パッケージ構成と依存方向

React Native アプリの src 配下は以下の構成とし、`features → shared` の一方向依存を厳守する。

```
src/
  features/
    network/           ← MOD-FE-HA-001 NetworkProvider
      outbox/          ← MOD-FE-HA-002 OutboxWorker
    navigation/
      step-engine/     ← MOD-FE-HA-003 StepEngine
    evidence/          ← MOD-FE-HA-004 EvidenceCapture
    suspension/        ← MOD-FE-HA-005 SuspensionFlow
    kaizen/            ← MOD-FE-HA-006 AndonKaizenFlow
  shared/
    ui/
      ElectronicSignPad/  ← MOD-FE-HA-007
    db/                   ← MOD-FE-HA-008 LocalDbService
    i18n/                 ← MOD-SH-001 LocaleResolver
    id/                   ← MOD-SH-002 IdGenerator
    clock/                ← MOD-SH-003 ClockService
    api/                  ← MOD-SH-004 ApiClient
```

依存禁止ルール:
- `shared/` 配下モジュールが `features/` 配下を import することを禁止する。
- `features/` 間の相互 import を禁止する。共通化が必要な場合は `shared/` に昇格させる。
