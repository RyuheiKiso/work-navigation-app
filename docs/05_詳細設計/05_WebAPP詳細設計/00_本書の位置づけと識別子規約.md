### 2-1. パッケージ間依存グラフ

依存方向は上流から下流への一方向とし、循環依存を禁止する。

```
features/sop-editor
  ├── features/approval
  └── features/sop-editor/dsl

features/dashboard
features/audit
features/hash-verify
features/outbox-mon
features/reports
features/user-mgmt

すべての features → shared/api (ApiClient / MOD-SH-004)
すべての features → shared/ui (共通コンポーネント)
すべての features → shared/hooks (共通カスタムフック)
すべての features → shared/i18n (LocaleResolver / MOD-SH-001)
```

- `shared/` 配下のモジュールは `features/` に依存しない
- `features/` 間の直接 import を禁止する（共通ロジックは `shared/` に切り出す）

## 3. FNC-FE-NNN 識別子採番規約

### 3-1. 採番対象

FNC-FE-NNN は以下の要素に対して採番する。採番は重要度の高いもの（状態遷移ハンドラ・カスタムフックのコアロジック・DSL 評価・Auto-Save デバウンス）に限定し、全コンポーネント関数の網羅は求めない。

| 採番対象 | 例 |
|---|---|
| Zustand ストアアクション | `useSopEditorStore.applyStep` |
| react-query カスタムフック | `useSopDraft`・`useApprovalMutation` |
| DSL 評価・ホワイトリスト検証 | `evaluateJsonLogic`・`validateDslAst` |
| Auto-Save デバウンス | `useAutoSave` |
| バージョン差分計算 | `computeStepDiff` |
| ハッシュチェーン検証 UI | `useHashChainVerification` |
| DLQ 再投入ハンドラ | `useOutboxRetry` |
| 帳票ダウンロード | `useReportDownload` |

## 5. MOD-FE-MA-005 UserRoleAdmin 型定義

UserRoleAdmin は SCR-MC-002（ユーザー管理）・SCR-MC-003（ロール/スキル管理）を担当する。

```typescript
// ロール定義（FR-AU-002 準拠 6 ロール固定）
export type UserRole =
  | 'operator'
  | 'supervisor'
  | 'quality_admin'
  | 'master_admin'
  | 'system_admin'
  | 'executive';

// ユーザー管理 Props（SCR-MC-002）
export interface UserAdminProps {
  onCreateUser: (dto: CreateUserDto) => Promise<void>;
  onUpdateUser: (userId: string, dto: UpdateUserDto) => Promise<void>;
  onDeactivateUser: (userId: string) => Promise<void>;
  onAssignRole: (userId: string, role: UserRole) => Promise<void>;
}

export interface CreateUserDto {
  loginId: string;
  displayName: string;
  role: UserRole;
  skillIds: string[];
  locale: 'ja' | 'en' | 'ja-simple';
}

export interface UpdateUserDto {
  displayName?: string;
  role?: UserRole;
  skillIds?: string[];
  locale?: 'ja' | 'en' | 'ja-simple';
}

// スキル管理 Props（SCR-MC-003）
export interface SkillAdminProps {
  onCreateSkill: (dto: CreateSkillDto) => Promise<void>;
  onUpdateSkill: (skillId: string, dto: UpdateSkillDto) => Promise<void>;
  onDeleteSkill: (skillId: string) => Promise<void>;
}

export interface CreateSkillDto {
  skillCode: string;
  nameJa: string;
  nameEn: string;
  description: string;
}

export interface UpdateSkillDto {
  nameJa?: string;
  nameEn?: string;
  description?: string;
}

// カスタムフック（FNC-FE-009）
export declare function useUserCrud(): {
  users: UserSummary[];
  isLoading: boolean;
  createUser: (dto: CreateUserDto) => Promise<void>;
  updateUser: (userId: string, dto: UpdateUserDto) => Promise<void>;
  deactivateUser: (userId: string) => Promise<void>;
};

// カスタムフック（FNC-FE-010）
export declare function useRoleMutation(): {
  assignRole: (userId: string, role: UserRole) => Promise<void>;
  isPending: boolean;
  error: Error | null;
};

// エラーコード対応
// ERR-VAL-014: ログイン ID 重複
// ERR-BIZ-009: 最終 system_admin の無効化禁止
// ERR-AUTH-003: RBAC 不足（自己より上位ロールへの昇格禁止）
```

## 6. 上流設計との対応

本サブの各章は以下の上流ドキュメントを入力とし、変更を行わない。

| 上流ドキュメント | 本サブへの制約 |
|---|---|
| `04_概要設計/02_ソフトウェア方式設計/11_モジュール一覧.md` | MOD-FE-MA-001〜005・MOD-FE-MC-001〜005 の責務は変更不可 |
| `04_概要設計/03_画面設計/01_画面一覧と機能配置.md` | SCR-MA/MC の主責務・アクセスロールは変更不可 |
| `04_概要設計/02_ソフトウェア方式設計/07_例外・エラーハンドリング統一方式.md` | ERR-NNN 識別子・HTTP ステータスマッピングは変更不可 |
| `04_概要設計/05_外部インターフェース設計/03_認証認可方式.md` | JWT RS256・RBAC 6 ロール定義は変更不可 |
| `05_詳細設計/01_詳細設計の識別子規約と採番台帳.md` | FNC-FE-NNN 採番規則を継承 |

---

**本節で確定した方針**
- **IPA 2.5.1 の全要求タスクを本サブの 11 章に割付け、カバレッジに漏れがないことを確認した。**
- **React 18+ / TypeScript strict / react-query v5 / Zustand v4 をコア技術スタックとして確定し、全モジュールに適用する `tsconfig.json` strict オプション（noUncheckedIndexedAccess・exactOptionalPropertyTypes）を本章で一元定義した。**
- **FNC-FE-001〜016 を初期採番し、次採番値 FNC-FE-017 を確定した。MOD-FE-MA-005 UserRoleAdmin の型定義を本章 §5 で完結させた。**
