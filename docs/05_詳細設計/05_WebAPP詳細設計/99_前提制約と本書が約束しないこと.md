## 2. 本書が約束しないこと（対象外と判断する）

### WA-DD-001: 実装ファイルの作成

**対象外と判断する。**

`src/features/` 配下の実装ファイルの作成は、本詳細設計の対象ではない。本書はコーディングに利用可能な型定義・フックシグネチャ・コンポーネントツリーを確定するにとどまる。実装は 06_実装フェーズで行う。

### WA-DD-002: CSS / スタイリングの詳細設計

**対象外と判断する。**

CSS クラス・Tailwind ユーティリティの詳細・デザイントークンの完全定義は `06_画面詳細設計/` の責務とする。本書のコードブロック内に登場する CSS クラス名（例: `bg-yellow-100`）は実装のヒントであり、設計の確定事項ではない。

### WA-DD-003: Storybook / コンポーネントカタログの作成

**対象外と判断する。**

Storybook による UI コンポーネントカタログの作成は実装フェーズで行う。本書は Props 型定義によってコンポーネントの契約を確定するが、ビジュアル仕様の確認ツールの整備は対象外とする。

### WA-DD-004: E2E テストシナリオの詳細設計

**対象外と判断する。**

E2E テストケース（TST-e2e-NNN）の詳細設計は `08_テストケース詳細設計/` で行う。本書は各コンポーネントの ERR-NNN 対応を記述するが、Playwright/Cypress のテストシナリオ記述は対象外とする。

### WA-DD-005: バックエンド API レスポンス型の定義

**対象外と判断する。**

`src/shared/api/generated/` 配下の OpenAPI 生成クライアントの型定義は `03_API詳細設計/` が確定し、utoipa による自動生成で作成する。本書のコードブロック内で `import type { XxxResponse } from '@/shared/api/generated'` として参照する型の定義は `03_API詳細設計/` に委ねる。

### WA-DD-006: 国際化（i18n）メッセージキーの全件定義

**対象外と判断する。**

`ja/en/ja-simple` の全翻訳キー・翻訳文字列の完全定義は、LocaleResolver（MOD-SH-001）の実装フェーズに委ねる。本書は `MultilingualText` 型の構造を確定するにとどまる。

## 4. 設計の品質判定基準

| 確認項目 | 判定基準 |
|---|---|
| 全 10 章の TypeScript 型定義 | Props・State・Actions の型が全モジュールに存在する |
| FNC-FE 採番 | FNC-FE-001〜016 が採番済み、次採番値 FNC-FE-017 が明示されている |
| JSON Logic ホワイトリスト | ALLOWED_OPERATORS が型定義と validateDslAst の両方に存在する |
| 承認フロー禁止遷移 | 全 6 禁止遷移が §2-1 遷移表に明記されている |
| SloStatus 逆方向メトリクス | availabilityPercent・offlineSyncRatePercent が明示的に分岐されている |
| 章末方針・参照業界分析 | 全 11 章（00〜09）に必置されている |

---

**本節で確定した方針**
- **本サブ（05_WebAPP詳細設計）が担保する 10 項目と、対象外と判断する 6 項目（WA-DD-001〜006）を明示し、実装フェーズとの責務境界を確定した。**
- **実装ファイル作成（WA-DD-001）・CSS 詳細設計（WA-DD-002）・E2E テストシナリオ（WA-DD-004）・OpenAPI 生成型（WA-DD-005）は明示的に後続フェーズに委譲し、本書はコーディング仕様の確定に集中することを確定した。**
- **TypeScript strict オプション群（strict / noUncheckedIndexedAccess / exactOptionalPropertyTypes）は全モジュール共通の前提制約として継承し、型定義の曖昧さを排除することを確定した。**
