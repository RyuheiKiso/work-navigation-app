# 06 セキュリティ・性能テストケース

本章はセキュリティテストケース TST-sec-001〜011 および性能テストケース TST-perf-001〜007 を確定する。セキュリティテストは RBAC・JWT・SQL インジェクション・個人ランキング禁止・ブルートフォース・HMAC 署名・バイナリ間エンドポイント越境アクセス防止の 7 領域、性能テストは k6 による負荷・スループット・ハッシュチェーン検証・SQLite 同期・帳票生成の 5 領域（バイナリ別ベースライン含む）を対象とする。

バックエンドは 2 バイナリ構成であり、セキュリティ・性能テストもバイナリ別に実施する。

| バイナリ | ポート | テスト観点 |
|---|---|---|
| `wnav_terminal_api` | 8080 | 作業ログ受信スループット・ハンディ端末向け RBAC・Idempotency-Key 検証 |
| `wnav_master_api` | 8081 | SOP 照会・マスタ更新スループット・管理系 RBAC・越境アクセス拒否 |

### 1-7. バイナリ間エンドポイント越境アクセス防止（TST-sec-011）

本テストは 2 バイナリ構成において、誤ったバイナリへのリクエストが確実に拒否されることを包括的に確認する。

| 項目 | 内容 |
|---|---|
| TST-ID | TST-sec-011 |
| テスト観点 | バイナリ間エンドポイント越境アクセス防止（セキュリティ境界の網羅的確認）|
| 検証スクリプト | `bash scripts/check-binary-boundary.sh` |
| 検証内容 | 以下の 6 組の越境リクエストが全て HTTP 404 を返すこと |

越境リクエスト一覧:

| 送信先 | リクエスト | 期待レスポンス |
|---|---|---|
| terminal-api（8080）| `POST /api/v1/sop`（SOP 編集）| HTTP 404 |
| terminal-api（8080）| `POST /api/v1/sops/{id}/versions`（SOP バージョンアップ）| HTTP 404 |
| terminal-api（8080）| `GET /api/v1/admin/users`（ユーザー管理）| HTTP 404 |
| master-api（8081）| `POST /api/v1/events`（作業ログ記録）| HTTP 404 |
| master-api（8081）| `POST /api/v1/step-events`（ステップイベント）| HTTP 404 |
| master-api（8081）| `POST /api/v1/andon-alerts`（アンドン発報）| HTTP 404 |

```bash
#!/usr/bin/env bash
# scripts/check-binary-boundary.sh
# バイナリ間越境アクセス防止確認スクリプト

set -euo pipefail

TERMINAL_BASE="http://localhost:8080"
MASTER_BASE="http://localhost:8081"
FAIL=0

check_404() {
  local url="$1"
  local method="${2:-GET}"
  local status
  status=$(curl -s -o /dev/null -w "%{http_code}" -X "$method" "$url" \
    -H "Authorization: Bearer ${TEST_JWT:-dummy}" \
    -H "Content-Type: application/json" \
    -d '{}' 2>/dev/null)
  if [ "$status" = "404" ]; then
    echo "PASS: $method $url → 404"
  else
    echo "FAIL: $method $url → $status (expected 404)"
    FAIL=1
  fi
}

# terminal-api（8080）への master-api 系リクエスト
check_404 "${TERMINAL_BASE}/api/v1/sop" POST
check_404 "${TERMINAL_BASE}/api/v1/sops/00000000-0000-0000-0000-000000000001/versions" POST
check_404 "${TERMINAL_BASE}/api/v1/admin/users" GET

# master-api（8081）への terminal-api 系リクエスト
check_404 "${MASTER_BASE}/api/v1/events" POST
check_404 "${MASTER_BASE}/api/v1/step-events" POST
check_404 "${MASTER_BASE}/api/v1/andon-alerts" POST

echo "----------------------------------------"
[ "$FAIL" -eq 0 ] && echo "TST-sec-011: ALL PASS" || { echo "TST-sec-011: FAILED"; exit 1; }
```

| 対応 NFR | NFR-SEC-001（エンドポイント分離）, NFR-SEC-048（セキュリティ境界）|

**本節で確定した方針**
- **TST-sec-001〜011 のセキュリティテストケースを 7 領域（RBAC・JWT・SQL インジェクション・個人ランキング・ブルートフォース・HMAC・バイナリ間越境アクセス防止）に分類し、各ケースの入力・期待出力・対応 FR を確定した。特に TST-sec-006 は BR-BUS-029 を API レスポンスの JSON 構造レベルで検証し、設計違反を防止する。**
- **TST-sec-011（バイナリ間エンドポイント越境アクセス防止）を新設し、`scripts/check-binary-boundary.sh` で 6 組の越境リクエストが全て HTTP 404 を返すことを自動確認することを確定した。**
- **TST-perf-001/002 は k6 の `constant-arrival-rate` エグゼキューターで `wnav_terminal_api`（8080）の 50 同時ユーザー・100 rps を厳密に制御し、P95 ≤ 2000 ms の NFR-PERF-001 達成を確認することを確定した。**
- **TST-perf-006/007 を新設し、`wnav_master_api`（8081）の SOP 照会・マスタ更新スループット（P95 ≤ 1500 ms、50 rps ベースライン）を確立することを確定した。terminal-api と独立して測定する。**
- **TST-perf-003 は Criterion ベンチマークで 100 万件のハッシュチェーン検証を 60 秒以内で完了することを確認し、週次バッチ（BAT-001）の実用性を保証することを確定した。**
