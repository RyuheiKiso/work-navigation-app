# 04 E2E テストケース

本章は全 22 ユースケース（UC-001〜022）のゴールデンパスに対する E2E テストケース TST-e2e-001〜022 を確定する。各テストケースは 1 UC に 1:1 で対応し、Playwright（Web / 管理コンソール）または Detox（React Native ハンディ APP）が実行するアクション列とアサーション条件を定義する。

---

## 1. E2E テストフレームワーク設定

| 対象アプリ | フレームワーク | 実行環境 |
|---|---|---|
| マスタメンテナンス（React Web）| Playwright（Chromium）| Docker Compose（CI）|
| 管理コンソール（React Web）| Playwright（Chromium）| Docker Compose（CI）|
| ハンディ APP（React Native）| Detox + iOS Simulator | CI（macOS ランナー）|

## 6. オフライン・同期 UC（TST-e2e-021〜022）

### TST-e2e-021: EMERGENCY_MODE 動作（UC-021）

| 項目 | 内容 |
|---|---|
| ツール | Detox |
| アクション | 1. ネットワーク断（Detox のネットワーク制御 API で模倣）→ 2. 5 分待機（シミュレーション）→ 3. EMERGENCY_MODE バナーが表示される → 4. ステップ完了が引き続き可能 → 5. Outbox に PENDING イベントが蓄積される |
| アサーション | NetworkStateManager.currentState = 'EMERGENCY_MODE' / API 呼び出しが 0 回 |
| 対応 FR | FR-OF-001/002 |

### TST-e2e-022: リカバリ（UC-022）

| 項目 | 内容 |
|---|---|
| ツール | Detox |
| アクション | 1. EMERGENCY_MODE 状態からネットワーク接続を復元 → 2. Outbox PENDING イベントが一括配信される → 3. マスタ同期が実行される → 4. バックエンド DB の work_events 件数がローカル SQLite と一致 |
| アサーション | local_outbox_events.status がすべて 'SENT' / NetworkStateManager.currentState = 'CONNECTED' |
| 対応 FR | FR-OF-003, FR-SY-001/002 |

---

**本節で確定した方針**
- **TST-e2e-001〜022 の 22 件は全 22 UC のゴールデンパスに 1:1 で対応し、各ケースに Playwright または Detox のアクション列とアサーション条件を確定した。**
- **TST-e2e-020 は XLSX 出力にワーカー別集計行が存在しないことをアサーションに含め、BR-BUS-029（個人別ランキング禁止）を E2E レベルで確認することを確定した。**
- **TST-e2e-021/022 はネットワーク断・EMERGENCY_MODE 遷移・リカバリの一連フローを Detox のネットワーク制御 API で模倣し、オフライン設計全体の統合動作を確認することを確定した。**
- **TST-e2e-023〜036（IQC/リワーク 14 件）を追加し、後工程ハードゲート・Two-Person Integrity・ALCOA+ Original 不変性を E2E レベルで確認することを確定した。**
