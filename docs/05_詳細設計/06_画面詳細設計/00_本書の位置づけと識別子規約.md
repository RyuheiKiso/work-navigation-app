# 00 本書の位置づけと識別子規約

本章は `06_画面詳細設計` ディレクトリ全体の責務・カバレッジ・識別子規約を確定する。IPA 共通フレーム 2013「2.5.1 ソフトウェアコンポーネントの詳細設計」に準拠し、全 35 画面（SCR-HA-001〜015 / SCR-MA-001〜011 / SCR-MC-001〜009）の UI 仕様をコーディング直前仕様として確定する。

## 3. 識別子規約

### 3-1. 画面項目 ID（F{SCR番号}-NNN）

画面項目 ID は `F{SCR番号3桁}-{連番3桁}` 形式で採番する。SCR-HA-001 の項目 ID は `F001-001` から開始する。

| 対象 SCR 範囲 | SCR 番号 | 項目 ID 先頭 |
|---|---|---|
| SCR-HA-001 | 001 | F001-001 |
| SCR-HA-002 | 002 | F002-001 |
| SCR-MA-001 | 101 | F101-001 |
| SCR-MC-001 | 201 | F201-001 |

### 3-2. 遷移 ID（TRN-NNN）

画面遷移は `TRN-{3桁連番}` で識別する。遷移方向・トリガー・前後画面を 05 章イベント定義テーブルで対応付ける。

### 3-3. FR-UI 要件カバレッジ

本ディレクトリが担保する FR-UI 要件を示す。

| FR-UI-ID | 要件概要 | 担保ファイル |
|---|---|---|
| FR-UI-001 | 多言語対応（日本語・英語・中国語）| 01 章 SCR-HA-015 |
| FR-UI-002 | タッチターゲット ≥ 44dp | 04 章 バリデーション仕様 |
| FR-UI-003 | 縮退表示（オフライン時）| 01 章 SCR-HA-015 / 06 章 NetworkContext |
| FR-UI-004 | ダークモード自動切替（20:00-6:00）| 06 章 状態管理実装設計 |

## 4. 完成判定基準

| 確認項目 | 判定基準 |
|---|---|
| 画面項目定義 | 全 35 画面の全項目に `F{SCR}-NNN` が採番済み |
| FR 対応 | 全項目テーブルの「対応 FR」列が空欄ゼロ |
| バリデーション | 全入力項目にバリデーションルール・ERR コードが紐付き済み |
| 遷移 | 全 TRN-NNN に遷移元・遷移先・トリガーが定義済み |
| イベント | 全 35 画面の主要イベントに API エンドポイントが紐付き済み |
| 状態管理 | 全 3 APP の状態型定義（TypeScript interface）が記載済み |

---

**本節で確定した方針**
- **本ディレクトリは全 35 画面（SCR-HA 15 / SCR-MA 11 / SCR-MC 9）を IPA 2.5.1 の要求に従いコーディング直前仕様として確定することを責務とする。**
- **画面項目 ID は `F{SCR番号3桁}-{連番3桁}` 形式で採番し、全項目が FR・ERR コード・遷移 ID と 1 対多でトレースできることを必須要件とする。**
- **上流（04_概要設計）が確定した SCR-ID・FR-ID・CMP-ID・デザイントークンは本書で変更せず、精緻化のみを行う。**
