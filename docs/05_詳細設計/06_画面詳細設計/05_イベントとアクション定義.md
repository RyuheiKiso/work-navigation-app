# 05 イベントとアクション定義

本章は全 35 画面のイベントカタログを確定する。各イベントについてトリガー・アクション・状態変更・対応 API エンドポイントを明示し、実装者が UI インタラクションとバックエンド呼び出しを一対一で対応付けられる仕様とする。

---

## 1. イベント定義の凡例

| 列名 | 説明 |
|---|---|
| SCR | 画面 ID |
| EVT-ID | イベント識別子（EVT-{SCR番号}-{3桁連番}）|
| イベント名 | camelCase のイベントハンドラ名 |
| トリガー | タップ / 入力変更 / フォーカスアウト / 自動（タイマー）/ カメラ検知 |
| アクション | 実行する関数・サービス呼び出し |
| 状態変更 | 変更される React State / Zustand Store / Context の具体的フィールド |
| API | 呼び出す API-NNN（`04_概要設計/付録/99_設計識別子採番台帳.md` 参照）|

**本節で確定した方針**
- **全 35 画面の主要イベントを EVT-{SCR番号}-{3桁連番} で採番し、トリガー・アクション・状態変更・API エンドポイントを一対一で対応付けることを確定した。**
- **SCR-MA-004（SOP 編集）の Auto-Save は入力停止 3 秒後のデバウンスで発動し、isDirty = true 状態を常時監視してデータ損失ゼロを確保することを確定した。**
- **SCR-MC-001（ダッシュボード）のポーリング間隔は SELECT で 10s / 30s / 60s / 5m から選択可能とし、OPS-041 の設定可能要件を確定した。**
