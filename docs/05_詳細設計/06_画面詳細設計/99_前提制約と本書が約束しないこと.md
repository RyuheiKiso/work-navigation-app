# 99 前提制約と本書が約束しないこと

本章は `06_画面詳細設計` ディレクトリの設計境界を確定し、下位フェーズ（実装・テスト実施）が本書を誤解なく継承できることを保証する。

| 担保事項 | 根拠章 |
|---|---|
| 全 35 画面（SCR-HA 15 / SCR-MA 11 / SCR-MC 9）の画面項目定義（F{SCR}-NNN・型・必須・制約・バリデーション・対応 FR）| 01〜03 章 |
| 全入力型（15 種別）のバリデーションルール・ERR コード・TypeScript validator 実装 | 04 章 |

## 2. 本書が約束しないこと（対象外と判断する）

### D-UI-001: 実装ファイルの作成

**対象外と判断する。**

`apps/handy/src/`・`apps/master/src/`・`apps/console/src/` 配下の実装ファイルの作成は、本詳細設計の対象ではない。本書はコーディングに利用可能な仕様を確定するにとどまる。実装は `06_実装` フェーズで行う。

### D-UI-002: 実際の画面表示確認・スタイル調整

**対象外と判断する。**

CSS / React Native StyleSheet の実際の適用確認・デバイス実機での表示確認・レイアウト微調整は、実装フェーズのフロントエンド実装作業に属する。本書はデザイントークン（カラー・スペーシング・タイポグラフィ）の仕様を `04_概要設計/03_画面設計/05_共通UIコンポーネントとデザインシステム.md` から継承するにとどまる。

### D-UI-003: Storybook / コンポーネントカタログの作成

**対象外と判断する。**

CMP-HA/MA/MC の Storybook ストーリーファイル作成は実装フェーズに委ねる。本書はコンポーネントの責務・使用 SCR・Props 型定義を確定するが、視覚的なカタログ生成は行わない。

### D-UI-004: E2E テストスクリプトの作成・実行

**対象外と判断する。**

Detox / Playwright / Cypress によるテストスクリプトの作成と実行は `07_テスト実施` フェーズで行う。本書は `05_章 イベントとアクション定義` においてテスト設計の根拠となるイベント・API 対応を確定するが、テストスクリプト自体の作成は行わない。

### D-UI-005: アクセシビリティの実機検証

**対象外と判断する。**

TalkBack / VoiceOver でのスクリーンリーダー実機検証は実装フェーズに委ねる。本書は WCAG 2.1 AA（コントラスト比 4.5:1）・タッチターゲット ≥ 44dp を仕様として確定（FR-UI-002・NFR-UX-007/009 準拠）するが、実機での適合確認は実装フェーズに属する。

### D-UI-006: 個人別ランキング表示機能の設計

**BR-BUS-029 により永続的に対象外。**

個人別ランキング表示・個人パフォーマンス比較を目的とした画面・コンポーネントは、業務ルール BR-BUS-029 により永続的に禁止されている。本詳細設計にも当該機能の設計は含まれない。この判断は実装フェーズ以降も変更しない。

### D-UI-007: 監視カメラ映像連動機能の設計

**BR-BUS-029 により永続的に対象外。**

監視カメラ映像の画面への組み込みは BR-BUS-029 により永続的に禁止されている。本詳細設計にも当該機能の設計は含まれない。

## 3. 前提制約の継承

| 制約 | 出典 |
|---|---|
| アスキー図の使用禁止。図は drawio-authoring スキルで作成し SVG エクスポートを使用 | `/docs/CLAUDE.md` |
| 曖昧表現（「考慮する」「検討する」「できる限り」等）の使用禁止 | `04_概要設計/00_本書の位置づけ` §4 |
| 全章末に「本節で確定した方針」（3 箇条以上）と「## 参照業界分析」を必置 | `04_概要設計/README.md` |
| eval() および動的コード生成の禁止（JSON Logic は安全な実装に限定）| `04_概要設計/02_ソフトウェア方式設計/99_前提制約` |
| タッチターゲット ≥ 44dp（NFR-UX-009 必須）| `04_概要設計/03_画面設計/05_共通UIコンポーネントとデザインシステム.md` |

| 縮退表示（Emergency Mode）でのオフライン動作継続 | FR-SY-008 / FR-UI-003 |

## 4. 下位フェーズへの引き渡し事項

本書の確定内容を以下の形式で `06_実装` フェーズに引き渡す。

| 引き渡し事項 | 参照ファイル | 形式 |
|---|---|---|
| 全 35 画面の画面項目定義テーブル | 01〜03 章 | Markdown テーブル |
| バリデーション TypeScript 実装 | 04 章 | TypeScript コードブロック |
| イベント・API 対応表 | 05 章 | Markdown テーブル |
| 状態管理 TypeScript 型定義・reducer・Store | 06 章 | TypeScript コードブロック |

実装者は本書に記載されたコードブロックを `packages/validators/`・`apps/handy/src/contexts/`・`apps/master/src/stores/`・`apps/console/src/stores/` へ配置する起点として使用する。

---

**本節で確定した方針**
- **本ディレクトリが担保する 8 項目と対象外と判断する 7 項目（D-UI-001〜007）を明示し、実装フェーズとの責務境界を確定した。**
- **BR-BUS-029 による個人別ランキング機能・監視カメラ連動機能の永続的非実装（D-UI-006/007）を本詳細設計においても明示的に継承し、スコープ侵食を防止した。**
- **eval() 禁止原則は UI 層においても JSON Logic DSL ホワイトリストバリデーター（04 章 `validateJsonLogicDsl`）により担保することを確定した。**
