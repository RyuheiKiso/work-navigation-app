# 02 マスタメンテ APP 画面項目定義（11 画面）

本章は SCR-MA-001〜011 の全 11 画面について、画面項目定義（項目 ID・型・必須・制約・バリデーション・対応 FR）を確定する。React 実装のコーディング直前仕様として機能する。

---

## 1. マスタメンテ APP 固有の項目型

| 型 | 説明 |
|---|---|
| TABLE | ページネーション付きデータグリッド |
| TREE | ツリー構造エディタ |
| DIFF | 差分表示（左右比較）|
| DROPZONE | ファイルドロップゾーン |
| DATE_PICKER | 日付選択ピッカー |

**本節で確定した方針**
- **SCR-MA-001〜011 の全 11 画面について、全項目（F1{SCR番号2桁}-NNN）・型・必須・制約・バリデーション・対応 FR・遷移 TRN を確定した。**
- **SCR-MA-004（SOP 編集）は CMP-MA-001（SopTreeEditor）・CMP-MA-002（DslConditionBuilder）を核として、Step ごとの JSON Logic DSL バリデーション（ホワイトリスト演算子・ネスト ≤ 5）を確定した。**
- **承認・廃止フロー（SCR-MA-008/011）は電子サイン（SIGN_PAD）と PIN の二要素を必須とし、FR-MA-009/012 および FR-AU-001 を担保することを確定した。**
- **SCR-MA-004（SOP 編集）は CMP-MA-005 SopFlowCanvas を中央ペイン DAG フローモードに統合し、Step-DAG の編集を CMP-MA-002 DslConditionBuilder と組み合わせて完結させることを確定した（FR-MA-016）。**
