# 04 入力バリデーション仕様

本章は全 35 画面の入力項目に適用するバリデーションルールを入力型別に確定し、ERR コードとの対応・TypeScript 実装関数・適用画面を明示する。実装者はここで確定したルールを変更なく実装しなければならない。

## 2. TypeScript バリデーション実装

### 2-1. 共通バリデーター群

```typescript
// packages/validators/src/index.ts
// 全 APP（ハンディ APP / マスタメンテ APP / 管理コンソール）が共有するバリデーター

/**
 * 文字列が空でないことを検証する。
 * 前後の空白を除去した後の長さが 0 より大きければ true を返す。
 */
export const notEmpty = (v: string): boolean =>
  v.trim().length > 0;

/**
 * 文字列の最大長を検証する。
 * @param n 最大文字数
 */
export const maxLength =
  (n: number) =>
  (v: string): boolean =>
    v.length <= n;

/**
 * 文字列の最小長を検証する。
 * @param n 最小文字数
 */
export const minLength =
  (n: number) =>
  (v: string): boolean =>
    v.length >= n;

/**
 * 数値が LSL 以上 USL 以下であることを検証する。
 * @param lsl 下限規格値（Lower Spec Limit）
 * @param usl 上限規格値（Upper Spec Limit）
 */
export const numericRange =
  (lsl: number, usl: number) =>
  (v: number): boolean =>
    v >= lsl && v <= usl;

/**
 * PIN 形式（4〜8 桁の数字のみ）を検証する。
 */
export const pinFormat = (v: string): boolean =>
  /^\d{4,8}$/.test(v);

/**
 * GS1 アプリケーション識別子形式を検証する。
 * AI (01) から始まる 14 桁 GTIN パターンを必須とする。
 */
export const gs1Format = (v: string): boolean =>
  /^\(01\)\d{14}/.test(v);

/**
 * 未来日（今日以降）であることを検証する。
 * 比較は日付部分のみ（時刻は無視）。
 */
export const futureDate = (v: Date): boolean => {
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  return v >= today;
};

/**
 * 写真解像度が最低要件（1280×720 px）を満たすことを検証する。
 * @param w 幅（px）
 * @param h 高さ（px）
 */
export const photoResolution = (w: number, h: number): boolean =>
  w >= 1280 && h >= 720;

/**
 * プロセスコード形式（英数字・アンダースコアのみ）を検証する。
 */
export const processCodeFormat = (v: string): boolean =>
  /^[A-Za-z0-9_]+$/.test(v);

/**
 * UUID 形式（RFC 4122）を検証する。
 */
export const uuidFormat = (v: string): boolean =>
  /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i.test(v);

/**
 * UCUM コード形式を検証する。
 * 英数字・記号（/、[、]、{、}、.）の組み合わせを許容する。
 */
export const ucumCodeFormat = (v: string): boolean =>
  /^[A-Za-z0-9/\[\]{}.]+$/.test(v) && v.length >= 1 && v.length <= 64;

/**
 * インポートファイルの拡張子を検証する。
 * @param allowedExts 許容拡張子リスト（例: ['csv', 'xlsx']）
 */
export const fileExtension =
  (allowedExts: string[]) =>
  (filename: string): boolean => {
    const ext = filename.split('.').pop()?.toLowerCase() ?? '';
    return allowedExts.includes(ext);
  };

/**
 * 日付範囲の最大日数を検証する。
 * @param maxDays 最大日数
 */
export const maxDateRange =
  (maxDays: number) =>
  (start: Date, end: Date): boolean => {
    const diffMs = end.getTime() - start.getTime();
    const diffDays = diffMs / (1000 * 60 * 60 * 24);
    return diffDays >= 0 && diffDays <= maxDays;
  };
```

### 2-2. JSON Logic DSL バリデーター

```typescript
// packages/validators/src/jsonLogicValidator.ts

/** JSON Logic ホワイトリスト演算子 */
export const JSON_LOGIC_ALLOWED_OPERATORS = new Set([
  '==', '===', '!=', '!==',
  '<', '<=', '>', '>=',
  '!', '!!',
  'and', 'or',
  'if',
  'var',
  'in',
  'cat',
  'substr',
  '+', '-', '*', '/', '%',
  'min', 'max',
  'merge',
  'all', 'none', 'some', 'filter', 'map', 'reduce',
  'missing', 'missing_some',
]) as ReadonlySet<string>;

/** JSON Logic DSL の最大ネスト深度 */
export const JSON_LOGIC_MAX_NEST = 5;

/**
 * JSON Logic DSL の仕様適合性を検証する。
 * ホワイトリスト外の演算子・最大ネスト深度超過を検出した場合 false を返す。
 * @param rule JSON Logic ルールオブジェクト
 * @param depth 現在のネスト深度（再帰用・初期値 0）
 */
export const validateJsonLogicDsl = (
  rule: unknown,
  depth: number = 0,
): boolean => {
  if (depth > JSON_LOGIC_MAX_NEST) return false;
  if (typeof rule !== 'object' || rule === null) return true;
  if (Array.isArray(rule)) {
    return rule.every((item) => validateJsonLogicDsl(item, depth + 1));
  }
  const keys = Object.keys(rule as Record<string, unknown>);
  if (keys.length !== 1) return false;
  const operator = keys[0];
  if (!JSON_LOGIC_ALLOWED_OPERATORS.has(operator)) return false;
  const operands = (rule as Record<string, unknown>)[operator];
  return validateJsonLogicDsl(operands, depth + 1);
};
```

### 2-3. フォームバリデーション統合（React Hook Form 用）

```typescript
// packages/validators/src/formRules.ts

export type ValidationRule<T> = {
  validate: (value: T) => boolean;
  errCode: string;
  message: string;
};

/** ログイン画面（SCR-HA-001）バリデーションルール */
export const loginFormRules = {
  userId: [
    {
      validate: notEmpty,
      errCode: 'ERR-AUTH-001',
      message: 'ユーザー ID を入力してください',
    },
    {
      validate: maxLength(128),
      errCode: 'ERR-AUTH-001',
      message: 'ユーザー ID は 128 文字以内で入力してください',
    },
  ],
  password: [
    {
      validate: notEmpty,
      errCode: 'ERR-AUTH-001',
      message: 'パスワードを入力してください',
    },
    {
      validate: minLength(8),
      errCode: 'ERR-AUTH-001',
      message: 'パスワードは 8 文字以上で入力してください',
    },
    {
      validate: maxLength(64),
      errCode: 'ERR-AUTH-001',
      message: 'パスワードは 64 文字以内で入力してください',
    },
  ],
} as const;

/** 電子サイン PIN（SCR-HA-010 / SCR-MA-008）バリデーションルール */
export const pinFormRules = {
  pin: [
    {
      validate: notEmpty,
      errCode: 'ERR-AUTH-002',

    },

      validate: pinFormat,

      message: 'PIN は 4〜8 桁の数字で入力してください',
    },
  ],

  ],
```

## 3. ERR コード対応表

| ERR コード | 説明 | 発生条件 | 適用バリデーター |
|---|---|---|---|
| ERR-AUTH-001 | 認証失敗 | ユーザー ID / パスワード不正、未入力 | `notEmpty`, `minLength(8)`, `maxLength(128)` |
| ERR-AUTH-002 | PIN 不一致 | PIN 形式不正、照合失敗 | `pinFormat` |
| ERR-AUTH-003 | アカウントロック | 認証失敗累積（ロック閾値超過）| — |
| ERR-VAL-002 | 数値範囲逸脱 | 測定値が LSL 未満または USL 超 | `numericRange(lsl, usl)` |
| ERR-VAL-003 | 形式不正 | GS1 / UCUM / 写真解像度 / 日付 / JSON Logic DSL / ファイル形式 / 期間 | 各 format バリデーター |
| ERR-VAL-004 | 必須未入力 / 業務ルール違反 | NOT NULL 違反、コード重複、選択必須未選択 | `notEmpty`, `processCodeFormat` |

## 4. バリデーションタイミング

| タイミング | 適用ルール |
|---|---|
| onChange（入力中）| maxLength・numericRange（リアルタイムフィードバック）|
| onBlur（フォーカスアウト）| 全ルール（エラーメッセージ表示）|
| onSubmit（送信時）| 全ルール（サーバー送信前の最終チェック）|
| サーバー応答後 | ERR-AUTH-001〜003 / ERR-BIZ-NNN（サーバー側検証結果表示）|

## 5. マルチ言語エラーメッセージ

バリデーションエラーメッセージは FR-UI-001（多言語対応）に従い日本語・英語・中国語の 3 言語を用意する。

| ERR コード | 日本語 | 英語 | 中国語 |
|---|---|---|---|
| ERR-AUTH-001 | ユーザー ID またはパスワードが正しくありません | Invalid user ID or password | 用户ID或密码不正确 |
| ERR-AUTH-002 | PIN が正しくありません | Incorrect PIN | PIN不正确 |
| ERR-AUTH-003 | アカウントがロックされています。管理者に連絡してください | Account is locked. Contact administrator | 账户已锁定，请联系管理员 |
| ERR-VAL-002 | 測定値が規格範囲外です（LSL: {lsl}, USL: {usl}）| Measured value out of spec (LSL: {lsl}, USL: {usl}) | 测量值超出规格范围（LSL: {lsl}, USL: {usl}） |
| ERR-VAL-003 | 入力形式が正しくありません | Invalid format | 输入格式不正确 |
| ERR-VAL-004 | 必須項目が入力されていません | Required field is empty | 必填项未填写 |

---

**本節で確定した方針**
- **全 35 画面の入力型を 15 バリデーション種別に分類し、各種別の ERR コード・TypeScript 実装関数・適用画面を確定した。**
- **JSON Logic DSL バリデーターはホワイトリスト演算子（39 種）とネスト深度上限（5）を確定し、eval() 禁止原則（`04_概要設計/99_前提制約`）を UI 層で担保することを確定した。**
- **バリデーションエラーメッセージは日本語・英語・中国語の 3 言語を確定し、FR-UI-001 多言語要件をバリデーション層でも充足することを確定した。**
