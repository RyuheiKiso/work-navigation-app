# 08 共通ライブラリ詳細設計（MOD-SH-001〜004）

本章は共通ライブラリ 4 種（LocaleResolver・IdGenerator・ClockService・ApiClient）の詳細設計を確定する。これらはバックエンド・フロントエンドを問わず全モジュールから利用される横断的関心事であり、テスト容易性と拡張性を最優先に設計する。

## 1. MOD-SH-001: LocaleResolver（i18n バックエンド統合）

バックエンドのエラーメッセージ多言語化を担当する。`rust-i18n` crate を使用し、翻訳ファイル（TOML）を `locales/` ディレクトリで管理する。

```rust
// shared/locale_resolver/src/lib.rs

/// バックエンドの翻訳設定を初期化する。
/// main.rs の起動処理の最初に呼び出す。
///
/// locales/
///   ja.toml   — 日本語（必須、100%翻訳率）
///   en.toml   — 英語（必須、100%翻訳率）
///   ja-simple.toml — やさしい日本語（JLPT N4 相当、FR-UI-002）
pub fn init_locale() {
    rust_i18n::i18n!("locales");
}

/// リクエストの Accept-Language ヘッダからロケールを解決する。
/// 対応言語: ja / en / ja-simple
/// 対応言語がない場合は "ja" にフォールバックする。
pub fn resolve_locale(accept_language: Option<&str>) -> &'static str {
    match accept_language {
        Some(lang) if lang.contains("en") => "en",
        Some(lang) if lang.contains("ja-simple") => "ja-simple",
        _ => "ja",
    }
}

/// 翻訳キーからメッセージを取得する。
/// キー形式: `{feature}.{component}.{key}`
/// 例: `error.biz.lock_step_violation`
pub fn t(key: &str, locale: &str) -> String {
    rust_i18n::t!(key, locale = locale).to_string()
}
```

## 2. MOD-SH-002: IdGenerator（UUID v7 生成）

```rust
// shared/id_generator/src/lib.rs

use uuid::Uuid;

/// UUID v7 を生成する。
/// UUID v7 はタイムスタンプ（ミリ秒精度）を先頭に埋め込むため、
/// ORDER BY id の性能がバージョン 4 より高い。
///
/// すべての主キー生成はこの関数経由で行う。
/// DB の DEFAULT 句による UUID 生成は使用しない（バージョン統一のため）。
pub fn new_id() -> Uuid {
    Uuid::now_v7()
}

/// Idempotency Key を生成する。
/// event_id と同一の UUID v7 を使用することで、
/// WorkEvent.event_id と outbox_events.idempotency_key が一致する。
pub fn new_idempotency_key() -> Uuid {
    Uuid::now_v7()
}

/// UUID を文字列で返す（API レスポンス等に使用）
pub fn id_to_string(id: &Uuid) -> String {
    id.to_string()
}

/// 文字列から UUID をパースする（API 入力値の検証）
pub fn parse_id(s: &str) -> Result<Uuid, uuid::Error> {
    Uuid::parse_str(s)
}
```

## 3. MOD-SH-003: ClockService（時刻抽象）

テスト時に時刻を固定できるように、時刻取得を trait で抽象化する。この設計により、日付境界・タイムアウト・TTL 検証のテストが決定論的に行える。

```rust
// shared/clock_service/src/lib.rs

use chrono::{DateTime, Utc};
use std::sync::Arc;

/// 時刻取得の抽象 Trait。
/// テスト時に FakeClock（固定時刻）に差し替えられる。
pub trait Clock: Send + Sync {
    /// 現在時刻を UTC で返す。
    fn now(&self) -> DateTime<Utc>;
}

/// 本番用: システム時刻を返す。
pub struct SystemClock;

impl Clock for SystemClock {
    fn now(&self) -> DateTime<Utc> {
        Utc::now()
    }
}

/// テスト用: 固定時刻を返す。
/// `Arc<Mutex<DateTime<Utc>>>` で保持するため、テスト中に時刻を更新できる。
pub struct FakeClock {
    fixed_time: std::sync::Mutex<DateTime<Utc>>,
}

impl FakeClock {
    pub fn new(t: DateTime<Utc>) -> Arc<Self> {
        Arc::new(Self {
            fixed_time: std::sync::Mutex::new(t),
        })
    }

    /// テスト中に時刻を進める。
    pub fn advance(&self, duration: chrono::Duration) {
        let mut t = self.fixed_time.lock().unwrap();
        *t = *t + duration;
    }

    /// テスト中に時刻をセットする。
    pub fn set(&self, t: DateTime<Utc>) {
        *self.fixed_time.lock().unwrap() = t;
    }
}

impl Clock for FakeClock {
    fn now(&self) -> DateTime<Utc> {
        *self.fixed_time.lock().unwrap()
    }
}

/// アプリケーションへの便利な注入用エイリアス
pub type ClockRef = Arc<dyn Clock>;

/// 本番用 ClockRef を生成する。
pub fn system_clock() -> ClockRef {
    Arc::new(SystemClock)
}
```

## 4. MOD-SH-004: ApiClient（OpenAPI 生成クライアント）

フロントエンド用の API クライアントは OpenAPI Generator（`@openapitools/openapi-generator-cli`）で自動生成する。バックエンド向けの内部クライアントは Rust の `reqwest` ベースで実装し、親機 API との通信に使用する。

### 4-1. フロントエンド向け自動生成クライアント

```
# 生成コマンド（CI/CD パイプラインで実行）
openapi-generator-cli generate \
  -i docs/05_詳細設計/03_API詳細設計/openapi.yaml \
  -g typescript-axios \
  -o src/shared/api/generated \
  --additional-properties=supportsES6=true,withSeparateModelsAndApi=true
```

### 4-2. Rust 内部 ApiClient（親機連携）

```rust
// shared/api_client/src/lib.rs

use reqwest::Client;
use serde::{Deserialize, Serialize};

/// 親機 API クライアント（IF-002 アウトバウンド）
pub struct ParentSystemApiClient {
    client: Client,
    base_url: String,
    /// OAuth 2.1 Client Credentials トークン（Bearer）
    access_token: tokio::sync::RwLock<Option<String>>,
    client_id: String,
    client_secret: String,
    token_endpoint: String,
}

impl ParentSystemApiClient {
    pub fn new(config: &ParentApiConfig) -> Self {
        let client = Client::builder()
            .timeout(std::time::Duration::from_secs(config.timeout_secs))
            .build()
            .expect("HTTP client build failed");

        Self {
            client,
            base_url: config.base_url.clone(),
            access_token: tokio::sync::RwLock::new(None),
            client_id: config.client_id.clone(),
            client_secret: config.client_secret.clone(),
            token_endpoint: config.token_endpoint.clone(),
        }
    }

    /// OAuth 2.1 Client Credentials でアクセストークンを取得・キャッシュする。
    /// トークンが期限切れの場合は自動更新する。
    async fn ensure_token(&self) -> Result<String, ApiClientError> {
        let token = self.access_token.read().await.clone();
        if let Some(t) = token {
            return Ok(t);
        }

        // Token Endpoint にリクエスト
        let response = self.client
            .post(&self.token_endpoint)
            .form(&[
                ("grant_type", "client_credentials"),
                ("scope", "wnav.outbox.write"),
                ("client_id", &self.client_id),
                ("client_secret", &self.client_secret),
            ])
            .send()
            .await
            .map_err(|e| ApiClientError::Http(e.to_string()))?;

        let token_response: TokenResponse = response.json().await
            .map_err(|e| ApiClientError::Deserialize(e.to_string()))?;

        let new_token = token_response.access_token.clone();
        *self.access_token.write().await = Some(new_token.clone());
        Ok(new_token)
    }

    /// アウトバウンド実績データを親機 Inbound API に送信する。
    pub async fn post_outbox_inbound(
        &self,
        payload: &serde_json::Value,
        idempotency_key: &str,
    ) -> Result<(), ApiClientError> {
        let token = self.ensure_token().await?;

        self.client
            .post(format!("{}/api/v1/sync/outbox/inbound", self.base_url))
            .bearer_auth(&token)
            .header("Idempotency-Key", idempotency_key)
            .json(payload)
            .send()
            .await
            .map_err(|e| ApiClientError::Http(e.to_string()))?;

        Ok(())
    }
}

#[derive(Debug, Deserialize)]
struct TokenResponse {
    access_token: String,
    #[allow(dead_code)]
    token_type: String,
}

#[derive(Debug, Clone, serde::Deserialize)]
pub struct ParentApiConfig {
    pub base_url: String,
    pub client_id: String,
    pub client_secret: String,
    pub token_endpoint: String,
    pub timeout_secs: u64,
}

#[derive(Debug, thiserror::Error)]
pub enum ApiClientError {
    #[error("HTTP error: {0}")]
    Http(String),
    #[error("Deserialize error: {0}")]
    Deserialize(String),
}
```

---

**本節で確定した方針**
- **LocaleResolver は `rust-i18n` crate で `ja`・`en`・`ja-simple` の 3 言語を提供し、Accept-Language ヘッダからロケールを解決して全エラーメッセージを多言語化する設計を確定した。**
- **ClockService は `trait Clock` として抽象化し、本番では SystemClock・テストでは FakeClock に DI することで時刻依存テストを決定論的に実行できる設計を確定した。**
- **ApiClient（MOD-SH-004）は OpenAPI Generator で TypeScript クライアントを自動生成し、Rust 側の親機連携は OAuth 2.1 Client Credentials フローで認証する設計を確定した。**
