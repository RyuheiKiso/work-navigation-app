# 02 詳細設計トレーサビリティマトリクス

本章は、要件定義で確定した全 86 件の機能要件（FR）に対して、05_詳細設計フェーズで採番した TST-NNN テストケースが少なくとも 1 件割り付けられていることを確認するトレーサビリティマトリクスである。

---

**本節で確定した方針**
- **全 87 FR に対して TST-NNN ≥ 1 件の割付けが完了していることを本マトリクスで確認し、05_詳細設計フェーズのテストカバレッジ完成条件を充足した。FR-SY-011（Case 端末排他占有）は 2026-05-18 追加。TST-intg-021〜023（型ラウンドトリップテスト）は PG↔SQLite スキーマ同期戦略対応として追加。**
- **NFR（ALCOA+ 9 属性・RBAC・性能・可用性）についても TST-alcoa / TST-sec / TST-perf の専用テストケースで 1:1 対応を確立し、非機能要件のトレーサビリティを確保した。**
- **本マトリクスは `08_テストケース詳細設計/00_本書の位置づけと識別子規約.md` §4 のカバレッジマトリクスと対をなし、付録として採番台帳と併せて参照する位置づけとする。**
