# 00 本書の位置づけと識別子規約

本章は IPA 共通フレーム 2013「2.5.2 ソフトウェアコンポーネントインターフェースの詳細設計」における本ディレクトリの責務範囲・識別子体系・39 エンドポイントの網羅表を確定する。

---

## 1. IPA 2.5.2 における位置づけ

IPA 2.5.2 タスクは「コンポーネント間のインターフェースを詳細化し、入力・出力・エラー処理・セキュリティ要件をコーディングが可能な精度で記述すること」を要求する。本ディレクトリはこの要求を REST API という形式で満たす。

| IPA 2.5.2 要求 | 本ディレクトリでの実現方式 |
|---|---|
| 入力インターフェース仕様 | 各章の requestBody JSON Schema（全フィールド・型・制約）|
| 出力インターフェース仕様 | 各章の responses JSON Schema（全フィールド・型）|
| エラー・例外処理の仕様 | `08_エラーコード完全定義.md`（26 コード × HTTP Status / log_level / retry）|
| セキュリティ要件 | `02_認証・認可API仕様.md`（JWT RS256 / RBAC 6 ロール）|
| 共通規約 | `01_OpenAPI共通仕様と設計原則.md`（Base URL / ヘッダ / エンベロープ）|

---

## 2. バイナリ分割方針

バックエンドは **`wnav_terminal_api`（ポート 8080）** と **`wnav_master_api`（ポート 8081）** の 2 バイナリで構成される。各 API-NNN には担当バイナリが割り当てられており、全エンドポイント網羅表（§ 4）の「担当バイナリ」列でこれを明示する。

| バイナリ | ポート | 役割 |
|---|---|---|
| `wnav_terminal_api` | 8080 | 現場端末からの書き込み・実施系（作業実行・エビデンス・電子サイン・アンドン発報・Kaizen 起票・IQC 検査実施・リワーク作業実施など）|
| `wnav_master_api` | 8081 | 管理系・承認系・閲覧系（マスタ管理・CAPA・Kaizen 管理・レポート・トレーサビリティ・DLQ 監視など）|
| 両バイナリ | 8080 + 8081 | 認証エンドポイント（`/api/v1/auth/*`）・ヘルスチェック（`/api/v1/healthz`, `/api/v1/readyz`）・OpenAPI 仕様（各バイナリが独立して公開）|

## 3. 識別子規約

### 3-1. API 識別子体系

`04_概要設計/付録/99_設計識別子採番台帳.md` § 4 で採番された識別子を本ディレクトリはそのまま継承する。形式を以下に示す。

```
API-{resource}-{NNN}
```

| フィールド | 規則 |
|---|---|
| `API-` | 固定プレフィックス |
| `{resource}` | エンドポイントのリソース名（英小文字・ハイフン区切り）|
| `{NNN}` | 3 桁連番（001 始まり、resource 別に独立採番）|

例: `API-auth-001`（POST /api/v1/auth/login）、`API-step-events-001`（POST /api/v1/work-executions/{id}/events）

### 3-2. 本ディレクトリでの新規採番

本ディレクトリは API-NNN を新規採番しない。上流台帳が権威である。

エラーコード（ERR-NNN）も同様に `04_概要設計/付録/99_設計識別子採番台帳.md` § 10 の体系を継承し、詳細記述のみ本書で行う。

## 5. 04_概要設計との差分整合

本ディレクトリで指示書が求める URL と採番台帳の URL が一部異なる。以下に整合ルールを示す。

| 指示書 URL | 採番台帳 URL | 本書での扱い |
|---|---|---|
| GET /api/v1/master/processes | （個別採番なし）| API-master 章で補足仕様として記述 |
| GET /api/v1/master/sops | （個別採番なし）| API-master 章で補足仕様として記述 |
| POST /api/v1/master/sops/{id}/versions | API-master-002 に相当 | 採番台帳識別子を優先 |
| GET /api/v1/system/healthz | /api/v1/healthz | 採番台帳識別子（API-system-001）を優先 |
| POST /api/v1/nonconformities | API-andon 章関連 | 06 章で補足仕様として記述 |

上流採番台帳が権威であるため、本書は採番台帳の URL を正式仕様とし、指示書との差異を各章で注記する。

---

**本節で確定した方針**
- **本ディレクトリは IPA 2.5.2 を担当し、39 エンドポイントの入出力・エラー・セキュリティ仕様をコーディング直前精度で記述することを責務として確定した。**
- **バックエンドは `wnav_terminal_api`（ポート 8080）と `wnav_master_api`（ポート 8081）の 2 バイナリに分割される。各エンドポイントの担当バイナリは § 4 の網羅表で明示する。**
- **API 識別子は `API-{resource}-{NNN}` 形式で `04_概要設計/付録/99_設計識別子採番台帳.md` から継承し、本書では新規採番を行わない。**
- **採番台帳 URL が指示書 URL と差異がある場合は採番台帳を権威とし、差異は各章の注記で明示する。**
