# 99 前提制約と本書が約束しないこと

本章は `03_API詳細設計/` ディレクトリ全体の前提制約・有効範囲・明示的な非カバー範囲を確定する。本ディレクトリを参照する実装者・レビュアーが本書の限界を正確に把握することを目的とする。

## 3. 本書が約束しないこと

### 3-1. 実装コード

本書はインターフェース仕様を記述するものであり、以下は対象外とする。

- Rust の実装コード（`crates/` / `src/` 配下のソースファイル）の作成。実装は 06_実装フェーズの責務。
- テストコードの作成。テストケース設計は `08_テストケース詳細設計/` の責務。
- OpenAPI YAML / JSON ファイルの手動作成。utoipa による自動生成が正式手段であり、手書き YAML は保守しない。

### 3-2. データベース詳細設計

SQL DDL の全文（CREATE TABLE・INDEX・VIEW）は `01_データベース詳細設計/` が担当する。本書はテーブル名・列名を参照するが DDL を定義しない。

### 3-3. 画面・フロントエンド仕様

TypeScript の型定義・React Native コンポーネント仕様は `04_ハンディAPP詳細設計/` および `05_WebAPP詳細設計/` が担当する。本書が定義する JSON スキーマはそれらの入力として使用される。

### 3-4. 非機能要件の実装詳細

| 非機能要件 | 担当文書 |
|---|---|
| レート制限の実装方式（トークンバケット実装）| `02_バックエンド詳細設計/` |
| TLS 証明書の管理手順 | `08_運用方式設計/` |
| PostgreSQL のパフォーマンスチューニング | `01_データベース詳細設計/` |
| ログの集約・保管基盤 | `08_運用方式設計/` |

### 3-5. 採番台帳未収録エンドポイントの正式化

本書で補足仕様として記述した以下のエンドポイントは `04_概要設計/付録/99_設計識別子採番台帳.md` に未収録である。実装前に採番台帳を更新して正式採番を取得すること。

### 3-6. ver1.0.0 後の変更

本書は ver1.0.0 リリース時点の仕様を確定する。以下は ver1.0.0 リリース後の判断事項であり、本書では定義しない。

- API v2 移行の判断（Breaking Change が発生した時点で別途設計）。
- 旧バージョン（v1）の deprecated 期間終了後の削除手順。
- 新規エンドポイントの追加（採番台帳への追記から開始）。
