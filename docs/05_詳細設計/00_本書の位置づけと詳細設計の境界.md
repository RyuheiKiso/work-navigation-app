# 00 本書の位置づけと詳細設計の境界

## 1. IPA 共通フレーム 2013 における位置づけ

本書（`05_詳細設計/`）は IPA 共通フレーム 2013「**2.5 ソフトウェア詳細設計プロセス**」の全タスクを担当する。

| IPA 2.5 タスク | 本書での担当サブ |
|---|---|
| 2.5.1 ソフトウェアコンポーネントの詳細設計 | 02_バックエンド（wnav_terminal_api + wnav_master_api の2バイナリ構成）/ 04_ハンディAPP / 05_WebAPP / 06_画面 / 07_アルゴリズム |
| 2.5.2 ソフトウェアコンポーネントインターフェースの詳細設計 | 03_API詳細設計 |
| 2.5.3 ソフトウェアコンポーネントデータの詳細設計 | 01_データベース詳細設計 |
| 2.5.4 要件の評価（テストケース設計）| 08_テストケース詳細設計 |

5 層開発モデルにおける本書の位置:

| 層 | 文書 | 問い | 内容 |

| 第 1 層 | 02_企画（構想）| なぜ | 価値命題・倫理スタンス |
| 第 2 層 | 02_企画（計画）| 何を・どう-骨格 | アーキテクチャ原則・技術スタック |

| **第 5 層（本書）** | **05_詳細設計** | **どう-詳細** | **コーディング直前仕様** |

## 3. 本書の境界（04 → 本書 → 06_実装）

### 3-1. 本書が確定すること

- 全 35 テーブルの CREATE TABLE 全文（DDL-NNN）
- 全インデックス仕様（IDX-NNN）
- 全ビュー定義（VW-NNN）
- JSONB キースキーマ（JSON Schema 形式）
- Rust: 各クレートの公開 struct・trait・fn シグネチャ（引数・戻り値・ライフタイム）
- TypeScript: 各コンポーネントの Props/State/Event 型定義
- OpenAPI 3.1: 全 39 エンドポイントの requestBody・responses・parameters 全フィールド
- アルゴリズム擬似コード（StepEngine・JSON Logic・ハッシュチェーン・Outbox）
- テストケース一覧（TST-NNN × FR/NFR/BR-BUS マッピング）

### 3-2. 本書が約束しないこと

- 実装ファイル（`crates/`, `apps/`, `src/` 配下）の作成 → 06_実装フェーズ
- 実際のデータ入力・テスト実行 → 07_テスト実施フェーズ
- 運用手順書の詳細 → `08_運用方式設計` 配下で完結済み
- OpenAPI YAML ファイルの生成 → utoipa による自動生成（`02_バックエンド詳細設計/01_wnav_terminal_api詳細設計.md` および `02_バックエンド詳細設計/02_wnav_master_api詳細設計.md` §7 参照）

## 4. 完成判定（妥協禁止の操作的定義）

| 確認項目 | 判定基準 |
|---|---|
| md ファイル | 全 98 件が章末「本節で確定した方針」「## 参照業界分析」を含む |
| DDL | 全 35 TBL が CREATE TABLE 全文で記述済み |
| IDX | 全インデックスが IDX-NNN で採番済み |
| VW | 全ビューが VW-NNN で採番済み |
| TST | 全 86 FR に対して TST-NNN が ≥1 件存在 |
| 図 | drawio-lint / svg-postcheck ERROR ゼロ |

---

**本節で確定した方針**
- **本書（05_詳細設計）は IPA 2.5 の全タスク（2.5.1〜2.5.4）を担当し、04_概要設計の骨格をコーディング直前仕様まで精緻化することを責務とする。**
- **SQL DDL 全文・Rust 型シグネチャ・TypeScript interface・OpenAPI 全フィールドをコードブロックで記載する「設計仕様コード含む」方針を確定した。**
- **04_概要設計が確定した MOD/TBL/API/FR/BR-BUS の体系は変更せず、本書は精緻化のみを担う。**
- **バックエンドは wnav_terminal_api（ポート 8080）と wnav_master_api（ポート 8081）の2バイナリ構成であることを本書の前提として確定した。ハンディ APP（OutboxWorker）は terminal-api にのみ通信し、各バッチジョブは2バイナリのいずれかに帰属する常駐 tokio task として動作する。**
