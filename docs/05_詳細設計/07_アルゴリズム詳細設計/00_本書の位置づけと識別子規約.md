# 00 本書の位置づけと識別子規約

本章は `05_詳細設計/07_アルゴリズム詳細設計` サブの位置づけ・IPA 2.5.1 タスクとの対応・FNC-FE-NNN 識別子採番規約・アルゴリズムカタログを確定する。

---

## 1. IPA 2.5.1 タスクカバレッジ

IPA 共通フレーム 2013「2.5.1 ソフトウェアコンポーネント詳細設計」が要求する「処理ロジック（アルゴリズム）の仕様化」タスクと、本サブの各章との対応を以下に示す。

| IPA 2.5.1 要求タスク | 担当章 | カバーする識別子 |
|---|---|---|
| ステップ進行制御ロジックの詳細設計 | `01_StepEngine` | FNC-FE-001〜004, FNC-BE-001〜005 |
| 条件分岐評価ロジックの詳細設計 | `02_JSON_Logic` | FNC-FE-005〜008 |
| 完全性・改ざん検知ロジックの詳細設計 | `03_ハッシュチェーン` | FNC-BE-009〜011 |
| 非同期メッセージ配信の詳細設計 | `04_Outbox` | FNC-BE-012, FNC-FE-009〜010 |
| 接続断・復旧制御ロジックの詳細設計 | `05_オフライン状態管理` | FNC-FE-011〜013 |
| バッチスケジュール・冪等性の詳細設計 | `06_バッチジョブ` | BAT-001〜010 |
| 帳票生成パイプラインの詳細設計 | `07_帳票生成` | RP-001/003/004/005/006 |

本サブは `02_バックエンド詳細設計`・`04_ハンディAPP詳細設計` の各クレート/モジュール設計で確定したシグネチャ（FNC-NNN）を引き継ぎ、処理ロジックの内部動作を擬似コードおよびコードブロックで確定する。

## 4. 上流設計との対応

本サブは以下の上流ドキュメントを変更しない。

| 上流ドキュメント | 本サブへの制約 |
|---|---|
| `04_概要設計/02_ソフトウェア方式設計/11_モジュール一覧.md` | MOD-FE-HA-002（StepEngine）・MOD-BE-002（wnav_domain）の責務は変更不可 |
| `03_要件定義/機能要件/10_業務ルール.md` | BR-BUS-001/003/022/024/029/040 は変更不可 |
| `03_要件定義/機能要件/` | FR-NV/EV/SY/OF/BS/RP 全件の要求は変更不可 |
| `04_概要設計/02_ソフトウェア方式設計/07_例外・エラーハンドリング統一方式.md` | ERR-NNN 体系は変更不可 |

---

**本節で確定した方針**
- **IPA 2.5.1 の処理ロジック仕様化タスクを本サブ 7 章（01〜07）に割付け、ALG-001〜024 の 24 アルゴリズムを全てカバーすることを確定した。**
- **FNC-FE-001〜013 および FNC-BE-001〜012 をアルゴリズム担当関数として対応付け、本サブは新規採番を行わず参照のみとすることを確定した。**
- **上流設計（MOD-NNN・BR-BUS-NNN・FR-NNN・ERR-NNN）の固定を宣言し、本サブが独断で変更しないことを確定した。**
