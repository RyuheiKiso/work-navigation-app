# 業務要件サブ

## 1. 本サブの責務

業務要件サブは IPA 共通フレーム 2013 要件定義プロセスの「業務要件の定義」アクティビティに相当する。

本サブが確定する内容は以下の 3 点に限定する。

- **業務フロー**: 工程/作業/Step の 3 階層における現場の業務の流れ
- **業務要件（BR）**: 「なぜその機能が必要か」の業務的根拠。機能要件（FR）の前提となる
- **業務ドメイン用語集**: 製造現場固有の業務用語・略語の一元定義

業務要件サブは「業務が何を達成したいか」を記述する。「システムがどう実装するか」は機能要件サブの責務であり、本サブでは扱わない。

### 機能要件サブとの分担境界

| 業務要件サブが扱う | 機能要件サブが扱う |
|---|---|
| 現場の業務フロー（工程・作業・Step の流れ） | システムの機能の検証可能命題 |
| 業務上の目的・課題・価値（BR の Why） | 機能の受入基準・優先度（FR の What） |
| ユースケース記述（アクター・事前条件・事後条件） | 機能の実装レベルの詳細 |
| 業務ドメイン用語の定義 | プロジェクト共通用語の定義（`02_用語集・略語・参照規約.md` が担当） |

---

**本節で確定した方針**
- 業務要件サブは「なぜ必要か（業務的根拠）」と「どのような業務フローか」を確定し、「どう実装するか」は機能要件サブに委任することを確定する。
- 業務ドメイン用語は `03_業務ドメイン用語集.md` に一元管理し、他サブは本ファイルを参照することを確定する。
- 業務要件 ID は `BR-BUS-xxx`（業務フロー系）と `BR-SC-xxx`（シナリオ系）の 2 系統で採番することを確定する。
