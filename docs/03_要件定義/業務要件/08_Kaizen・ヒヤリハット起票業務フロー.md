# 08 Kaizen・ヒヤリハット起票業務フロー

本章の責務は、作業員による改善提案（Kaizen Teian）および危険発見（ヒヤリハット）の起票から採否決定・SOP 改訂連動までの業務フロー（BF-04）を確定することである。Just Culture 保護原則の業務手続・匿名性の担保・用途三限定の運用規約を合わせて定義する。

## 1. 起票主体と保護原則

### 1-1. 起票主体の確定

Kaizen Teian・ヒヤリハットの起票主体は以下のとおり確定する。

| 起票種別 | 主な起票主体 | 副次的起票主体 |
|---|---|---|
| Kaizen Teian（改善提案） | 作業員（SH-01） | 現場監督（SH-02）・品質担当（SH-03） |
| ヒヤリハット（危険発見・未遂事象） | 作業員（SH-01） | 現場監督（SH-02） |

作業員が最も多くの業務知識（現場で実際に発生することの実態）を持っており、起票の主体として設計する。現場監督・品質担当も起票権を持つが、起票を現場監督・品質担当に委ねる運用は禁止する。

### 1-2. Just Culture 保護の業務手続

Just Culture の原則に基づき、ヒヤリハット起票・Kaizen Teian の起票が「報告することで不利益を受けない」ことを制度・技術の両面で保証する業務手続を確定する。

**制度的保護**

- 就業規則・労使協定に「ヒヤリハット起票を理由とする不利益処分の禁止」を明記する（導入先企業の義務として運用方針に含める）
- ヒヤリハット件数の増加を「安全文化の改善指標」として月次経営会議で報告する
- 件数目標（「毎月 N 件以上起票する」等）の設定を業務運用として禁止する

**技術的保護**

- ヒヤリハット起票の匿名化オプションを提供する（後述 4 節参照）
- 起票者 ID は品質担当のみが RBAC でアクセス可能とし、現場監督・経営層の直接閲覧を禁止する
- 個人別の起票件数ランキング・達成率の表示を管理 Web から削除する

### 1-3. 起票操作の業務手順

作業員がタブレット APP から起票する業務手順を確定する。

1. SOP 実行中または待機中にタブレット APP の「起票」ボタンを押下する
2. 起票種別（Kaizen Teian / ヒヤリハット）を選択する
3. 内容を入力する（テキスト・写真・定型タグの組み合わせ）
4. 匿名化オプションを選択する（任意）
5. 「送信」を押下する
6. 起票時刻・作業員 ID（匿名化設定時は非表示）・対象工程・SOP ステップ（任意）が自動付与される
7. 現場監督・品質担当に自動通知が送信される

**本節で確定した方針**
起票主体は作業員を主とし、現場監督・品質担当への委任を禁止する業務原則を確定する。
Just Culture 保護を制度的（就業規則）・技術的（RBAC・匿名化・ランキング削除）の両面で実装することを確定する。
起票者への不利益を防ぐ業務環境の整備を導入先企業の義務として運用方針に含めることを確定する。

## 2. 起票〜評価〜採否

### 2-1. 五段階フローの定義

Kaizen Teian・ヒヤリハットの処理フローを以下の五段階で確定する。

**第一段階: 起票（作業員）**

作業員が起票を完了する。起票レコードに以下の情報が記録される。

- 起票種別（kaizen_teian / near_miss）
- タイトル（任意・空欄の場合はシステムが「起票 YYYY-MM-DD HH:MM」を自動付与）
- 本文（必須・最大 1,000 文字）
- 添付写真（任意・複数枚可）
- 定型タグ（「手順に関する提案」「設備に関する提案」「安全に関する発見」等）
- 対象工程 ID（任意）
- 関連する SOP ステップ ID（任意）
- 匿名化フラグ（true の場合、品質担当以外には作業員 ID を非表示）
- 起票時刻・端末 ID

**第二段階: 現場監督スクリーニング（現場監督）**

起票から 3 営業日以内に現場監督がスクリーニングを行う。

- 内容が明確でない場合: 起票者に「補足情報の追記依頼」通知を送信する
- 重複・既知の課題の場合: 既存の CAPA/Kaizen レコードへの参照を付与し「統合」とする
- 緊急性が高い場合: 「緊急フラグ」を付与し品質担当への通知を即時送信する
- スクリーニング完了後、「品質担当評価待ち」状態に遷移させる

現場監督がスクリーニングを 5 営業日以内に実施しない場合、品質担当に「スクリーニング遅延通知」が送信される。

**第三段階: 品質担当評価（品質担当）**

品質担当が起票内容を評価する。

- 安全・品質・コスト・作業性の四軸で影響を評価する
- 既存の CAPA レコードとの関連性を確認する
- 実現可能性・優先度を評価し、採否判断の根拠を記録する

**第四段階: 採否決定（品質担当）**

品質担当が採否を決定する。

| 採否 | 条件 | 起票者への通知内容 |
|---|---|---|
| 採用 | 改善効果が認められ、実施が現実的な場合 | 採用通知（採用理由・次のアクション・担当者・期限） |
| 条件付き採用 | 内容の修正・追加情報の提供を条件に採用する場合 | 条件付き採用通知（必要な修正・追加情報の詳細） |
| 保留 | 他の改善活動との優先度調整が必要な場合 | 保留通知（保留理由・再評価時期の目安） |
| 不採用 | 技術的・業務的に実施不可能な場合 | 不採用通知（不採用理由の説明・代替対応の提案） |

いずれの採否決定においても、起票者へのフィードバック通知を必ず返信する。起票が黙殺されることを業務として禁止する。

**第五段階: フォローアップ（品質担当・現場監督）**

採用された提案の実施状況を追跡する。

- 「採用」から実施完了まで担当者・期限を CAPA レコードに記録する
- 実施完了後、起票者に「実施完了通知」を送信する
- ヒヤリハットの場合: 適切な予防措置が講じられたことを現場監督が確認・電子サインで記録する

**本節で確定した方針**
起票〜評価〜採否フローを五段階（起票・スクリーニング・評価・採否・フォロー）で確定する。
採否決定時の起票者へのフィードバック通知を必須とし、黙殺を業務として禁止する。
スクリーニング 5 営業日以内未実施の場合の遅延通知を業務ルールとして確定する。

## 3. マスタ改訂連動

### 3-1. Kaizen 採用から SOP 改訂への連動手続

採用された Kaizen Teian が SOP の改訂を必要とする場合、以下の連動手続を確定する。

1. 品質担当が「採用」操作を実施する際、「SOP 改訂連動」オプションを有効化する
2. 影響する SOP を選択する（一件または複数件）
3. システムが自動的に BF-02（マスタ管理業務フロー）を起動し、改訂提案レコードを生成する
4. 改訂提案レコードに Kaizen Teian レコード ID が参照リンクとして設定される
5. BF-02 の六段階フロー（改訂提案→影響分析→ドラフト→レビュー→承認→公開）が進行する
6. SOP 改訂版の公開完了後、Kaizen Teian レコードの状態が「実施完了」に更新される
7. 起票者（作業員）に「Kaizen 実施完了通知」が送信される

### 3-2. ヒヤリハットから予防措置への連動手続

採用されたヒヤリハット報告が予防措置（EWI 追加・SOP 改訂・設備改善等）を必要とする場合、以下の連動手続を確定する。

1. 品質担当が予防措置の種類を選択する（SOP 改訂 / 設備改善 / EWI 追加 / 教育）
2. 選択した措置種別に応じて関連するマスタ管理フロー（BF-02）または保全フローが起動される
3. 措置完了後、ヒヤリハットレコードに「予防措置完了」の電子サインが記録される

**本節で確定した方針**
採用 Kaizen から SOP 改訂への連動は BF-02 の自動起動と Kaizen レコード ID の参照リンクで管理することを確定する。
ヒヤリハットから予防措置への連動は措置種別選択→関連フロー起動→完了確認の三ステップで確定する。
連動フローの完了後に起票者への完了通知を必須とすることを確定する。

## 4. 匿名性の担保と用途三限定

### 4-1. 匿名化オプションの業務手続

起票者が匿名化オプションを選択した場合、以下の業務手続を確定する。

- 品質担当のみが起票者 ID を参照できる（RBAC による技術的制限）
- 現場監督・経営層・IT 担当は起票者 ID を参照できない
- システム管理者（IT 担当）であっても、業務目的外での起票者 ID の参照は就業規則上の禁止事項とする
- 匿名化された起票レコードに対する採否通知は、起票者のみに受信可能な形で送信される

### 4-2. 用途三限定の業務運用規約

Kaizen Teian・ヒヤリハットの起票データおよび起票行動データ（起票日時・起票頻度・起票内容）の利用は、以下の三用途に限定することを業務運用規約として確定する。

| 用途 | 許可される利用内容 | 禁止される利用内容 |
|---|---|---|
| 品質保証 | 不適合調査・CAPA 連動・トレサビ照会 | 個人の「問題発見能力」の評価 |
| 工程改善 | 工程・ライン単位の改善課題集計・トレンド分析 | 個人別の起票件数ランキング・達成率 |
| 教育設計 | 工程全体の教育ニーズ分析・カリキュラム設計 | 個人を特定した「教育が必要な作業員」の評価レポート |

**禁止される転用用途**（技術的制限と就業規則の双方で禁止）

- 人事評価・昇給・昇格・解雇の根拠
- 個人別 Kaizen 起票件数の上長への自動報告
- 「Kaizen を起票しない作業員」の特定・通知

### 4-3. 運用監査の業務手続

用途三限定の遵守状況を年次で監査する業務手続を確定する。

- 品質担当が年次で「アクセスログ監査」を実施する（管理 Web のアクセスログから禁止転用の痕跡を確認する）
- 監査結果を経営層・労働者代表に報告する
- 違反が発見された場合は是正措置を実施し、労働者代表に報告する

**本節で確定した方針**
起票者 ID の匿名化オプションを技術的に実装し、品質担当以外のアクセスを RBAC で禁止することを確定する。
行動データの用途を品質保証・工程改善・教育設計の三用途に限定し、人事評価・個人ランキングへの転用を技術的・制度的双方で禁止することを確定する。
年次の用途三限定遵守監査を業務手続として確定し、結果を経営層・労働者代表に報告することを確定する。
