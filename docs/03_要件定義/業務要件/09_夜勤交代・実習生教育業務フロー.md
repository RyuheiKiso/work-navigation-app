# 09 夜勤交代・実習生教育業務フロー

本章の責務は、シフト交代時の引継ぎ業務（BF-05）と実習生教育・習熟度管理業務（BF-06）を確定することである。プレースキーパーによる中断引継ぎ・ダークモード運用規律・多言語支援・スキル等級管理の各業務手続を定義する。

## 1. シフト交代の引継ぎ業務

### 1-1. 引継ぎ業務の業務的位置づけ

製造現場の三交代・二交代勤務では、シフト交代時の引継ぎの質が製品品質・安全・生産性に直接影響する。引継ぎ漏れによる工程スキップ・異常の見落としは不適合の原因となる。本システムは I-PASS 構造の電子引継ぎ票で引継ぎの標準化・完全化を実現する。

### 1-2. 引継ぎ必須情報の確定

シフト交代引継ぎ票に必ず含まれる情報を以下のとおり確定する。

| I-PASS 項目 | 引継ぎ必須情報 |
|---|---|
| I（Illness / 状況概要） | 現在の作業状態（工程名・品番・ロット番号・完了ステップ数 / 全ステップ数） |
| P（Patient / 案件詳細） | 当日の作業特記事項（SOP 改訂直後フラグ・4M 変更情報・仕掛品の特記） |
| A（Assessment / 評価） | 現時点での品質状態（不適合発生の有無・アンドン発報履歴・測定値トレンドの状況） |
| S（Situation / 未完課題） | 未完ステップ・未回収の中断ログ・保留中のアンドン・未回答の Kaizen フィードバック |
| S（Synthesis / 引継ぎ指示） | 次シフトへの具体的指示（「ステップ 14 の測定を先に確認してから続行すること」等） |

### 1-3. 引継ぎ業務の手順

引継ぎ業務の業務手順を以下のとおり確定する。

**引渡し側（前シフト作業員）の手順**

1. シフト終了 10 分前にタブレット APP が「引継ぎ票作成」を促す通知を表示する
2. 前シフト作業員が引継ぎ票の各項目を入力する（自動生成された情報を確認・修正する）
3. プレースキーパーから取得した中断状態（ステップ番号・進捗状況）が自動で I 欄に表示される
4. 特記事項・未完課題・引継ぎ指示を手入力する
5. 前シフト作業員が電子サインで引継ぎ票を確定する
6. 引継ぎ票が次シフト作業員のタブレット APP にプッシュ通知で送信される

**受取側（次シフト作業員）の手順**

1. 次シフト作業員がタブレット APP にログインする
2. 未読の引継ぎ票通知が表示される
3. 引継ぎ票の全項目を確認する（スクロールで全内容を読み進めるよう設計する）
4. 確認完了後、次シフト作業員が電子サインで「引継ぎ確認」を記録する
5. 引継ぎ票の確認電子サインが完了するまで、対象作業の開始画面への遷移がブロックされる

**本節で確定した方針**
引継ぎ必須情報を I-PASS の五項目（状況概要・案件詳細・評価・未完課題・引継ぎ指示）で確定する。
引継ぎ票の受取側確認電子サインが完了するまで作業開始をブロックすることを業務ルールとして確定する。
プレースキーパーからの中断状態の自動取得を引継ぎ票作成の補完機能として確定する。

## 2. プレースキーパーによる中断引継ぎ

### 2-1. プレースキーパーの中断引継ぎ手続

作業の途中でシフト交代が発生した場合（工程の途中で作業員が交代する場合）、プレースキーパーによる中断引継ぎを以下の手続で行う。

1. 前シフト作業員が「中断」操作を行う（シフト交代種別: 「シフト交代」を選択）
2. プレースキーパーが以下の情報を引継ぎスナップショットとして記録する
   - 中断ステップ番号
   - 完了済みステップの電子サインリスト
   - 未完了ステップの一覧
   - 中断直前 3 ステップの記録値サマリ
   - 使用中の設備 ID・計測器 ID
3. 中断スナップショットが引継ぎ票の I 欄（状況概要）に自動転記される
4. 次シフト作業員が引継ぎ票を確認し、電子サインで確認する
5. 次シフト作業員が「再開」操作を行う
6. Welcome 画面が表示される（前シフト作業員の中断情報・経過時間・直前 3 ステップのサマリ）
7. ロックステップが中断ステップへ強制復帰し、次シフト作業員が作業を継続する

### 2-2. クリティカルステップ中断時の特例手続

クリティカルステップ（`fmea_rpn_flag = true`）の実行途中でシフト交代が発生した場合、以下の特例手続を確定する。

- 中断前に現場監督の「中断確認」電子サインを必須とする
- 次シフト作業員が再開する際に「目視再確認プロンプト」が強制表示される
- 「目視再確認完了」の電子サインを押下してからステップの実施に進む

**本節で確定した方針**
プレースキーパーによる中断引継ぎスナップショット（ステップ番号・完了済みリスト・記録値サマリ）の自動生成を業務手続として確定する。
クリティカルステップ中断時の現場監督確認電子サインと再開時の目視再確認プロンプトを特例手続として確定する。

## 3. ダークモード運用規律

### 3-1. 夜勤帯の定義と自動切り替え

夜勤帯を 20:00〜6:00 と定義し、この時間帯にタブレット APP が自動的にダークモードに切り替わることを業務ルールとして確定する。

ダークモードの業務的意義は以下のとおりである。

- ブルーライトの低減による眼精疲労の軽減
- サーカディアンリズムへの影響を最小化し、夜勤作業員の生理的負担を軽減する
- 暗い工場環境での視認性向上

### 3-2. ダークモードと夜勤固有の業務規律

夜勤帯（20:00〜6:00）において以下の業務規律を追加で適用する。

| 業務規律 | 適用条件 | 業務手続 |
|---|---|---|
| クリティカルステップ二重確認 | 連続夜勤 3 日目以降かつ深夜帯（0:00〜5:00） | クリティカルステップ完了時に「もう一度確認してください」の 2 秒長押しプロンプトを表示する |
| 異常高速操作の検知 | 通常完了時間の 50% 以下で Step を完了した場合 | 「操作が速すぎます。確認しましたか？」のプロンプトで一時停止し、「確認した」タップで再開する |
| 振動通知の強化 | 夜勤帯全体 | 通知の振動パターンを 2 段階強化パターンに切り替える |

### 3-3. 夜勤帯の手動モード切り替え許可

昼夜を問わない特殊シフト（早番・遅番・変形労働時間制）への配慮として、作業員がプロファイル設定からダークモードを手動で有効化・無効化できる業務設定を提供する。自動切り替えを手動で上書きした場合、上書き理由の入力は不要とする。

**本節で確定した方針**
夜勤帯（20:00〜6:00）の自動ダークモード切り替えを業務ルールとして確定する。
連続夜勤 3 日目・深夜帯でのクリティカルステップ二重確認プロンプトを業務規律として確定する。
異常高速操作検知（通常の 50% 以下での完了）での一時停止を業務規律として確定する。

## 4. 実習生向け多言語・読み書き支援

### 4-1. 多言語 SOP の業務手続

外国人技能実習生・特定技能労働者が在籍する工場での多言語 SOP 運用業務手続を確定する。

**SOP 翻訳の業務フロー**

1. 品質担当が新版 SOP の翻訳依頼を対象言語の母語話者レビュアーに行う
2. 母語話者レビュアーが翻訳内容をレビューし、承認または修正依頼を行う
3. 安全指示を含むステップの翻訳は人間レビューを承認条件とし、機械翻訳のみでの有効化を禁止する
4. 翻訳版の承認後、品質担当が管理 Web で「多言語版承認」操作を行う
5. 承認された翻訳版が当該言語を設定した作業員のタブレット APP に配信される

**外国語版が未承認の場合の業務手続**

当該言語の翻訳版が未承認の場合、作業員のタブレットには以下の表示を行う。

- 日本語版のみを表示する
- 「この SOP の XX 語版は準備中です。現場監督の補助のもとで作業を実施してください」の通知を表示する
- 作業員が「現場監督を呼ぶ」ボタンを押すと、現場監督に通知が送信される

### 4-2. ピクトグラム・動画の活用業務手続

言語に依存しない手順理解を支援するため、以下の業務手続を確定する。

- ISO 7010 安全標識ピクトグラムを安全上の重要指示を含む全ステップに表示する
- 動画字幕は承認済み翻訳版と連動し、字幕言語を作業員のプロファイル設定に従って自動選択する
- クリティカルステップ（fmea_rpn_flag = true）には日本語原文と翻訳版を併記し、疑義が生じた場合に原文参照できる状態を維持する

**本節で確定した方針**
多言語 SOP の有効化には母語話者レビューの承認を必須とし、機械翻訳のみでの有効化を禁止することを確定する。
翻訳未承認時の「現場監督を呼ぶ」ボタン配置を業務要件として確定する。
ISO 7010 ピクトグラムを安全指示ステップに必須表示することを確定する。

## 5. スキル等級と作業可否ゲート

### 5-1. スキル等級の定義

作業員のスキル等級を以下の四段階で確定する。

| 等級 | 名称 | 業務上の意味 |
|---|---|---|
| L1 | 見習い（Novice） | 指導員の立会いのもとで作業を実施できる。単独作業は禁止 |
| L2 | 基礎（Competent） | 指導員の立会いなしで標準 SOP の作業を実施できる |
| L3 | 熟練（Proficient） | 複数の SOP・工程を担当でき、習熟度差があるメンバーへの指導支援ができる |
| L4 | 専門（Expert） | SOP の改訂提案・Kaizen 起票の品質が高く、新人教育を担当できる |

### 5-2. スキル等級の認定業務手続

スキル等級の認定は現場監督が行い、以下の手続で実施する。

1. 現場監督が管理 Web から「スキル認定」操作を行う
2. 対象作業員・対象工程・認定等級・認定理由を入力する
3. 認定後、作業員のスキルマスタに等級・取得日・有効期限が記録される
4. 有効期限は以下の条件で自動失効する
   - 有効期限（現場監督が設定）が経過した場合
   - 対象 SOP がメジャー改訂された場合（メジャー改訂後 N 回完了で等級が自動復帰）
   - 最終実施日から 30 日以上経過した場合（Skill Decay 検知）

### 5-3. 作業可否ゲートの業務手続

SOP の各ステップには `skill_level_required`（L1〜L4）が設定される。作業員が対象ステップを実施しようとした際に、以下の業務手続でゲートを適用する。

- 作業員のスキル等級が `skill_level_required` を充足している場合: 通常通り実施できる
- 充足していない場合: 「このステップを実施するには L3 以上の認定が必要です。現場監督を呼んでください」のブロック画面が表示される
- 現場監督が立会い電子サインを押下した場合（L1 作業員の立会い条件充足）: ブロックが解除されステップを実施できる

**本節で確定した方針**
スキル等級を L1〜L4 の四段階で確定し、各等級の業務上の意味を定義する。
スキル等級の有効期限自動失効条件（有効期限経過・メジャー改訂・30 日ブランク）を確定する。
作業可否ゲートをスキル等級不足時のブロック＋現場監督立会い電子サインによる解除で確定する。
