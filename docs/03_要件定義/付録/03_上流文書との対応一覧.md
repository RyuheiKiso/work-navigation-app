# 03 上流文書との対応一覧

本章の責務は、本書（要件定義書）と上流文書（システム化構想・システム化計画・90_業界分析）との対応関係を明示することである。本章は「本書のどの章がどの上流文書の意思決定を受けているか」を一覧として提供し、設計根拠の追跡を容易にする。

---

## 1. システム化構想との対応

### 1-1. システム化構想の章構成（推定）と本書の対応

システム化構想は `docs/02_企画/システム化構想/` に格納されている。構想書はシステム化計画 README §1 が示すとおり「Why — なぜ作るか、解決すべき課題、実現する価値、倫理スタンス」を扱う文書である。

| 構想の章（推定）| 扱う問い | 本書の対応章 |
|---|---|---|
| 構想: 課題と背景 | なぜ作るか。製造現場の作業ナビゲーション・トレサビの現状課題 | `01_要件定義の前提継承と上流リンケージ.md` H2-1 |
| 構想: 倫理スタンス・境界宣言 | 行動データの用途三限定・データオーナーシップ・AI/ML 非提供の根拠 | `非機能要件/06_倫理品質要件.md` / `99_前提制約と本書が約束しないこと.md` |
| 構想: ステークホルダーと価値仮説 | 誰のために作るか。8 種ステークホルダーの価値命題 | `03_ステークホルダーと要件責任マトリクス.md` |
| 構想: 非スコープの倫理的宣言 | 個人別生産性ランキング・監視機能の永続非提供 | `99_前提制約と本書が約束しないこと.md` / `非機能要件/06.md` |

---

**本節で確定した方針**
- 構想書の「なぜ」（課題・倫理スタンス・価値仮説）が本書の前提継承・倫理品質要件・ステークホルダーマトリクスに対応することを確定する。
- 構想書の倫理的宣言（永続非提供項目）は `99_前提制約` と `NFR-ETH` の双方に反映されていることを確定する。
- 構想書の完成後に本対応表を更新し、全章の対応を完全版として記録することを確定する。
