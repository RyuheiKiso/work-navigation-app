# 移行要件サブ

## 1. 本サブの責務

移行要件サブは IPA 共通フレーム 2013 要件定義プロセスの「移行要件の定義」アクティビティに相当する。

本サブが確定する内容は以下の 3 点に限定する。

- **マスタ初期投入要件（MIG-INI）**: 工程/作業/Step の SOP マスタを初回に電子化する際の品質基準・受入基準
- **既存データ移行要件（MIG-MIG）**: 紙記録・既存電子記録の移行方針・データクレンジング基準
- **カットオーバー要件**: 本番切替のロールバック条件・縮退期間の設計

### 計画書からの継承

本サブは計画書 `13_データ移行とマスタ初期投入戦略.md` の方針を要件として具体化する。
計画書が確定した「マスタ初期投入の品質基準・ロールバック方針」を受入基準付き要件として再記述する。

---

**本節で確定した方針**
- 移行要件サブは「初回マスタ投入の品質基準」「既存データ移行の受入基準」「カットオーバーのロールバック条件」の 3 点を確定する。
- 移行要件は MIG-INI-xxx・MIG-MIG-xxx として採番する。
- 移行完了の受入基準は「移行後の記録完全性率 100%」「カットオーバー後 72 時間の縮退運転なし」等の検証可能命題で記述する。
