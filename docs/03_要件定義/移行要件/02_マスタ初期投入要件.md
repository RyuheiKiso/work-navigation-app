# 02 マスタ初期投入要件

本書の責務は、本システムを導入初日から業務利用可能な状態にするために必要なマスタデータの初期投入要件を確定することである。マスタ初期投入は導入失敗の最大要因であり、品質担当・IT 担当の協働による厳密な品質保証が不可欠である。上流文書「システム化計画 13 章 1〜2 節」の方針を受けて、数値基準・バリデーション規則・受入確認手順を要件として確定する。

---

## 1. マスタ初期投入の対象

### 1-1. 投入対象マスタの 5 種確定

MIG-010: 以下の 5 種類のマスタを初期投入対象として確定する。

| マスタ種別 | 主要エンティティ | 必須フィールド | 備考 |
|---|---|---|---|
| SOP マスタ | processes / operations / steps / step_params | process_no, process_name, op_name, step_no, step_content | QC 工程表を工程定義の正源とする |
| ユーザーマスタ | users / user_roles / user_skills | username, display_name, role, hire_date | パスワードは初期値（初回変更必須）で投入する |
| 設備マスタ | equipment | equipment_no, equipment_name, process_id | 保全記録は含まない |
| 計測器マスタ | instruments | instrument_no, instrument_name, calibration_cert_no, calibration_due_date | 有効期限切れは警告フラグ付与 |
| ロットマスタ（直近 1 年） | lots | lot_no, external_lot_id, product_code, started_at | is_legacy=true を強制付与する |

MIG-011: 初期投入の実施順序は以下の通りとする。参照整合性制約（外部キー制約）の関係上、順序を逆にした投入を禁止する。

1. ユーザーマスタ
2. 設備マスタ
3. 計測器マスタ
4. SOP マスタ（工程 → 作業 → Step の順）
5. ロットマスタ（直近 1 年）

### 1-2. 各マスタの投入前準備要件

MIG-012: 各マスタの投入前に品質担当が以下の準備を完了させることを必須とする。

| マスタ種別 | 投入前準備 |
|---|---|
| SOP マスタ | 電子化対象 SOP の優先度付け完了・方式選択完了（MIG-006 参照） |
| ユーザーマスタ | 全作業員の役割アサイン確認・初期スキル情報の書面確認 |
| 設備マスタ | 設備台帳の最新化確認・廃棄済み設備の除外処理 |
| 計測器マスタ | 校正証明書の現物確認・有効期限の再確認 |
| ロットマスタ | 引継ぎ対象ロットの選定・external_lot_id の重複チェック |

**本節で確定した方針**
- SOP マスタ・ユーザーマスタ・設備マスタ・計測器マスタ・ロットマスタの 5 種を初期投入対象として確定する。
- 投入順序を参照整合性に従って確定し、順序逸脱を禁止する。
- 各マスタの投入前準備を品質担当が完了させることを必須条件として確定する。

## 2. 品質基準

### 2-1. 電子署名必須要件

MIG-013: 初期投入されたすべての SOP マスタに対して、投入完了後 24 時間以内に品質担当の電子署名を付与することを必須とする。電子署名の構成要素は以下の通りとする。

| 構成要素 | 仕様 |
|---|---|
| ユーザー ID | 品質担当のシステムログイン ID |
| タイムスタンプ | サーバー側 UTC タイムスタンプ（クライアント時刻は不使用） |
| サーバー署名 | HMAC-SHA256 による署名（JWT と同等の強度） |
| 署名対象ハッシュ | 全 Step の内容ハッシュ（SHA-256） |

MIG-014: 電子署名未付与の SOP マスタは「未承認」ステータスとして管理し、作業員のナビゲーション画面に表示しない。電子署名なしの SOP を現場作業に使用することを禁止する。

### 2-2. OCR 誤変換許容率

MIG-015: 方式 3（OCR 変換）で取込んだ SOP の OCR 誤変換は、本番投入時点でゼロ件であることを必須とする。OCR 誤変換のゼロ件確認は品質担当の全文目視照合によって実施する。自動照合ツールのみによる確認を認めない。

OCR 誤変換の定義: 原文と異なる文字・数値・単位が変換後テキストに含まれている状態。

MIG-016: OCR 変換後の照合記録（照合担当者・照合日時・照合結果）を本システムの電子記録として保存する。照合記録の保存期間は本番稼働後 7 年以上とする。

### 2-3. 必須フィールド完全性

MIG-017: 初期投入時点で、すべての必須フィールドが 100% 入力されていることを必須とする。必須フィールドに NULL 値または空文字が存在する状態での本番投入を禁止する。

| マスタ種別 | 必須フィールド完全性の確認方法 |
|---|---|
| SOP マスタ | バルク取込 CLI ツールのバリデーションレポートで「必須フィールドエラー 0 件」を確認する |
| ユーザーマスタ | 投入後に管理画面のユーザー一覧を全件確認し、空欄なしを目視確認する |
| 設備マスタ・計測器マスタ | 投入後に管理画面の一覧で必須フィールドの表示を全件確認する |

### 2-4. 参照整合性チェック合格要件

MIG-018: 初期投入完了後、本システムが提供するデータ整合性チェックコマンドを実行し、全チェック項目が合格であることを本番投入の必須条件とする。チェック内容は以下の通りとする。

| チェック項目 | 内容 |
|---|---|
| 外部キー整合性 | Step が参照する Operation ID・設備 ID・計測器 ID が存在することを確認する |
| Step 番号連続性 | 各 Operation 内の Step 番号に欠番・重複がないことを確認する |
| ロット番号一意性 | external_lot_id が全ロット間で一意であることを確認する |
| ユーザーロール整合 | 全ユーザーに少なくとも 1 つの有効なロールが割り当てられていることを確認する |

**本節で確定した方針**
- SOP 電子署名を投入完了後 24 時間以内に付与することを必須とし、未署名 SOP の現場使用を技術的に禁止する。
- OCR 誤変換ゼロ件・必須フィールド完全性 100%・参照整合性チェック全通過の 3 基準を品質保証の最低要件として確定する。
- 整合性チェックの全合格を本番投入の前提条件として確定し、不合格での投入を禁止する。

## 3. 取込テンプレート仕様

### 3-1. テンプレートの種別

MIG-019: 以下の 4 種類の Excel テンプレートを ver1.0.0 成果物として提供する。

| テンプレート名 | 対応マスタ | 形式 |
|---|---|---|
| SOP_TEMPLATE_v1.xlsx | SOP マスタ（工程・作業・Step） | Excel（xlsx） / CSV |
| USER_TEMPLATE_v1.xlsx | ユーザーマスタ | Excel（xlsx） / CSV |
| EQUIPMENT_TEMPLATE_v1.xlsx | 設備マスタ | Excel（xlsx） / CSV |
| INSTRUMENT_TEMPLATE_v1.xlsx | 計測器マスタ | Excel（xlsx） / CSV |

ロットマスタは直近 1 年分のデータを対象とするため、専用のロットテンプレート（LOT_TEMPLATE_v1.xlsx）も提供する。

### 3-2. SOP テンプレートの列定義

MIG-020: SOP_TEMPLATE の必須列・任意列の定義は以下の通りとする。

| 列名 | 必須/任意 | データ型 | バリデーション規則 |
|---|---|---|---|
| process_no | 必須 | 文字列（最大 20 文字） | 同一テンプレート内で一意・英数字とハイフンのみ |
| process_name | 必須 | 文字列（最大 100 文字） | 空文字不可 |
| op_no | 必須 | 文字列（最大 20 文字） | process_no + op_no が一意 |
| op_name | 必須 | 文字列（最大 100 文字） | 空文字不可 |
| step_no | 必須 | 整数 | 1 以上・同一 op_no 内で連番（欠番不可） |
| step_content | 必須 | 文字列（最大 1000 文字） | 空文字不可 |
| step_type | 必須 | 文字列（列挙型） | {confirm, measure, select, photo, text, custom} のいずれか |
| judgment_criterion | 任意 | 文字列（最大 500 文字） | 測定 Step では入力推奨 |
| equipment_no | 任意 | 文字列（最大 20 文字） | 設備マスタに存在する equipment_no を参照 |
| instrument_no | 任意 | 文字列（最大 20 文字） | 計測器マスタに存在する instrument_no を参照 |
| image_filename | 任意 | 文字列（最大 255 文字） | img/ 配下の存在確認を行う |
| note | 任意 | 文字列（最大 2000 文字） | 制約なし |

### 3-3. バリデーション規則とエラーレポート形式

MIG-021: バルク取込 CLI ツールは投入前に以下のバリデーションを実行し、エラーがある場合は投入を中断してレポートを出力する。

| バリデーション種別 | 内容 |
|---|---|
| 必須フィールド検証 | すべての必須列が空でないことを確認する |
| データ型検証 | 整数型フィールドに非整数値が入っていないことを確認する |
| 列挙型検証 | step_type が定義済み列挙値のいずれかであることを確認する |
| 参照整合性検証 | equipment_no / instrument_no が参照先マスタに存在することを確認する |
| 一意性検証 | Step 番号の欠番・重複・process_no の重複を検出する |
| 文字数検証 | 最大文字数を超過するフィールドを検出する |

MIG-022: エラーレポートは以下の CSV 形式で出力する。

```
row_number,column_name,error_type,error_message,value
3,step_no,DUPLICATE_STEP_NO,"Step番号が重複しています",1
7,equipment_no,REFERENCE_NOT_FOUND,"設備マスタに存在しないequipment_noです","EQ-999"
```

MIG-023: バリデーションレポートの「エラー（ERROR 以上）」が 1 件以上存在する状態での本番投入を禁止する。警告（WARNING）は人的確認後に投入を継続できる。

**本節で確定した方針**
- 4 種類の Excel テンプレートを ver1.0.0 成果物として確定し、各テンプレートの必須列・任意列・バリデーション規則を確定する。
- バルク取込 CLI ツールが投入前にバリデーションを実行し、エラー時は投入中断・CSV レポート出力を行う設計を確定する。
- エラー 1 件以上での本番投入を禁止し、エラーゼロを投入実行の必須条件として確定する。

## 4. SOP 紙→電子化の優先度付け基準

### 4-1. 優先度マトリクス

MIG-024: SOP の電子化優先度は以下のマトリクスに従って品質担当が決定する。移行対象 SOP のリスト作成時に各 SOP の優先度を記録し、書面化する。

| 優先度 | 判定基準 | 対象の例 |
|---|---|---|
| 最優先（P1） | 規制管理特性が指定された工程（ISO 9001・顧客仕様・計量法対象） | 最終検査工程・法定計量器使用工程 |
| 高（P2） | 過去 12 ヶ月にクレームが 1 件以上発生した工程 | 外観検査・組立工程 |
| 中（P3） | 工程能力指数 Cpk が 1.33 未満の工程 | ばらつきが大きい加工工程 |
| 低（P4） | 安定工程・非製品接触工程 | 梱包・補助作業 |

MIG-025: P1 工程の SOP は本番カットオーバ日までに全件電子化を完了することを必須とする。P2 以下は計画に基づいて順次電子化し、本番稼働開始後も段階的に追加投入できる。

### 4-2. クリティカル工程の追加要件

MIG-026: P1（最優先）と判定された工程の SOP には以下の追加要件を課す。

- 方式 1（手入力）を必須とし、方式 3（OCR 変換）の単独使用を禁止する
- 電子署名を投入完了後 8 時間以内に付与する（24 時間ルールより厳格に適用する）
- 現場監督による全 Step 確認を品質担当承認の前提条件とする
- 初期投入後 1 週間以内に代表ロットでのナビゲーション実行確認を実施する

**本節で確定した方針**
- 規制管理特性・クレーム件数・工程能力指数・安定性の 4 軸による P1〜P4 の優先度マトリクスを確定する。
- P1 工程の SOP はカットオーバ日までに全件電子化を完了することを必須とする。
- P1 工程は手入力方式必須・8 時間以内電子署名・現場監督確認の 3 点を追加要件として確定する。

## 5. 初期投入後の受入確認基準

### 5-1. 受入確認の 4 条件

MIG-027: 初期投入後の受入確認は以下の 4 条件がすべて充足された時点で完了とする。

| 条件 | 確認内容 | 確認担当者 |
|---|---|---|
| 条件 1 | 投入件数確認（投入予定件数と実際の登録件数が一致する） | IT 担当 |
| 条件 2 | 整合性チェック全通過（MIG-018 のチェックコマンドが全項目 OK） | IT 担当 |
| 条件 3 | 代表 SOP のナビゲーション実行確認（各優先度区分から 1 件以上） | 現場監督 |
| 条件 4 | 品質担当の受入承認署名（電子署名で初期投入完了を承認） | 品質担当 |

### 5-2. 件数確認の方法

MIG-028: 投入件数確認は以下の SQL クエリ（またはシステム管理画面の件数表示）で実施する。確認対象は工程・作業・Step・ユーザー・設備・計測器・ロットの 7 エンティティとする。

| エンティティ | 確認項目 |
|---|---|
| 工程（processes） | 計画投入件数 vs 実際の登録件数 |
| 作業（operations） | 計画投入件数 vs 実際の登録件数 |
| Step（steps） | 計画投入件数 vs 実際の登録件数 |
| ユーザー（users） | 計画投入件数 vs 実際の登録件数 |
| 設備（equipment） | 計画投入件数 vs 実際の登録件数 |
| 計測器（instruments） | 計画投入件数 vs 実際の登録件数 |
| ロット（lots） | 計画投入件数 vs 実際の登録件数 |

MIG-029: 件数差異（計画件数 vs 実際件数の不一致）が 1 件以上ある場合は、受入確認を保留し原因を調査する。調査完了と差異解消の確認なしに受入承認署名を行うことを禁止する。

**本節で確定した方針**
- 投入件数確認・整合性チェック全通過・代表 SOP ナビゲーション確認・品質担当承認署名の 4 条件をすべて充足することを受入確認完了の定義として確定する。
- 7 エンティティの件数確認を IT 担当が実施し、件数差異ゼロを条件として確定する。
- 件数差異発生時は保留・調査・解消確認を経てからの承認を必須とする。
