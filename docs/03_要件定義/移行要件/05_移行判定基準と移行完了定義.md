# 05 移行判定基準と移行完了定義

本書の責務は、移行作業の完了を客観的に判定するための基準と、移行完了の正式宣言に必要な条件を確定することである。曖昧な完了定義は移行後の問題発生時に責任の所在を不明確にし、品質保証の根拠を失わせる。本書で確定する判定基準はすべて検証可能な命題として記述する。

---

## 1. 移行完了の定義

### 1-1. 移行完了の 4 条件

MIG-060: 移行完了は以下の 4 条件がすべて充足された時点で宣言する。1 条件でも未充足の場合は移行完了を宣言しない。

| 条件 | 定義 | 確認担当者 |
|---|---|---|
| 条件 1：全マスタ投入完了 | MIG-027 が定める受入確認の 4 条件（投入件数確認・整合性チェック全通過・ナビゲーション実行確認・品質担当承認署名）がすべて充足されている | IT 担当 + 品質担当 |
| 条件 2：整合性チェック全通過 | バルク取込 CLI ツールの整合性チェックコマンドが全チェック項目「OK」を返し、エラーが 0 件である | IT 担当 |
| 条件 3：受入テスト合格 | MIG-045 で定めるリハーサル確認項目 7 件が本番環境で全件合格である | 品質担当・現場監督・IT 担当 |
| 条件 4：並行運用期間終了 | MIG-047 で定める 2 週間の並行運用期間が終了し、段階 3（紙 SOP のフォールバック用途転換）が完了している | 品質担当 |

MIG-061: 移行完了の宣言は品質担当が移行完了報告書に電子署名を付与することで正式化する。署名なしの口頭による移行完了宣言は認めない。

### 1-2. 各条件の詳細確認方法

MIG-062: 条件 3（受入テスト合格）の本番環境での確認は以下の通り実施する。

| 確認項目 | 本番環境での実施内容 |
|---|---|
| P1 工程の SOP ナビゲーション | P1 工程から最低 1 件を実際の作業員がタブレット APP で操作し、全 Step 表示と電子署名完了を確認する |
| P2〜P4 工程のサンプル確認 | 各優先度区分から 1 件ずつ代表 SOP のナビゲーション表示を確認する |
| 作業記録の PostgreSQL への格納 | 確認用 Step 記録が本番 PostgreSQL の audit_log テーブルに正しく格納されていることを IT 担当が確認する |

**本節で確定した方針**
- 全マスタ投入完了・整合性チェック全通過・受入テスト合格・並行運用期間終了の 4 条件を移行完了定義として確定する。
- 移行完了の宣言は品質担当の電子署名付き移行完了報告書で正式化し、口頭宣言を認めない。
- 4 条件の 1 条件でも未充足の場合は移行完了を宣言しないことを確定する。

## 2. 移行 GO/NO-GO 判定基準

### 2-1. GO/NO-GO 判定の時点

MIG-063: 移行 GO/NO-GO 判定を以下の 2 時点で実施する。

| 判定時点 | 目的 | 判定権限者 |
|---|---|---|
| フェーズ 2（リハーサル）終了後 | 本番移行の実施可否を判定する | 品質担当（最終権限）・IT 担当・現場監督 |
| 段階 1 完了後（3 者合意） | 段階 2（全ライン展開）への移行可否を判定する | 品質担当・IT 担当・現場監督（3 者全員合意必須） |

### 2-2. GO 判定の数値基準

MIG-064: 本番移行 GO 判定には以下の数値基準をすべて満たすことを必須とする。

| 基準項目 | GO 判定の数値 | 計測方法 |
|---|---|---|
| リハーサルの実施回数 | 2 回以上 | リハーサル実施記録で確認する |
| リハーサル問題点票のオープン件数 | 0 件 | 問題点票の全件クローズを確認する |
| バリデーションレポートのエラー件数 | 0 件 | CLI ツールの出力レポートで確認する |
| 整合性チェックの全項目 OK | 100%（全項目 OK） | チェックコマンドの出力で確認する |
| P1 工程の電子署名完了率 | 100% | 管理画面の SOP 一覧で確認する |
| リハーサル全体所要時間 | 計画時間の 120% 以内 | リハーサル時計測値で確認する |

MIG-065: 上記基準の 1 項目でも未達の場合は NO-GO 判定とし、本番移行を保留する。保留理由を問題点票に記録し、解消後に再判定を実施する。

**本節で確定した方針**
- 移行 GO/NO-GO 判定をリハーサル終了後と段階 1 完了後の 2 時点で実施することを確定する。
- 6 項目の数値基準（すべて厳密な閾値）を GO 判定の必須条件として確定する。
- 1 項目でも未達の場合は NO-GO とし、問題解消後の再判定を必須とする。

## 3. 移行完了記録

### 3-1. 移行完了報告書の必須項目

MIG-066: 移行完了報告書は以下の必須項目を含み、本システムの電子記録として保存する。保存期間は本番稼働後 7 年以上とする。

| 必須項目 | 内容 |
|---|---|
| 移行完了宣言日時 | 品質担当が電子署名を付与した日時（サーバー UTC タイムスタンプ） |
| 4 条件の充足確認記録 | 各条件の確認担当者・確認日時・確認結果の一覧 |
| 投入件数サマリ | 工程・作業・Step・ユーザー・設備・計測器・ロットの 7 エンティティの投入件数 |
| リハーサル実施記録の参照 | リハーサル実施日・回数・発見問題点数・クローズ件数 |
| 残存課題（あれば） | 移行完了後も継続対応が必要な課題とその対応期限・担当者 |
| 品質担当の電子署名 | ユーザー ID・タイムスタンプ・サーバー署名 |

MIG-067: 移行完了報告書は本システムの文書管理機能（管理 Web）で保管し、ALCOA+ 準拠の電子記録として扱う。is_legacy フラグは付与しない（移行後に新規作成する正式記録のため）。

MIG-068: 移行完了後 90 日以内に移行の事後レビューを実施する。事後レビューでは移行プロセスの問題点・改善点を文書化し、将来の横展開時の移行手順改善に活用する。事後レビュー報告書も本システムの電子記録として保存する。

**本節で確定した方針**
- 移行完了報告書の 6 必須項目を確定し、本番稼働後 7 年以上の保存を要件として確定する。
- 移行完了報告書を ALCOA+ 準拠の正式電子記録として管理 Web で保存することを確定する。
- 移行完了後 90 日以内の事後レビュー実施と文書化を必須として確定する。
