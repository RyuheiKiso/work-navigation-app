# 機能要件サブ

## 1. 本サブの責務

機能要件サブは IPA 共通フレーム 2013 要件定義プロセスの「システム要件の定義（機能要件）」アクティビティに相当する。

本サブが確定する内容は以下の 2 点に限定する。

- **機能要件（FR）**: システムが実行する機能の検証可能命題。「何をするか」を「ID・検証可能命題・受入基準・MoSCoW・上流リンク」の 5 要素で確定する
- **明示的非機能**: 「〜を実装しない」形式の要件。倫理制約 EC-01〜EC-06 に基づく禁止実装項目を含む

機能要件サブは「システムが何をするか」を記述する。「どのように実装するか（アルゴリズム・画面レイアウト・DB スキーマ詳細）」は概要設計・詳細設計の責務であり、本サブでは扱わない。

### 業務要件サブとの分担境界

| 機能要件サブが扱う | 業務要件サブが扱う |
|---|---|
| 「システムが〜する」形式の検証可能命題 | 「業務として〜が必要である」形式の業務的根拠 |
| 受入基準（AC）の具体的検証条件 | ユースケース記述（アクター・事前条件・事後条件） |
| MoSCoW 優先度の確定 | 業務フローの全体像 |
| 上流リンク（BR・計画書章） | 業務ドメイン用語の定義 |

---

**本節で確定した方針**
- 機能要件サブは「何をするか（検証可能命題）」を確定し、「どう実装するか」は概要設計書に委任することを確定する。
- 全 FR は「ID・検証可能命題・受入基準・MoSCoW・上流リンク」の 5 要素が揃った状態でのみ Agreed 状態に遷移できることを確定する。
- 禁止実装要件（「〜を実装しない」形式）も FR として採番し、倫理制約の設計反映を明示することを確定する。
