# 09 画面・機能対応表

本章は、ハンディ APP・マスタメンテナンス APP・管理コンソールの全画面を識別子付きで列挙し、各画面と機能要件（FR）のクロスマトリクス、画面遷移概要、ロール別アクセス可否を確定する。概要設計フェーズにおける UI 設計の入力文書として機能する。

---

## 3. 画面遷移概要

### 3-1. ハンディ APP の主要遷移

**作業実行フロー（正常系）**
SCR-HA-001（ログイン）→ SCR-HA-002（ホーム）→ SCR-HA-003（作業指示一覧）→ SCR-HA-004（作業実行メイン）→ SCR-HA-005/006/007（Step 入力）→ SCR-HA-008（電子サイン）→ SCR-HA-004（次 Step）→ … → SCR-HA-003（完了）

**中断・再開フロー**
SCR-HA-004（作業実行中）→ SCR-HA-009（中断入力）→ SCR-HA-002（ホームに戻る）→ SCR-HA-003（作業指示一覧）→ SCR-HA-009（再開確認）→ SCR-HA-004（中断 Step から再開）

**Kaizen 起票フロー**
SCR-HA-004（作業中）または SCR-HA-002（ホーム）→ SCR-HA-011（Kaizen 起票）→ SCR-HA-004 または SCR-HA-002（起票完了・戻る）

**アンドン発報フロー**
SCR-HA-004（作業中）→ SCR-HA-010（アンドン発報）→ SCR-HA-004（発報後・作業継続または中断）

### 3-2. マスタメンテナンス APP の主要遷移

**SOP 編集・承認フロー**
SCR-MA-001（ログイン）→ SCR-MA-002（ダッシュボード）→ SCR-MA-003（SOP 一覧）→ SCR-MA-004（SOP 編集 Draft）→ SCR-MA-005（Step 詳細編集）→ SCR-MA-006（DSL エディタ）→ SCR-MA-004（保存）→ SCR-MA-007（承認フロー）→ SCR-MA-003（承認完了・PUBLISHED）

### 3-3. 管理コンソールの主要遷移

**監査・トレサビフロー**
SCR-MC-001（ログイン）→ SCR-MC-002（ダッシュボード）→ SCR-MC-004（トレサビ照会）→ SCR-MC-005（監査ログ閲覧・エクスポート）

**CAPA 管理フロー**
SCR-MC-002（ダッシュボード）→ SCR-MC-007（Kaizen 一覧）→ SCR-MC-006（CAPA 管理：起票・進捗・クローズ）

図: fig_screen_function_matrix（img/ 配下）を参照

**本節で確定した方針**
- ハンディ APP の作業実行フロー・中断再開フロー・Kaizen/アンドンへの分岐を確定する。
- マスタメンテナンス APP の SOP 編集→承認の一方向フローを確定し、承認なしでの PUBLISHED 遷移を禁止する。
- 管理コンソールの監査・CAPA フローを確定する。
