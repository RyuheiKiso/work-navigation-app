# 運用要件サブ

## 1. 本サブの責務

運用要件サブは IPA 共通フレーム 2013 要件定義プロセスの「運用要件の定義」アクティビティに相当する。

本サブが確定する内容は以下の 4 点に限定する。

- **バックアップ・リストア要件（OPS-BKP）**: バックアップ頻度・保存期間・リストア手順の自動化基準
- **監視・アラート要件（OPS-MON）**: サービス死活監視・ログ監視・アラート通知の基準
- **縮退運転要件（OPS-DGR）**: Offline-First による業務継続・紙フォールバックへの縮退条件
- **SOP 改訂ループ要件（OPS-SOP）**: SOP の改訂起票・承認・公開・通知サイクルの運用基準

### 計画書からの継承

本サブは計画書 `09_運用・組織・教育方針.md` の方針を要件として具体化する。
IT 担当 1 名前提（O-C02 制約）での運用を前提とし、自動化・自己診断・ドキュメント化を運用要件の設計原則とする。

---

**本節で確定した方針**
- 運用要件サブは IT 担当 1 名が単独で運用できる複雑度に留めた OPS-BKP/MON/DGR/SOP 要件を確定する。
- バックアップは「日次フルバックアップ + PostgreSQL WAL アーカイブ」を Must 要件として確定する。
- SOP 改訂ループは「改訂起票→品質担当承認→作業員通知→次回作業から適用」の自動通知フローを OPS-SOP 要件として確定する。
