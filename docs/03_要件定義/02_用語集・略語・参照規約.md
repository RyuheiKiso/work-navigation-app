# 02 用語集・略語・参照規約

## 1. 用語の階層と命名規約

### プロジェクト共通用語と業務ドメイン用語の棲み分け

本書が管理する用語は以下の 2 層に分類する。

| 層 | 種別 | 管理場所 | 対象範囲 |
|---|---|---|---|
| 第一層 | プロジェクト共通用語 | 本章（本ファイル） | 要件定義書全体・構想・計画・概要設計にわたって共通して使用する用語 |
| 第二層 | 業務ドメイン用語 | `業務要件/03_業務ドメイン用語集.md` | 製造現場固有の業務用語・工程名・作業名・製品区分等 |

第一層の用語は本章で定義し、要件記述中に初出時は本章を参照することを義務とする。
第二層の業務ドメイン用語は本章では定義せず、すべて [`業務要件/03_業務ドメイン用語集.md`](業務要件/03_業務ドメイン用語集.md) を参照すること。

- 用語の追加・変更は本章の改訂として管理し、全要件への影響確認を行うことを確定する。

## 3. 引用規約

### 業界分析参照形式

業界分析ファイルへの参照は以下の形式を使用する。

```
[`90_業界分析/XX_*.md`](../../90_業界分析/XX_*.md)
```

### 上流文書参照形式

システム化構想への参照は以下の形式を使用する。

```
[`02_企画/システム化構想/XX_*.md`](../../02_企画/システム化構想/XX_*.md)
```

システム化計画への参照は以下の形式を使用する。

```
[`02_企画/システム化計画/XX_*.md`](../../02_企画/システム化計画/XX_*.md)
```

### 本書内部の章間参照形式

本書内部の他章への参照は以下の相対パス形式を使用する。

```
[`機能要件/01_ナビゲーション機能要件.md`](機能要件/01_ナビゲーション機能要件.md)
[`99_前提制約と本書が約束しないこと.md`](99_前提制約と本書が約束しないこと.md)
```

### 必須 / 関連の区別

各章末の「## 参照業界分析」節では以下の区分を明示する。

| 区分 | 意味 | 記載方針 |
|---|---|---|
| **必須** | その章の論理が成立するために不可欠な分析 | 省略不可。章の本文で直接引用する |
| **関連** | 理解を深めるために参照を推奨する分析 | 最大 4 件まで。章本文での引用は任意 |

「関連」は最大 4 件とする。関連性が薄い文書を羅列することは禁止する。

### 要件 ID の参照形式

他の要件 ID を本文中で参照する場合は以下の形式を使用する。

```
FR-NV-001（`機能要件/01_ナビゲーション機能要件.md` 参照）
NFR-ETH-001（`非機能要件/06_倫理品質要件.md` 参照）
```

---

**本節で確定した方針**
- 引用規約を業界分析参照・上流文書参照・内部章間参照・必須/関連区別の 4 種類で確定する。
- 過剰引用防止として「関連」を最大 4 件に制限することを確定する。
- 要件 ID 参照時は対応ファイルパスを明示することを確定する。

## 4. 規制表記三段階の詳細定義

### 三段階の定義と使い分け

本書内で規制・標準への対応状況を表現する場合、以下の三段階のみを使用する。

| 表現 | 定義 | 使用条件 |
|---|---|---|
| **準拠する** | 当該規制・標準の要求事項をすべて満たすことを設計目標として確定している | 具体的な適合基準を同一文書内で明示できる場合のみ |
| **対応する** | 当該規制・標準の関連条項を設計上考慮し、ベストエフォートで満たすことを確定している | 全条項への適合を保証しないが、対応範囲を明示できる場合 |
| **対象外と判断する** | 当該規制・標準への適合を本システムの責務と判断しない | 対象外の根拠を同一文書内で明示できる場合のみ |

### 禁止表現リスト（規制表記）

以下の表現は規制・標準への対応状況の記述において使用を禁止する。

| 禁止表現 | 禁止理由 | 代替表現 |
|---|---|---|
| 考慮する | 対応の程度が不明確。受入基準を設定できない | 対応する（対応範囲を明示する） または 対象外と判断する |
| 検討する | 将来の決定を示唆し、現時点での確定を回避している | 準拠する / 対応する / 対象外と判断する のいずれかに確定する |
| 参考にする | 拘束力の程度が不明確 | 対応する（参考にする条項を対応範囲として明示する） |
| 可能性がある | 確定でない状態を記述している | 確定する（YES/NO を明示する） |
| 場合がある | 条件が不明確 | 条件を明示して要件として記述する |
| なるべく準拠する | 「準拠する」の意味を希薄化している | 対応する（対応範囲を明示する） |
| 準拠を目指す | 「準拠する」との区別が不明確 | 対応する（達成基準が確定できない場合） |
| 適宜対応する | 対応の基準が不明確 | 対応する（対応基準を明示する） |

### 本書における規制表記の適用例

本書において規制表記を用いる主な場面と適用例を以下に示す。

| 規制・標準 | 適用表現 | 根拠 |
|---|---|---|
| ALCOA+ 9 原則 | 対応する（9 原則すべてに対応する実装を確定する） | 計画 06章で ALCOA+ を設計品質基準として採用。GMP 文脈の GxP 全条項への適合は対象外と判断する |
| ISO 9001:2015 第 7.5 条（文書化した情報） | 対応する（電子記録の完全性・可読性・検索可能性に対応する） | 適用業種において ISO 9001 内部監査が現実的に発生する |
| 労働安全衛生法 | 対応する（ヒヤリハット・アンドン機能を提供する。法的報告義務の代行は対象外と判断する） | 構想 14章の国内規制立ち位置の継承 |
| 個人情報保護法（2022 年改正） | 対応する（データ最小化・利用目的の明示・同意取得を設計で担保する） | 構想 04章の行動データ用途三限定の継承 |
| GDPR 第 5 条第 1 項 (c) | 対応する（データ最小化原則を設計に組み込む） | 計画 01章の透明性公約の継承 |
| GMP / FDA 21 CFR Part 11 | 対象外と判断する（想定導入環境が規制対象業種ではないため） | 計画 11章 P06 の継承 |
| IATF 16949 | 対象外と判断する（同上） | 計画 11章 P06 の継承 |
| AS9100 | 対象外と判断する（航空宇宙業種は対象範囲外のため） | 計画 02章の対象業種定義の継承 |
| HACCP | 対象外と判断する（食品業種に特化した規制適合は本書の責務外のため） | 計画 11章 P06 の継承 |
| EU AI Act | 対象外と判断する（AI/ML 機能を永続非提供のため） | 構想 08章・11章の継承 |

---

**本節で確定した方針**
- 規制表記は「準拠する」「対応する」「対象外と判断する」の三段階のみとし、禁止表現 8 件を確定する。
- GMP/FDA 21 CFR Part 11/IATF 16949/AS9100/HACCP/EU AI Act を対象外と判断することを本章で宣言する。
- ALCOA+ 9 原則・ISO 9001 第 7.5 条・労働安全衛生法・個人情報保護法・GDPR 第 5 条への対応を確定する。
