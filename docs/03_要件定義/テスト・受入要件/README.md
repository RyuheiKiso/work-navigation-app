# テスト・受入要件サブ

## 1. 本サブの責務

テスト・受入要件サブは IPA 共通フレーム 2013 要件定義プロセスの「受入れテスト要件の定義」アクティビティに相当する。

本サブが確定する内容は以下の 3 点に限定する。

- **ユーザー受入テスト（UAT）要件（TST-UAT）**: Must 要件の全受入基準を検証するテスト種別・テスト環境・合格基準
- **ALCOA+ 検証要件（TST-ALC）**: ALCOA+ 9 原則の各原則が要件定義書の受入基準を満たしていることを検証するテスト計画
- **テスト種別定義**: ユニットテスト・統合テスト・E2E テスト・Detox モバイル E2E・負荷テストの適用方針

本サブはテストケースの実装・テストコードの記述は行わない。これらは詳細設計書・実装の責務である。
本サブは「何を・どの基準で・どの手順で検証するか」の要件を確定する。

---

**本節で確定した方針**
- テスト・受入要件サブは「UAT 合格基準」「ALCOA+ 9 原則の検証計画」「テスト種別と適用方針」の 3 点を TST 要件として確定する。
- Must 要件の全受入基準に対して TST-UAT 要件を対応付け、RTM を完成させることを確定する。
- ALCOA+ Contemporaneous 原則（タイムスタンプ精度）の検証は自動化テストで実施することを Must 要件として確定する。
