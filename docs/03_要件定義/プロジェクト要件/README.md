# プロジェクト要件サブ

## 1. 本サブの責務

プロジェクト要件サブは IPA 共通フレーム 2013 要件定義プロセスの「プロジェクト要件の定義」アクティビティに相当する。

本サブが確定する内容は以下の 3 点に限定する。

- **変更管理要件（PRJ-CHG）**: Frozen 状態の要件を変更する際のプロセス・承認者・影響確認の義務
- **意思決定プロセス要件（PRJ-DEC）**: 個人開発における技術選定・設計変更・非スコープ判断の意思決定権限と ADR 記録義務
- **個人開発制約要件**: Bus Factor = 1 の開発体制における引継ぎ品質・ドキュメント化基準

### 計画書からの継承

本サブは計画書 `10_開発スケジュールと意思決定プロセス.md` の意思決定プロセス設計と R-O01（Bus Factor = 1 対策）を PRJ 要件として具体化する。

個人開発（O-C01 制約）・phase なし（O-C03 制約）という制約を前提とし、ver1.0.0 でプロダクションレベル品質を達成するための開発プロセス要件を確定する。

---

**本節で確定した方針**
- プロジェクト要件サブは「変更管理プロセス」「意思決定権限の ADR 記録」「引継ぎドキュメントのみで環境再構築可能」の 3 点を PRJ 要件として確定する。
- Frozen 状態の要件変更は「影響範囲確認 → A の再承認 → ADR 作成」の 3 ステップを PRJ-CHG の Must 要件として確定する。
- 技術選定・設計変更の意思決定は全て ADR として記録することを PRJ-DEC の Must 要件として確定する。
