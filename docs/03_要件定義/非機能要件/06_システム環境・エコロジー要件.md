# 06 システム環境・エコロジー要件

本章は、タブレット・サーバーのハードウェア最低スペック・ソフトウェア環境・ネットワーク・設置環境・ライセンス・廃棄・エコロジーの 7 節で、システムが稼働する物理・論理環境の要件を確定する。計画 07 章（技術スタック選定根拠）・計画 11 章（前提 T01〜T05）の方針を受入基準として具体化する。

---

## 1. ハードウェア環境要件

### 1-1. タブレット最低スペック

| 要件 ID | NFR-ENV-001 |
|---|---|
| 要件名 | タブレット防護等級 |
| 要件内容 | タブレット端末は IP54 以上（粉塵・水飛沫保護）かつ MIL-STD-810H（耐衝撃・耐振動・耐温度）準拠であること |
| 根拠 | 製造現場での水飛沫・粉塵・落下リスクに対応する |

| 要件 ID | NFR-ENV-002 |
|---|---|
| 要件名 | タブレット画面サイズ |
| 要件内容 | 画面サイズは 8 インチ以上であること。手袋着用状態で操作可能なタッチターゲット（44dp 以上）を表示するために必要な最低解像度は 1280×800（WXGA）以上とする |

| 要件 ID | NFR-ENV-003 |
|---|---|
| 要件名 | タブレット輝度 |
| 要件内容 | 屋外・高照度環境での可読性のため、最大輝度 800nits 以上を推奨する。最低要件は 400nits とし、夜勤環境での使用に対応するため輝度調整機能を必須とする |

| 要件 ID | NFR-ENV-004 |
|---|---|
| 要件名 | タブレット RAM・ストレージ |
| 要件内容 | RAM 4GB 以上（React Native + SQLite + バックグラウンド Outbox Consumer の同時稼働を保証）。内部ストレージ 64GB 以上（SQLite キャッシュ・写真バッファ・オフライン SOP データを格納）|

### 1-2. サーバー最低スペック

| 要件 ID | NFR-ENV-005 |
|---|---|
| 要件名 | サーバー CPU |
| 要件内容 | CPU: 4 コア以上（Rust/axum の非同期処理で tokio が有効活用できる最低コア数）|

| 要件 ID | NFR-ENV-006 |
|---|---|
| 要件名 | サーバー RAM |
| 要件内容 | RAM: 16GB 以上（PostgreSQL の shared_buffers + Docker コンテナ + OS の合計メモリ要件）|

| 要件 ID | NFR-ENV-007 |
|---|---|
| 要件名 | サーバー SSD |
| 要件内容 | SSD: 1TB 以上（OS・ミドルウェア・PostgreSQL データ・Docker イメージ・ログの合計容量）|

**本節で確定した方針**
- タブレット要件を IP54/MIL-STD-810H・8 インチ以上・400nits 最低（800nits 推奨）・RAM 4GB・ストレージ 64GB として確定する。
- サーバー最低スペックを CPU 4 コア・RAM 16GB・SSD 1TB として確定する。
- 本節で確定したスペックを下回る機器での稼働動作は保証対象外とすることを確定する。

## 2. ソフトウェア環境要件

### 2-1. OS・ランタイム要件

| 要件 ID | NFR-ENV-010 |
|---|---|
| 要件名 | サーバー OS |
| 要件内容 | Windows Server 2022（IIS + WSL2 + Docker 構成）を正式サポート対象とする。Linux（Ubuntu 22.04 LTS または Rocky Linux 9）も正式サポート対象とする |

| 要件 ID | NFR-ENV-011 |
|---|---|
| 要件名 | タブレット OS |
| 要件内容 | Android 12 以上（React Native Expo の最低サポートバージョン）および iOS 16 以上（iPhone / iPad）を正式サポート対象とする。Windows 10 以上（react-native-windows 対応端末）を準サポートとする |

| 要件 ID | NFR-ENV-012 |
|---|---|
| 要件名 | ミドルウェア要件 |
| 要件内容 | PostgreSQL 16 以上。Docker 24 以上。Docker Compose v2 以上。Rust Edition 2024 (1.80+)。Node.js 20 LTS（フロントエンドビルド用）|

### 2-2. OSS ライセンス互換性

| 要件 ID | NFR-ENV-013 |
|---|---|
| 要件名 | OSS ライセンス互換性確認 |
| 要件内容 | `cargo license` および `license-checker` で全依存ライブラリのライセンスを確認する。GPL-2.0・GPL-3.0 ライセンスの依存を禁止する（LGPL は商用利用条件を確認の上で許可する）|

**本節で確定した方針**
- サーバー OS を Windows Server 2022 と Linux（Ubuntu 22.04 / Rocky Linux 9）の両方を正式サポートとして確定する。
- タブレット OS を Android 12+ / iOS 16+ を正式・Windows タブレットを準サポートとして確定する。
- GPL-2.0/3.0 ライセンス依存を禁止し、OSS ライセンス互換性確認ツールの使用を確定する。

## 3. ネットワーク環境要件

### 3-1. LAN・Wi-Fi 要件

| 要件 ID | NFR-ENV-020 |
|---|---|
| 要件名 | LAN 帯域 |
| 要件内容 | サーバーへの有線接続 LAN 帯域は 100Mbps 以上（推奨 1Gbps）とする |

| 要件 ID | NFR-ENV-021 |
|---|---|
| 要件名 | Wi-Fi 帯域 |
| 要件内容 | タブレット向け Wi-Fi 帯域は 25Mbps 以上（AP 1 基あたりの実効帯域）を推奨する。WPA3 または WPA2-AES を使用する |

| 要件 ID | NFR-ENV-022 |
|---|---|
| 要件名 | Wi-Fi 断での縮退保証 |
| 要件内容 | Wi-Fi が切断された状態でも、ハンディ APP は SQLite ローカルキューへの記録を継続できること（Offline-First 設計。機能要件 FR-OFF 参照）。Wi-Fi 復旧後に自動同期が再開されること |

| 要件 ID | NFR-ENV-023 |
|---|---|
| 要件名 | Wi-Fi デッドゾーンの許容 |
| 要件内容 | 工場内の一部エリアで Wi-Fi カバレッジが不完全（前提 P04）な場合でも、Offline-First 設計によって業務継続を保証する。Wi-Fi デッドゾーンでの作業記録は端末に保持し、電波圏内に戻った時点で同期する |

**本節で確定した方針**
- LAN 100Mbps 以上・Wi-Fi 25Mbps 以上・WPA3/WPA2-AES を推奨スペックとして確定する。
- Wi-Fi 断での縮退保証（ローカルキュー継続 + 復旧後自動同期）を必須要件として確定する。
- Wi-Fi デッドゾーンの存在を前提 P04 として許容し、Offline-First 設計で対応することを確定する。

## 4. 設置環境要件

### 4-1. 工場現場動作環境

| 要件 ID | NFR-ENV-030 |
|---|---|
| 要件名 | 動作温度範囲 |
| 要件内容 | タブレット端末の動作温度範囲: 0〜50℃（製造現場の通常温度範囲を網羅）|

| 要件 ID | NFR-ENV-031 |
|---|---|
| 要件名 | 動作湿度範囲 |
| 要件内容 | タブレット端末の動作湿度範囲: 非結露状態で 10〜90% RH（製造現場の湿度変動に対応）|

| 要件 ID | NFR-ENV-032 |
|---|---|
| 要件名 | 耐粉塵要件 |
| 要件内容 | IP54 以上（粉塵の侵入から保護）を必須とする。切削粉・溶接スパッタ・研磨粉が発生する環境での使用を想定する。クリーンルーム（クラス 100 以下）環境は対象外と判断する（前提 P01 の製造現場範囲を超える）|

| 要件 ID | NFR-ENV-033 |
|---|---|
| 要件名 | 耐衝撃要件 |
| 要件内容 | MIL-STD-810H の落下試験（1.2m 落下）に準拠した端末を選定する。タブレット専用ケースの装着は推奨とし、必須要件としない |

**本節で確定した方針**
- 動作温度 0〜50℃・湿度 10〜90% RH・IP54 耐粉塵・MIL-STD-810H 耐衝撃を設置環境の必須要件として確定する。
- クリーンルーム（クラス 100 以下）環境は本プロジェクトの対象外と判断する。
- タブレット専用ケースを推奨要件として位置づけ、必須要件としないことを確定する。

## 5. ライセンス要件

### 5-1. OSS ライセンス適合性

| 要件 ID | NFR-ENV-040 |
|---|---|
| 要件名 | GPL 禁止・MIT/Apache 許可 |
| 要件内容 | 本システムが依存する OSS ライブラリのライセンスに GPL-2.0・GPL-3.0 が含まれる場合、当該ライブラリの使用を禁止する。MIT・Apache 2.0・BSD 系・LGPL 2.1（動的リンク条件を満たす場合）は許可する |

| 要件 ID | NFR-ENV-041 |
|---|---|
| 要件名 | OSS ライセンス一覧の管理 |
| 要件内容 | リリース前に `cargo license` と `license-checker` で全依存ライブラリのライセンス一覧を生成し、docs/oss-licenses/ に格納する。ライセンス違反が検出された場合は代替ライブラリへの変更または依存の削除を行う |

**本節で確定した方針**
- GPL-2.0/3.0 依存を禁止し、MIT / Apache 2.0 / BSD / LGPL（動的リンク）を許可するライセンスポリシーを確定する。
- リリース前の OSS ライセンス一覧生成・docs/oss-licenses/ への格納を必須プロセスとして確定する。

## 6. 廃棄・データ消去要件

### 6-1. 退職者 PII の匿名化

| 要件 ID | NFR-ENV-050 |
|---|---|
| 要件名 | 退職者 PII の 60 日以内匿名化 |
| 要件内容 | ユーザーの退職・削除申請から 60 日以内に、当該ユーザーの PII（氏名・worker_id 実名）を内部 UUID に置換する。WorkEvent の worker_id は変更しない（証拠保全）。User テーブルの氏名・メールアドレスを削除または匿名化テキストに置換する |

### 6-2. 端末廃棄手順

| 要件 ID | NFR-ENV-051 |
|---|---|
| 要件名 | 端末廃棄時のデータ消去 |
| 要件内容 | タブレット端末を廃棄・返却する場合、以下の手順を実施する。SQLCipher 暗号化済み DB の鍵を廃棄する / OS の工場出荷状態へのリセットを実施する / 消去手順の実施記録を IT 担当が保管する |

**本節で確定した方針**
- 退職者 PII の 60 日以内匿名化（UUID 置換）を必須とし、WorkEvent の証拠保全と両立させることを確定する。
- 端末廃棄時の SQLCipher 鍵廃棄 + 工場リセット + 記録保管の 3 ステップを廃棄手順として確定する。

## 7. エコロジー要件

### 7-1. 消費電力・設計寿命

| 要件 ID | NFR-ENV-060 |
|---|---|
| 要件名 | サーバー消費電力目標 |
| 要件内容 | サーバーの通常稼働時の消費電力を 250W 以下とすることを目標とする（中小規模オンプレサーバーの一般的水準）。計測は IT 担当が年次確認する |

| 要件 ID | NFR-ENV-061 |
|---|---|
| 要件名 | タブレット設計寿命 |
| 要件内容 | タブレット端末の設計寿命を 4 年以上とする。選定基準として「メーカーサポート期間 4 年以上」を必須とし、EOL 前のリプレース計画を IT 担当が管理する |

**本節で確定した方針**
- サーバー消費電力目標を 250W 以下として確定し、年次確認を IT 担当の義務とする。
- タブレット設計寿命を 4 年以上・メーカーサポート期間 4 年以上として確定し、EOL 管理を IT 担当の責務とする。
