# 04 移行性要件

本章は、開発→ステージング→本番の環境間移行・マスタ初期投入・バージョン移行・Excel/CSV テンプレート仕様・子機モード初期同期の各要件を確定する。計画 13 章（データ移行とマスタ初期投入戦略）の方針を SLO・受入基準として具体化する。

---

## 1. 環境間移行要件

### 1-1. 3 環境構成

| 要件 ID | NFR-MIG-001 |
|---|---|
| 要件名 | 環境構成 |
| 要件内容 | 開発（DEV）・ステージング（STG）・本番（PRD）の 3 環境を維持する。DEV は個人開発用ローカル環境。STG は本番同等構成のテスト環境。PRD は実業務環境 |

| 要件 ID | NFR-MIG-002 |
|---|---|
| 要件名 | 環境間移行手順の標準化 |
| 要件内容 | DEV→STG および STG→PRD の移行は Docker Compose の設定切替とデータベースマイグレーション（sqlx migrate）のみで実施できること。手動スクリプト作業を必要としない |

### 1-2. 移行前検証基準

| 要件 ID | NFR-MIG-003 |
|---|---|
| 要件名 | STG での事前検証 |
| 要件内容 | STG での検証が以下の全項目を通過した後でのみ PRD への移行を許可する |
| 検証項目 | cargo test 全パス / Detox E2E テスト全パス / 負荷テスト（同時接続 500 名シミュレーション）/ マスタ初期投入データの整合性確認 / バックアップ・復旧テスト（RPO 15 分・RTO 1 時間）|

**本節で確定した方針**
- DEV / STG / PRD の 3 環境を確定し、環境間移行を Docker Compose 設定切替 + sqlx migrate のみで実施することを確定する。
- STG→PRD 移行の受入基準 5 項目（テスト・負荷・整合性・復旧テスト）全通過を必須とすることを確定する。
- 手動スクリプト作業を移行手順から排除し、再現性を保証することを確定する。

## 2. データ移行性要件

### 2-1. マスタ初期投入の品質基準

| 要件 ID | NFR-MIG-010 |
|---|---|
| 要件名 | マスタ初期投入の OCR 誤変換許容率 |
| 目標値 | 0%（許容しない）|
| 要件内容 | 紙 SOP・既存 Excel からの初期投入データに OCR 誤変換が存在する場合、投入を中止し人手確認を経て再投入する |

| 要件 ID | NFR-MIG-011 |
|---|---|
| 要件名 | 電子署名必須の初期投入 |
| 要件内容 | 本番環境への SOP マスタ初期投入は、品質担当の電子署名（承認操作）を経た PUBLISHED 版として投入すること。未承認の Draft 版を本番環境にそのまま移行することを禁止する |

| 要件 ID | NFR-MIG-012 |
|---|---|
| 要件名 | 移行データのバリデーション |
| 要件内容 | マスタ初期投入ツールはデータ投入前に以下のバリデーションを実施する。Step の instruction_text（日本語・必須）が空でないこと / input_type が有効な列挙値であること / skill_level_required が L1〜L4 の範囲内であること / evidence_required が true の場合に input_type が photo_capture / numeric_input / qr_scan のいずれかであること |

### 2-2. 移行ツール提供要件

| 要件 ID | NFR-MIG-013 |
|---|---|
| 要件名 | CSV 一括投入ツール |
| 要件内容 | 本章 4 節で確定した Excel/CSV テンプレートからのマスタ一括投入 CLI ツール（または管理コンソールの CSV インポート機能）を提供する。投入エラーの件数・項目名を CSV に出力して確認を促す |

**本節で確定した方針**
- マスタ初期投入の OCR 誤変換許容率を 0% として確定し、品質担当の電子署名を必須とする。
- 投入前バリデーション 4 条件（instruction_text 非空・input_type 有効・スキルレベル範囲・evidence_required 整合）を確定する。
- エラー件数・項目名を CSV 出力するバリデーション機能を提供ツールの必須要件として確定する。

## 3. バージョン移行要件

### 3-1. ver1.x マイグレーション方針

| 要件 ID | NFR-MIG-020 |
|---|---|
| 要件名 | マイグレーションの停止時間目標 |
| 要件内容 | ver1.x のマイグレーション（パッチバージョン・マイナーバージョン）は無停止または計画停止（最大 15 分・夜間実施）で完了すること |

| 要件 ID | NFR-MIG-021 |
|---|---|
| 要件名 | マイグレーションの後退可能性 |
| 要件内容 | ver1.x のマイグレーションは前バージョンへのロールバック手順を事前に用意すること。ロールバックは 1 時間以内に完了可能であること |

| 要件 ID | NFR-MIG-022 |
|---|---|
| 要件名 | DB スキーマ変更の後方互換性 |
| 要件内容 | sqlx migrate での DB スキーマ変更は以下の原則に従う。既存カラムの型変更・削除は禁止（新カラムの追加のみ許容）。データの再構成が必要な変更は専用マイグレーションスクリプトで実施し、STG で事前検証する |

**本節で確定した方針**
- ver1.x マイグレーションの停止時間目標を無停止または最大 15 分（夜間）として確定する。
- ロールバック手順の事前用意と 1 時間以内完了を必須要件として確定する。
- DB スキーマの既存カラム型変更・削除禁止を sqlx migrate の設計制約として確定する。

## 4. Excel/CSV 取込テンプレートの仕様凍結要件

### 4-1. テンプレートのスキーマ凍結

| 要件 ID | NFR-MIG-025 |
|---|---|
| 要件名 | マスタ初期投入テンプレートのスキーマ凍結 |
| 要件内容 | SOP マスタ初期投入用 CSV テンプレートのカラム定義を ver1.0.0 リリース時点で凍結する。カラムの追加・削除・名称変更はマイナーバージョンアップとして扱い、変更前後の移行手順を提供する |

| 要件 ID | NFR-MIG-026 |
|---|---|
| 要件名 | テンプレートの必須カラム |
| 要件内容 | SOP 初期投入 CSV テンプレートの必須カラムは以下のとおりとする |
| 必須カラム | process_code（工程コード）/ operation_code（作業コード）/ step_number（ステップ番号）/ instruction_text_ja（日本語手順テキスト）/ input_type（入力種別）/ evidence_required（証拠要否: true/false）/ skill_level_required（スキルレベル: L1-L4）|

| 要件 ID | NFR-MIG-027 |
|---|---|
| 要件名 | テンプレートのバリデーション規則文書化 |
| 要件内容 | テンプレートのバリデーション規則（各カラムの型・必須/任意・許容値）を README またはヘッダー行のコメントとして CSV ファイルに同梱する |

**本節で確定した方針**
- SOP 初期投入 CSV テンプレートの 7 必須カラムを確定し、ver1.0.0 時点でスキーマを凍結する。
- カラム変更はマイナーバージョンアップとして扱い、移行手順の提供を義務付ける。
- バリデーション規則を CSV ファイルに同梱することで、投入者が仕様を確認できる設計を確定する。

## 5. 子機モード初期同期要件

### 5-1. 初回同期の完了条件

| 要件 ID | NFR-MIG-030 |
|---|---|
| 要件名 | 初回親機→子機同期の完了条件 |
| 要件内容 | 子機（本システム）の初回導入時、以下の全マスタが親機から取得されること。SOP マスタ（全 PUBLISHED 版）/ ユーザーマスタ / 設備マスタ / ロットマスタ（直近 90 日分）/ 外部一意キーマッピング |

| 要件 ID | NFR-MIG-031 |
|---|---|
| 要件名 | 初回同期の完了確認 |
| 要件内容 | 管理コンソールの「初期同期状態」画面で、各マスタの取得件数・最終同期タイムスタンプ・欠損確認結果を表示する。全マスタの同期完了が確認されてから本番稼働を開始する |

| 要件 ID | NFR-MIG-032 |
|---|---|
| 要件名 | 初回同期の所要時間目標 |
| 要件内容 | SOP マスタ 1,000 件・ユーザーマスタ 500 件・設備マスタ 200 件の初回同期を 30 分以内に完了すること |

**本節で確定した方針**
- 初回同期の完了条件を 5 種マスタ（SOP / ユーザー / 設備 / ロット / 外部キー）の取得完了として確定する。
- 管理コンソールでの同期状態確認画面を提供し、全マスタ同期確認後の本番稼働開始を義務付ける。
- 標準規模データ（SOP 1,000 件・ユーザー 500 件・設備 200 件）の初回同期 30 分以内完了を目標として確定する。
