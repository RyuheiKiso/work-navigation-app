# 03 要件定義

## 1. 本書の位置づけ

本書（要件定義書）は IPA 共通フレーム 2013 (SLCP-JCF2013) の要件定義プロセスに相当する。
上位文書としてシステム化構想（`docs/02_企画/システム化構想/`）とシステム化計画（`docs/02_企画/システム化計画/`）が存在し、本書はその成果物を受けて作成する。

### IPA 共通フレーム 2013 上の三層分担

| 層 | 扱う問い | 文書 |
|---|---|---|
| システム化構想 | Why — なぜ作るか、課題、価値命題、倫理スタンス | `docs/02_企画/システム化構想/` |
| システム化計画 | What/How-Frame — 対象範囲・機能スコープ・アーキテクチャ原則・技術スタック | `docs/02_企画/システム化計画/` |
| **本書（要件定義）** | **What 詳細 — 検証可能命題としての機能要件・受入基準・ユースケース記述・品質目標値** | **本書（`docs/03_要件定義/`）** |
| 概要設計（未作成） | How 骨格 — 画面構成・API 設計・DB スキーマ骨格 | `docs/04_概要設計/`（未作成） |

### 上流・下流との分担マトリクス

| 区分 | 扱う問い | 担当文書 |
|---|---|---|
| 構想 → 計画への引き渡し | Why（倫理・ビジョン・課題）→ What/How-Frame（スコープ・技術）の変換 | 構想 → 計画 |
| 計画 → 要件定義への引き渡し | What/How-Frame（スコープ・方針）→ What 詳細（検証可能命題・受入基準）の変換 | 計画 → **本書** |
| 要件定義 → 概要設計への引き渡し | What 詳細（要件・受入基準）→ How 骨格（画面・API・DB）の変換 | **本書** → 概要設計 |

本書は「する論」を継承する。「〜すべきである」という規範的命題は構想書の言語であり、本書の言語ではない。本書の全要件は「〜する」という検証可能命題として記述する。

---

**本節で確定した方針**
- 本書は IPA 共通フレーム 2013 要件定義プロセスに相当し、構想・計画の Why/What-Frame を受けて検証可能な What 詳細を確定する。
- 「する論」を継承し、全要件は受入基準・優先度（MoSCoW）・上流リンクを備えた検証可能命題として記述する。
- 本書の権威は ver1.0.0 リリースをもって概要設計書に移行する。

## 5. 引用規約

### 業界分析参照形式

業界分析ファイルへの参照は以下の形式を使用する。

```
[`90_業界分析/XX_*.md`](../../90_業界分析/XX_*.md)
```

### 上流文書参照形式

構想・計画の章への参照は以下の形式を使用する。

```
[`02_企画/システム化計画/XX_*.md`](../../02_企画/システム化計画/XX_*.md)
[`02_企画/システム化構想/XX_*.md`](../../02_企画/システム化構想/XX_*.md)
```

### 必須 / 関連の区別

各章末の「## 参照業界分析」節では以下の区分を明示する。

| 区分 | 意味 | 記載方針 |
|---|---|---|
| **必須** | その章の論理が成立するために不可欠な分析 | 省略不可。章の本文で直接引用する |
| **関連** | 理解を深めるために参照を推奨する分析 | 最大 4 件まで。章本文での引用は任意 |

## 6. 規制表記の三段階定義

本書内で規制・標準への対応状況を表現する場合、以下の三段階のみを使用する。
「考慮する」「検討する」「参考にする」「可能性がある」「場合がある」「〜したい」「思われる」等の曖昧表現は全章を通じて禁止する。

| 表現 | 定義 | 使用条件 |
|---|---|---|
| **準拠する** | 当該規制・標準の要求事項をすべて満たすことを設計目標として確定している | 具体的な適合基準を同一文書内で明示できる場合のみ |
| **対応する** | 当該規制・標準の関連条項を設計上考慮し、ベストエフォートで満たすことを確定している | 全条項への適合を保証しないが、対応範囲を明示できる場合 |
| **対象外と判断する** | 当該規制・標準への適合を本システムの責務と判断しない | 対象外の根拠を同一文書内で明示できる場合のみ |

## 7. 図の管理規約

本書（および全下位章）における図の扱いは以下のとおりとする。

- **作成ツール**: draw.io のみ。アスキー図・手書きスキャン・PowerPoint エクスポートは禁止
- **ファイル形式**: `.drawio` と `.svg` を対で管理する。`.svg` のみの単独配置は禁止
- **配置場所**: `img/` ディレクトリ配下に格納する（例: `img/fig_req_traceability.drawio` と `img/fig_req_traceability.svg`）
- **検証必須**: `drawio-lint` と `svg-postcheck` を両方通過したファイルのみコミットする
- **本文参照形式**: `図: fig_xxx（img/ 配下）を参照` の形式で本文から参照する。`<img>` タグによる直接埋め込みは禁止
