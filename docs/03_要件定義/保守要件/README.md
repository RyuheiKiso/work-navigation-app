# 保守要件サブ

## 1. 本サブの責務

保守要件サブは IPA 共通フレーム 2013 要件定義プロセスの「保守要件の定義」アクティビティに相当する。

本サブが確定する内容は以下の 3 点に限定する。

- **技術的負債管理要件（MNT-TDB）**: 技術的負債の計測・ADR（アーキテクチャ決定記録）の管理・負債返済方針
- **依存バージョン管理要件（MNT-DEP）**: Cargo.lock/package-lock の固定・マイナーバージョンアップ手順・メジャーアップの ADR 作成義務
- **テストカバレッジ・CI/CD 要件（NFR-MNT との連携）**: ユニットテスト 80% 以上・E2E コアフロー 100% の維持

### 計画書からの継承

本サブは計画書 `10_開発スケジュールと意思決定プロセス.md` の「ドキュメント駆動設計・ADR 蓄積」方針と計画書 `11_前提・制約・リスク.md` の R-O01（Bus Factor = 1 対策）を保守要件として具体化する。

IT 担当 1 名（O-C02 制約）が将来のチーム開発・SIer 移管を可能にするための引継ぎ品質を保守要件として確定する。

---

**本節で確定した方針**
- 保守要件サブは「ドキュメントのみで環境再構築が可能」「テストカバレッジを ver1.0.0 で達成」「ADR で全アーキテクチャ決定を記録」の 3 点を MNT 要件として確定する。
- `Cargo.lock` および `package-lock.json` による依存バージョン固定を MNT-DEP の Must 要件として確定する。
- メジャーバージョンアップは ADR 作成を義務とし、全 CI テスト通過を確認してから適用することを確定する。
