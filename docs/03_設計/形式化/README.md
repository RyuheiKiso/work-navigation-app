§3.4「分析方法論」の最終ステップである **形式化** を、§3.4.1 で確定した notation で具体化する成果物の格納先。沈黙の妥協（§2）を防ぐため、選択した記法・対象・検証手段を本書で固定し、CI で常時緑を維持する（将来 CI 整備時）。

## 1. 採用 notation（§3.4.1 再掲）

| 対象 | notation | 採用理由 | 不採用案 |
| --- | --- | --- | --- |
| 単一作業の状態 | **階層型有限状態機械（HSM, Harel statecharts 1987）** | 中断・再開・例外を入れ子で表現可能 | UML state machine（HSM の方が階層・並行を簡潔） |
| 並行・同期・順序情報の流れ | **着色 Petri net（CPN, Jensen & Kristensen 2009）** | 並行・資源・カラー（属性）を一体表現 | プレース変遷ネット（属性表現困難） |
| データ・整合性 | **TLA+（Lamport 1999）** | 同期競合・原子性・前進性を網羅検証 | Alloy（バウンデッド過ぎる） |
| 動的トポロジ（プロセス代数） | π計算／CSP は **不採用** | アプリ層に動的チャネル生成中心の処理がない | — |
