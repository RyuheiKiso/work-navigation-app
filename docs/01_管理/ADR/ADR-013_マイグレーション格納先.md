# ADR-013: マイグレーション格納先

日付: 2026-05-19
状態: 確定

## 背景

sqlx migrate のマイグレーションファイルの格納先として、複数の候補ディレクトリが検討された。
- `migrations/`（リポジトリルート直下）: sqlx のデフォルト
- `src/infra/database/migrations/`: インフラ層との整合性を重視した配置
- `db/migrations/`: 慣用的な配置

`docs/08_移行/導入手順/03_PostgreSQL初期化とマイグレーション手順.md` では `/opt/wnav/migrations/` という参照が存在していたが、ソースツリー上の格納先が明示されていなかった。
また `src/infra/README.md` では `src/infra/database/` が「DB マイグレーションファイル（sqlx migrate）」を管理すると記述されていた。

## 決定

マイグレーションファイルの格納先を `src/infra/database/migrations/` に確定する。

## 理由

- `src/infra/README.md` が `src/infra/database/` を「sqlx migrate 形式のマイグレーションファイルを管理する」ディレクトリと定義しており、整合性がある
- インフラ層（`src/infra/`）にデータベース関連ファイルを集約することで、Dockerfile・config・scripts・migrations が一箇所に揃い保守性が向上する
- リポジトリルート直下への配置はフロントエンド・バックエンドコードと混在してしまいディレクトリ構成の可読性を損なう
- sqlx は `--source` オプションで格納先を明示できるため、デフォルト外のパスを使用しても問題ない

## 影響

- `scripts/apply_migrations.sh` の `MIGRATION_DIR` デフォルト値: `src/infra/database/migrations`
- sqlx 実行コマンド: `sqlx migrate run --source src/infra/database/migrations`
- CI/CD パイプラインのマイグレーション実行ステップは `--source src/infra/database/migrations` を明示する
- `docs/08_移行/導入手順/03_PostgreSQL初期化とマイグレーション手順.md` §3-1 の `ls -la /opt/wnav/migrations/` は `src/infra/database/migrations/` への参照に合わせること
