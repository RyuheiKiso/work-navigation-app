# ADR-016: 固定 6 ロール確定

日付: 2026-05-19
状態: 確定

## 背景

アプリケーションのユーザーロール（業務上の役職に対応するシステムロール）について、ドキュメント間で定義が不一致だった。

`docs/08_移行/導入手順/03_PostgreSQL初期化とマイグレーション手順.md` §4-1 では以下の 6 ロールが期待される初期データとして記載されていた。
- `operator`（オペレーター）
- `line_leader`（ラインリーダー）
- `master_admin`（マスタ管理者）
- `quality_admin`（品質管理者）
- `system_admin`（システム管理者）
- `viewer`（閲覧者）

一方、`docs/02_企画/システム化計画/15_セキュリティ深堀り.md` では以下の 6 ロールが RBAC 設計として記載されていた。
- `operator`
- `supervisor`
- `master_admin`
- `quality_admin`
- `system_admin`
- `executive`

`line_leader` と `supervisor` が同一概念か別概念かが不明確であり、`viewer` と `executive` も定義が混在していた。

## 決定

固定 6 ロールを以下の通り確定する。

| ロールコード | 名称 | 主な権限概要 |
|---|---|---|
| `operator` | オペレーター | 作業手順閲覧・作業ログ記録・電子署名 |
| `supervisor` | 監督者 | operator の全権限 + ライン状況確認・簡易集計閲覧 |
| `master_admin` | マスタ管理者 | SOP / 工程 / 製品 / マスタデータの作成・更新・論理削除 |
| `quality_admin` | 品質管理者 | SOP 承認・IQC 判定・リワーク指示・品質レポート閲覧 |
| `system_admin` | システム管理者 | ユーザー管理・設定変更・バックアップ実行・フェイルオーバー |
| `executive` | 経営層 | ダッシュボード閲覧・KPI レポート閲覧（更新権限なし） |

`line_leader` は `supervisor` に統一する（役職名の英語訳として supervisor が汎用的）。
`viewer` は `executive` に統合する（参照専用という意味では同義であり、経営層向けダッシュボードが主用途）。

## 理由

- `docs/02_企画/システム化計画/15_セキュリティ深堀り.md` の RBAC 設計（`supervisor` / `executive`）は上流のセキュリティ方式設計であり、導入手順書（下流）より権威が高い
- `supervisor`（監督者・工程管理者）は工場現場の役職として `line_leader` より適切な英語表現である
- `executive`（経営層）は参照専用ロールとして、製造現場の KPI ダッシュボードを閲覧する経営層ユーザーを表現するのに適切である
- 6 ロールという数は RBAC の複雑さを最小限に保ちながら、製造業の役職階層を網羅するのに十分な粒度である

## 影響

- `src/infra/database/migrations/` の初期データ INSERT は 6 ロール（`operator` / `supervisor` / `master_admin` / `quality_admin` / `system_admin` / `executive`）を挿入する
- `docs/08_移行/導入手順/03_PostgreSQL初期化とマイグレーション手順.md` §4-1 の `line_leader` / `viewer` の記述は次回改訂時に本 ADR に合わせて修正する
- アプリケーションの RBAC 実装（axum ミドルウェア・React 権限判定）はこの 6 ロールを権威とする
- 確認 SQL の期待値を「6 件（`operator` / `supervisor` / `master_admin` / `quality_admin` / `system_admin` / `executive`）」に更新する
