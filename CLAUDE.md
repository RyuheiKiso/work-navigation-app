# コメント方針

コメントは **WHY** を一行で書く。自明な **WHAT**（コード行の言い換え）は書かない。
`// 状態を Running に遷移` のような直下行の機械的言い換えは禁止。
非自明な不変条件・制約・参照（ロードマップ §、ADR、外部仕様）はモジュール／関数レベルの doc コメントに集約する。

# 行数上限

プログラムのソースファイルの行数の上限は 500 行とする。
超える場合は適切にファイルを分割する。CI で機械的に強制する。

# 設計原則

クリーンアーキテクチャを採用する。
ドメイン駆動設計を採用する。
テスト駆動開発を採用する。
不変条件は型または property test で守る。

# コミット

Conventional Commits 形式とする (`feat:` / `fix:` / `refactor:` / `docs:` / `test:` / `chore:` / `perf:` / `ci:`)。
本文は WHAT ではなく WHY を 1〜3 行で記す。`新しい機能を追加` のような無情報メッセージは禁止。
