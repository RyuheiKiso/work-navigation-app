# 変更管理手順

> 対応規格: GMP Annex 11 §10／FDA 21 CFR Part 11 §11.10
> 対応 §: ロードマップ §22.4 §32

本システムへの変更（バージョンアップ／設定変更／フロー変更）を **計画／評価／承認／実装／検証** の 5 段階で管理する。

## 1. 変更分類

| 区分 | 例 | 影響度 | 必要文書 |
| --- | --- | --- | --- |
| Type 1（不可逆） | スキーマ MAJOR 変更／ライセンス変更 | 高 | ADR ／ 全レビュー ／ 場合により再 PQ |
| Type 2（可逆） | UI 微調整／既定値の変更 | 低 | PR レビューのみ |
| Type 3（緊急） | CVSS ≥ 7.0 の脆弱性修正 | 高 | ホットフィックス（72h 以内） |

## 2. 変更フロー

```
変更要求
  ↓
影響度評価（QA／IT／業務）
  ↓
承認／差戻し
  ↓ 承認
試行版発行（パイロット端末）
  ↓
本番昇格（影響範囲事前提示）
  ↓
事後検証（必要時に再 OQ／PQ 部分実施）
```
