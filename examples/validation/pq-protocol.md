# PQ プロトコル（パフォーマンスクオリフィケーション）

> 対応規格: GMP Annex 11 §4.4／FDA 21 CFR Part 11 §11.10(b)
> 対応 §: ロードマップ §22.1 §13.4
> 想定実施時間: 2 日／環境

## 1. 目的

本システムが、**実運用想定の負荷／業務シナリオで継続的に仕様を満たす** ことを文書化する。

## 2. 前提条件

- IQ／OQ 合格
- 実運用相当のデータセット（マスタ／フロー／ユーザ）が投入済み

## 3. 試験シナリオ

### 3.1. 業務フロー連続実行（48 時間）

| 項目 | 期待結果 | 実測 | 合否 |
| --- | --- | --- | --- |
| 同時稼働端末数 | 200（中規模） | | |
| 完了作業数 | 5,000 件以上／日 | | |
| 同期遅延 中央値 | ≤ 5 秒 | | |
| 監査ログ追記不変性違反 | 0 件 | | |
| エラー率 | ≤ 0.5% | | |

### 3.2. 中断・再開（§10.1）

シフト交代をシミュレートし、Suspended 状態の作業を翌シフトの担当者が再開できることを確認。

| 試験 | 期待結果 | 合否 |
| --- | --- | --- |
| 50 件の suspend → resume | 全件データ損失なし | |

### 3.3. 障害シナリオ（§13.4.2）

`scripts/chaos/s01-s08.sh` を順次実行し、各シナリオで業務影響と復旧時間を計測。

| シナリオ | 復旧時間目標 | 実測 | 合否 |
| --- | --- | --- | --- |
| S-01 ネットワーク断 24h | 復旧後 5 分以内に同期完了 | | |
| S-02 DB 接続断 | 指数バックオフで自動回復 | | |
| S-03 ストレージ満杯 | 警告 → ブロックで業務影響なし | | |
| S-04 時刻ジャンプ | NTP ズレ警告発火 | | |
| S-08 端末電源断 | 直前状態へ復帰 | | |

### 3.4. DR drill（§15.2）

`scripts/dr-drill.sh medium` を実行し、RPO ≤ 15 分／RTO ≤ 1 時間 の達成を確認。

### 3.5. ユーザ受入試験（UAT）

代表ユーザ（§3.2 各ペルソナ）が業務フローを実施し、所要時間・誤操作率・主観評価を計測。

| ペルソナ | 想定 | 実測 | 合否 |
| --- | --- | --- | --- |
| 作業者 | 中断・再開がデータ損失なし | | |
| 班長 | 12 名同時監視で過剰提示なし | | |
| 生産技術 | 5 分以内に最初のフロー発行（§10.2.4） | | |
| 監査・規制対応 | 監査画面で改ざん検出可能 | | |
