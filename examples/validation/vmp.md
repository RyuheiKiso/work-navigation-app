# バリデーションマスタープラン（VMP）

> 対応規格: GMP Annex 11 §1, §4.1／FDA 21 CFR Part 11 §11.10(a)
> 対応 §: ロードマップ §12 §4.9
> 改訂サイクル: 年 1 回／重大変更時

## 1. 目的

work-navigation-app（以下「本システム」）を、製造現場の作業ナビゲーションシステムとして
GxP 環境で運用するための **バリデーション活動の全体計画** を定める。

## 2. 適用範囲

- 端末作業ナビアプリ（§7.1 Tauri+React）
- 設定 Web UI（§7.2 React）
- バックエンド（§7.3 Rust+tokio+PostgreSQL）
- 公式アドオン（§17）

社外秘情報・基幹システム本体・ネットワーク機器は **本 VMP の範囲外**。

## 3. バリデーション戦略

### 3.1. ライフサイクルアプローチ

GAMP 5 の V モデルに従う。

| 左辺（要求／設計） | 右辺（試験） |
| --- | --- |
| URS（ユーザ要件仕様） | PQ（パフォーマンスクオリフィケーション） |
| FS（機能仕様） | OQ（オペレーショナルクオリフィケーション） |
| DS（設計仕様） | IQ（インストールクオリフィケーション） |
| 実装（コード／設定） | 単体／統合テスト（§13） |

### 3.2. リスクベース

ICH Q9 に従い、製品品質・データ完全性・患者安全に対する影響度を評価する。

## 4. 役割と責任（RACI）

| 活動 | 導入企業 QA | システム部門 | 本プロジェクト |
| --- | --- | --- | --- |
| URS 作成 | A／R | C | I |
| FS／DS レビュー | A | R | C |
| IQ／OQ 実施 | C | R／A | C |
| PQ 実施 | A／R | C | I |
| 変更管理 | A | R | C |
| 定期レビュー | A／R | C | I |

凡例: R=実行、A=最終責任、C=諮問、I=報告
