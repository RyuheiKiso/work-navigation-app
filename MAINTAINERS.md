## 3. PR 受入権限

- メンテナ 1 名以上のレビュー承認で merge 可能。
- 創設メンテナが利益相反のある PR（例: 自分自身が author）の場合、副メンテナ昇格まで `competitive-score` 反論プロセス（§4.6.3）または GitHub Discussions の意見募集を経ること。
- ライセンス（§19.1）／用語集主要用語（§28）／公開 API（§17.3）等の Type 1 決定（§30.1）を含む PR は ADR を必須要件とする。

## 4. 資金チャネル採否（§19.5.1）

| チャネル | 採否 | 備考 |
| --- | --- | --- |
| GitHub Sponsors | 採用 | 個人開発の継続的支援 |
| Open Collective | 採用 | 法人寄付の透明な会計 |
| 商用サポート（任意） | 評価中 | 導入企業向け SLA 付き有償サポート（OSS 本体は無料維持） |
| 認定アドオン制度（任意） | 評価中 | サードパーティアドオン認定／審査料 |
| クラウドファンディング（特定機能） | 評価中 | 大型機能（例: 多言語拡張、§11.3.1）の資金調達 |
| 助成金（OSS Funder／Sovereign Tech Fund 等） | 評価中 | 公益性高い機能の助成金応募 |

四半期ごとの収支は [`docs/governance/financial-report-template.md`](./docs/governance/financial-report-template.md) に従い `docs/governance/financial-report-<年期>.md` に公開する。

## 5. 単独大口寄付の取り扱い

特定企業・個人からの単独大口（年次総額の 50% 超）は **編集権独占リスク** として §29 R-017 候補で起票し、複数チャネル分散を優先する（§19.5.1）。

## 6. 寄贈・中立化

公開 +24 ヶ月時点で OpenSSF／Linux Foundation Manufacturing 等への寄贈を評価する（§19.4「公開 +24 ヶ月：中立化検討」）。

## 7. 連絡経路

- 一般質問: GitHub Discussions
- バグ・機能要望: GitHub Issues
- セキュリティ脆弱性: GitHub Security Advisories（[`SECURITY.md`](./SECURITY.md)）
- 行動規範違反: メンテナへの GitHub Security Advisories（[`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md)）

## 8. ライセンス再ライセンスの禁止

本プロジェクトは Apache-2.0 から **より制限的なライセンスへ後退する再ライセンスを禁止** する（§19.5.3）。フォークの自由は保証する。

## 9. 受入観点（§19.4.4）

- 本表の各項目が公開済みであること。
- メンテナ可用時間と PR レビュー SLA が整合し、§19.4.4 達成度で集計されていること。
- 単独大口寄付発生時に §29 リスク登録簿が更新されていること（R-017）。
- 寄贈先候補が §19.4 ＋ §22.3 撤退条件と双方向参照されていること。
