# app/frontend のガイドライン

本ディレクトリは **端末アプリ（Tauri、Android／Windows、再生専用）** と **メンテナンス Web SPA（React）** の実装を保持する。
親 `src/app/CLAUDE.md`（語彙・楔射程）と `src/framework/frontend/CLAUDE.md`（a11y・物理制約）を継承する。

## 1. 二系統の役割分離

| 系統 | 役割 | 配布 | 永続化 |
|---|---|---|---|
| 端末アプリ（Tauri） | 単位作業の **再生専用** | APK 直配布／MSI・MSIX 直配布（Q23-C／F） | SQLite ローカル（Q39-H） |
| メンテナンス Web | 手順の作成・編集・承認・改訂・実績ダッシュボード | Docker（自前ホスト Q7-B） | PostgreSQL（`src/database`） |

UI コードは可能な限り共有する（企画書 §8.2）が、**端末側に手順編集 UI を持ち込まない**。`if (isTauri) { 編集 UI }` のような条件分岐で再生専用境界を破らない。

## 2. 端末アプリの不変条件

- **完全オフラインで動作**（企画書 §8.7、Q32-E）。サーバ停止・ネットワーク断で機能停止しない。
- 起動 = 1 単位作業の `Session`。複数単位作業を 1 セッションに混ぜない。
- ステップ完了の確証方式は **手順設計者が選択しない**。種別から自動決定する（企画書 §8.6 表に厳密に従う）:
  - 通常作業 → 「完了」タップ（Q22-A）
  - 標準時間が短い作業 → 最低滞在時間警告（標準時間の 50% 未満で警告。閾値は ADR-0008 で確定）
  - 安全確認 → 写真証跡 + 数値・チェックリスト（Q22-C／E）
  - 品質確認 → 数値入力 + 写真証跡（Q22-C／E）
  - トレーサビリティ → バーコード／QR スキャン（Q22-D）
- ハプティック・視覚フラッシュ通知を備える（騒音下の補助）。
- 障害時のフェイルセーフ: PDF 印刷出力で紙バックアップ／隣端末への即時呼出（企画書 §8.7、Q32-A／B／C）。

## 3. 物理制約（楔現場の実装）

`src/framework/frontend` の数値（48dp、WCAG 2.2、200% リサイズ）に加え、本層では以下を満たす。

- **片手保持・手袋装着** を前提に、主要操作はサムリーチ可能な領域に置く。
- 視線距離 30〜60cm を想定したフォントサイズ既定。
- 直射日光下と暗所の **両立** を意識した配色（最大コントラスト ／ ナイトモード）。
- ロングプレス必須／ピンチ前提／ホバー前提の操作を採用しない。

## 4. メンテナンス Web SPA の規律

- 認証は Keycloak（OIDC、ADR-0011）に委譲。SPA 側に独自ログインフォームを実装しない。
- 手順編集 UI で TWI-JI 三段（主要ステップ／急所／急所の理由）の入力を **必須化**。空のまま保存できる経路を作らない（`src/app` §3）。
- 実績ダッシュボードは **読み取り専用**。集計データの編集経路を作らない（イベントソーシングの不変条件、`src/database` §2）。
- 改訂・承認は不可逆イベントとして表現する。「下書き保存して承認状態を直接書き換える」UI を作らない。

## 5. 状態と同期

- サーバ絶対正本（`src/app` §4）。端末側のローカル変更は **実績・メモ・写真添付のみ**。
- 同期は `src/framework/backend` の差分同期エンジンに委ねる。本層で独自再実装しない。
- idempotency key 付与・時刻二重記録（端末／サーバ）を本層で **省略しない**。

## 6. やらないこと

- 3D ／ AR ／ SVG 動的ハイライト（企画書 §12 NS-3）。
- ストア配布／MDM 配布（NS-5）。Play Store／App Store／MS Store 向けの分岐を持ち込まない。
- 班長俯瞰／IE 専用／品証専用の画面（J-E／J-F／J-G、NS-19／NS-20 系）。
- 動画 10 秒超のメディア再生（企画書 Q11-A／B／C）。
