# framework/frontend のガイドライン

本ディレクトリは **ドメイン非依存の UI プリミティブ層** である。
親 `src/framework/CLAUDE.md` の依存方向・語彙制約を継承する。

## 1. 提供範囲

- ボタン・入力・モーダル・トースト・リスト・スケルトン等の汎用コンポーネント。
- ルーティング基盤・i18n 基盤・テーマ／配色・アクセシビリティユーティリティ。
- Tauri と Web SPA で共用可能な抽象（端末固有 API は薄いアダプタ越しのみ）。
- 提供しないもの: 手順表示／単位作業／TWI-JI 三段／段取り替え専用 UI。これらは `src/app/frontend` 専用。

## 2. アクセシビリティと現場物理制約（本層で実装する強制値）

企画書 §6.3 の数値要件はドメインに依存しない物理事実であるため、本層に内製化する。

- タッチターゲット: **48×48dp 以上**（Material Design Guidelines、企画書 R-7）。
- コントラスト: **WCAG 2.2 SC 1.4.6 (Enhanced) を既定**。SC 1.4.4 Resize Text に準拠（R-5）。
- フォントサイズ: 200% リサイズで横スクロールが発生しないレイアウトを既定。
- 片手操作・手袋装着を阻害する UI（極小ホバー、ロングプレス必須、ピンチ前提）を本層から提供しない。

## 3. 技術選定

- TypeScript strict（`any` 禁止、`unknown` を強制）。
- フレームワークは React 系（企画書 ADR-0004）。React に密結合する API は `framework/frontend` 直下ではなく `framework/frontend/react/` 配下に隔離し、ビュー層差し替えに耐える境界を持たせる。
- 状態管理ライブラリは選択肢を本層で固定しない。本層は **状態管理に依存しないコンポーネント** を提供する。

## 4. Tauri と Web の共用

- 端末（Tauri）／メンテナンス Web の **UI コードは共有する**（企画書 §8.2）。
- プラットフォーム差は `Capabilities` インタフェース越しに注入し、コンポーネントから直接 `window.__TAURI__` 等を触らない。
- ファイルアクセス・カメラ・QR・ハプティックは本層で抽象化し、Web 側はモック実装で動くこと。

## 5. テスト

- ストーリー駆動（コンポーネント単位の視覚的回帰テスト）を必須とする。
- a11y チェックを CI に組み込み、コントラスト・タッチ領域違反を機械的に弾く。

## 6. やらないこと

- 3D／AR／SVG 動的ハイライト等の重量描画は本層で扱わない（企画書 §12 NS-3 に整合）。
- アニメーションは情報伝達の補助に限定し、装飾アニメーションを本層から提供しない。
